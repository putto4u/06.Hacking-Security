## 시큐어 코딩(Secure Coding, 안전한 소프트웨어 개발) 보안 함수 활용 교육 과정 --- ### **[Module 1] 입력 데이터 검증 및 필터링 보안 함수** * **개요 (Overview)** * 사용자 입력값에 포함된 위험 문자열을 식별하고 제거하여 **XSS(Cross-Site Scripting, 크로스 사이트 스크립팅)** 및 **SQLi(SQL Injection, SQL 인젝션)** 공격을 원천 차단하는 기법입니다. * 외부 유입 데이터에 대한 **화이트리스트(Whitelist, 허용 목록)** 기반 검증 함수를 설계하고 적용합니다. * **핵심 보안 함수 및 메커니즘** * **정규 표현식(Regular Expression, 문자열 패턴 매칭)**을 활용한 데이터 형식 검증. * **HTML Entity Encoding(HTML 엔티티 인코딩, 특수 문자를 안전한 문자 코드로 변환)** 함수 적용. * **Parameterized Query(매개변수화된 질의, SQL 쿼리 구조와 데이터를 분리)** 기능을 통한 DB(Database, 데이터베이스) 보안 강화. > **용어 주석** > * **XSS(Cross-Site Scripting):** 공격자가 웹 페이지에 악성 스크립트를 삽입하여 사용자의 브라우저에서 실행되게 하는 공격 방식입니다. > * **SQL Injection:** 응용 프로그램의 보안 허점을 이용하여 악의적인 SQL 문을 실행시켜 데이터베이스를 비정상적으로 조작하는 공격입니다. --- ### **[Module 2] 암호화 및 해시 함수(Cryptography & Hash Functions) 활용** * **개요 (Overview)** * 중요 데이터를 저장하거나 전송할 때 데이터의 **기밀성(Confidentiality, 인가되지 않은 접근 차단)**과 **무결성(Integrity, 데이터 변조 방지)**을 보장하기 위한 암호화 API(Application Programming Interface, 응용 프로그램 인터페이스) 활용법을 다룹니다. * 안전하지 않은 알고리즘(MD5, SHA-1 등)을 배제하고 표준 보안 알고리즘을 선택하는 기준을 제시합니다. * **핵심 보안 함수 및 메커니즘** * **PBKDF2(Password-Based Key Derivation Function 2, 비밀번호 기반 키 유도 함수)**를 이용한 비밀번호 해싱. * **Salt(솔트, 해시 함수 적용 전 추가하는 임의의 데이터)** 및 **Iteration(이테레이션, 반복 횟수)** 설정을 통한 무차별 대입 공격 방어. * **AES-256(Advanced Encryption Standard 256, 고급 암호화 표준 256비트)** 대칭키 암호화 함수를 이용한 데이터 보호. > **용어 주석** > * **PBKDF2:** 솔트를 적용하고 해시 함수를 수만 번 반복하여 비밀번호를 안전하게 저장하는 키 유도 함수입니다. > * **AES-256:** 미국 표준 기술 연구소(NIST)에 의해 제정된 블록 암호 방식 중 가장 보안성이 높은 대칭키 암호화 표준입니다. --- ### **[Module 3] 인증 및 세션 관리(Authentication & Session Management) 보안** * **개요 (Overview)** * 사용자의 신원을 확인하고 권한을 부여하는 과정에서 발생할 수 있는 보안 취약점을 예방하기 위한 함수 구현 방안입니다. * 세션 탈취 및 **CSRF(Cross-Site Request Forgery, 사이트 간 요청 위조)** 공격을 방어하는 안전한 세션 관리 라이브러리를 사용합니다. * **핵심 보안 함수 및 메커니즘** * **Session ID(세션 식별자)**의 안전한 생성 및 재생성(Regeneration) 함수. * **Secure / HttpOnly** 속성이 적용된 **Set-Cookie(쿠키 설정)** 함수 사용. * **Anti-CSRF Token(CSRF 방지 토큰)** 발행 및 검증 함수 구현. > **용어 주석** > * **CSRF(Cross-Site Request Forgery):** 사용자가 자신의 의지와 무관하게 공격자가 의도한 행위(수정, 삭제 등)를 특정 웹사이트에 요청하게 하는 공격입니다. > * **HttpOnly:** 자바스크립트를 통해 쿠키에 접근하는 것을 차단하여 세션 탈취(XSS 활용)를 예방하는 옵션입니다. --- ### **[Module 4] 에러 처리 및 자원 관리(Error Handling & Resource Management)** * **개요 (Overview)** * 시스템 에러 발생 시 내부 정보(Stack Trace, 경로 정보 등)가 외부에 노출되지 않도록 하는 예외 처리 기법을 학습합니다. * 사용한 자원(Memory, File Descriptor 등)을 안전하게 해제하여 **DoS(Denial of Service, 서비스 거부)** 공격을 예방합니다. * **핵심 보안 함수 및 메커니즘** * **Try-Catch-Finally(예외 처리 블록)**를 이용한 안전한 예외 처리. * 사용자 정의 에러 페이지 반환을 통한 정보 노출 차단. * **Resource Leak(자원 누수)** 방지를 위한 자동 자원 해제(Auto-Closeable) 패턴 적용. > **용어 주석** > * **DoS(Denial of Service):** 시스템의 자원을 고갈시켜 정상적인 서비스를 제공하지 못하게 만드는 공격입니다. > * **Stack Trace:** 에러 발생 시 프로그램의 실행 흐름과 소스 코드 위치를 보여주는 리포트로, 공격자에게 시스템 구조 정보를 제공할 위험이 있습니다. --- **Next Step: 시큐어 코딩 실습 예제 코드 작성**
