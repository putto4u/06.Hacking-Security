경로 조작 (Path Traversal)

Unrestricted File System Access

취약점 개요 (Overview)

**경로 조작(Path Traversal, 디렉터리 탐색)** 취약점은 애플리케이션이 파일 경로를 생성할 때 외부 입력값을 적절히 검증하지 않아 발생합니다. 공격자는 ../ (상위 디렉터리 이동 문자열)을 사용하여 웹 서비스가 제한한 디렉터리(Web Root)를 탈출하고, 시스템의 민감한 파일(/etc/passwd, 환경 설정 파일 등)에 접근할 수 있습니다.

Vulnerable Controller (취약한 파일 다운로드 로직) 

// 파일 다운로드 서비스를 제공하는 Spring Boot 컨트롤러 예제
@GetMapping("/api/download")
public void downloadFile(@RequestParam String fileName, HttpServletResponse response) {
    try {
        // [위험] 사용자가 입력한 fileName이 직접 경로 생성에 사용됨
        // 예: fileName이 "../../../etc/passwd"일 경우 의도치 않은 경로로 확장됨
        String baseDir = "/app/storage/uploads/"; // 서비스 허용 루트
        File targetFile = new File(baseDir + fileName); // 경로 조작 발생 지점

        // 파일 존재 여부만 체크하고 상위 디렉터리 이동 여부는 검증하지 않음
        if (targetFile.exists() && targetFile.isFile()) {
            // 파일 콘텐츠를 읽어 응답 스트림으로 전송
            Files.copy(targetFile.toPath(), response.getOutputStream());
            response.flushBuffer(); // 클라이언트로 데이터 전송 완료
        }
    } catch (IOException ex) {
        log.error("파일 처리 중 오류 발생: {}", ex.getMessage());
    }
}

[실전 공격 시뮬레이션 및 실행 결과]

**분석 결과:** 공격자가 입력한 ../ 기호가 운영체제 레벨에서 처리되면서, 애플리케이션이 /app/storage/uploads/../../../../../etc/passwd를 최종 경로로 인식하여 시스템 파일을 그대로 노출시켰습니다.

명령어 삽입 (Command Injection)

OS Command Execution Breach

취약점 개요 (Overview)

**운영체제 명령어 삽입(Command Injection)**은 웹 애플리케이션이 쉘(Shell) 명령어를 실행할 때 외부 입력값을 필터링 없이 인자로 사용할 때 발생합니다. 공격자는 세미콜론(;), 파이프(|), 앰퍼샌드(&) 등의 쉘 메타문자를 사용하여 원래 의도하지 않은 추가 명령어를 서버 권한으로 실행할 수 있습니다.

Network Diagnostic Utility (위험한 네트워크 진단 도구) 

// 관리자용 네트워크 상태 확인 유틸리티
public String checkNetworkStatus(String targetHost) {
    StringBuilder output = new StringBuilder();
    try {
        // [위험] 쉘을 호출하여 운영체제 명령어를 문자열 결합 방식으로 생성
        // 만약 targetHost가 "8.8.8.8; ls -al" 이라면 쉘은 두 개의 명령어로 인식
        String cmd = "sh -c 'ping -c 3 " + targetHost + "'"; // 명령어 주입 지점
        
        // Runtime.exec는 전달된 문자열 전체를 OS 쉘로 전달함
        Process process = Runtime.getRuntime().exec(cmd);
        
        // 명령어 실행 결과(Standard Output)를 읽어와서 반환
        BufferedReader reader = new BufferedReader(new InputStreamReader(process.getInputStream()));
        String line;
        while ((line = reader.readLine()) != null) {
            output.append(line).append("\n"); // 실행 결과를 화면에 노출
        }
    } catch (Exception e) {
        return "진단 실패: " + e.getMessage();
    }
    return output.toString();
}

자격 증명 노출 (Hardcoded Secrets)

Sensitive Data Leakage in Code

취약점 개요 (Overview)

**하드코딩된 자격 증명(Hardcoded Credentials)**은 DB 비밀번호, 클라우드(Cloud) API 키, 암호화 키 등을 소스코드 내부에 평문 문자열로 직접 기입하는 행위입니다. 이는 코드 관리 도구(Git 등)를 통해 외부에 유출될 위험이 매우 높으며, 유출 시 인프라 전체에 대한 마스터 키를 넘겨주는 것과 같습니다.

Configuration Source (노출된 환경 설정 코드) 

// [보안 약점] 중요 정보가 코드 레벨에서 평문으로 관리됨
@Configuration
public class CloudInfrastructureConfig {

    // AWS 자원 접근을 위한 마스터 액세스 키 (매우 위험)
    private static final String AWS_ACCESS_KEY = "AKIA5F7EXAMPLE_KEY_123"; 
    
    // AWS 자원 접근을 위한 비밀 키 (코드 유출 시 바로 침해 사고)
    private static final String AWS_SECRET_KEY = "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"; 

    // 데이터베이스 마스터 계정 패스워드
    private static final String DB_PASSWORD = "Admin_DB_P@ssw0rd!_2026"; 

    @Bean
    public AmazonS3 s3Client() {
        // 하드코딩된 키값을 사용하여 클라우드 클라이언트 생성
        BasicAWSCredentials credentials = new BasicAWSCredentials(AWS_ACCESS_KEY, AWS_SECRET_KEY);
        return AmazonS3ClientBuilder.standard()
                .withCredentials(new AWSStaticCredentialsProvider(credentials))
                .build();
    }
}

안전하지 않은 역직렬화 (Insecure Deserialization)

Untrusted Object Reconstruction

취약점 개요 (Overview)

**안전하지 않은 역직렬화(Insecure Deserialization)**는 공격자가 변조한 객체 데이터(바이트 스트림)를 서버로 전송할 때, 서버가 이를 복원(역직렬화)하는 과정에서 악의적인 코드를 실행하게 되는 취약점입니다. 이는 원격 코드 실행(RCE, Remote Code Execution)으로 이어지는 가장 치명적인 취약점 중 하나입니다.

Session Recovery Logic (취약한 세션 복구 로직) 

// 클라이언트의 영구 세션 데이터를 복원하는 클래스
public class UserSessionManager {

    public UserProfile restoreProfileFromCookie(byte[] serializedData) {
        try {
            // 1. 외부(쿠키 등)에서 받은 바이트 데이터를 입력 스트림으로 변환
            ByteArrayInputStream bais = new ByteArrayInputStream(serializedData);
            
            // 2. [치명적 위협] 화이트리스트 검증 없이 ObjectInputStream 생성
            ObjectInputStream ois = new ObjectInputStream(bais);

            // 3. readObject() 호출 시, 데이터 내부에 포함된 객체의 코드가 즉시 실행됨
            // 공격자가 가젯 체인(Gadget Chain)을 엮어 보냈다면 이 지점에서 서버가 장악됨
            UserProfile profile = (UserProfile) ois.readObject(); // Sink (실행 지점)
            
            return profile;
        } catch (Exception e) {
            log.error("세션 복원 중 심각한 보안 오류 발생");
        }
        return null;
    }
}
