Technical Manual Attacker Infrastructure Setup

Kali Linux
환경 구축 및 시스템 최적화

본 문서는 오펜시브 시큐리티(Offensive Security) 진단을 위한 공격자 운영체제인 칼리 리눅스(Kali Linux)의 가상 머신 구성부터, 원활한 실무 수행을 위한 한글화, 패키지 최적화, 그리고 네트워크 편의 설정 절차를 기술합니다.

Chapter 1. 칼리 리눅스 확보 및 가상 환경 배포

1. 공식 이미지 다운로드

모의 해킹 도구의 무결성 유지를 위해 운영체제 이미지는 반드시 공식 홈페이지를 통해서만 다운로드해야 합니다.

[칼리 리눅스 공식 홈페이지 Get Kali 다운로드 화면]

참조: Kali Linux Official Site (kali.org/get-kali)

설치 이미지 선택 (Pre-built vs Installer)

가상 머신 환경(VirtualBox, VMware)에 배포할 경우, 초기 설치 과정을 생략할 수 있는 Pre-built Virtual Machines 이미지를 사용하는 것이 표준입니다. 이는 VirtualBox용 .ova 또는 압축된 .vdi 형태로 제공되며 환경 구성 시간을 대폭 단축합니다.

2. 가상 머신(VirtualBox) 배포 및 하드웨어 할당

다운로드한 Pre-built 이미지를 버추얼박스에 가져오기(Import)하여 시스템 리소스를 할당합니다.

1 가상 시스템 가져오기: 버추얼박스 파일 메뉴에서 '가상 시스템 가져오기'를 선택하고 다운로드한 .ova 파일을 로드합니다.

2 운영체제 종류: Linux / Debian (64-bit) 설정 확인.

3 리소스 할당(권장): 원활한 해킹 툴 구동을 위해 최소 RAM 2048MB (2GB) 이상, CPU 2코어 이상을 할당합니다.

3. 초기 접속 및 루트(Root) 권한 확보

2020.1 버전 이후 칼리 리눅스는 기본 root 계정 로그인을 보안상 차단하고 일반 사용자 계정을 사용합니다.

# GUI 로그인 창 초기 자격 증명
Username: kali
Password: kali

# 터미널 실행 후 실무를 위한 Root 권한 획득
kali@kali:~$ sudo su -
[sudo] password for kali: # kali 입력
root@kali:~# # 루트 프롬프트 활성화 완료

Chapter 2. 실무 편의성 및 한글화 최적화

영문판을 기본으로 배포되는 칼리 리눅스의 특성상, 국내 환경에서의 원활한 정보 수집(OSINT) 및 보고서 작성을 위해 한글 폰트 및 입력기 설정이 필수적입니다.

1. 시스템 저장소 및 패키지 최신화

모든 환경 설정 이전에 패키지 인덱스를 동기화하고 시스템을 최신 상태로 업그레이드합니다. 이 과정은 네트워크 연결(NAT 또는 브리지 모드)이 필수적입니다.

# 저장소 목록 업데이트 및 설치된 패키지 업그레이드 (-y 옵션으로 프롬프트 자동 수락)
root@kali:~# apt update && apt full-upgrade -y

# 불필요한 패키지 자동 정리
root@kali:~# apt autoremove -y

2. 한글 폰트 및 입력기(Fcitx5) 구축

기본 상태에서는 웹 브라우저 등에서 한글이 깨지는(네모 박스) 현상이 발생합니다. 나눔 폰트와 Fcitx5 한글 입력기를 설치합니다.

# 1. 한글 나눔 폰트 설치 (웹 폰트 깨짐 방지)
root@kali:~# apt install -y fonts-nanum fonts-nanum-coding

# 2. Fcitx5 기반 한글 입력기 설치
root@kali:~# apt install -y fcitx5 fcitx5-hangul fcitx5-frontend-gtk3 fcitx5-frontend-qt5

입력기 활성화 터미널에서 im-config 명령어를 입력하고, 기본 입력기를 fcitx5로 설정한 뒤 시스템을 재부팅(reboot)합니다.

단축키 설정 재부팅 후 우측 상단 키보드 아이콘 우클릭 > Fcitx5 Configuration 메뉴에서 Hangul을 추가하고 '한/영' 전환 단축키(기본 Ctrl+Space 또는 Shift+Space)를 확인합니다.

3. 시스템 시간대(Timezone) 변경

로그 분석 및 공격 시간의 정확한 타임라인 구성을 위해 시스템 시간대를 한국 시간(KST)으로 변경합니다.

# 시스템 시간대를 서울(Asia/Seoul)로 설정
root@kali:~# timedatectl set-timezone Asia/Seoul

# 설정 확인
root@kali:~# date
2026. 03. 17. (화) 09:15:00 KST

4. 편의 기능: 가상 머신 스냅샷(Snapshot) 생성

스냅샷 백업 필수

모든 기본 설정과 업데이트가 완료된 이 시점이 가장 '깨끗한' 베이스라인 환경입니다. 향후 악성코드 분석이나 시스템 설정 오류 발생 시 즉시 복구할 수 있도록 버추얼박스 관리자 화면에서 반드시 '스냅샷'을 생성해 두어야 합니다.

Chapter 3. 다중 PC 환경 네트워크 구성 (Red Team 설정)

교실 내에서 다수의 물리적 PC가 각각 공격(Kali)과 방어/타겟(Metasploitable) 역할을 수행하는 훈련 환경을 위한 네트워크 어댑터 구성 방법입니다.

단일 PC 실습 시

칼리 리눅스의 어댑터를 '호스트 전용 어댑터(Host-Only)'로 설정하여 내부 가상 스위치망을 구축합니다. 인터넷이 단절되므로 업데이트는 진행할 수 없습니다.

다중 PC 연동 (교실 실습) 시

칼리 리눅스의 어댑터를 '브리지 어댑터(Bridged)'로 설정합니다. 이를 통해 강의실 네트워크 대역폭(예: 192.168.0.X)의 물리적 IP를 부여받아 다른 PC의 타겟 가상 머신에 접근할 수 있습니다.

네트워크 어댑터 재시작 및 통신 검증

# 브리지 모드 설정 후 네트워크 인터페이스 재시작 및 IP 확인
root@kali:~# systemctl restart NetworkManager
root@kali:~# ip addr show eth0

# 원격 타겟 PC(예: 다른 학생 PC의 Metasploitable 2)와의 통신 확인
root@kali:~# ping -c 4 [Target_IP]

이 문서는 칼리 리눅스 시스템의 아키텍처 설정 및 최적화 과정을 가이드합니다. 설정된 시스템을 허가되지 않은 외부 시스템 공격에 사용하는 행위는 엄격히 금지됩니다.
