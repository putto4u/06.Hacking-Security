## 1. root 홈, 패스 디렉터리 권한 및 패스 설정

**🚨 보안 위협:** 최고 관리자 계정의 PATH 환경변수에 현재 디렉터리를 지칭하는 `.`(마침표)가 정상적인 시스템 경로보다 우선적으로 설정되어 있으면, 현재 디렉터리에 변조된 악성 명령어가 삽입되었을 때 관리자 권한으로 해당 코드가 실행될 위험이 존재합니다.
**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음.

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) PATH 환경변수 확인
# echo $PATH

Step 2) 환경설정 파일 내 PATH 변수값 수정
PATH=$PATH:$HOME/bin:<상대 경로> 또는 상대 경로 삭제
```

**[ Shell 종류별 환경설정 파일 ]**

| Shell(쉘) 종류 | 환경설정 파일 |
| --- | --- |
| Bourne Shell(sh) | /etc/profile, $HOME/.profile |
| C Shell(csh) | /etc/csh.cshrc, /etc/csh.login, $HOME/.cshrc, $HOME/.login |
| Korn Shell(ksh) | /etc/profile, $HOME/.profile, $HOME/.kshrc |
| Bash Shell(bash) | /etc/profile, $HOME/.bash_profile, $HOME/.bashrc, /etc/bash.bashrc |

> **📖 전문 용어 해설**
> * **PATH 환경변수(PATH Environment Variable, 패스 환경변수):** 운영체제가 실행 가능한 명령어나 프로그램을 찾을 때 참조하는 디렉터리 경로들의 집합을 나타내는 동적 변수입니다.

## 2. 파일 및 디렉터리 소유자 설정

**🚨 보안 위협:** 퇴직자나 삭제된 계정으로 인해 소유자가 없는 Orphan File(소유자가 없는 파일)이 방치될 경우, 비인가자가 동일한 UID(사용자 식별자)를 생성하여 해당 파일의 소유권을 획득한 후 시스템 자원을 변조하거나 침입 경로로 악용할 위험이 존재합니다.
**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음.

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) 소유자와 그룹이 존재하지 않는 파일 및 디렉터리 확인
# find / \( -nouser -o -nogroup \) -xdev -ls 2>/dev/null

Step 2) 소유자가 존재하지 않는 파일 또는 디렉터리 제거
# rm <파일 이름>
# rm -r <디렉터리 이름>

Step 3) 사용중인 파일 및 디렉터리의 경우 소유자 및 그룹 변경
# chown <사용자 이름> <파일 및 디렉터리 이름>
# chgrp <그룹 이름> <파일 및 디렉터리 이름>

# ※ 소유자 또는 그룹이 존재하지 않는 파일은 파일 속성의 해당 필드에 UID, GID가 숫자로 표시됨
```

> **📖 전문 용어 해설**
> * **Orphan File(오펀 파일, 소유자가 없는 파일):** 시스템 상에 파일은 존재하지만, 해당 파일에 할당된 소유자 계정이나 그룹이 이미 시스템에서 삭제되어 주인이 불명확해진 파일을 의미합니다.

## 3. /etc/passwd 파일 소유자 및 권한 설정

**🚨 보안 위협:** 시스템의 핵심 계정 정보가 담긴 `/etc/passwd` 파일에 일반 사용자의 쓰기 권한이 부여되어 있다면, 비인가자가 임의로 계정을 추가하거나 기존 계정의 UID(사용자 식별자)를 0(root)으로 변조하여 최고 관리자 권한을 탈취할 위험이 존재합니다.
**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음.

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) /etc/passwd 파일 소유자 및 권한 확인
# ls -l /etc/passwd

Step 2) /etc/passwd 파일 소유자 및 권한 변경
# chown root /etc/passwd
# chmod 644 /etc/passwd
```

> **📖 전문 용어 해설**
> * **/etc/passwd:** 유닉스/리눅스 시스템에서 사용자 계정명, 비밀번호(주로 x로 표시), UID, GID(그룹 식별자), 사용자 설명, 홈 디렉터리, 기본 Shell(쉘) 정보를 평문 텍스트 형태로 저장하는 핵심 구성 파일입니다.

<br><br><br>

## 4. 시스템 시작 스크립트 권한 설정

**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음.

## 5. /etc/shadow 파일 소유자 및 권한 설정

**🚨 보안 위협:** 사용자의 실제 암호화된 패스워드 해시값이 저장된 `/etc/shadow` 파일의 권한 관리가 미흡하여 비인가자에게 읽기 권한이 노출될 경우, Brute Force Attack(무차별 대입 공격) 등의 오프라인 크래킹 툴을 이용해 비밀번호가 쉽게 해독될 위험이 존재합니다.
**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음.

### 점검 및 조치 사례

**[ SOLARIS, LINUX ]**
```bash
Step 1) /etc/shadow 파일 소유자 및 권한 변경
# chown root /etc/shadow
# chmod 400 /etc/shadow
```

**[ AIX ]**
```bash
Step 1) /etc/security/passwd 파일 소유자 및 권한 변경
# chown root /etc/security/passwd
# chmod 400 /etc/security/passwd
```

**[ HP-UX ]**
```bash
Step 1) /tcb/files/auth/ 디렉터리 소유자 및 권한 변경
# chown root /tcb/files/auth
# chmod 400 /tcb/files/auth
```

> **📖 전문 용어 해설**
> * **Brute Force Attack(브루트 포스 공격, 무차별 대입 공격):** 가능한 모든 문자열의 조합을 논리적인 순서대로 하나씩 대입하여 올바른 비밀번호를 찾아낼 때까지 끝까지 시도하는 해킹 기법입니다.

<br><br><br>

## 6. /etc/hosts 파일 소유자 및 권한 설정

**🚨 보안 위협:** 로컬 도메인 네임 해석을 담당하는 `/etc/hosts` 파일이 악의적으로 변조될 경우, 사용자가 정상적인 URL 주소를 입력하더라도 해커가 준비해 놓은 가짜 사이트로 연결되는 Pharming(파밍) 공격을 통해 민감한 정보가 유출될 위험이 존재합니다.
**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음.

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) /etc/hosts 파일 소유자 및 권한 변경
# chown root /etc/hosts
# chmod 644 /etc/hosts
```

> **📖 전문 용어 해설**
> * **Pharming(파밍):** 사용자가 올바른 웹 사이트 주소를 입력해도, 시스템 설정 변조(hosts 파일 조작 등)나 DNS 서버를 조작하여 해커가 의도한 위장 웹 사이트로 접속되게 한 후 개인정보를 탈취하는 사이버 범죄입니다.

<br><br><br>

## 7. /etc/(x)inetd.conf 파일 소유자 및 권한 설정

**🚨 보안 위협:** 네트워크 서비스들을 통합 관리하는 데몬 설정 파일인 `inetd.conf` 또는 `xinetd.conf`에 부적절한 권한이 설정되어 있으면, 공격자가 임의의 백도어 포트를 개방하거나 불필요한 취약 서비스들을 활성화하여 시스템에 백도어를 설치할 위험이 존재합니다.
**⚙️ 조치 시 영향:** 설정 변경 후 해당 서비스 데몬을 재시작해야 설정이 올바르게 반영됩니다.

### 점검 및 조치 사례

**[ SOLARIS, AIX, HP-UX ]**
```bash
Step 1) /etc/inetd.conf 파일 소유자 및 권한 변경
# chown root /etc/inetd.conf
# chmod 600 /etc/inetd.conf
```

**[ LINUX - inetd ]**
```bash
Step 1) /etc/inetd.conf 파일 소유자 및 권한 변경
# chown root /etc/inetd.conf
# chmod 600 /etc/inetd.conf
```

**[ LINUX - xinetd ]**
```bash
Step 1) /etc/xinetd.conf 파일 소유자 및 권한 변경
# chown root /etc/xinetd.conf
# chmod 600 /etc/xinetd.conf

Step 2) /etc/xinetd.d/ 디렉터리 내 모든 파일의 소유자 및 권한 변경
# chown -R root /etc/xinetd.d/
# chmod -R 600 /etc/xinetd.d/
```

**[ LINUX - systemd ]**
```bash
Step 1) /etc/systemd/system.conf 파일 소유자 및 권한 변경
# chown root /etc/systemd/system.conf
# chmod 600 /etc/systemd/system.conf

Step 2) /etc/systemd/ 디렉터리 내 모든 파일의 소유자 및 권한 변경
# chown -R root /etc/systemd/
# chmod -R 600 /etc/systemd/
```

> **📖 전문 용어 해설**
> * **Internet Service Daemon(인터넷 서비스 데몬, 슈퍼 데몬):** Telnet(텔넷), FTP 등 다양한 개별 네트워크 서비스의 요청을 중앙에서 대기하고 있다가, 요청이 들어오면 해당 하위 서비스를 연결하여 구동시켜 주는 중계 역할의 관리 데몬입니다.

## 8. /etc/(r)syslog.conf 파일 소유자 및 권한 설정

**🚨 보안 위협:** 시스템 로그 기록 정책을 정의하는 `syslog.conf` 설정 파일이 비인가자에 의해 변조되면, 해커가 자신의 침입 이력이나 시스템 마비 행위를 기록되지 않게 은폐하여 추적과 감사(Audit)를 불가능하게 만들 위험이 존재합니다.
**⚙️ 조치 시 영향:** 권한 변경 후 일반적인 운영엔 영향이 없으나, 로그 데몬을 재시작해야 완전한 무결성이 유지됩니다.

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) /etc/(r)syslog.conf 파일 소유자 및 권한 변경
# chown root /etc/(r)syslog.conf
# chmod 640 /etc/(r)syslog.conf
```

> **📖 전문 용어 해설**
> * **System Logging Protocol(시스템 로깅 프로토콜):** 네트워크 장비나 시스템에서 발생하는 다양한 이벤트(에러, 경고, 시스템 정보 등) 메시지들을 표준화된 형식으로 기록, 수집 및 전송하기 위한 프로토콜 및 데몬입니다.

<br><br><br>

## 9. /etc/services 파일 소유자 및 권한 설정

**🚨 보안 위협:** 서버에서 제공하는 각종 서비스와 매핑되는 기본 네트워크 포트 번호를 정의한 `/etc/services` 파일이 변조되면, 정상적인 통신 포트가 임의의 악성 포트로 우회되어 서비스 거부 상태에 빠지거나 공격용 트래픽 채널로 남용될 위험이 존재합니다.
**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음.

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) /etc/services 파일 소유자 및 권한 변경
# chown root /etc/services
# chmod 644 /etc/services
```

> **📖 전문 용어 해설**
> * **/etc/services:** 서비스 이름(예: ssh, http)과 그에 대응하는 포트 번호(예: 22, 80) 및 프로토콜(TCP/UDP) 정보를 매핑시켜 놓은 일종의 포트 안내소 역할을 하는 텍스트 파일입니다.
