10. SUID, SGID, Sticky bit 설정 파일 점검

[cite_start]🚨 보안 위협: SUID, SGID, Sticky bit 설정이 적절하지 않을 경우, 비인가자가 해당 파일로 특정 명령어를 실행하여 최고 관리자(root) 권한을 획득할 수 있는 심각한 위험이 존재합니다. [cite: 74] [cite_start]⚙️ 조치 시 영향: SUID, SGID, Sticky bit 설정 파일의 권한을 제거할 시, OS 및 응용프로그램 등 연계된 서비스의 정상 작동 여부를 반드시 확인해야 합니다. [cite: 74]

점검 및 조치 사례

[ SOLARIS, LINUX, AIX, HP-UX ]

Step 1) SUID, SGID가 설정된 파일 확인
# find / -user root -type f \( -perm -04000 -o -perm -02000 \) -xdev -exec ls -al {} \;

Step 2) 불필요한 특수 권한 제거
# chmod -s <파일 이름>

Step 3) 반드시 사용이 필요한 경우 특정 그룹에서만 사용하도록 제한하여 일반 사용자의 Setuid 사용 제한
# chgrp <그룹 이름> <SUID를 설정할 파일>
# chmod 4750 <SUID를 설정할 파일>

📖 전문 용어 해설 * SUID(Set User ID, 셋 유저 아이디): 설정된 파일이 실행될 때, 특정 작업 수행을 위해 일시적으로 해당 파일 소유자(주로 root)의 권한을 얻게 되는 특수 접근 권한입니다. * SGID(Set Group ID, 셋 그룹 아이디): 설정된 파일이 실행될 때, 일시적으로 해당 파일이 속한 그룹의 권한을 얻게 되는 특수 접근 권한입니다. * Sticky bit(스티키 비트): 디렉터리에 설정되며, 이 권한이 부여된 디렉터리 내의 파일은 오직 파일의 소유자나 관리자만 수정 및 삭제가 가능하도록 제한하는 권한입니다.




11. 사용자, 시스템 환경변수 파일 소유자 및 권한 설정

[cite_start]🚨 보안 위협: 홈 디렉터리 내 사용자 파일 및 시스템 시작 파일 등 환경변수 파일의 접근 권한이 적절하지 않을 경우, 비인가자가 이를 변조하여 정상 사용자의 서비스 이용을 제한하거나 악용할 위험이 존재합니다. [cite: 74] [cite_start]⚙️ 조치 시 영향: 일반적인 경우 영향 없음. [cite: 74]

점검 및 조치 사례

[ SOLARIS, LINUX, AIX, HP-UX ]

Step 1) 홈 디렉터리 환경변수 파일 소유자 및 권한 확인
# ls -l <홈 디렉터리 환경변수 파일>
# ※ 환경변수 파일 종류: .profile, .kshrc, .cshrc, .bashrc, .bash_profile, .login, .exrc, .netrc 등

Step 2) 홈 디렉터리 환경변수 파일 소유자 및 권한 변경
# chown <root 또는 파일 소유자> <홈 디렉터리 환경변수 파일>
# chmod o-w <홈 디렉터리 환경변수 파일>

📖 전문 용어 해설 * 환경변수 파일(Environment Variable File): 사용자가 시스템에 로그인할 때 쉘(Shell)의 구동 환경과 각종 경로 정보, 기본 설정값 등을 초기화해주는 파일입니다.




12. world writable 파일 점검

[cite_start]🚨 보안 위협: 시스템 파일과 같은 중요 파일에 world writable 권한이 적용될 경우, 일반 사용자 및 비인가자가 해당 파일을 임의로 수정하거나 제거하여 시스템에 치명적인 장애를 유발할 위험이 존재합니다. [cite: 74] [cite_start]⚙️ 조치 시 영향: 일반적인 경우 영향 없음. [cite: 74]

점검 및 조치 사례

[ SOLARIS, LINUX, AIX, HP-UX ]

[일반 사용자 쓰기 권한 제거]

Step 1) world writable 파일 확인
# find / -type f -perm -2 -exec ls -l {} \;

Step 2) 일반 사용자 쓰기 권한 제거
# chmod o-w <파일 이름>

Step 3) 불필요한 world writable 파일 제거
# rm <파일 이름>

📖 전문 용어 해설 * world writable 파일(월드 라이터블 파일, 모든 사용자 쓰기 허용 파일): 소유자나 특정 그룹에 속하지 않은 시스템 내의 모든 일반 사용자(Others)에게 쓰기(Write) 권한이 허용된 취약한 상태의 파일을 의미합니다.

14. $HOME/.rhosts, hosts.equiv 사용 금지

[cite_start]🚨 보안 위협: r-command(rlogin, rsh 등)에 대한 보안 설정이 적용되지 않을 경우, 원격지의 공격자가 별도의 인증 절차 없이 관리자 권한으로 시스템에 접속해 임의의 명령을 실행하거나 주요 정보를 유출할 위험이 존재합니다. [cite: 74] [cite_start]⚙️ 조치 시 영향: 일반적인 경우 영향 없음. [cite: 74]

점검 및 조치 사례

[ SOLARIS, LINUX, AIX, HP-UX ]

Step 1) /etc/hosts.equiv, $HOME/.rhosts 파일 소유자 및 권한 변경
# chown <root 또는 해당 계정> /etc/hosts.equiv
# chmod 600 /etc/hosts.equiv

# chown <root 또는 해당 계정> $HOME/.rhosts
# chmod 600 $HOME/.rhosts

Step 2) /etc/hosts.equiv, $HOME/.rhosts 파일 내 "+" 옵션이 부여된 계정 확인
# cat /etc/hosts.equiv
# cat $HOME/.rhosts

Step 3) /etc/hosts.equiv, vi $HOME/.rhosts 파일 내 "+" 옵션 제거 후 허용 호스트 및 계정 등록

📖 전문 용어 해설 * r-command(알 커맨드, 원격 제어 명령어): 인증 없이 다른 시스템에 원격으로 접속하여 명령을 내릴 수 있게 해주는 유닉스 명령어(rlogin, rsh, rexec 등)로, 보안상 매우 취약하여 사용이 제한됩니다.




15. 접속 IP 및 포트 제한

[cite_start]🚨 보안 위협: 허용할 호스트에 대한 IP주소 및 포트 제한이 적용되지 않을 경우, Telnet이나 FTP 등 보안에 취약한 네트워크 프로토콜을 통하여 외부 비인가자의 불법적인 접근 및 침해사고가 발생할 위험이 존재합니다. [cite: 74] [cite_start]⚙️ 조치 시 영향: 허용 목록에 등록되지 않은 IP는 서비스 사용이 완전히 차단됩니다. [cite: 74]

점검 및 조치 사례

[ SOLARIS ]

[TCP Wrapper]
Step 1) TCP Wrapper에 설정된 접근제한 확인
Step 2) 서비스 차단 및 허용 설정값 수정
# vi /etc/hosts.deny
ALL:ALL
# vi /etc/hosts.allow
<허용할 서비스> : <허용할 IP주소>
# 예시) sshd: 192.168.18.129, 192.168.18.180
# ※ TCP Wrapper 접근제어 가능 서비스: SYSTAT, FINGER, FTP, TELNET, RLOGIN, RSH, TALK, EXEC, TFTP, SSH
# ※ hosts.allow, hosts.deny 두 파일이 존재하지 않는 경우 모든 접근을 허용함

[Packet Filter]
Step 1) Packet Filter에 설정된 접근제한 확인 및 수정
Step 2) /etc/firewall/pf.conf 파일에 허용할 IP 및 포트 정책 추가
# 예시) SSH 서비스 제한
# pass in quick proto tcp from 192.168.1.0/24 to any port = 22 keep state
# block in quick proto tcp from any to any port = 22 keep state
Step 3) 설정한 접근제한 정책 적용
# svcadm refresh svc:/network/firewall:default

19. 홈 디렉토리로 지정한 디렉토리의 존재 관리

[cite_start]🚨 보안 위협: /etc/passwd 파일에 설정된 사용자의 홈 디렉터리가 시스템 내에 존재하지 않을 경우, 해당 계정으로 로그인 시 자동으로 최상위 루트 디렉터리(/)가 홈으로 할당되어 비인가자가 시스템 시스템 명령어에 무단으로 접근할 위험이 존재합니다. [cite: 74] [cite_start]⚙️ 조치 시 영향: 일반적인 경우 영향 없음. [cite: 74]

점검 및 조치 사례

[ SOLARIS, LINUX, AIX, HP-UX ]

# cat /etc/passwd

Step 2) 홈 디렉토리가 존재하지 않는 사용자 계정이 불필요한 계정일 경우, 해당 계정 삭제
# userdel <사용자 이름>

Step 3) 사용중인 계정일 시, 해당 계정의 홈 디렉토리 설정
# vi /etc/passwd
# 예시) example:x:1000:1000::/home/example:/bin/bash

📖 전문 용어 해설 * 루트 디렉터리(Root Directory, /): 유닉스/리눅스의 트리형 파일 시스템 구조에서 가장 꼭대기(최상위)에 위치한 디렉터리로 시스템 운영에 필수적인 모든 폴더가 이곳에서 시작됩니다.




20. 숨겨진 파일 및 디렉토리 검색 및 제거

[cite_start]🚨 보안 위협: 악성 파일이나 백도어 프로그램 등이 숨겨진 파일 및 디렉터리로 존재할 경우, 운영자가 이를 육안으로 쉽게 탐지하지 못하여 시스템 장악 및 정보 유출 등의 치명적인 사고로 이어질 위험이 존재합니다. [cite: 74] [cite_start]⚙️ 조치 시 영향: 일반적인 경우 영향 없음. [cite: 74]

점검 및 조치 사례

[ SOLARIS, LINUX, AIX, HP-UX ]

Step 1) 특정 디렉토리 내 불필요한 파일 확인
# ls -al <디렉토리 이름>

Step 2) 숨겨진 파일 및 디렉토리 확인
# find / -type f -name ".*"
# find / -type d -name ".*"

Step 3) 불필요하거나 의심스러운 숨겨진 파일 및 디렉토리 제거
# rm <파일 이름>
# rm -r <디렉토리 이름>

📖 전문 용어 해설 * 숨겨진 파일(Hidden File): 파일 이름이 마침표(.)로 시작하여 기본 명령어(ls)로는 화면에 출력되지 않는 파일로, 악성 해커들이 시스템 침입 흔적이나 백도어를 은닉하기 위해 자주 악용합니다.
