## 10. SUID, SGID, Sticky bit 설정 파일 점검

[cite_start]**🚨 보안 위협:** SUID, SGID, Sticky bit 설정이 적절하지 않을 경우, 비인가자가 해당 파일로 특정 명령어를 실행하여 최고 관리자(root) 권한을 획득할 수 있는 심각한 위험이 존재합니다. [cite: 74]
[cite_start]**⚙️ 조치 시 영향:** SUID, SGID, Sticky bit 설정 파일의 권한을 제거할 시, OS 및 응용프로그램 등 연계된 서비스의 정상 작동 여부를 반드시 확인해야 합니다. [cite: 74]

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) SUID, SGID가 설정된 파일 확인
# find / -user root -type f \( -perm -04000 -o -perm -02000 \) -xdev -exec ls -al {} \;

Step 2) 불필요한 특수 권한 제거
# chmod -s <파일 이름>

Step 3) 반드시 사용이 필요한 경우 특정 그룹에서만 사용하도록 제한하여 일반 사용자의 Setuid 사용 제한
# chgrp <그룹 이름> <SUID를 설정할 파일>
# chmod 4750 <SUID를 설정할 파일>
```

> **📖 전문 용어 해설**
> * **SUID(Set User ID, 셋 유저 아이디):** 설정된 파일이 실행될 때, 특정 작업 수행을 위해 일시적으로 해당 파일 소유자(주로 root)의 권한을 얻게 되는 특수 접근 권한입니다.
> * **SGID(Set Group ID, 셋 그룹 아이디):** 설정된 파일이 실행될 때, 일시적으로 해당 파일이 속한 그룹의 권한을 얻게 되는 특수 접근 권한입니다.
> * **Sticky bit(스티키 비트):** 디렉터리에 설정되며, 이 권한이 부여된 디렉터리 내의 파일은 오직 파일의 소유자나 관리자만 수정 및 삭제가 가능하도록 제한하는 권한입니다.

## 18. 홈 디렉토리 소유자 및 권한 설정

[cite_start]**🚨 보안 위협:** 사용자별 홈 디렉터리에 타 사용자의 접근이나 수정이 허용되어 있을 경우, 임의의 비인가자가 악의적인 목적으로 접근해 설정 파일을 변조하고 정상적인 서비스를 제한할 위험이 존재합니다. [cite: 74]
[cite_start]**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음. [cite: 74]

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) 사용자별 홈 디렉토리 확인
# cat /etc/passwd

Step 2) 사용자별 홈 디렉토리 소유자 및 권한 확인
# ls -ald <사용자 홈 디렉토리>

Step 3) 사용자별 홈 디렉토리 소유자를 해당 사용자로 변경 및 일반 사용자 권한 제거
# chown <사용자 이름> <사용자 홈 디렉토리>
# chmod o-w <사용자 홈 디렉토리>
```

> **📖 전문 용어 해설**
> * **홈 디렉토리(Home Directory):** 시스템에 계정을 부여받은 사용자가 로그인 직후 진입하게 되며, 자신의 데이터와 개인 설정 파일들을 보관하는 독립된 디렉터리 공간입니다.

## 19. 홈 디렉토리로 지정한 디렉토리의 존재 관리

[cite_start]**🚨 보안 위협:** `/etc/passwd` 파일에 설정된 사용자의 홈 디렉터리가 시스템 내에 존재하지 않을 경우, 해당 계정으로 로그인 시 자동으로 최상위 루트 디렉터리(`/`)가 홈으로 할당되어 비인가자가 시스템 시스템 명령어에 무단으로 접근할 위험이 존재합니다. [cite: 74]
[cite_start]**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음. [cite: 74]

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) 사용자별 홈 디렉토리 확인
# cat /etc/passwd

Step 2) 홈 디렉토리가 존재하지 않는 사용자 계정이 불필요한 계정일 경우, 해당 계정 삭제
# userdel <사용자 이름>

Step 3) 사용중인 계정일 시, 해당 계정의 홈 디렉토리 설정
# vi /etc/passwd
# 예시) example:x:1000:1000::/home/example:/bin/bash
```

> **📖 전문 용어 해설**
> * **루트 디렉터리(Root Directory, `/`):** 유닉스/리눅스의 트리형 파일 시스템 구조에서 가장 꼭대기(최상위)에 위치한 디렉터리로 시스템 운영에 필수적인 모든 폴더가 이곳에서 시작됩니다.

## 20. 숨겨진 파일 및 디렉토리 검색 및 제거

[cite_start]**🚨 보안 위협:** 악성 파일이나 백도어 프로그램 등이 숨겨진 파일 및 디렉터리로 존재할 경우, 운영자가 이를 육안으로 쉽게 탐지하지 못하여 시스템 장악 및 정보 유출 등의 치명적인 사고로 이어질 위험이 존재합니다. [cite: 74]
[cite_start]**⚙️ 조치 시 영향:** 일반적인 경우 영향 없음. [cite: 74]

### 점검 및 조치 사례

**[ SOLARIS, LINUX, AIX, HP-UX ]**
```bash
Step 1) 특정 디렉토리 내 불필요한 파일 확인
# ls -al <디렉토리 이름>

Step 2) 숨겨진 파일 및 디렉토리 확인
# find / -type f -name ".*"
# find / -type d -name ".*"

Step 3) 불필요하거나 의심스러운 숨겨진 파일 및 디렉토리 제거
# rm <파일 이름>
# rm -r <디렉토리 이름>
```

> **📖 전문 용어 해설**
> * **숨겨진 파일(Hidden File):** 파일 이름이 마침표(`.`)로 시작하여 기본 명령어(`ls`)로는 화면에 출력되지 않는 파일로, 악성 해커들이 시스템 침입 흔적이나 백도어를 은닉하기 위해 자주 악용합니다.
