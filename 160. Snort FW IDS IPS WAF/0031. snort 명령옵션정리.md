# Snort 운영 및 주요 실행 옵션(Command Line Options) 체계

네트워크 보안 관제 아키텍처를 설계하거나 **IDS (Intrusion Detection System, 침입 탐지 시스템)**를 운영할 때, **Snort (Open Source Network Intrusion Detection System, 오픈 소스 네트워크 침입 탐지 시스템)**의 실행 옵션을 정확히 이해하는 것은 매우 중요합니다. Snort는 옵션 조합에 따라 단순한 패킷 수집기부터 정교한 침입 차단 시스템까지 다양한 역할을 수행할 수 있습니다.



---

## 1. 패킷 스니퍼 모드 (Packet Sniffer Mode) 관련 옵션

네트워크 인터페이스를 통과하는 트래픽을 실시간으로 화면에 출력하여 가시성을 확보할 때 사용하는 옵션들입니다. 주로 **Troubleshooting (트러블슈팅, 문제 해결)**이나 트래픽의 기본 구성을 확인할 때 활용됩니다.

| 옵션 | 기능 명칭 | 세부 설명 |
| :--- | :--- | :--- |
| **-v** | **Verbose (상세 출력)** | 패킷의 기본 헤더(IP/TCP/UDP/ICMP) 정보를 화면에 표시합니다. |
| **-d** | **Dump Data (데이터 덤프)** | 패킷의 응용 계층 데이터인 **Payload (페이로드, 데이터 본문)**를 함께 출력합니다. |
| **-e** | **Display Link Layer (링크 계층 표시)** | 데이터 링크 계층(Layer 2)의 이더넷 헤더 정보(MAC 주소 등)를 표시합니다. |
| **-X** | **Hex/ASCII Dump (16진수 덤프)** | 페이로드의 내용을 16진수와 ASCII 문자로 상세히 출력합니다. |

> **Verbose (상세 출력)**: 프로그램 실행 과정이나 데이터의 내용을 생략 없이 자세하게 보여주는 모드입니다.
> 
> **Payload (페이로드, 데이터 본문)**: 전송되는 데이터 패킷에서 제어 정보를 제외하고 실제 전달하고자 하는 핵심 데이터를 의미합니다.
> 
> **MAC Address (Media Access Control Address, 매체 접근 제어 주소)**: 네트워크 인터페이스 카드(NIC)에 부여된 전 세계에서 유일한 하드웨어 식별 번호입니다.

---

## 2. 패킷 로거 모드 (Packet Logger Mode) 관련 옵션

수집된 패킷을 디스크에 저장하여 사후 분석이나 법적 증거 확보를 위해 사용하는 옵션들입니다. **Binary (바이너리, 이진 데이터)** 형태로 저장하면 **Wireshark (와이어샤크)** 등 외부 도구와 완벽히 호환됩니다.

* **-l (Log Directory, 로그 저장 경로)**: 로그 파일이 생성될 디렉토리를 지정합니다. (예: `-l ./sntlog`)
* **-K (Logging Format, 로그 형식)**: 로그의 저장 형식을 지정합니다. 주로 `-K pcap`을 사용하여 표준 **PCAP (Packet Capture, 패킷 캡처)** 파일로 기록합니다.
* **-b (Binary Logging, 바이너리 로깅)**: 모든 패킷을 바이너리 형태로 저장합니다. 텍스트 기록보다 시스템 부하가 적어 고속 네트워크 환경에 적합합니다.



> **PCAP (Packet Capture, 패킷 캡처)**: 네트워크 트래픽을 캡처하여 파일로 저장하기 위한 업계 표준 API 및 파일 형식입니다.
> 
> **Binary (바이너리, 이진 데이터)**: 0과 1로 구성된 데이터 형식으로, 컴퓨터가 직접 처리하기에 최적화되어 있으며 텍스트보다 저장 공간을 적게 차지합니다.
> 
> **Log Directory (로그 저장 경로)**: 시스템에서 발생하는 각종 이벤트 기록(로그)이 물리적으로 저장되는 디렉토리 위치입니다.

## 3. IDS/IPS 모드 및 운영 설정 옵션

설정 파일(`snort.conf`)을 로드하여 실제 탐지 규칙(Rule)을 적용하고 보안 정책을 가동할 때 사용하는 핵심 옵션들입니다.

| 옵션 | 기능 명칭 | 세부 설명 |
| :--- | :--- | :--- |
| **-c** | **Configuration (설정 파일 로드)** | Snort 설정 파일의 경로를 지정하여 탐지 엔진을 가동합니다. (예: `-c /etc/snort/snort.conf`) |
| **-T** | **Test Configuration (설정 테스트)** | 설정 파일의 문법 오류나 규칙의 정당성을 검사한 후 종료합니다. |
| **-A** | **Alert Mode (경고 출력 모드)** | 경고 메시지의 출력 형식을 지정합니다. (`fast`, `full`, `console`, `none` 등) |
| **-i** | **Interface (인터페이스 선택)** | 패킷을 수집할 네트워크 카드(NIC)를 선택합니다. (예: `-i eth0`) |
| **-r** | **Read PCAP (파일 읽기)** | 저장된 PCAP 파일을 읽어 들여 탐지 규칙을 재시뮬레이션합니다. |

> **Configuration (설정, 구성)**: 소프트웨어가 작동하는 방식이나 환경을 정의한 파라미터들의 모음입니다.
> 
> **Alert Mode (경고 출력 모드)**: 탐지된 위협 정보를 사용자에게 어떤 수준과 속도로 보여줄지 결정하는 설정입니다.
> 
> **Network Interface Card (NIC, 네트워크 인터페이스 카드)**: 컴퓨터를 네트워크에 연결하여 데이터를 주고받을 수 있게 해주는 하드웨어 장치입니다.

## 4. 고급 성능 및 관리 옵션

대규모 인프라 운영 시 시스템 자원 관리와 데이터 무결성을 위해 설계된 고급 옵션들입니다.

* **-U (Packet Buffer Flush, 패킷 버퍼 즉시 비우기)**: 패킷이 버퍼에 쌓일 때까지 기다리지 않고 즉시 디스크에 기록합니다. 실시간 분석이 중요할 때 유용합니다.
* **-q (Quiet Mode, 저소음 모드)**: 초기 로고나 상태 정보를 출력하지 않고 조용히 실행됩니다. 백그라운드 서비스 운영 시 사용합니다.
* **-D (Daemon Mode, 데몬 모드)**: Snort를 백그라운드 프로세스로 실행하여 터미널을 종료해도 상주하도록 합니다.
* **-n (Packet Count, 패킷 개수 제한)**: 지정된 개수의 패킷만 처리한 후 자동으로 종료됩니다. (예: `-n 100`)



> **Daemon (데몬, 백그라운드 프로세스)**: 사용자가 직접 제어하지 않아도 시스템 백그라운드에서 상주하며 특정 서비스를 제공하는 프로그램입니다.
> 
> **Buffering (버퍼링, 데이터 일시 저장)**: 데이터 처리 속도의 차이를 극복하기 위해 전송할 데이터를 일시적으로 메모리에 보관하는 기술입니다.
> 
> **Background Process (백그라운드 프로세스)**: 사용자 인터페이스(터미널 등)와 연결되지 않고 뒤에서 실행되는 작업 상태를 의미합니다.
