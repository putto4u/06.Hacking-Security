## 와이어샤크(Wireshark) 설치 및 네트워크 트래픽 분석 실무

네트워크 보안 분석 및 트러블슈팅의 핵심 도구인 와이어샤크를 우분투 20.04 LTS(Long Term Support, 장기 지원 버전) 환경에서 설치하고, 침입 탐지 시스템인 Snort(스노트)의 로그를 연동하여 분석하는 심화 과정을 다룹니다.

---

### 1. 우분투 20.04 환경에서 와이어샤크 설치 및 설정

와이어샤크는 GUI(Graphical User Interface, 그래픽 사용자 인터페이스) 기반의 네트워크 프로토콜 분석 도구입니다. 우분투에서 가장 최신 버전의 안정적인 패키지를 설치하기 위해 공식 PPA(Personal Package Archive, 개인 패키지 저장소)를 사용합니다.

#### 1.1 리포지토리 추가 및 패키지 설치

터미널을 열고 아래 명령어를 순차적으로 입력합니다.

```bash
# 최신 안정화 버전 유지를 위한 PPA 추가
sudo add-apt-repository ppa:wireshark-dev/stable

# 패키지 리스트 업데이트
sudo apt update

# 와이어샤크 설치
sudo apt install wireshark
```

#### 1.2 비-루트(Non-root) 사용자 권한 설정

보안상 이유로 일반 사용자 계정에서 네트워크 인터페이스에 접근하여 패킷을 캡처할 수 있도록 설정해야 합니다. 설치 중 나타나는 설정 창에서 **"Yes"**를 선택합니다.

만약 설정 창을 놓쳤다면 다음 명령어로 재설정할 수 있습니다.

```bash
sudo dpkg-reconfigure wireshark-common
```

그 후, 현재 사용자를 `wireshark` 그룹에 추가합니다.

```bash
sudo usermod -aG wireshark $USER
```
> **설정 적용을 위해 반드시 로그아웃 후 다시 로그인하거나 시스템을 재부팅해야 합니다.**

### 2. 와이어샤크 주요 메뉴 구성 및 인터페이스 이해

와이어샤크의 인터페이스는 분석 효율성을 극대화하기 위해 세분화되어 있습니다.



#### 2.1 3단 인터페이스 구성

1.  **Packet List Pane (패킷 리스트 창):** 캡처된 모든 패킷이 한 줄씩 표시됩니다. 시간, 출발지/목적지 IP 주소, 프로토콜, 길이, 요약 정보를 한눈에 볼 수 있습니다.
2.  **Packet Details Pane (패킷 상세 창):** 리스트에서 선택한 특정 패킷의 계층별 프로토콜 헤더 정보를 트리 구조로 보여줍니다. OSI 7 Layer(Open Systems Interconnection 7 Layer, 개방형 시스템 상호 연결 7계층) 모델에 따른 분석이 가능합니다.
3.  **Packet Bytes Pane (패킷 바이트 창):** 실제 패킷 데이터의 16진수(Hexadecimal) 표현과 ASCII(American Standard Code for Information Interchange, 미국 정보 교환 표준 부호) 문자를 대조하여 보여줍니다.

#### 2.2 핵심 메뉴 위치

* **Capture (캡처):** 네트워크 인터페이스를 선택하고 캡처를 시작/중지합니다. `Options` 메뉴에서 무차별 모드(Promiscuous Mode) 등을 설정할 수 있습니다.
* **Analyze (분석):** 프로토콜 계층 통계, 전문가 정보(Expert Info)를 확인할 수 있습니다.
* **Statistics (통계):** 트래픽 양, 엔드포인트(Endpoint, 통신 종단점) 통계, IO 그래프 등을 생성하여 네트워크 전반의 흐름을 시각화합니다.

### 3. Snort 로그 연동 및 분석 기법

Snort(스노트)는 실시간 트래픽 분석과 패킷 로깅을 수행하는 IDS(Intrusion Detection System, 침입 탐지 시스템)입니다. Snort에서 탐지된 위협 데이터를 와이어샤크로 가져와 정밀 분석하는 것은 보안 아키텍트의 필수 역량입니다.

#### 3.1 Snort 로그 생성 확인 (Pcap 형식)

Snort는 로그를 텍스트(Alert) 형태뿐만 아니라 바이너리 패킷 파일인 `pcap` (Packet Capture, 패킷 캡처) 형식으로 저장할 수 있습니다.

```bash
# Snort 실행 시 -b 옵션을 사용하여 바이너리 로그 생성
sudo snort -A fast -b -i eth0 -c /etc/snort/snort.conf
```
* **Pcap (Packet Capture):** 네트워크 패킷 데이터를 저장하기 위한 표준 파일 형식입니다.

#### 3.2 와이어샤크에서 Snort 로그 읽기

1.  와이어샤크를 실행합니다.
2.  **File > Open** 메뉴를 선택합니다.
3.  Snort 로그가 저장된 경로(기본값: `/var/log/snort/`)로 이동하여 `snort.log.xxxxxxx` 파일을 선택합니다.

### 4. 와이어샤크를 활용한 심층 분석 방법

가져온 데이터를 바탕으로 보안 위협이나 네트워크 장애 원인을 규명하는 단계입니다.

#### 4.1 디스플레이 필터(Display Filter) 활용

수만 개의 패킷 중 원하는 것만 골라내는 기능입니다. 상단 필터 바에 입력합니다.

* `ip.addr == 192.168.1.10`: 특정 IP 주소와 관련된 모든 패킷 출력
* `tcp.flags.syn == 1 && tcp.flags.ack == 0`: TCP SYN(Synchronize, 동기화) 패킷만 필터링 (포트 스캔 탐지 시 유용)
* `http.request.method == "POST"`: 웹 서버로 데이터를 전송하는 요청만 확인

#### 4.2 Follow Stream (스트림 추적)

파편화된 패킷들을 하나로 묶어 실제 통신 내용을 재구성하는 기능입니다.
1.  분석할 패킷에서 마우스 우클릭을 합니다.
2.  **Follow > TCP Stream** (또는 UDP/HTTP)을 선택합니다.
3.  공격자가 입력한 명령어 문자열이나 서버의 응답 메시지를 평문으로 확인할 수 있습니다.



#### 4.3 전문가 정보 (Expert Info) 분석

와이어샤크는 패킷의 오류나 비정상적인 흐름을 자동으로 감지합니다.
* **위치:** 하단 왼쪽의 동그란 아이콘 혹은 `Analyze > Expert Information`
* **분류:**
    * **Chat:** 정상적인 통신 흐름
    * **Note:** 주의가 필요한 사항 (예: TCP Retransmission)
    * **Warn:** 경고 (예: 의심스러운 프로토콜 사용)
    * **Error:** 명백한 오류 (예: Checksum 에러)

### 5. 전문 용어 해설 및 주석

* **PPA (Personal Package Archive):** 우분투의 공식 저장소 외에 개발자가 직접 패키지를 업로드하여 배포할 수 있는 개인 저장소입니다.
* **Promiscuous Mode (무차별 모드):** 네트워크 카드(NIC)가 자신에게 전달되지 않은 패킷까지 모두 수신하여 운영체제로 넘겨주는 설정입니다. 스니핑(Sniffing)의 기본 설정입니다.
* **TCP Retransmission (Transmission Control Protocol Retransmission, TCP 재전송):** 보낸 패킷에 대한 확인 응답(ACK)이 오지 않아 데이터를 다시 보내는 현상으로, 네트워크 혼잡이나 패킷 손실을 의미합니다.
* **IDS (Intrusion Detection System, 침입 탐지 시스템):** 네트워크나 시스템의 미인가된 사용, 오용, 남용 등을 실시간으로 탐지하는 시스템입니다.
* **Hexadecimal (16진수):** 0-9와 A-F를 사용하여 수를 표현하는 방식으로, 컴퓨터 분야에서 바이트 데이터를 표기할 때 주로 사용됩니다.
