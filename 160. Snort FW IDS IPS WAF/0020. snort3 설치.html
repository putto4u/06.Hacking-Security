리눅스 환경에서의 Snort 3 설치 및 기본 환경 구성

Snort 3는 기존 버전보다 향상된 성능과 유연성을 제공하기 위해 C++ (C Plus Plus, C 언어의 기능을 확장한 객체 지향 프로그래밍 언어) 기반으로 완전히 재설계되었습니다. 본 과정은 가장 널리 사용되는 Ubuntu (우분투, 데비안 계열의 오픈 소스 리눅스 운영체제) 환경을 기준으로 실습을 진행합니다.

1. 시스템 업데이트 및 필수 의존성 패키지 설치

Snort 3를 빌드하고 실행하기 위해서는 다양한 라이브러리와 빌드 도구가 필요합니다. 시스템의 패키지 목록을 최신화하고 필수 구성 요소를 먼저 설치합니다.

# 시스템 패키지 업데이트
sudo apt update && sudo apt upgrade -y

# 빌드 도구 및 필수 라이브러리 설치
sudo apt install -y build-essential cmake curl git libpcap-dev libpcre3-dev \
libhwloc-dev pkg-config flex bison zlib1g-dev libluajit-5.1-dev \
libssl-dev liblzma-dev libunwind-dev

설명: APT (Advanced Package Tool, 데비안 계열 리눅스의 패키지 관리 시스템)를 사용하여 시스템을 최신 상태로 유지합니다. Cmake (Cross Platform Make, 여러 플랫폼에서 빌드 프로세스를 제어하는 도구)와 libpcap (Packet Capture Library, 네트워크 패킷 캡처를 위한 라이브러리) 등은 Snort의 핵심 엔진을 컴파일하는 데 필수적인 요소입니다.

2. LibDAQ (데이터 수집 라이브러리) 설치

Snort 3는 패킷을 수집하기 위해 DAQ (Data Acquisition, 데이터 수집 라이브러리)의 특정 버전이 필요합니다. Snort 엔진 설치 전에 반드시 먼저 완료되어야 합니다.

# LibDAQ 소스 코드 다운로드 및 빌드
git clone https://github.com/snort3/libdaq.git
cd libdaq
./bootstrap
./configure
make
sudo make install
sudo ldconfig

sudo apt update # 시스템의 패키지 저장소 목록을 최신으로 업데이트합니다.
sudo apt install -y libdumbnet-dev # dnet 기능을 제공하는 dumbnet 개발 라이브러리를 설치합니다.
sudo ln -sf /usr/include/dumbnet.h /usr/include/dnet.h # dumbnet 헤더를 dnet 헤더로 인식하도록 심볼릭 링크를 생성합니다.
sudo ln -sf /usr/lib/x86_64-linux-gnu/libdumbnet.so /usr/lib/x86_64-linux-gnu/libdnet.so # 라이브러리 파일도 dnet 이름으로 링크를 생성합니다.

sudo apt update # 시스템 패키지 저장소 정보를 최신으로 갱신합니다.
sudo apt install -y libpcre2-dev # PCRE2 개발용 라이브러리 및 헤더 파일을 설치합니다.

설명: LibDAQ (Data Acquisition Library, 데이터 수집 라이브러리)는 Snort가 하드웨어 네트워크 인터페이스와 통신하여 패킷을 가로채는 중간 계층 역할을 합니다. ldconfig 명령은 설치된 공유 라이브러리를 시스템이 즉시 인식할 수 있도록 캐시를 갱신하는 역할을 수행합니다.

3. Snort 3 엔진 설치

이제 본격적으로 Snort 3 소스 코드를 가져와 컴파일 및 설치를 진행합니다.

# Snort 3 소스 코드 다운로드
cd ~
git clone https://github.com/snort3/snort3.git
cd snort3

# 빌드 및 설치 진행
./configure_cmake.sh --prefix=/usr/local/snort
cd build
make -j$(nproc)
sudo make install

설명: Prefix (접두사, 소프트웨어가 설치될 최상위 경로) 옵션을 통해 특정 디렉토리에 설치를 집중시켜 관리를 용이하게 합니다. -j$(nproc) 옵션은 현재 시스템의 CPU (Central Processing Unit, 중앙 처리 장치) 코어 수를 자동으로 확인하여 병렬 컴파일을 수행함으로써 설치 시간을 대폭 단축시킵니다.

4. Snort 3 실행 환경 및 초기 설정

설치가 완료되면 Snort가 정상적으로 작동하는지 확인하고, 네트워크 환경에 맞는 설정을 진행해야 합니다.

# 설치 버전 확인
/usr/local/snort/bin/snort -V

# 기본 설정 파일(snort.lua) 수정
# 주로 네트워크 인터페이스 및 룰 경로 설정
sudo vi /usr/local/snort/etc/snort/snort.lua

설명: Snort 3는 기존의 텍스트 기반 설정 방식에서 벗어나 Lua (루아, 가볍고 확장성이 뛰어난 스크립트 언어) 기반의 설정 파일을 사용합니다. snort.lua 파일 내에서 HOME_NET (홈 네트워크, 보호 대상이 되는 내부 네트워크 범위) 대역을 실제 환경(예: 192.168.1.0/24)에 맞춰 수정해야 정확한 탐지가 가능합니다.

5. 탐지 규칙(Rule) 설정 및 유료 서비스 안내

Snort는 엔진만으로는 작동하지 않으며, 공격을 식별할 Ruleset (룰셋, 탐지 규칙 모음)이 필요합니다.

  • Community Rules (커뮤니티 룰): 무료로 제공되는 기본적인 탐지 규칙입니다.
  • Registered Rules (등록자 룰): 회원 가입 시 무료로 제공되나 최신 업데이트가 30일 지연됩니다.
  • Subscriber Rules (유료) (구독자 룰): Cisco Talos (시스코 탈로스, 위협 인텔리전스 그룹)에서 배포하는 최신 보안 규칙을 실시간으로 제공받는 유료 서비스입니다.
# 룰 저장을 위한 디렉토리 생성
sudo mkdir -p /usr/local/snort/etc/snort/rules

# 커뮤니티 룰 다운로드 예시
wget https://www.snort.org/downloads/community/snort3-community-rules.tar.gz
tar -xvzf snort3-community-rules.tar.gz -C /usr/local/snort/etc/snort/rules

설명: IPS (Intrusion Prevention System, 침입 방지 시스템)의 핵심 성능은 최신 위협 정보를 담은 룰셋에 달려 있습니다. 실무 환경이나 엔터프라이즈 레벨에서는 보안 사고 예방을 위해 Subscriber Rules (구독자 룰셋, 유료 정기 구독 서비스) 사용을 권장합니다.

6. 설치 무결성 검사 및 인터페이스 실행

모든 설정이 완료되면 설정 파일의 오류 여부를 검사하고 실제 네트워크 인터페이스에서 구동합니다.

# 설정 파일 오류 검사
/usr/local/snort/bin/snort -c /usr/local/snort/etc/snort/snort.lua -T

# 특정 인터페이스(eth0)에서 Snort 실행
sudo /usr/local/snort/bin/snort -c /usr/local/snort/etc/snort/snort.lua -i eth0 -A alert_fast

설명: -T 옵션은 Test Mode (테스트 모드)를 의미하며, 실제 패킷을 탐지하기 전에 설정 파일의 문법적 오류를 점검합니다. -i 옵션 뒤에는 시스템의 실제 NIC (Network Interface Card, 네트워크 인터페이스 카드) 명칭을 입력하여 해당 경로로 흐르는 트래픽을 모니터링하도록 지정합니다.
