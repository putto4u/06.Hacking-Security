### 네트워크 보안 분석 및 탐지 로그 관리 가이드

네트워크 침입 탐지 시스템인 **Snort**(Open Source Network Intrusion Detection System, 오픈 소스 네트워크 침입 탐지 시스템)와 패킷 분석 도구인 **Wireshark**(Network Protocol Analyzer, 네트워크 프로토콜 분석기)는 보안 전문가에게 필수적인 도구입니다. 본 교재에서는 실시간 데이터 저장 기법부터 심층 패킷 분석까지의 과정을 다룹니다.

---

### Snort 로그 및 실시간 데이터 저장 아키텍처

Snort는 탐지된 이벤트를 다양한 방식으로 기록합니다. 대규모 네트워크 환경에서는 탐지 성능 저하를 방지하기 위해 로그 생성과 저장을 분리하는 아키텍처를 권장합니다.

#### 1. 로그 저장 방식의 분류
* **Packet Logging Mode**(패킷 로깅 모드): 패킷 전체를 **PCAP**(Packet Capture, 패킷 캡처) 형식으로 저장합니다. 분석 데이터가 방대해지지만 사후 포렌식에 유리합니다.
* **Alert Mode**(경고 모드): 설정된 규칙에 부합하는 이벤트 발생 시 요약된 경고 메시지만 기록합니다. **Syslog**(System Logging Protocol, 시스템 로그 프로토콜)나 텍스트 파일로 저장됩니다.
* **Unified2**(유니파이드2): Snort 전용 이진 로그 형식입니다. Snort의 성능을 극대화하기 위해 직접 DB에 쓰지 않고 이 형식으로 먼저 저장한 뒤 별도의 도구로 처리합니다.

#### 2. 실시간 데이터 저장 및 시각화 (Barnyard2 & ELK Stack)
과거에는 **Barnyard2**(Spooler for Snort Unified2, Snort 전용 로그 스풀러)를 사용하여 **MySQL**(My Structured Query Language, 오픈 소스 관계형 데이터베이스)에 저장했으나, 현재는 실시간 분석을 위해 **ELK Stack**(Elasticsearch, Logstash, Kibana, 엘라스틱서치/로그스태시/키바나 통합 로그 관리 솔루션)을 주로 활용합니다.



**전문 용어 해설**
> **IDS (Intrusion Detection System, 침입 탐지 시스템):** 네트워크나 시스템에 대한 비정상적인 행위를 실시간으로 탐지하여 관리자에게 알리는 보안 솔루션입니다.
> **SIEM (Security Information and Event Management, 보안 정보 및 이벤트 관리):** 다양한 보안 장비의 로그를 통합 관리하고 상관관계를 분석하여 위협을 탐지하는 시스템입니다.

### Wireshark 설치 및 환경 구성

**Wireshark**(와이어샤크)는 실시간 네트워크 트래픽을 가로채고 분석할 수 있는 세계에서 가장 널리 쓰이는 네트워크 분석 도구입니다.

#### 1. 운영체제별 설치 방법
* **Windows**: [공식 홈페이지](https://www.wireshark.org/)에서 설치 파일을 내려받아 실행합니다. 설치 과정 중 **Npcap**(Network Packet Capture Driver, 윈도우용 패킷 캡처 드라이버) 설치가 반드시 포함되어야 합니다.
* **Linux (Ubuntu)**: 터미널에서 다음 명령어를 입력합니다.
    ```bash
    sudo apt update
    sudo apt install wireshark
    ```
    설치 중 일반 사용자도 패킷을 캡처할 수 있도록 설정(Should non-superusers be able to capture packets?)에 'Yes'를 선택합니다.
* **macOS**: **Homebrew**(Package Manager for macOS, 맥용 패키지 관리자)를 사용하여 설치하거나 공식 홈페이지의 `.dmg` 파일을 이용합니다.

#### 2. 실행 및 인터페이스 확인
설치 후 실행하면 현재 활성화된 **NIC**(Network Interface Card, 네트워크 인터페이스 카드) 목록이 나타납니다. 트래픽이 발생하는 인터페이스를 더블 클릭하여 캡처를 시작합니다.



**전문 용어 해설**
> **Promiscuous Mode (무차별 모드):** 자신에게 오는 패킷뿐만 아니라 해당 네트워크 세그먼트를 통과하는 모든 패킷을 받아들이도록 설정하는 랜카드 모드입니다.
> **Pcap (Packet Capture, 패킷 캡처):** 네트워크 트래픽을 캡처하고 파일로 저장하기 위한 API(Application Programming Interface)입니다.

### Wireshark를 이용한 패킷 분석 방법론

캡처된 데이터는 방대하므로 원하는 정보만 추출하는 능력이 핵심입니다.

#### 1. 디스플레이 필터 (Display Filters)
분석 시 가장 많이 사용하는 필터 예시입니다.
* `ip.addr == 192.168.1.10`: 특정 IP 주소와 관련된 모든 패킷을 조회합니다.
* `tcp.port == 80`: HTTP 통신과 관련된 80번 포트 패킷만 조회합니다.
* `http.request.method == "POST"`: 웹 서버로 데이터를 전송하는 POST 요청만 확인합니다.

#### 2. TCP Stream 추적 (Follow TCP Stream)
조각난 패킷들을 하나의 완성된 대화 형태로 재구성하는 기능입니다.
* 패킷 리스트에서 마우스 오른쪽 버튼 클릭 -> **Follow** -> **TCP Stream** 선택.
* 이를 통해 로그인 시도 시 전송된 ID/PW나 웹 페이지의 소스 코드를 평문(Plaintext) 상태로 확인할 수 있습니다.

#### 3. 통계 기능을 활용한 이상 징후 파악
* **Statistics -> Conversations**: 어떤 호스트끼리 가장 많은 데이터를 주고받는지 확인하여 **C&C**(Command and Control Server, 명령 제어 서버) 통신 여부를 의심할 수 있습니다.
* **Statistics -> Protocol Hierarchy**: 현재 트래픽 중 특정 프로토콜의 비중이 비정상적으로 높다면 공격의 징후로 판단합니다.

### 공격 패턴별 패킷 특징 분석

보안 전문가는 특정 공격이 발생했을 때 나타나는 패킷의 '시그니처(Signature)'를 이해해야 합니다.

#### 1. 포트 스캐닝 (Port Scanning)
공격자가 대상 서버의 열린 포트를 확인하기 위해 수행합니다.
* **TCP SYN Scan**: 공격자가 **SYN**(Synchronize, 연결 요청) 패킷을 보냈을 때 서버가 **SYN/ACK**를 보내면 포트가 열린 것으로 간주하고, 즉시 **RST**(Reset, 연결 강제 종료) 패킷을 보내 연결을 끊습니다.
* **Wireshark 패턴**: 짧은 시간 내에 다수의 포트로 향하는 SYN 패킷과 이에 대응하는 RST 또는 SYN/ACK 패킷이 관찰됩니다.



#### 2. ICMP Flooding (DoS 공격)
서버에 대량의 ICMP Echo Request를 보내 시스템 자원을 고갈시킵니다.
* **패킷 패턴**: 동일한 출발지 IP 혹은 위조된 IP로부터 초당 수만 개의 **ICMP**(Internet Control Message Protocol, 인터넷 제어 메시지 프로토콜) 패킷이 유입됩니다.

#### 3. SQL Injection (SQL 인젝션)
웹 애플리케이션의 입력값을 조작하여 데이터베이스 정보를 탈취합니다.
* **Wireshark 패턴**: HTTP GET/POST 요청 패킷 내부의 URI 또는 Body 영역에서 `SELECT`, `UNION`, `' OR '1'='1` 과 같은 SQL 예약어 및 특수문자가 포함된 문자열이 탐지됩니다.

#### 4. Brute Force (무차별 대입 공격)
* **패킷 패턴**: 특정 계정(예: admin)에 대해 짧은 시간 동안 수많은 로그인 시도(Telnet, SSH, HTTP 등)가 발생하며, 대부분 'Login Failed' 응답 패킷이 뒤따릅니다.

**전문 용어 해설**
> **3-Way Handshake (3단계 핸드셰이크):** TCP 통신을 시작하기 전, 두 호스트 간에 신뢰성 있는 연결을 설정하기 위해 SYN, SYN/ACK, ACK 패킷을 주고받는 과정입니다.
> **Payload (페이로드):** 전송되는 패킷의 실제 데이터 부분을 의미하며, 공격 코드가 주로 숨겨지는 곳입니다.

### 실전 분석 팁
1. **Time Display Format**: 패킷 분석 시 '도착 시간'을 절대 시간(UTC)이나 이전 패킷과의 시간 차이로 설정하여 공격의 지속 시간을 파악하세요.
2. **Coloring Rules**: Wireshark의 색상 규칙을 활용하여 오류가 발생한 패킷(빨간색/검은색)을 시각적으로 빠르게 구분하세요.
3. **GeoIP 설정**: 외부 IP 주소의 물리적 위치를 파악하여 해외로부터의 비정상적인 접속을 식별하세요.
