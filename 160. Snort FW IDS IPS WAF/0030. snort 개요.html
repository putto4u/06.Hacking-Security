1. 네트워크 보안과 침입 탐지 시스템(IDS)의 이해

네트워크 보안의 핵심은 외부 또는 내부의 비정상적인 활동을 실시간으로 감지하고 대응하는 것입니다. 이를 위해 가장 중추적인 역할을 하는 솔루션이 **IDS (Intrusion Detection System, 침입 탐지 시스템)**와 **IPS (Intrusion Prevention System, 침입 방지 시스템)**입니다.

[도식: 보안 시스템 분류(Security System Classification)]

NIDS

Network-based (네트워크 기반)

HIDS

Host-based (호스트 기반)

NIDS

네트워크 세그먼트 전체의 트래픽을 모니터링하며, 패킷의 헤더와 페이로드를 분석합니다.

HIDS

개별 서버나 워크스테이션에 설치되어 시스템 호출, 로그 파일, 파일 시스템의 변경 사항을 감시합니다.

주석: 패킷 페이로드 (Packet Payload)

네트워크를 통해 전송되는 데이터 패킷에서 헤더 정보를 제외한 실제 전송 데이터를 의미합니다. 공격자는 주로 이 페이로드에 악성 코드를 삽입합니다.

2. 탐지 방법론: 시그니처 기반 vs 이상 탐지

보안 관제에서 가장 중요한 것은 '어떻게 공격을 정의하느냐'입니다. Snort를 포함한 대부분의 IDS는 다음 두 가지 방식을 혼합하여 사용합니다.

시그니처 기반 탐지 (Knowledge-based Detection)

기존에 알려진 공격 패턴(Signature)을 데이터베이스화하여 유입되는 트래픽과 비교하는 방식입니다.

  • 장점: 오탐률(False Positive)이 매우 낮음
  • 단점: Zero-day Attack(제로 데이 공격)에 취약

이상 탐지 (Behavior-based Detection)

정상적인 네트워크의 상태를 학습(Baseline)한 후, 이를 벗어나는 과도한 트래픽이나 비정상적인 연결 시도를 탐지하는 방식입니다.

  • 장점: 새로운 유형의 공격 탐지 가능성
  • 단점: 정상 활동 오인으로 오탐률이 높음

3. 오픈 소스 IDS/IPS 시장의 흐름

솔루션 명칭 주요 특징 운영 주체
Snort (스노트) 사실상의 표준(De Facto Standard), 방대한 시그니처 보유 Cisco (시스코)
Suricata (수리카타) 멀티스레딩 지원, 고속 네트워크 환경 최적화 OISF
Zeek (지크, 구 Bro) 네트워크 트래픽 분석 및 로깅에 특화된 프레임워크 Corelight

Snort의 위상

Snort는 1998년 마틴 로슈(Martin Roesch)에 의해 개발된 이후, 전 세계에서 가장 널리 사용되는 침입 탐지 및 방지 시스템입니다. 단순한 패킷 스니퍼에서 시작하여 현재는 복잡한 차세대 방화벽의 핵심 엔진으로 활용되고 있습니다.

4. Snort 2.x vs Snort 3.x 비교 분석

최근 Snort는 아키텍처의 근본적인 변화를 겪으며 **Snort 3.0(Snort++)**으로 진화했습니다. 이는 현대의 고속 네트워크 환경에 대응하기 위한 필수적인 변화입니다.

주석: Lua (루아)

가볍고 확장성이 뛰어난 스크립트 언어로, Snort 3에서는 설정의 유연성을 높이기 위해 채택되었습니다.

[주요 차이점 요약(Key Differences)]
Architecture (아키텍처)
2.x: Single-thread 3.x: Multi-thread
Configuration (설정 방식)
snort.conf snort.lua
Rule Loading (규칙 로딩)
Slow (Text) Fast (Shared Memory)

5-8. Snort 핵심 4단계 아키텍처 (The 4-Layer Architecture)

01. Sniffer
02. Preprocessor
03. Detection
04. Alerting

01. Packet Sniffer (패킷 스니퍼)

네트워크 인터페이스(NIC, Network Interface Card)를 Promiscuous Mode (무차별 모드)로 설정하여 패킷을 가로챕니다. DAQ (Data Acquisition Library, 데이터 수집 라이브러리)를 사용하여 로우 레벨 패킷을 수집합니다.

02. Preprocessor (전처리기)

IP Defragmentation (IP 파편화 재조합), TCP Stream Reassembly (TCP 스트림 재구성) 등을 통해 공격의 우회를 차단하고 데이터를 Normalization (정규화)합니다.

03. Detection Engine (탐지 엔진)

스노트의 심장부로 Snort Rules(스노트 규칙)와 패킷 데이터를 대조합니다. 애플리케이션 계층(L7) 분석이 가능합니다.

04. Alerting/Logging (경고 및 로그)

탐지 결과를 Unified2 바이너리 포맷이나 데이터베이스(MySQL, Syslog)에 기록하고 사용자에게 Alert (경고)를 전달합니다.

주석: 정규화 (Normalization)

다양한 형태로 변형된 데이터를 일관된 형식으로 변환하여 분석의 정확도를 높이는 과정입니다.

9. 실습 환경 준비: Kali Linux에서 Snort 설치

실습은 침투 테스트 및 보안 분석 전문 배포판인 **Kali Linux (칼리 리눅스)** 환경에서 진행합니다.

# 패키지 목록 업데이트 및 Snort 설치(Update & Install)
$ sudo apt-get update
$ sudo apt-get install snort -y
# 설치 버전 확인(Version Check)
$ snort -V
CONFIG (설정) /etc/snort
LOG (로그) /var/log/snort
BINARY (실행파일) /usr/sbin/snort

10. 실습: Snort 기본 모드 실행 (Sniffer & Logger)

01

Sniffer Mode (스니퍼 모드)

네트워크 인터페이스를 통과하는 트래픽을 실시간으로 화면(Console)에 출력합니다. 다음은 페이로드까지 확인하는 가장 대표적인 실행 예제입니다.

Command Prompt
# 페이로드 데이터까지 포함하여 실시간 출력
$ snort -vd

터미널 출력 예시 (Execution Output)

03/30-07:35:10.123456 192.168.1.10:443 -> 192.168.1.100:54321
TCP TTL:64 TOS:0x0 ID:1234 IpLen:20 DgmLen:1500 DF
***A**** Seq: 0x12345678 Ack: 0x87654321 Win: 0xFFAF TcpLen: 32
Payload (HEX & ASCII):
47 45 54 20 2f 20 48 54 54 50 2f 31 2e 31
GET / HTTP/1.1
-v (Verbose) 상세 정보 모드. 패킷의 TCP/IP 헤더를 화면에 출력합니다.
-d (Dump) 데이터 덤프. 페이로드(Application Layer) 내용을 보여줍니다.
03/30-07:35:10 패킷이 탐지된 날짜(월/일)와 시간(시:분:초)입니다.
TTL (Time to Live) 패킷이 네트워크에서 생존할 수 있는 홉(Hop) 수입니다.
DF (Don't Fragment) 패킷을 더 이상 쪼개지 말라는 설정 플래그입니다.
Win (Window Size) 한 번에 받을 수 있는 데이터의 양을 나타냅니다.
02

Logger Mode (로거 모드)

화면 출력을 넘어, 사후 분석을 위해 수집된 패킷을 디스크에 저장하는 모드입니다. -l 옵션으로 경로를 지정합니다.

Command Prompt
# 모든 정보(-dev)를 포함하여 ./log 디렉토리에 저장
$ snort -dev -l ./log
-e (Ethernet) L2 헤더 정보(MAC 주소 등)까지 포함하여 기록합니다.
-l [dir] (Log) 로그 파일이 저장될 '폴더 경로'를 지정합니다.

생성되는 로그 파일 구조

./log/
├── snort.log.1712345678 (바이너리 패킷 데이터)
└── alert (텍스트 형태의 경고 메시지 요약)
03

로그 확인 및 리딩 모드 (Reading Mode)

바이너리(Binary) 형식으로 저장된 로그는 일반 텍스트 편집기(vi, cat)로 읽을 수 없습니다. Snort 엔진을 통해 다시 해석해야 합니다.

Command Prompt
# 저장된 로그 파일을 읽어 들여 분석 결과 출력
$ snort -r ./log/snort.log.1712345678

분석 결과 요약 (Packet Statistics)

Snort Packet Analyzer Summary:
Total Packets:
10,240
TCP Breakdown:
9,800 (95.7%)
UDP Breakdown:
400 (3.9%)
ICMP Breakdown:
40 (0.4%)
* 위 통계는 -r 옵션 실행 종료 시 Snort가 자동으로 생성하는 패킷 분석 보고서입니다.
-r (Read) Tcpdump/Wireshark 형식의 바이너리 로그 파일을 읽어 들입니다.
snort.log.[timestamp] 로그 파일 뒤의 숫자는 유닉스 타임스탬프(Unix Timestamp)입니다.

11. Snort 주요 명령행 옵션 요약 (Summary of Snort Command-line Options)

Snort의 동작 방식을 결정하는 주요 옵션들을 범주별로 정리한 요약 레퍼런스입니다.

옵션 (Option) 설명 및 용도 (Description & Usage)
-v **Verbose (상세 정보):** 패킷의 TCP/IP 헤더 정보를 표준 출력(화면)에 표시합니다.
-d **Dump (덤프):** 응용 계층의 데이터(페이로드)까지 포함하여 출력합니다.
-e **Ethernet (이더넷):** 데이터 링크 계층(Layer 2)의 헤더 정보를 함께 보여줍니다.
-l [dir] **Log (로그 디렉토리):** 패킷 데이터를 기록할 로그 디렉토리 경로를 지정합니다.
-c [file] **Config (설정 파일):** 특정 설정 파일(snort.conf)을 읽어 IDS 모드로 실행합니다.
-r [file] **Read (읽기):** 바이너리 포맷(tcpdump 형식)의 로그 파일을 읽어 분석합니다.
-i [dev] **Interface (인터페이스):** 감시할 네트워크 인터페이스(eth0 등)를 직접 지정합니다.
-A [mode] **Alert (경고 모드):** 경고 출력 방식(fast, full, none, console 등)을 설정합니다.
-T **Test (테스트):** 설정 파일의 문법적 오류 및 시스템 구성 요소의 자가 진단을 수행합니다.
-q **Quiet (정적 모드):** 실행 시 초기 배너 정보나 종료 시 통계 데이터를 출력하지 않습니다.
-V **Version (버전 확인):** 현재 설치된 Snort의 버전 및 엔진 정보를 확인합니다.

실무 팁: 옵션 조합 활용

대부분의 보안 전문가는 실시간 분석 시 **`snort -vde`** (헤더+데이터+L2 정보) 조합을 선호하며, 운영 환경에서 IDS 모드로 구동할 때는 **`snort -c /etc/snort/snort.conf -A fast -D`** (설정 파일 적용, 빠른 경고, 데몬 모드) 형태를 주로 사용합니다.

Security & Admin Notice

보안 공지: 실습 중 생성되는 로그 데이터는 시스템 자원을 소모할 수 있으므로, 주기적인 정리가 필요합니다. Unified2 포맷 사용 시 별도의 파서가 필요할 수 있음에 유의하십시오. (참조: Snort Official Documentation v2.9.x 및 Snort 3 User Manual)
