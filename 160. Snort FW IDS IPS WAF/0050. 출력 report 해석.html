1. Snort 3 시스템 아키텍처 (System Architecture)

Snort 3는 DAQ (Data Acquisition, 데이터 수집 계층)를 통해 하드웨어 추상화를 실현하며, 수집된 패킷은 Inspector (인스펙터, 프로토콜 분석기)를 거쳐 정규화된 후 탐지 엔진으로 전달됩니다.

GID가 중요한 이유: 같은 경고 메시지라도 GID가 다르면 대응 방식이 달라집니다.
- GID 1: 일반 탐지 규칙 위반 (정책 검토 필요)
- GID 116: 프로토콜 분석기 탐지 (애플리케이션 취약점 공격 가능성 높음)

4. 아키텍트 인사이트 (Expert Insight)

Performance Optimization

로그 분석 결과 Outstanding 패킷 비율이 지속적으로 20%를 상회한다면, 이는 탐지 엔진의 처리 속도보다 패킷 유입 속도가 빠름을 의미합니다. 이 경우 Snort 3의 Worker Threads 설정을 코어 수에 맞게 확장하거나, 불필요한 인스펙터(Inspector)를 비활성화하여 오버헤드를 줄여야 합니다.

Security Posture

Bad Checksum 비율이 높게 나타나는 것은 단순히 패킷 손상이 아니라, 공격자가 탐지 장비를 교란하기 위해 의도적으로 생성한 패킷이거나 하드웨어 가속 기능 간의 충돌일 수 있습니다. 보안 아키텍트는 -k none 옵션의 사용 여부를 신중히 결정해야 합니다.
