1. 아키텍처 심층 개요 (Architecture In-depth Overview)

네트워크 위협은 단순히 텍스트 로그를 확인하는 것만으로는 완벽히 대응할 수 없습니다. 대량의 트래픽 중에서 유의미한 위협을 골라내고 이를 시각적으로 인지하기 위해서는 강력한 분석 도구가 필요합니다. 본 실습에서는 업계 표준인 **Snort(Intrusion Detection System, 침입 탐지 시스템)**와 이를 현대적인 웹 인터페이스로 시각화해주는 **Snorby(스노비)**를 결합합니다. **Snort**가 패킷을 분석하여 탐지 이벤트를 생성하면, **Barnyard2(반야드2)**가 이 로그를 해석하여 **MySQL(마이에스큐엘)** 데이터베이스에 기록하고, 최종적으로 **Snorby** 대시보드가 이를 세련된 그래프와 통계로 변환하여 실시간 보안 가시성을 제공합니다.

Terminology Insight

IDS (Intrusion Detection System, 침입 탐지 시스템): 네트워크 트래픽을 감시하여 악의적인 활동이나 정책 위반을 탐지하고 보고하는 보안 시스템입니다.
Snorby (스노비): 루비 온 레일즈(Ruby on Rails) 기반의 오픈소스 웹 애플리케이션으로, Snort의 탐지 로그를 시각화하고 관리할 수 있게 돕는 보안 대시보드입니다.
Barnyard2 (반야드2): Snort의 출력 데이터를 해석하여 데이터베이스(MySQL 등)에 효율적으로 저장해주는 오픈소스 해석기(Interpreter)이자 스풀러(Spooler)입니다.

2. 핵심 구성 요소 상세 정의 (Detailed Components Definition)

BACKEND: SNORT & BY2

Snort 2.9 Engine

  • Unified2 Binary Output:

    Snorby와의 연동을 위해 고속의 이진 로그 포맷인 Unified2를 사용합니다.

  • Barnyard2 Spooler:

    Snort의 성능 저하를 방지하기 위해 별도의 프로세스로 DB에 데이터를 입력합니다.

FRONTEND: SNORBY UI

Snorby Web Application

  • Ruby on Rails Framework:

    Snorby는 루비 기반으로 작동하며, 세련된 대시보드와 이벤트 분류 기능을 제공합니다.

  • MySQL Database:

    탐지된 모든 보안 이벤트와 패킷 페이로드(Payload)를 구조화하여 저장합니다.

Terminology Expert

Unified2 (유니파이드2): Snort에서 패킷 탐지 결과를 저장하기 위해 설계된 전용 이진 포맷입니다. 성능 오버헤드가 매우 적어 실무 환경에서 권장됩니다.
Ruby on Rails (루비 온 레일즈): 루비 프로그래밍 언어로 작성된 오픈소스 웹 애플리케이션 프레임워크입니다. Snorby의 핵심 엔진으로 사용됩니다.

3. 단계별 실증 및 구현 (Step-by-Step Implementation)

01

Snort 2.9 및 DAQ 라이브러리 설치

Snorby와 가장 안정적으로 호환되는 Snort 2.9 버전을 소스 컴파일 방식으로 설치합니다. 패킷 캡처 추상화 레이어인 DAQ (Data Acquisition, 데이터 수집 라이브러리) 설치가 선행되어야 합니다.

Snort Source Compilation Console
# DAQ 라이브러리 컴파일 및 설치
$ wget https://www.snort.org/downloads/snort/daq-2.0.7.tar.gz
$ tar -xvzf daq-2.0.7.tar.gz && cd daq-2.0.7
$ ./configure && make && sudo make install

# Snort 2.9 설치 (Snorby 호환 최적화)
$ ./configure --enable-sourcefire --enable-gre --enable-mpls --enable-targetbased
$ make && sudo make install
02

MySQL 데이터베이스 보안 스키마 구축

Snorby가 사용할 데이터베이스 공간을 생성하고, Barnyard2가 데이터를 입력할 수 있도록 권한을 설정합니다.

MySQL Root Console
-- Snorby 전용 데이터베이스 및 사용자 생성 --
CREATE DATABASE snort;
CREATE USER 'snorbyuser'@'localhost' IDENTIFIED BY 'password123';
GRANT ALL PRIVILEGES ON snort.* TO 'snorbyuser'@'localhost';
FLUSH PRIVILEGES;
03

Barnyard2 소스 설치 및 브릿지 구성

Barnyard2(반야드2)는 Snort가 생성한 이진 로그 파일을 실시간으로 모니터링하여 MySQL 데이터베이스에 최적화된 형태로 입력하는 역할을 수행합니다. 이를 위해 MySQL 개발 라이브러리와 연동하여 직접 소스 컴파일을 진행해야 합니다.

Barnyard2 Installation & Configuration Console
# 1. 필수 의존성 패키지 및 MySQL 라이브러리 설치
$ sudo apt-get install -y libmysqlclient-dev libpcap-dev autoconf libtool

# 2. Barnyard2 소스 다운로드 및 컴파일
$ git clone https://github.com/firnsy/barnyard2.git && cd barnyard2
$ ./autogen.sh && ./configure --with-mysql --with-mysql-libraries=/usr/lib/x86_64-linux-gnu
$ make && sudo make install

# 3. 설정 파일 및 데이터베이스 연동 구성
$ sudo cp etc/barnyard2.conf /etc/snort/
# /etc/snort/barnyard2.conf 파일 하단에 다음 내용 추가
output database: log, mysql, user=snorbyuser password=password123 dbname=snort host=localhost

# 4. Barnyard2 데몬 실행 (Spool 모드)
$ barnyard2 -c /etc/snort/barnyard2.conf -d /var/log/snort -f snort.log -w /etc/snort/bylog.wal
Barnyard2 Key Options

-c [Config]: Barnyard2의 설정 파일 경로를 지정합니다.
-d [Directory]: Snort가 생성하는 Unified2 로그 파일이 저장된 디렉토리를 지정합니다.
-f [File]: 모니터링할 기본 로그 파일 명칭(Prefix)을 지정합니다.
-w [Wal File]: "Write Ahead Log" 파일로, Barnyard2가 마지막으로 읽은 로그 위치를 기록하여 재시작 시 중복 입력을 방지합니다.

04

Snorby 웹 애플리케이션 배포 및 초기화

Ruby (루비) 환경에서 Snorby 소스를 내려받고 필요한 **Gem(젬)**들을 설치합니다. 이후 데이터베이스 마이그레이션을 통해 Snorby의 UI 테이블을 생성합니다.

Snorby App Deployment Terminal
# 루비 번들러를 통한 의존성 설치
$ bundle install

# Snorby 데이터베이스 설정 및 초기화 (Rake 작업)
$ bundle exec rake snorby:setup

# Snorby 서버 실행 (포트 3000)
$ bundle exec rails server -e production

4. 실시간 보안 모니터링 실습 (Live Monitoring Practice)

구축된 파이프라인이 정상적으로 작동하는지 확인하기 위해 가상의 공격을 발생시키고 Snorby 대시보드에 실시간으로 반영되는지 확인합니다.

공격 시나리오 (Nmap Scan)

공격자 PC에서 Snort 센서가 설치된 IP를 대상으로 **Nmap(앤맵)** 스캔을 수행합니다. Snort는 이를 포트 스캐닝 시도로 탐지하여 로그를 생성합니다.

$ nmap -sS -A [SNORT_IP]

Snorby 분석

Snorby 대시보드에서 **"NMAP Scan Detected"** 또는 관련 시그니처 로그가 상단에 배치되는지 확인합니다. 발생 빈도와 심각도(Severity)가 색상별로 구분되어 출력됩니다.

5. 아키텍트 심층 분석 (Architect's Insight)

Barnyard2의 중요성

Snort가 직접 DB에 데이터를 입력하게 되면, 트래픽이 몰릴 때 DB 처리 지연으로 인해 Snort의 패킷 분석이 멈출 수 있습니다. Barnyard2는 이를 비동기적으로 처리하여 Snort의 탐지 성능을 100% 보장하는 역할을 합니다.

보안 및 로그 최적화

Snorby는 모든 패킷의 원시 데이터(Raw Payload)를 DB에 저장할 수 있습니다. 하지만 이는 저장 공간을 급격히 소모하므로, 실무에서는 주요 탐지 이벤트에 대해서만 페이로드를 기록하도록 Snort 룰의 `threshold(임계값)`을 튜닝해야 합니다.

AWS Environment & Billing Notice

**AWS (Amazon Web Services, 아마존 웹 서비스)**에서 IDS 환경을 구축할 경우, 패킷 미러링을 위해 **VPC Traffic Mirroring (브이피씨 트래픽 미러링)** 기능을 사용하게 됩니다. 이 서비스는 시간당 및 미러링되는 트래픽 용량당 과금이 발생하므로, 실습 종료 후에는 반드시 Mirror Filter와 Session을 삭제해야 불필요한 과금을 방지할 수 있습니다.
