네트워크 전수 패킷 캡처 및 단계별 심층 분석 가이드

네트워크 보안 관제 및 Troubleshooting (트러블슈팅, 문제 해결) 과정에서 가시성을 확보하기 위해, 특정 탐지 규칙(Rule)에 매칭되는 패킷뿐만 아니라 인터페이스를 통과하는 모든 트래픽을 실시간으로 확인하고 기록하는 과정은 필수적입니다. 특히 바이너리 형태로 저장된 로그를 Wireshark (와이어샤크, 네트워크 패킷 분석 도구)로 넘기기 전, Snort (Open Source Network Intrusion Detection System, 오픈 소스 네트워크 침입 탐지 시스템) 자체의 읽기 모드를 통해 빠르게 1차 검증을 수행하는 아키텍처는 분석의 효율성을 극대화합니다.

1. 실시간 패킷 로깅(Logging) 및 전수 캡처 실행

네트워크상의 모든 데이터를 가로채어 실시간으로 출력하고 파일로 저장하기 위해서는 Snort 실행 시 상세 출력 옵션과 저장 경로 지정을 병행해야 합니다.

1.1. 로그 저장 디렉토리(Directory) 준비

로그가 저장될 공간을 미리 생성하고, 권한 문제로 인해 기록이 중단되지 않도록 소유권과 권한을 설정합니다.

# 로그 디렉토리 생성 및 권한 설정
mkdir -p ./sntlog
chmod 755 ./sntlog

1.2. 실시간 전수 캡처 및 바이너리(Binary) 저장

다음 명령어를 통해 네트워크의 모든 패킷을 실시간으로 화면에 출력함과 동시에 ./sntlog 경로에 업계 표준 형식으로 저장합니다.

# 실시간 상세 출력 및 pcap 저장 명령어
sudo snort -vde -l ./sntlog -K pcap
  • -v (Verbose, 상세 정보 출력): 패킷의 기본 헤더 정보를 터미널에 표시합니다.
  • -d (Dump Application Layer, 응용 계층 데이터 덤프): 패킷 내의 실제 데이터 본문(Payload)을 16진수 및 ASCII 형태로 화면에 출력합니다.
  • -e (Display Link Layer, 데이터 링크 계층 정보 표시): 계층 2(L2)인 이더넷 헤더 정보(MAC 주소 등)까지 상세히 보여줍니다.
  • -l (Log Directory, 로그 저장 경로): 생성된 로그 파일이 저장될 위치를 ./sntlog로 지정합니다.
  • -K pcap (Packet Capture Format, 패킷 캡처 형식): 로그를 텍스트가 아닌 바이너리 형태인 PCAP (Packet Capture, 패킷 캡처) 파일로 강제 저장하여 호환성을 확보합니다.

Troubleshooting (트러블슈팅, 문제 해결): 시스템이나 네트워크에서 발생한 결함이나 문제를 논리적으로 찾아내어 해결하는 과정입니다.

Payload (페이로드, 데이터 본문): 전송되는 데이터 패킷에서 헤더 정보를 제외하고 실제 전달하고자 하는 핵심 정보입니다.

PCAP (Packet Capture, 패킷 캡처): 네트워크 트래픽을 가로채어 파일로 저장하기 위한 API이자 파일 데이터 형식입니다.

2. Snort 읽기 모드를 이용한 1차 로그 분석

와이어샤크와 같은 무거운 GUI (Graphical User Interface, 그래픽 사용자 인터페이스) 도구를 실행하기 전, 텍스트 기반의 터미널 환경에서 Snort를 사용하여 저장된 바이너리 로그를 빠르게 훑어볼 수 있습니다. 이는 특정 시간대의 패킷 존재 여부나 대략적인 공격 패턴을 즉시 파악하는 데 유리합니다.

2.1. 저장된 바이너리 로그 읽기

Snort의 -r 옵션을 사용하면 이미 저장된 PCAP (Packet Capture, 패킷 캡처) 파일을 읽어 들여 화면에 출력할 수 있습니다.

# 특정 로그 파일 읽기 (파일명은 생성된 타임스탬프 확인 필요)
snort -vde -r ./sntlog/snort.log.1711930000

2.2. 조건부 로그 읽기 (BPF 필터 활용)

모든 패킷을 보는 대신, 특정 조건에 맞는 패킷만 골라서 읽을 수 있습니다. 이는 BPF (Berkeley Packet Filter, 버클리 패킷 필터) 표현식을 사용합니다.

# 로그 파일 내에서 TCP 80번 포트(HTTP) 트래픽만 골라 보기
snort -vde -r ./sntlog/snort.log.1711930000 tcp and port 80

GUI (Graphical User Interface, 그래픽 사용자 인터페이스): 사용자가 그래픽 아이콘과 시각적 지표를 통해 컴퓨터와 상호작용할 수 있게 해주는 환경입니다.

BPF (Berkeley Packet Filter, 버클리 패킷 필터): 데이터 링크 계층의 패킷을 필터링하기 위한 기술로, 대부분의 네트워크 분석 도구에서 필터링 문법의 표준으로 사용됩니다.

Binary Log (바이너리 로그, 이진 로그): 텍스트가 아닌 0과 1의 이진 데이터로 기록된 파일로, 직접 편집은 불가능하지만 데이터 보존율이 높습니다.

3. 프로세스(Process) 정지 및 데이터 검증

패킷 수집이 완료되었거나 분석에 충분한 데이터가 확보되었다면, 안전하게 프로세스를 종료하고 생성된 파일의 무결성을 확인해야 합니다.

3.1. 캡처 중단 (Interrupt)

실행 중인 Snort 프로세스를 중단하려면 키보드의 Ctrl + C 조합키를 입력합니다. 이는 운영체제에 SIGINT (Signal Interrupt, 시그널 인터럽트)를 전달하여 프로세스가 현재까지 기록된 데이터를 파일에 안전하게 저장하고 종료되도록 유도합니다.

3.2. 생성된 로그 파일 리스트 확인

종료 후 ./sntlog 디렉토리를 확인하여 파일의 용량과 생성 시간을 체크합니다.

# 생성된 로그 파일 상세 리스트 확인
ls -lh ./sntlog

SIGINT (Signal Interrupt, 시그널 인터럽트): 실행 중인 프로세스에 중단 신호를 보내어 작업을 멈추게 하는 유닉스 계열 시스템의 제어 신호입니다.

Timestamp (타임스탬프, 시간 정보): 데이터가 생성되거나 기록된 시점의 정확한 날짜와 시각을 숫자로 표현한 값입니다.

4. 와이어샤크(Wireshark)를 이용한 최종 정밀 분석

Snort를 통해 1차적으로 확인된 의심 패킷을 시각화하여 정밀하게 해부(Dissecting)하는 단계입니다. 와이어샤크의 강력한 프로토콜 디코딩 능력을 활용합니다.

4.1. 파일 로드(Load) 및 필터링

  1. 와이어샤크 실행: GUI 환경에서 와이어샤크 프로그램을 구동합니다.
  2. 파일 열기 (Open): File -> Open 메뉴를 통해 ./sntlog에 저장된 로그 파일을 불러옵니다.
  3. Display Filter (디스플레이 필터, 표시 필터): 상단 필터창에 ip.src == [공격자IP] 또는 http.request.method == "POST" 등을 입력하여 심층 분석 대상을 선별합니다.

4.2. 스트림 분석 (Stream Analysis)

단일 패킷 분석에서 나아가, 세션 전체의 흐름을 파악하기 위해 Follow -> TCP Stream 기능을 사용합니다. 이를 통해 공격자가 전송한 실제 데이터와 서버의 응답을 인간이 읽을 수 있는 텍스트 형태로 재구성하여 확인할 수 있습니다.

Dissecting (디섹팅, 패킷 해부): 캡처된 원시 바이너리 데이터를 각 프로토콜의 표준 규격에 맞춰 필드별로 쪼개어 분석하는 과정입니다.

Display Filter (디스플레이 필터, 표시 필터): 수집된 전체 데이터 중 분석가가 설정한 특정 조건에 맞는 데이터만 화면에 노출하는 기능입니다.

TCP Stream (TCP 스트림, TCP 흐름): 두 종단 간의 연속적인 데이터 교환 과정을 하나의 논리적인 흐름으로 묶어 보여주는 기능입니다.
