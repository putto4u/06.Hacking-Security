1. 아키텍처 심층 개요 (Architecture In-depth Overview)

현대의 사이버 보안 위협은 고도로 지능화되고 있으며, 이를 탐지하고 대응하기 위해서는 단순히 로그를 남기는 것을 넘어 실시간으로 분석하고 시각화할 수 있는 인프라가 필수적입니다. 본 실습에서는 업계 표준인 Snort (Intrusion Detection System, 침입 탐지 시스템)와 강력한 빅데이터 분석 솔루션인 Splunk (Security Information and Event Management, 보안 정보 및 이벤트 관리)를 결합합니다. Snort가 수집한 네트워크 원시 데이터(Raw Data)를 JSON (JavaScript Object Notation, 자바스크립트 객체 표기법) 형식으로 변환하여 Splunk에 파이프라인 방식으로 전송함으로써, 보안 관리자는 위협 발생 즉시 대시보드를 통해 상황을 직관적으로 파악할 수 있습니다.

Terminology Insight

IDS (Intrusion Detection System, 침입 탐지 시스템): 네트워크 트래픽을 실시간으로 감시하여 비정상적이거나 악의적인 활동을 탐지하고 보고하는 시스템입니다. 공격 시그니처와 대조하는 방식과 비정상 행위를 탐지하는 방식을 혼용합니다.
SIEM (Security Information and Event Management, 보안 정보 및 이벤트 관리): 기업 내 산재한 보안 장비의 로그를 중앙에서 수집하여 연관 분석(Correlation Analysis)을 수행하고, 위협에 대한 가시성을 제공하는 통합 보안 관제 플랫폼입니다.
JSON (JavaScript Object Notation, 자바스크립트 객체 표기법): 사람이 읽기 쉽고 기계가 파싱하기 용이한 데이터 교환 표준 형식입니다. 키-값(Key-Value) 구조로 되어 있어 Splunk와 같은 빅데이터 엔진에서 매우 효율적으로 색인됩니다.

2. 핵심 구성 요소 상세 정의 (Detailed Components Definition)

NODE 01: SENSOR

Snort 3 IDS

  • Ubuntu 22.04 LTS (우분투 22.04 장기 지원 버전):

    최신 보안 패치와 안정성이 보장된 우분투 서버 환경을 사용합니다.

  • Snort 3 Engine:

    멀티스레딩(Multi-threading)을 지원하여 대규모 트래픽 처리가 가능합니다.

NODE 02: SIEM

Splunk Enterprise

  • Real-time Indexing:

    유입되는 데이터를 실시간으로 인덱싱하여 검색 즉시 결과를 반환합니다.

  • Dashboard Visualizer:

    복잡한 데이터를 그래프와 차트로 변환하여 상황 인지 능력을 극대화합니다.

Terminology Expert

Promiscuous Mode (무차별 모드): 네트워크 인터페이스가 자신에게 할당된 MAC 주소가 아닌 다른 패킷들도 모두 수신하도록 설정하는 모드입니다. 전체 네트워크 모니터링을 위해서는 반드시 이 모드가 활성화되어야 합니다.
LTS (Long Term Support, 장기 지원 버전): 오픈소스 소프트웨어에서 보안 패치 및 기술 지원을 최장 기간 보장하는 안정적인 배포판을 의미합니다.

3. 단계별 실증 및 구현 (Step-by-Step Implementation)

01

Snort 3 및 종속성 패키지 설치

Snort는 실시간 패킷 처리를 위해 고도로 최적화된 라이브러리들을 필요로 합니다. libpcap (Packet Capture Library, 패킷 캡처 라이브러리) 및 pcre (Perl Compatible Regular Expressions, 펄 호환 정규식) 등을 포함한 필수 요소들을 먼저 구성합니다.

Ubuntu 22.04 LTS Shell Console
# Ubuntu 패키지 저장소 업데이트 및 필수 라이브러리 설치
$ sudo apt update

$ sudo apt install -y build-essential libpcap-dev libpcre3-dev libdumbnet-dev zlib1g-dev liblzma-dev openssl libssl-dev snort
02

JSON 로그 출력 인터페이스 구성

Snort의 탐지 결과를 Splunk가 즉시 이해할 수 있는 JSON 구조로 출력하도록 환경 설정 파일을 수정합니다.

/etc/snort/snort.lua
-- Snort 3 SIEM Integration Plugin Configuration --
alert_json = {
    file = true,
    limit = 100, -- MB 단위 로그 제한
    fields = 'timestamp pkt_num proto src_addr src_port dst_addr dst_port action msg'
}

아키텍트 해설: 데이터 보존 및 동시성 (Data Persistence & Concurrency)

JSON 출력 플러그인의 특성: Snort 3는 모듈형 아키텍처를 채택하고 있어 여러 개의 출력 플러그인을 동시에 가동할 수 있습니다. alert_json을 활성화하더라도 기존의 텍스트 기반 로그(alert_fast 등)는 파일 경로와 형식을 그대로 유지하며 병렬로 기록됩니다. 이는 SIEM (Security Information and Event Management, 보안 정보 및 이벤트 관리)을 통한 원격 분석과 로컬에서의 직접적인 분석을 동시에 수행할 수 있게 하여 보안 운영의 유연성을 보장합니다.

출처: Snort Official Documentation
03

Splunk Enterprise 서버 구축 (Ubuntu Optimized Installation)

Splunk Enterprise (스플렁크 엔터프라이즈) 우분투 전용 .deb 패키지를 사용하여 설치합니다. 우분투 환경에서는 dpkg의 종속성 문제를 방지하기 위해 apt install 명령어를 사용하여 로컬 패키지를 설치하는 것이 표준입니다.

[도식: Ubuntu 기반 Splunk 배포 및 초기화 워크플로우]
DEB 패키지 확보
apt 로컬 설치
ufw 방화벽 개방
Web UI 접속
Ubuntu Server Terminal
# 우분투 환경에서 의존성 해결을 포함한 패키지 설치 (추천 방식)
$ sudo apt install ./splunk-9.x.x-linux-2.6-amd64.deb

# Splunk 서비스 초기 실행 및 라이선스 수락 (ID/PW 설정 필수)
$ sudo /opt/splunk/bin/splunk start --accept-license

# Ubuntu ufw 방화벽 설정 (Web UI 포트 8000 개방)
$ sudo ufw allow 8000/tcp

# 부팅 시 자동 시작 설정
$ sudo /opt/splunk/bin/splunk enable boot-start
Ubuntu Specific Terminology

apt install ./ (고급 패키지 도구 설치): 단순한 dpkg와 달리, 설치하려는 패키지가 필요로 하는 추가 라이브러리(Dependencies)를 우분투 저장소에서 자동으로 찾아 함께 설치해주는 방식입니다.
ufw (Uncomplicated Firewall, 간편한 방화벽): 우분투의 기본 방화벽 관리 도구로, 보안을 위해 필요한 포트만을 선택적으로 개방할 때 사용합니다.
boot-start (부트 스타트): 시스템 전원이 켜질 때 수동 조작 없이 서비스가 자동으로 실행되도록 환경을 구성하는 기능입니다.

05

보안 관제 대시보드 및 SPL 쿼리 구현

수집된 데이터를 유의미한 정보로 가공하기 위해 SPL (Search Processing Language, 검색 처리 언어)을 사용하여 주요 위협 근원지를 시각화합니다.

Splunk Search Bar (SPL Editor)
index=* sourcetype="_json"
| stats count by src_addr
| sort -count
| rename src_addr as "Attacker IP Address", count as "Detection Frequency"
1. Select 'Visualization' Tab
2. Choose 'Bar Chart'
3. Save As > Dashboard Panel
Expert Glossary

SPL (Search Processing Language, 검색 처리 언어): Splunk 플랫폼에서 데이터를 검색, 분석 및 시각화하기 위해 사용되는 전용 언어입니다. 파이프(|)를 통해 명령어를 전달하는 구조로 설계되었습니다.
Parsing (구문 분석): 비정형 데이터를 정형화된 데이터 필드로 변환하는 과정입니다. JSON은 필드 이름과 값이 명확히 정의되어 있어 별도의 복잡한 파싱 과정 없이 SIEM에서 즉시 활용 가능합니다.

4. 아키텍트 심층 분석 및 최적화 (Architect's Insight)

성능 최적화 및 확장성

대규모 엔터프라이즈 환경에서 Snort의 로그 발생량이 하루 수백 기가바이트에 달할 수 있습니다. 이때 Splunk 인덱서의 부하를 줄이기 위해 Splunk Universal Forwarder (전용 데이터 송신 도구)를 Snort 센서 노드에 설치하여 압축 및 암호화 전송을 수행하는 구조를 권장합니다.

보안 권한 체계 설계

Splunk 서비스 계정이 Snort 로그 디렉터리에 접근할 때 최소 권한 원칙(Principle of Least Privilege)을 준수해야 합니다. Splunk 유저를 별도의 보안 그룹에 추가하고 로그 파일에 읽기 전용 ACL(Access Control List, 액세스 제어 목록)을 부여하여 시스템 권한 탈취를 방지하십시오.
