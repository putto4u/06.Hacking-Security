아키텍처 개요 (Architecture Overview)

현대의 엔터프라이즈 네트워크 아키텍처에서 웹 애플리케이션은 공격자들의 가장 주요한 타겟이 됩니다. OWASP(Open Web Application Security Project, 오픈 웹 애플리케이션 보안 프로젝트)에서 발표하는 Top 10 취약점은 가장 빈번하게 발생하며 치명적인 영향을 미치는 보안 위협의 표준 지표입니다.

이러한 위협을 네트워크 계층에서 선제적으로 식별하고 차단하기 위해 NIDS(Network Intrusion Detection System, 네트워크 침입 탐지 시스템)인 Snort(스노트)를 활용합니다. Snort는 패킷의 페이로드(Payload, 전송되는 실제 데이터)를 심층 분석하여 악의적인 패턴(시그니처)을 매칭해내는 강력한 룰 기반 엔진을 제공합니다.

전문 용어 해설
  • NIDS (Network Intrusion Detection System, 네트워크 침입 탐지 시스템): 네트워크 스위치의 미러 포트(Mirror Port)를 통해 흐르는 트래픽의 복사본을 수집하여, 실시간으로 해킹 시도를 탐지하고 보안 관리자에게 경고(Alert)를 보내는 보안 솔루션입니다.
  • Payload (페이로드, 전송 데이터): 네트워크 패킷에서 출발지와 목적지 주소 같은 헤더(Header) 정보를 제외한, 실제로 전송되는 핵심 데이터 본문을 의미합니다. 해커의 공격 코드는 대부분 이 페이로드 안에 은닉됩니다.

실증 및 구현 (Implementation & Rule Tuning)

탐지 룰 01 (Rule 01)

A03:2021 - 인젝션 (Injection) 방어

/etc/snort/rules/sql.rules

SQL 인젝션(SQL Injection)은 사용자의 입력값을 검증하지 않고 데이터베이스 쿼리로 직접 연결할 때 발생하는 가장 고전적이면서도 치명적인 아키텍처 결함입니다. 공격자는 논리적으로 무조건 참(True)이 되는 구문인 ' OR 1=1 -- 등을 주입하여 인증을 우회하거나 모든 데이터를 열람합니다. 아래 룰은 웹 요청(HTTP Request)에서 이러한 보편적인 우회 시도를 탐지합니다.

스노트 콘솔 (Snort Console) / sql.rules
alert tcp $EXTERNAL_NET any -> $HTTP_SERVERS $HTTP_PORTS (
    msg:"SQL Injection - Universal Bypass Attempt";
    flow:to_server,established;
    content:"' or 1=1"; nocase; http_uri;
    classtype:web-application-attack;
    sid:1000001; rev:1;
)

각 행 상세 분석 (Line-by-Line Analysis)

  • alert tcp ... 헤더 (Header) 및 방향 설정: 외부망(EXTERNAL_NET)의 임의의 출발지 포트(any)에서 내부망 보호 대상 웹 서버(HTTP_SERVERS)의 지정된 웹 포트(HTTP_PORTS)로 유입되는 전송 제어 프로토콜(TCP, Transmission Control Protocol) 트래픽에 대해 시스템 관리자에게 경고(Alert) 액션을 취하도록 지시합니다.
  • msg:"..." 경고 메시지 (Message): 탐지 엔진이 룰을 트리거했을 때, 통합 로그 모니터링 시스템(SIEM) 화면이나 로그 파일에 출력될 직관적인 알람 제목을 정의합니다. 아키텍트 관점에서는 위협의 본질을 즉시 파악할 수 있도록 구체적으로 작성해야 합니다.
  • flow:to_server... 흐름 제어 (Flow Control): TCP 3-Way Handshake가 완벽히 맺어진 세션(established)에 한해서, 클라이언트가 서버로(to_server) 보내는 방향의 패킷만을 선별적으로 검사합니다. 이는 네트워크 검사 장비의 부하(CPU Overhead)를 획기적으로 줄이고, 반쪽짜리 비정상 연결로 인한 오탐을 원천 방지하는 핵심 옵션입니다.
  • content:"..." 페이로드 심층 검사 (Payload Inspection): HTTP 요청의 주소창 및 파라미터 영역(http_uri) 내에서 대소문자를 무시하고(nocase) 악성 문자열 패턴(' or 1=1)이 존재하는지 바이트 단위로 딥 패킷 인스펙션(DPI)을 수행합니다.
  • classtype:... 메타데이터 (Metadata): 룰의 공격 유형을 '웹 애플리케이션 공격(web-application-attack)'으로 분류합니다. 또한 커스텀 룰을 의미하는 고유 식별자(sid: 1000001)와 버전 관리용 리비전 번호(rev: 1)를 부여하여 대규모 엔터프라이즈 환경에서의 시그니처 형상 관리를 용이하게 합니다.
탐지 룰 02 (Rule 02)

A03:2021 - 교차 사이트 스크립팅 (XSS)

/etc/snort/rules/web-attacks.rules

교차 사이트 스크립팅(XSS, Cross-Site Scripting)은 게시판이나 입력 폼을 통해 악성 자바스크립트를 주입하고, 다른 사용자의 브라우저에서 해당 스크립트가 실행되도록 유도하여 세션 쿠키(Session Cookie) 등을 탈취하는 공격입니다. 주로 <script> 태그의 삽입을 모니터링합니다.

스노트 콘솔 (Snort Console) / web-attacks.rules
alert tcp $EXTERNAL_NET any -> $HTTP_SERVERS $HTTP_PORTS (
    msg:"XSS - Script Tag Attempt";
    flow:to_server,established;
    content:"<script>"; nocase; http_client_body;
    classtype:web-application-attack; sid:1000002; rev:1;
)

각 행 상세 분석 (Line-by-Line Analysis)

  • content:"<script>" 탐지 문자열 (Detection String): 브라우저에서 동적인 클라이언트 사이드 언어를 실행시키는 가장 기본적인 HTML 태그 형태를 문자열 매칭의 핵심 조건으로 설정합니다. 해커는 이를 통해 세션 하이재킹(Session Hijacking)을 시도합니다.
  • http_client_body HTTP 클라이언트 본문 검사: 이 룰에서 가장 눈여겨봐야 할 옵션입니다. 일반적인 GET 방식의 URI 파라미터가 아닌, POST 방식 등으로 전송되는 폼 데이터(Form Data), JSON, XML 형태의 HTTP 요청 본문(Body) 영역 안에서만 문자열을 탐지하도록 지정하여 효율을 극대화합니다.
탐지 룰 03 (Rule 03)

A01:2021 - 취약한 접근 제어 (Broken Access Control)

/etc/snort/rules/web-misc.rules

취약한 접근 제어는 인증된 사용자라도 자신의 권한을 벗어난 파일이나 디렉터리에 접근할 수 있는 설계 결함입니다. 대표적으로 상위 디렉터리로 이동하는 ../ 문자를 연속적으로 입력하여 리눅스의 핵심 설정 파일인 /etc/passwd 등에 접근하는 디렉터리 트래버셜(Directory Traversal) 공격이 있습니다.

스노트 콘솔 (Snort Console) / web-misc.rules
alert tcp $EXTERNAL_NET any -> $HTTP_SERVERS $HTTP_PORTS (
    msg:"Exploit - Directory Traversal Attempt";
    flow:to_server,established;
    content:"../"; http_uri;
    classtype:attempted-recon; sid:1000003; rev:1;
)

각 행 상세 분석 (Line-by-Line Analysis)

  • content:"../" 경로 조작 문자열 탐지: 웹 서버의 기본 서비스 경로(Web Root)를 강제로 벗어나기 위해 운영체제(OS)에서 통용되는 상위 폴더 이동 기호인 ../ (Dot-Dot-Slash)의 유입을 면밀히 감시합니다.
  • classtype:attempted-recon 정보 수집 시도 분류: 시스템 탈취라는 본 공격에 앞서 서버의 내부 디렉터리 구조나 백업 파일 존재 여부를 파악하려는 정찰(Reconnaissance) 시도 범주로 이 이벤트를 매핑합니다. 보안 관제 조직은 이 로그를 통해 본격적인 공격의 전조 증상(Indicator)을 식별합니다.
탐지 룰 04 (Rule 04)

A05: 보안 설정 오류

/etc/snort/rules/policy.rules

관리자 페이지(Admin Panel)가 특정 IP 대역으로 제한되지 않고 외부에 노출되어 접근이 시도되는 정책 위반을 감시합니다.

스노트 콘솔 (Snort Console) / policy.rules
alert tcp $EXTERNAL_NET any -> $HTTP_SERVERS $HTTP_PORTS (
  msg:"Policy - Unauthorized Admin Access";
  content:"/admin/"; http_uri;
  classtype:policy-violation; sid:1000004;
)

각 행 상세 분석

  • content:"/admin/"; 경로 노출 검증: URI 내에 인가된 인원만 접속해야 할 관리자용 백오피스 디렉터리 접근 시도를 탐지합니다.
  • classtype:policy-violation; 정책 위반 분류: 직접적인 익스플로잇(해킹) 행위라기보다는 사내 접근 제어 리스트(ACL, Access Control List) 정책을 우회한 위반 사항으로 로그를 기록합니다.
탐지 룰 05 (Rule 05)

A04: 불안전한 설계

/etc/snort/rules/web-attacks.rules

비즈니스 로직의 허점을 노려, 희생자가 자신의 의도와 무관하게 특정 액션(송금, 정보 변경)을 실행하게 만드는 위조 요청을 탐지합니다.

스노트 콘솔 (Snort Console) / web-attacks.rules
alert tcp $EXTERNAL_NET any -> $HTTP_SERVERS $HTTP_PORTS (
  msg:"Potential CSRF Attempt";
  content:"transfer.php?amount="; http_uri;
  classtype:web-application-attack; sid:1000005;
)

각 행 상세 분석

  • content:"transfer.php?amount="; 비즈니스 로직 매개변수 탐지: 사이트 간 요청 위조(CSRF, Cross-Site Request Forgery) 공격에 자주 이용되는, 주요 자금 이체나 권한 승격 등 민감한 동작을 수행하는 매개변수(Parameter)의 비정상적인 호출을 심층 검사합니다.

아키텍트 인사이트 (Expert Insight)

페이로드 인코딩 우회와 PCRE의 필수성

위 본문에서 다룬 룰들은 교육 아키텍처 목적상 가장 원형(Raw)에 가까운 공격 패턴을 단순화한 문자열 매칭(content:"...")에 불과합니다. 실제 해킹 그룹(APT)은 침입 탐지 시스템을 우회하기 위해 페이로드를 URL 인코딩 (URL Encoding) (예: < 를 %3c 로 변환)이나 16진수 인코딩 (Hex Encoding) 기법으로 치환하여 전송합니다.

이러한 난독화된 공격을 방어하기 위해 아키텍트는 반드시 정규 표현식 엔진인 PCRE (Perl Compatible Regular Expressions, 펄 호환 정규 표현식)를 활용해야 합니다. Snort 룰 옵션에 pcre:"/(%3C|<)script(%3E|>)/i"; 와 같이 구성하여 복합적인 인코딩 변이 공격까지 하나의 룰로 우아하게 통제해야 합니다.

오탐(False Positive) 통제와 룰 튜닝

보안 룰의 민감도를 최대로 높이면 탐지율은 올라가지만, 정상적인 비즈니스 로직(예: 사용자가 블로그에 코드를 포스팅하기 위해 <script> 글자를 타이핑하는 행위)마저 공격으로 간주하여 차단하는 오탐 (False Positive, 긍정 오류)가 급증하게 됩니다.

초기 침입 탐지 시스템(IDS) 도입 시에는 바로 차단(Drop) 모드로 운영하지 않고, 경고(Alert) 모드로 최소 2~4주간 트래픽 기준선(Baseline)을 분석해야 합니다. 이후 특정 출발지 IP 대역의 예외 처리, 특정 URI 파라미터의 검사 제외 등 정밀한 룰 튜닝(Rule Tuning) 과정을 거치는 것이 성공적인 인프라 보안 구축의 핵심 역량입니다.

전문가 부록 용어
  • APT (Advanced Persistent Threat, 지능형 지속 위협): 특정 타겟(기업, 국가기관 등)을 대상으로 장기간에 걸쳐 다양한 해킹 기술을 총동원하여 은밀하게 침투하고 데이터를 유출하는 고도화된 해킹 공격을 의미합니다.
  • False Positive (오탐, 긍정 오류): 시스템이 정상적인 네트워크 활동이나 파일을 잘못 판단하여 악의적인 것으로 분류(경고 발생)하는 상황입니다. 보안 담당자의 피로도를 높이는 주된 원인입니다.
