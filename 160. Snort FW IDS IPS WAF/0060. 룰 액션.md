## Snort 탐지 정책의 핵심: 규칙 액션(Rule Actions)의 다양성

Snort 규칙의 가장 첫 번째 필드를 구성하는 **규칙 액션(Rule Actions, 규칙 동작)**은 특정 패킷이 설정된 조건(Header 및 Options)과 일치했을 때 시스템이 수행할 구체적인 행위를 정의합니다. 단순히 위협을 알리는 `alert` 외에도, 네트워크 트래픽의 흐름을 제어하거나 후속 분석을 위한 데이터를 수집하기 위해 다양한 액션이 존재합니다. 보안 아키텍처 설계 시 어떤 액션을 선택하느냐에 따라 시스템의 부하, 탐지 효율성, 그리고 실제 차단 성능이 결정됩니다.



| 액션 명칭 | 주요 기능 | 운영 모드 |
| :--- | :--- | :--- |
| **alert** | 경고를 생성하고 패킷의 상세 내용을 로그로 기록 | IDS / IPS |
| **log** | 경고 없이 패킷의 내용만 로그로 저장 | IDS / IPS |
| **pass** | 해당 패킷을 검사하지 않고 통과시킴 (무시) | IDS / IPS |
| **drop** | 패킷을 차단하고 로그를 기록 (Inline 모드 전용) | IPS (Inline) |
| **reject** | 패킷 차단 후 로그 기록 및 연결 종료 신호 송신 | IPS (Inline) |
| **sdrop** | 패킷을 차단하되 로그를 남기지 않음 (Silent Drop) | IPS (Inline) |
| **activate** | 경고 발생 후 연관된 dynamic 규칙을 활성화 | IDS / IPS |
| **dynamic** | activate 규칙에 의해 호출될 때까지 대기 | IDS / IPS |

> **Rule Actions (규칙 액션, 규칙 동작)**: 패킷이 규칙 조건에 부합할 때 Snort 엔진이 수행해야 할 명령을 지정하는 지시어입니다.
>
> **IDS (Intrusion Detection System, 침입 탐지 시스템)**: 네트워크 트래픽을 감시하여 위협을 탐지하고 알리는 수동적인 보안 시스템입니다.
>
> **IPS (Intrusion Prevention System, 침입 방지 시스템)**: 탐지된 위협을 실시간으로 차단하고 네트워크 흐름을 제어하는 능동적인 보안 시스템입니다.

## 기본 및 로깅 액션: alert, log, pass

기본적인 세 가지 액션은 주로 **IDS (Intrusion Detection System, 침입 탐지 시스템)** 환경에서 트래픽의 가시성을 확보하기 위해 사용됩니다. 

1.  **alert (알람, 경고)**: 가장 표준적인 액션으로, 지정된 탐지 방식(파일, 데이터베이스, Syslog 등)에 따라 경고 메시지를 생성하고 패킷 데이터를 기록합니다.
2.  **log (로그, 기록)**: 보안 담당자가 특정 트래픽의 패턴을 장기적으로 분석해야 할 때 사용합니다. 사용자에게 즉각적인 경보를 띄우지는 않지만, 나중에 포렌식(Forensics) 자료로 활용할 수 있도록 데이터를 조용히 저장합니다.
3.  **pass (패스, 통과)**: 특정 트래픽이 신뢰할 수 있는 대상이거나, 잦은 오탐(False Positive)으로 인해 검사 제외가 필요한 경우 사용합니다. 일종의 '화이트리스트(Whitelist)' 기능을 수행하며, 해당 규칙에 매칭된 패킷은 이후의 다른 규칙 검사를 거치지 않고 즉시 통과됩니다.

> **Syslog (System Logging Protocol, 시스템 로그 프로토콜)**: 컴퓨터 시스템에서 발생하는 다양한 이벤트 메시지를 전송하고 저장하기 위한 표준 프로토콜입니다.
>
> **Forensics (Digital Forensics, 디지털 포렌식)**: 디지털 기기에 저장된 데이터를 수집, 분석하여 법적 증거력을 확보하는 기술 및 절차를 의미합니다.
>
> **Whitelist (화이트리스트, 허용 목록)**: 보안상의 이유로 안전하다고 판단되어 접근이나 실행이 명시적으로 허용된 대상의 목록입니다.

## 인라인 차단 액션: drop, reject, sdrop

Snort가 패킷의 경로상에 직접 위치하는 **Inline Mode (인라인 모드, 직렬 모드)**로 동작할 때, 즉 **IPS (Intrusion Prevention System, 침입 방지 시스템)** 기능을 수행할 때 사용하는 강력한 제어 액션들입니다.

* **drop (드롭, 폐기)**: 패킷이 목적지에 도달하지 못하도록 폐기하고 해당 이벤트를 로그에 남깁니다. 공격자 입장에서는 패킷이 단순히 사라진 것처럼 보입니다.
* **reject (리젝트, 거부)**: 패킷을 차단함과 동시에 송신자에게 연결이 거부되었음을 명시적으로 알립니다. TCP 환경에서는 **RST (Reset, 연결 초기화)** 패킷을, UDP 환경에서는 **ICMP Port Unreachable (Internet Control Message Protocol Port Unreachable, 인터넷 제어 메시지 프로토콜 포트 도달 불가)** 메시지를 전송하여 세션을 강제로 종료시킵니다.
* **sdrop (Silent Drop, 사이런트 드롭, 조용한 폐기)**: `drop`과 동일하게 패킷을 차단하지만 로그를 전혀 남기지 않습니다. 대량의 **DDoS (Distributed Denial of Service, 분산 서비스 거부 공격)** 트래픽처럼 로그 기록 자체가 시스템 성능에 심각한 저해를 줄 때 유용하게 활용됩니다.



> **Inline Mode (인라인 모드, 직렬 모드)**: 보안 장비가 네트워크 트래픽의 경로 한가운데 설치되어 모든 패킷을 직접 통과시키며 실시간 제어를 수행하는 구성 방식입니다.
>
> **RST (Reset, 연결 초기화)**: TCP 연결에서 예기치 못한 오류가 발생하거나 강제로 연결을 종료해야 할 때 사용하는 제어 플래그입니다.
>
> **ICMP (Internet Control Message Protocol, 인터넷 제어 메시지 프로토콜)**: IP 패킷 전송 중 발생하는 오류 보고 및 네트워크 상태 진단을 위해 사용되는 프로토콜입니다.

## 지능형 탐지 액션: activate, dynamic

고급 보안 설계를 위해 제공되는 이 기능은 특정 이벤트가 발생했을 때만 추가적인 감시를 시작하는 **조건부 탐지** 체계를 구축할 때 사용합니다.

* **activate (액티베이트, 활성화)**: 일종의 트리거(Trigger) 역할을 합니다. 특정 패킷을 탐지하여 알람을 발생시킨 후, 이와 연결된 `dynamic` 규칙들을 즉시 활성화합니다.
* **dynamic (다이내믹, 동적)**: 평상시에는 비활성화 상태로 시스템 자원을 소모하지 않다가, 연관된 `activate` 규칙이 동작하는 순간부터 패킷 검사를 시작합니다. 주로 공격의 전조 증상이 발견된 후, 공격자의 후속 행위를 정밀하게 추적(Tracing)하기 위해 사용됩니다.

이러한 동적 규칙 구성은 수천 개의 규칙을 상시 구동해야 하는 대규모 IDC 환경에서 CPU 효율성을 극대화하고, 불필요한 로그 생성을 억제하는 전략적 아키텍처 구성 요소로 평가받습니다.

> **Trigger (트리거, 방사기)**: 특정 조건이 만족되었을 때 자동으로 연쇄적인 반응이나 동작을 일으키는 장치나 로직을 의미합니다.
>
> **IDC (Internet Data Center, 인터넷 데이터 센터)**: 기업의 서버, 네트워크 장비 등을 집중적으로 관리하고 운영하는 전문 시설입니다.
>
> **Tracing (트레이싱, 추적)**: 데이터의 흐름이나 시스템의 동작 과정을 순차적으로 기록하여 분석하는 행위입니다.
