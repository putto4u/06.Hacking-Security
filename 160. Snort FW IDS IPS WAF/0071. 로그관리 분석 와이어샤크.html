Snort와 Wireshark를 활용한 지능형 침입 탐지 및 패킷 포렌식

CORE: SECURITY 2.0

Senior Infrastructure Architect

현대적인 사이버 보안 환경에서 네트워크 침입 탐지 시스템인 Snort(Open Source Network Intrusion Detection System, 오픈 소스 네트워크 침입 탐지 시스템)와 패킷 분석 도구인 Wireshark(Network Protocol Analyzer, 네트워크 프로토콜 분석기)는 보안 전문가가 갖춰야 할 필수 무기입니다. 네트워크 상에서 발생하는 보이지 않는 위협을 가시화하고, 공격의 근원을 추적하기 위해서는 실시간 데이터 저장 기법부터 심층 패킷 분석까지의 유기적인 연동 과정이 필수적입니다.

본 가이드는 Snort가 탐지한 보안 이벤트를 효율적으로 관리하고, 이를 Wireshark를 통해 상세히 해부하여 실전적인 방어 전략을 수립하는 과정을 상세히 다룹니다.

1. Snort 로그 및 실시간 데이터 저장 아키텍처 (Storage Architecture)

Snort는 탐지된 이벤트를 다양한 방식으로 기록합니다. 대규모 네트워크 환경에서는 탐지 성능 저하를 방지하기 위해 로그 생성(Detection Engine)과 저장(Output Logging)을 분리하는 비대칭형 아키텍처를 권장합니다. 특히 고속 네트워크 환경에서는 탐지 엔진이 DB에 직접 쓰는 행위 자체가 병목 현상을 유발할 수 있으므로, 중간 저장소(Spooler)를 두는 방식이 핵심입니다.

① 로그 저장 방식의 분류

Packet Logging Mode

패킷 전체를 PCAP(Packet Capture, 패킷 캡처) 형식으로 저장합니다. 분석 데이터가 방대해지지만 사후 포렌식 및 증거 확보에 매우 유리합니다.

Alert Mode

설정된 규칙에 부합하는 이벤트 발생 시 요약된 경고 메시지만 기록합니다. Syslog(System Logging Protocol, 시스템 로그 프로토콜)나 일반 텍스트 파일로 저장됩니다.

Unified2 (유니파이드2)

Snort 전용 이진 로그 형식입니다. Snort의 성능을 극대화하기 위해 직접 DB에 쓰지 않고 이 형식으로 먼저 저장한 뒤 별도의 도구로 처리합니다.

② 실시간 데이터 저장 및 시각화 (Barnyard2 & ELK Stack)

과거에는 Barnyard2(Spooler for Snort Unified2, Snort 전용 로그 스풀러)를 사용하여 MySQL(My Structured Query Language, 오픈 소스 관계형 데이터베이스)에 저장하는 방식이 표준이었습니다. 하지만 현대의 대용량 트래픽 환경에서는 검색 속도와 시각화 유연성을 위해 ELK Stack(Elasticsearch, Logstash, Kibana, 엘라스틱서치/로그스태시/키바나 통합 로그 관리 솔루션)을 주로 활용합니다. 이를 통해 보안 이벤트의 지리적 위치(Geo-IP), 시간대별 공격 추이 등을 대시보드로 즉시 확인할 수 있습니다.

• IDS (Intrusion Detection System, 침입 탐지 시스템): 네트워크나 시스템에 대한 비정상적인 행위를 실시간으로 탐지하여 관리자에게 알리는 보안 솔루션입니다.

• SIEM (Security Information and Event Management, 보안 정보 및 이벤트 관리): 다양한 보안 장비의 로그를 통합 관리하고 상관관계를 분석하여 위협을 탐지하는 시스템입니다.

2. Wireshark 설치 및 환경 구성 (Protocol Analyzer Setup)

Wireshark(와이어샤크)는 실시간 네트워크 트래픽을 가로채고 분석할 수 있는 세계에서 가장 널리 쓰이는 네트워크 분석 도구입니다. Snort가 경고(Alert)를 발생시키면, 분석가는 Wireshark를 이용해 실제 해당 패킷의 페이로드에 어떤 공격 코드가 담겨 있었는지를 세밀하게 검증하게 됩니다.

Windows 환경

공식 홈페이지에서 설치 파일을 내려받아 실행합니다. 설치 과정 중 Npcap(Network Packet Capture Driver, 윈도우용 패킷 캡처 드라이버) 설치가 반드시 포함되어야 인터페이스 수집이 가능합니다.

# Check Npcap Service Status
sc query npcap

Linux (Ubuntu) 환경

APT 패키지 관리자를 통해 설치하며, 설치 과정에서 'Should non-superusers be able to capture packets?' 질문에 'Yes'를 선택해야 일반 계정으로 캡처가 가능합니다.

sudo apt update && sudo apt install wireshark -y
sudo usermod -aG wireshark $USER

실행 및 인터페이스 확인

  • 01. 설치 후 Wireshark를 실행하면 현재 시스템에서 활성화된 NIC(Network Interface Card, 네트워크 인터페이스 카드) 목록이 실시간 그래프와 함께 나타납니다.
  • 02. 실제 트래픽이 발생하는 인터페이스를 더블 클릭하여 캡처를 시작합니다. (예: eth0, ens33, Wi-Fi)

• Promiscuous Mode (무차별 모드): 자신에게 오는 패킷뿐만 아니라 해당 네트워크 세그먼트를 통과하는 모든 패킷을 받아들이도록 설정하는 랜카드 모드입니다.

• Pcap (Packet Capture, 패킷 캡처): 네트워크 트래픽을 캡처하고 파일로 저장하기 위한 API(Application Programming Interface)입니다.

3. Wireshark를 이용한 패킷 분석 방법론 (Analysis Methodology)

캡처된 데이터는 초당 수천 개 이상의 패킷이 쌓이므로 방대한 정보 속에서 공격 징후를 골라내는 능력이 핵심입니다. Wireshark는 이를 위해 강력한 필터링 및 추적 기능을 제공합니다.

[도식: 효율적인 패킷 분석 단계]

① 디스플레이 필터 (Display Filters)

ip.addr == 192.168.1.10 특정 IP 패킷 전체 조회
tcp.port == 80 HTTP 통신 필터링
http.request.method == "POST" 웹 서버 전송 데이터(POST) 확인

② TCP Stream 추적 (Follow TCP Stream)

네트워크상에서 조각나서 전달되는 개별 패킷들을 하나의 완성된 대화 형태로 재구성하는 기능입니다. 분석가는 패킷 리스트에서 마우스 오른쪽 버튼 클릭 -> Follow -> TCP Stream을 선택하여 공격자가 전송한 명령어나 서버의 응답 코드를 평문(Plaintext) 상태로 한눈에 파악할 수 있습니다.

③ 통계 기능을 활용한 이상 징후 파악

  • Statistics -> Conversations: 어떤 호스트끼리 가장 많은 데이터를 주고받는지 확인하여 C&C(Command and Control Server, 명령 제어 서버) 통신 여부를 의심할 수 있는 정황을 포착합니다.
  • Statistics -> Protocol Hierarchy: 현재 캡처된 전체 트래픽 중 특정 프로토콜(예: ICMP, DNS)의 비중이 비정상적으로 높다면 이는 공격의 강력한 징후가 됩니다.

4. 공격 패턴별 패킷 특징 분석 (Signature Analysis)

보안 전문가는 특정 공격이 발생했을 때 나타나는 패킷의 고유한 '시그니처(Signature)'를 즉각적으로 식별할 수 있어야 합니다.

포트 스캐닝 (Port Scanning)

공격자가 TCP SYN Scan을 수행할 경우, 짧은 시간 내에 다수의 목적지 포트로 향하는 SYN(연결 요청) 패킷이 관찰됩니다.

Pattern: SYN -> (Wait for SYN/ACK) -> RST (Connection Aborted)

ICMP Flooding (DoS)

서버에 대량의 ICMP Echo Request를 보내 시스템 자원을 고갈시키는 DoS(Denial of Service, 서비스 거부 공격)의 전형입니다.

Pattern: Thousands of ICMP Type 8 packets per second

SQL Injection (SQL 인젝션)

HTTP 요청 패킷의 URI 또는 Body 영역에서 데이터베이스 탈취용 구문이 탐지됩니다.

Payload: SELECT, UNION, ' OR '1'='1'

Brute Force (무차별 대입 공격)

특정 계정에 대해 매우 짧은 시간 동안 수많은 로그인 패킷이 발생하며, 대부분 실패 응답이 뒤따릅니다.

Pattern: Continuous POST /login -> 401 Unauthorized

• 3-Way Handshake (3단계 핸드셰이크): TCP 통신을 시작하기 전, 두 호스트 간에 신뢰성 있는 연결을 설정하기 위해 SYN, SYN/ACK, ACK 패킷을 주고받는 과정입니다.

• Payload (페이로드): 전송되는 패킷의 실제 데이터 부분을 의미하며, 공격 코드가 주로 숨겨지는 곳입니다.

5. 아키텍트 인사이트 (Expert Insight & Tips)

Time Display Format

패킷 분석 시 '도착 시간'을 UTC(Coordinated Universal Time, 협정 세계시)나 이전 패킷과의 시간 차이로 설정하여 공격의 지속성 및 정밀도를 파악하십시오.

Coloring Rules

Wireshark의 색상 규칙을 최적화하여 RST/ACK 충돌이나 오류 패킷(빨간색/검은색)을 시각적으로 빠르게 구분하여 분석 속도를 높이십시오.

AWS Billing Warning

AWS 환경에서 VPC Traffic Mirroring을 사용하여 실시간 패킷 캡처를 수행할 경우, 인스턴스당 시간별 사용료 및 데이터 전송량에 따른 과금이 즉시 발생하므로 실습 종료 후 반드시 설정을 해제하십시오.
