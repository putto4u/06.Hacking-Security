VMware Workstation Pro 다운로드 및 최적화 가이드

본 문서는 IT 인프라 환경 구축을 위한 가장 기본적인 단계인 VM (Virtual Machine, 가상 머신) 하이퍼바이저 소프트웨어의 설치 및 구성 방법을 다룹니다. 성공적인 실습 환경 구축을 위해 스텝바이스텝으로 정확하게 따라오시기 바랍니다.

* VM (Virtual Machine, 가상 머신): 물리적인 컴퓨터 하드웨어를 소프트웨어적으로 모방하여 만든 독립적인 가상 컴퓨터 환경입니다. 하나의 물리적인 호스트 PC 위에서 여러 개의 독립된 OS (Operating System, 운영 체제)를 동시에 실행할 수 있게 해주는 핵심 기술입니다.
[그림 1] Type 2 하이퍼바이저 (VMware Workstation) 아키텍처 구조도
호스트 OS 위에서 소프트웨어 형태로 동작하며 여러 개의 게스트 OS(손님 운영체제)를 관리합니다.

[중요 라이선스 및 과금 정책 변경 안내]

VMware가 Broadcom(브로드컴)에 인수된 이후 라이선스 정책이 대폭 변경되었습니다. 과거 상용 소프트웨어였던 VMware Workstation Pro 제품은 현재 개인 사용자(Personal Use) 및 학습 목적에 한해 전면 무료로 제공됩니다.
단, 기업 환경이나 영리 목적으로 사용할 경우 Broadcom의 구독형 유료 라이선스가 반드시 필요합니다. 실습용으로는 무료 버전을 안심하고 사용하셔도 됩니다.

최신 VMware Workstation Pro 다운로드 방법

과거의 VMware 공식 홈페이지 다운로드 링크는 더 이상 작동하지 않습니다. 모든 소프트웨어 다운로드는 VCF (VMware Cloud Foundation, 클라우드 인프라 통합 플랫폼) 통합 포털인 Broadcom Support Portal을 통해 진행해야 합니다.

* VCF (VMware Cloud Foundation, 클라우드 인프라 통합 플랫폼): 서버, 스토리지, 네트워크 가상화를 하나로 통합하여 프라이빗 클라우드나 하이브리드 클라우드를 쉽게 구축하고 관리할 수 있도록 해주는 브로드컴의 핵심 소프트웨어 제품군입니다.
  1. Broadcom 포털 가입 및 로그인: Broadcom Support Portal에 접속하여 우측 상단의 Register를 눌러 개인 이메일로 회원가입을 진행한 후 로그인합니다.
  2. 소프트웨어 메뉴 진입: 로그인 후 상단 메뉴바 좌측에서 Software를 선택하고, 드롭다운 메뉴에서 VMware Cloud Foundation을 클릭합니다.
  3. 다운로드 센터 이동: 화면 우측 상단의 My Downloads 메뉴를 클릭합니다.
  4. 제품 검색: 우측 상단의 검색창에 VMware Workstation Pro를 입력하여 검색합니다.
  5. 개인용 버전 선택: 검색 결과 목록에서 반드시 VMware Workstation Pro 17 for Personal Use (또는 최신 버전의 Personal Use)를 클릭합니다.
  6. 약관 동의 및 검증: 다운로드 테이블 좌측 상단의 I agree to the terms and conditions (이용 약관 동의) 체크박스를 선택합니다. 이때 영문 이름과 주소를 입력하는 검증(Verification) 팝업이 뜰 수 있으며, 정확히 입력해야 승인됩니다.
  7. 설치 파일 다운로드: 검증이 완료되면 우측의 구름 모양 아이콘(Download)이 활성화됩니다. 클릭하여 .exe (Windows용) 설치 파일을 다운로드합니다.

[자주 오해하거나 실수하는 부분]

Broadcom 포털 가입 후 주소 및 개인정보를 대충 기입(예: 1234, asdf 등)할 경우, 미국의 수출 통제 및 보안 정책 시스템에 의해 계정이 즉각 차단되거나 다운로드 권한 검증(Verification)에서 영구 실패할 수 있습니다. 반드시 실제 거주하는 도시와 영문 주소 형식을 맞추어 입력해야 합니다.

VMware Workstation Pro 설치 과정

다운로드한 설치 파일을 실행하여 호스트 OS (Operating System, 운영 체제)에 하이퍼바이저를 설치합니다.

  1. 관리자 권한 실행: 다운로드 폴더에서 VMware-workstation-full-xxx.exe 파일을 우클릭하고 관리자 권한으로 실행을 클릭합니다.
  2. 설치 유형 선택: Typical(일반)과 Custom(사용자 지정) 중 Typical을 선택합니다.
  3. Enhanced Keyboard Driver 설치 (선택): 설치 도중 Enhanced Virtual Keyboard Driver 체크 박스가 나타납니다. 호스트와 가상 머신 간의 다국어 키보드 충돌을 방지하므로 체크하는 것을 권장합니다. (체크 시 설치 완료 후 PC 재부팅이 필요합니다.)
  4. 라이선스 입력 단계: 설치 마지막 화면에 License Key를 묻는 창이 나옵니다. 하단의 Use VMware Workstation 17 for Personal Use (개인용으로 사용) 라디오 버튼을 선택하고 Continue를 누릅니다.
  5. 설치가 완료되면 Finish를 누르고 PC를 재부팅합니다.
* OS (Operating System, 운영 체제): 컴퓨터의 하드웨어와 소프트웨어 자원을 관리하고, 사용자가 컴퓨터와 통신할 수 있도록 도와주는 가장 기본적이고 핵심적인 시스템 소프트웨어입니다. (예: Windows, Linux, macOS)

핵심 네트워크(Network) 설정 이해하기

가상 머신을 생성하기 전, IT 인프라 아키텍트로서 반드시 정확히 이해해야 하는 것이 가상 네트워크 토폴로지입니다. VMware는 GUI (Graphical User Interface, 그래픽 사용자 인터페이스) 기반의 Virtual Network Editor를 제공합니다.

* GUI (Graphical User Interface, 그래픽 사용자 인터페이스): 사용자가 텍스트 명령어(CLI)를 타이핑하는 대신, 마우스와 같은 포인팅 장치를 이용하여 화면의 아이콘이나 메뉴 등의 시각적인 요소를 클릭하여 컴퓨터를 제어하는 방식입니다.
[그림 2] 가상 머신 네트워크 토폴로지 (Network Topology)
NAT(공유기 역할), Bridged(직접 연결), Host-Only(폐쇄망) 모드의 트래픽 흐름과 가상 스위치(VMnet) 구조입니다.
네트워크 모드 설명 및 특징 실무 활용 사례
NAT (VMnet8) 호스트 PC의 IP 주소를 가상 머신이 공유하여 외부(인터넷)와 통신합니다. 가상 머신은 자체적인 사설 IP를 할당받습니다. 인터넷 연결이 필요하면서도, 외부에서 가상 머신으로 직접 접근하는 것을 막고 싶을 때 (가장 보편적인 기본 설정)
Bridged (VMnet0) 가상 머신이 호스트 PC와 동일한 물리적 네트워크(공유기 등)에 직접 연결됩니다. 호스트 PC와 동등한 네트워크 대역의 IP를 할당받습니다. 가상 머신을 실제 물리 서버처럼 네트워크상에 노출하여 다른 팀원이나 장비와 통신해야 할 때
Host-Only (VMnet1) 외부 인터넷과 완전히 단절되며, 오직 호스트 PC와 가상 머신들 사이에서만 통신이 가능한 격리된 네트워크입니다. 보안이 매우 중요한 폐쇄망 테스트, 악성코드 분석, 독립적인 내부 DB 클러스터 구성 시
* NAT (Network Address Translation, 네트워크 주소 변환): 하나의 공인 IP 주소를 여러 대의 내부 장치(가상 머신 등)가 공유하여 사용할 수 있게 해주는 기술입니다. IP 주소 부족 문제를 해결하고 내부망을 외부로부터 숨기는 보안 역할도 수행합니다.
sudo apt update # 패키지 리스트 업데이트
sudo apt install open-vm-tools-desktop -y # 클립보드 공유/데스크톱 환경용 VMware 도구 설치
sudo reboot # 시스템 재부팅 (설정 적용)

[전문가의 실전 팁: DHCP 충돌 방지]

실습 과정에서 고정 IP(Static IP)를 할당하여 서버 인프라를 구축할 경우, VMware의 내장 DHCP (Dynamic Host Configuration Protocol, 동적 호스트 설정 프로토콜) 서비스가 자동으로 IP를 할당하면서 IP 충돌이 발생할 수 있습니다.
서버 구축 실습 시에는 상단 메뉴의 Edit > Virtual Network Editor에 들어가서 NAT 네트워크의 Use local DHCP service to distribute IP address to VMs 체크를 해제하고 수동으로 IP를 관리하는 것이 아키텍트의 올바른 접근법입니다.

* DHCP (Dynamic Host Configuration Protocol, 동적 호스트 설정 프로토콜): 네트워크에 접속하는 컴퓨터나 스마트폰 등의 장치들에게 IP 주소, 서브넷 마스크, 게이트웨이 등의 네트워크 통신 설정 정보를 자동으로 할당해 주는 프로토콜입니다.
