🔒 네트워크 1계층/2계층 보안 관련 명령어

시스템 관리자가 네트워크의 가장 기본적인 계층인 1계층(물리 계층)과 2계층(데이터 링크 계층)에서 보안 문제를 점검하고 대응하기 위해 알아야 할 핵심 명령어들을 정리했습니다.

이 계층의 보안은 주로 ARP Spoofing 방지, MAC 주소 관리, 그리고 물리적 접근 통제에 중점을 둡니다.

1. ARP 및 MAC 주소 관리 (2계층 보안 핵심)

ARP 캐시는 2계층 MAC 주소와 3계층 IP 주소를 연결하는 핵심이며, 스푸핑 공격의 주 대상입니다.

기능 명령어 (OS별) 사용 목적 및 보안 중요성
ARP 캐시 확인 $ arp -a 현재 시스템에 저장된 IP-MAC 매핑을 확인하여 ARP 스푸핑 징후(비정상적인 MAC 주소 매핑)를 탐지합니다.
정적 ARP 추가 $ arp -s [IP] [MAC] 게이트웨이와 같이 중요한 장치의 ARP 정보를 고정시켜 공격자가 임의로 MAC 주소를 위변조하는 것을 원천적으로 차단합니다.
ARP 캐시 삭제 $ arp -d 또는 $ arp -d * (Windows) ARP 스푸핑 공격이 의심되거나 발생했을 때, 오염된 캐시 항목을 즉시 초기화합니다.
인터페이스 상태 확인 $ ip addr (Linux) / $ ifconfig (Legacy Linux/macOS) / $ ipconfig /all (Windows) 시스템에 연결된 모든 네트워크 인터페이스의 MAC 주소, IP 주소, 상태(up/down) 등을 확인하여 설정 오류나 예상치 못한 인터페이스 활성화 여부를 점검합니다.

2. 스위치 및 포트 보안 (2계층 장비 관리)

대부분의 2계층 보안 조치는 스위치(Switch)에서 이루어집니다. Cisco IOS 기반 스위치를 예시로 듭니다.

기능 명령어 (Cisco IOS 예시) 사용 목적 및 보안 중요성
포트 보안 활성화 (config-if)# switchport port-security 특정 포트에서 허용되는 MAC 주소의 개수를 제한하고, 허용되지 않은 MAC 주소가 접근할 때 포트를 차단(Shutdown)하도록 설정합니다.
허용 MAC 개수 설정 (config-if)# switchport port-security maximum 1 한 포트당 하나의 MAC 주소만 허용하도록 설정하여, 비인가 장치 연결이나 사설 스위치/허브 연결을 방지합니다.
MAC 주소 Sticky 설정 (config-if)# switchport port-security mac-address sticky 포트에 처음 연결된 장치의 MAC 주소를 자동으로 학습하고 이 주소를 정적 항목처럼 유지시켜, 다른 MAC 주소의 접근을 차단합니다.
포트 보안 위반 조치 설정 (config-if)# switchport port-security violation shutdown 보안 위반(Maximum MAC 초과 등) 발생 시 해당 포트를 즉시 차단(Shutdown)하여 공격을 중단시킵니다.
포트 상태 및 보안 확인 # show port-security interface [인터페이스 번호] 포트 보안 설정, 현재 학습된 MAC 주소, 위반 횟수 등을 확인하여 보안 상태를 점검합니다.

3. 물리 계층(1계층) 및 연결 상태 확인

1계층은 물리적인 연결 자체를 다루며, 장치 자체의 상태 확인이 중요합니다.

기능 명령어 사용 목적 및 보안 중요성
링크 상태 확인 $ ethtool [인터페이스 이름] (Linux) 네트워크 인터페이스의 물리적 연결 상태(Link Status), 속도(Speed), 듀플렉스 모드(Duplex) 등을 확인하여 물리적 문제나 비정상적인 연결을 확인합니다.
핑 테스트 $ ping [IP 주소] 대상 장치까지의 물리적 연결 상태와 네트워크 활성 여부를 가장 간단하게 확인합니다. 연결 실패 시 1계층 케이블 문제나 2계층 스위치 포트 문제일 수 있습니다.
로그 확인 $ tail -f /var/log/syslog 또는 $ dmesg (Linux) 인터페이스의 연결/단절(Link Up/Down) 이벤트 기록, 비정상적인 전송 오류 등을 확인하여 물리적 변동 사항을 추적합니다.

ARP 스푸핑과 스위치 포트 보안은 2계층에서 발생하는 가장 흔하면서도 치명적인 위협이므로, arp -s를 통한 정적 매핑 관리와 스위치의 port-security 설정은 시스템 관리자에게 필수적인 작업입니다.
