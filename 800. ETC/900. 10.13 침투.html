1. 침투 테스트 아키텍처 (Pentest Architecture)

본 프로젝트의 공격 대상인 192.168.100.3 (VM3)는 다양한 인프라 서비스(FTP, SMB, NFS, SSH)가 활성화되어 있는 리눅스 시스템입니다. 우리는 공격자(Attacker) 관점에서 네트워크를 스캐닝하고, 발견된 취약한 설정을 통해 데이터 탈취 및 시스템 제어권을 획득하는 과정을 실습합니다.

2. Nmap 스캔 데이터 해석 (Port Analysis)

제공된 Nmap (Network Mapper) 스캔 결과는 타겟 시스템의 공격 표면(Attack Surface)을 명확히 보여줍니다. 각 포트의 상태와 배너(Banner) 정보를 바탕으로 서비스의 버전 정보를 확인하여 취약점 데이터베이스와 매칭하는 과정이 필수적입니다.

PORT 21
vsftpd 3.0.5

Anonymous Login Allowed (익명 로그인 허용)

PORT 445
Samba smbd 4

Shared Drive Enumeration (공유 드라이브 열거 가능)

PORT 2049
NFS (Network File System)

Remote Mounting (원격 마운팅 노출)

Nmap Report Summary
Target IP: 192.168.100.3
Open Ports: 21, 22, 111, 139, 445, 2049
OS: Linux 5.X (MikroTik/Ubuntu Based)

3. 취약점 진단 로드맵 (Exploitation Roadmap)

단계별 침투 전략 (Step-by-Step Strategy)

1
정보 수집 및 열거 (Enumeration)

FTP 익명 로그인, Samba 공유 폴더 리스트 확인, NFS 마운트 포인트 조회를 통한 민감 정보 획득

2
취약점 분석 (Vulnerability Research)

vsftpd 3.0.5 및 OpenSSH 9.6p1 버전에 대한 기공개 취약점(CVE) 및 익스플로잇 코드 검색

3
침투 및 권한 획득 (Gaining Access)

수집된 자격 증명(Credentials) 혹은 취약점을 이용해 SSH 쉘 획득 또는 리버스 쉘 생성

4. FTP 취약점 실증 (FTP Exploitation)

STEP 01

익명 접속 (Anonymous Login)

Nmap 결과에서 ftp-anon이 활성화된 것을 확인했습니다. 이를 통해 사용자명 없이 서버에 접속하여 파일을 탐색합니다.

Kali Linux Terminal
$ ftp 192.168.100.3
Connected to 192.168.100.3.
Name (192.168.100.3:kali): anonymous
331 Please specify the password.
Password: (공백 엔터 혹은 아무 이메일)
230 Login successful.
ftp> ls -R

전문가 분석 (Analyst Tip)

FTP 서버에서 익명 접근을 허용하면 서버 내의 중요한 설정 파일, 소스 코드, 혹은 백업 데이터가 외부로 유출될 위험이 매우 큽니다. 발견 시 즉시 비활성화해야 합니다.

5. Samba 서비스 분석 (SMB Enumeration)

Samba 서비스는 파일 공유를 위해 사용되며, 잘못된 설정 시 권한 없는 사용자가 공유된 폴더의 목록을 보거나 접근할 수 있습니다.

STEP 02

공유 목록 확인 (Listing Shares)

$ smbclient -L //192.168.100.3 -N
Sharename Type Comment
--------- ---- -------
print$ Disk Printer Drivers
Backup Disk Critical Backup Files
IPC$ IPC IPC Service (VM3 server)

* 주석: -N 옵션은 패스워드 없이(Null Session) 접속을 시도하는 옵션입니다.

6. NFS 원격 마운트 실습 (NFS Hijacking)

NFS (Network File System)는 네트워크를 통해 파일 시스템을 마운트하게 해줍니다. 포트 2049가 열려있을 때, 어떤 디렉토리가 외부에 노출되어 있는지 확인합니다.

STEP 03

마운트 포인트 조회 (Showmount)

$ showmount -e 192.168.100.3
Export list for 192.168.100.3:
/home/user/public (everyone)
/mnt/data 192.168.100.0/24

공격자 마운트 시도 (Mounting)

$ mkdir /tmp/target_nfs
$ mount -t nfs 192.168.100.3:/home/user/public /tmp/target_nfs

7. RPCBind 및 포트맵 분석 (RPC Mapping)

RPCBind (포트 111)는 어떤 RPC 프로그램이 어떤 포트에서 대기 중인지 정보를 제공합니다. 이를 통해 추가적인 숨겨진 서비스나 버전 정보를 식별할 수 있습니다.

$ rpcinfo -p 192.168.100.3
program vers proto port service
100000 4 tcp 111 portmapper
100003 3 tcp 2049 nfs
100005 3 tcp 39655 mountd

보안 위협 (Security Risk)

RPCBind를 통해 내부 서비스의 정보를 외부에서 과도하게 수집할 수 있는 정보 노출 취약점(Information Disclosure)이 존재합니다. 방화벽에서 외부 차단이 권고됩니다.

8. 오프라인 취약점 DB 검색 (Searchsploit)

Nmap에서 식별된 버전 정보를 기반으로 칼리 리눅스 내장 도구인 Searchsploit을 사용하여 가용 공격 코드를 검색합니다.

$ searchsploit vsftpd 3.0.5
----------------------------------------- ---------------------------------
Exploit Title | Path
----------------------------------------- ---------------------------------
vsftpd 3.0.x - Denial of Service | linux/remote/49740.py
----------------------------------------- ---------------------------------

* 결과 분석: vsftpd 3.0.5 버전은 원격 코드 실행(RCE) 보다는 서비스 거부(DoS) 공격에 취약함이 확인됩니다. 즉, 서비스 구성상의 허점(익명 로그인 등)을 노리는 것이 더 효과적일 수 있습니다.

9. SSH 자격 증명 공격 (Brute Force)

익명 로그인을 통해 획득한 파일에서 사용자 이름 정보를 유추했을 경우, Hydra와 같은 도구를 이용해 패스워드 크래킹을 시도할 수 있습니다.

Hydra Console
$ hydra -l user -P /usr/share/wordlists/rockyou.txt ssh://192.168.100.3
Hydra v9.5 (c) 2023 by van Hauser/THC
[22][ssh] host: 192.168.100.3 login: user password: password123

대응 방안 (Mitigation)

SSH 무차별 대입 공격을 방어하기 위해서는 Fail2Ban 설치, 강력한 패스워드 정책, 혹은 비밀번호 대신 SSH 키 기반 인증을 사용하는 것이 필수입니다.

단일 취약점으로는 침투가 어려울 수 있으나, 여러 취약점을 결합하면 시스템 장악이 가능합니다.

11. 권한 상승 진단 (Privilege Escalation)

시스템에 접속한 후, 일반 사용자(User) 권한에서 관리자(Root) 권한으로 상승할 수 있는 요소가 있는지 진단합니다.

$ sudo -l
Matching Defaults entries for user on VM3:
env_reset, mail_badpass
User user may run the following commands on VM3:
(ALL : ALL) NOPASSWD: /usr/bin/python3

* 위 결과는 python3 명령을 패스워드 없이 sudo(관리자 권한)로 실행 가능함을 나타냅니다. 이는 즉각적인 루트 권한 획득이 가능함을 의미합니다.

12. 민감 데이터 유출 (Data Exfiltration)

침투 성공 후, 가장 중요한 목표는 데이터의 유출입니다. /etc/shadow 파일을 탈취하여 시스템 내 다른 사용자들의 해시 정보를 획득합니다.

$ sudo cat /etc/shadow | grep -E "root|user"
root:$6$Xp2...:19803:0:99999:7:::
user:$6$zB9...:19803:0:99999:7:::

13. 안티포렌식 및 지속성 (Post-Exploitation)

실제 환경에서는 침투 성공 후 흔적을 지우거나 차후 재접속을 위한 백도어를 심는 행위가 이루어집니다.

  • 로그 파일 조작 (Log Manipulation):

    /var/log/auth.log 등의 파일에서 공격자 IP와 관련된 기록을 삭제합니다.

  • 지속성 유지 (Persistence):

    공격자의 SSH Public Key를 ~/.ssh/authorized_keys에 등록하여 패스워드 없이 재접속을 허용합니다.

14. 인프라 보안 강화 대책 (Security Hardening)

단기 조치 (Quick Fix)

  • - FTP 익명 로그인 비활성화
  • - NFS 마운트 포인트에 대한 IP 접근 제어
  • - Samba Guest 접근 차단

장기 전략 (Strategic Hardening)

  • - 정기적인 취약점 스캔 (Vulnerability Scanning)
  • - 최소 권한의 법칙(Principle of Least Privilege) 적용
  • - EDR(Endpoint Detection and Response) 도입

15. 최종 결과 요약 (Executive Summary)

VM3 취약점 진단 프로젝트 완료
