Cyber Security Lab SYS-SEC-01 | VULNERABILITY ANALYSIS

BoF (Buffer Overflow)
버퍼 오버플로우: 제어 흐름 가로채기

SCORE: 9.8 / CRITICAL

System Architect Pro

아키텍처 개요 (Architecture Overview)

프로그램이 데이터를 메모리에 저장할 때, 사전에 할당된 메모리 공간인 버퍼 (Buffer, 임시 데이터 저장 공간)의 크기를 초과하여 데이터를 기록하려는 현상입니다. 입력값에 대한 경계 검사가 제대로 이루어지지 않을 때 발생하며, 넘쳐흐른 데이터가 인접한 메모리 주소를 덮어쓰게 됩니다. 이는 단순한 프로그램 충돌(Crash, 비정상 종료)을 넘어, 공격자가 시스템의 제어권을 탈취하고 악의적인 코드를 실행하게 만드는 가장 전통적이고 치명적인 시스템 해킹 취약점 중 하나입니다.

메모리 구조와 공격 원리 (Memory Structure & Principle)

STACK MEMORY

스택(Stack)의 기본 구조

  • 지역 변수 (Local Variables): 함수 내에서 선언된 임시 데이터가 쌓이는 공간입니다. 공격자는 주로 이 공간의 버퍼를 초과시킵니다.
  • EBP (Extended Base Pointer, 베이스 포인터): 스택의 기준점 역할을 하는 레지스터로, 함수 호출 전의 스택 상태를 복원하는 데 사용됩니다.
  • RET (Return Address, 반환 주소): 함수가 종료된 후 원래 돌아가야 할 실행 주소입니다. 공격의 핵심 타겟입니다.
ATTACK VECTOR

제어 흐름 가로채기 (Hijacking)

  • 페이로드(Payload) 주입: 지정된 버퍼 크기를 고의로 초과하는 악성 데이터를 밀어 넣습니다.
  • RET 변조: 넘쳐흐른 데이터가 EBP를 지나 RET(반환 주소) 값을 덮어씁니다.
  • 셸코드(Shellcode) 실행: 프로그램의 실행 흐름을 공격자가 미리 삽입해 둔 악성 코드 위치로 강제로 변경하여 시스템 권한을 장악합니다.

취약한 함수와 시큐어 코딩 (Vulnerable Functions & Secure Coding)

CODE

입력값 검증 부재와 안전한 대체 함수 적용

C/C++ 언어에서 데이터 입력 길이의 경계를 검증하지 않는 표준 라이브러리 함수(strcpy(), gets() 등)는 보안상 매우 취약합니다. 실무에서는 반드시 복사할 데이터의 최대 길이를 명시적으로 제한하는 안전한 함수(strncpy(), fgets() 등)로 대체하여 시큐어 코딩 (Secure Coding, 안전한 소프트웨어 개발 구현)을 적용해야 합니다.

Vulnerable Code
// 취약점: 버퍼 크기(10)를 확인하지 않음
char buffer[10];
strcpy(buffer, user_input);
Secure Code
// 보완: 복사할 최대 길이를 명시함
char buffer[10];
strncpy(buffer, user_input, sizeof(buffer)-1);

아키텍트 인사이트 (Expert Insight)

실전 팁: 레거시 코드 진단

레거시 (Legacy, 과거에 개발되어 현재까지 사용 중인 시스템) 코드를 클라우드로 마이그레이션하거나 보안 진단을 수행할 때, 가장 먼저 정적 분석 자동화 도구(SAST)를 활용하여 소스 코드 내에 취약한 문자열 처리 함수가 사용되었는지 전수 조사해야 합니다. 방치된 레거시 코드는 침해 사고의 가장 흔한 진입점입니다.

자주 오해하는 부분: OS 보호 기법 맹신 금지

"최신 운영체제는 자체 방어 기능이 뛰어나므로 코딩 시 신경 쓰지 않아도 된다"는 것은 매우 위험한 착각입니다. OS 차원의 우회 기법(ROP 등)들이 지속적으로 연구되고 있으므로, 근본적인 방어는 개발 단계에서의 입력값 길이 통제에서 출발해야 합니다.

[참고] OS 기반의 보호 기법 (Mitigation, 완화 기술)
  • ASLR (Address Space Layout Randomization, 주소 공간 배치 난수화): 메모리 주소를 무작위로 변경하여 공격자의 주소 예측을 방해합니다.
  • DEP/NX (Data Execution Prevention / No-eXecute, 데이터 실행 방지): 스택 등 데이터 영역에 '실행 불가' 속성을 부여하여 셸코드 실행을 차단합니다.
  • Stack Canary (스택 카나리아, 버퍼 오버플로우 탐지 기법): 버퍼와 RET 사이에 검증 값을 삽입하여, 함수 종료 시 값의 변조 여부를 확인합니다. *탄광의 카나리아 새에서 유래
