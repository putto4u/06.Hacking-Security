Infrastructure Architect

1. 인프라스트럭처 설계 (Infrastructure Design)

본 실습 환경은 LAMP Stack (Linux, Apache, MySQL/MariaDB, PHP; 리눅스 기반 웹 서비스 묶음) 아키텍처를 기반으로 구성됩니다. 공격자는 외부 네트워크에서 접근하여 서비스의 설정 오류 및 코드 취약점을 분석합니다.

[도식: 타겟 서버 서비스 구조 및 네트워크 흐름]

Attacker (공격자)

Kali Linux

Victim (타겟 서버)

192.168.100.80

Apache2 MariaDB PHP

2. 타겟 서버 환경 구성 (Server Environment Setup)

STEP 01

기본 서비스 설치 및 방화벽 설정

운영체제를 최신화하고 웹 서비스 운영에 필요한 핵심 패키지들을 설치합니다. 또한 UFW (Uncomplicated Firewall; 복잡하지 않은 방화벽)를 통해 80번 포트를 개방합니다.

Ubuntu Terminal
$ sudo su -
$ apt update && apt install -y apache2 mariadb-server php php-mysql
# 서비스 가동 및 방화벽 허용
$ systemctl start apache2 && systemctl start mariadb
$ ufw allow 80/tcp && ufw reload
STEP 02

취약 웹 애플리케이션 배포

실습용 취약 사이트 소스를 다운로드하여 배포하고, MariaDB (마리아 데이터베이스)에 기초 데이터를 주입합니다.

App Setup
$ cd /var/www/html
$ wget -O webhack.zip https://bit.ly/43Dxh9g && unzip webhack.zip
# DB 데이터 주입 및 관리자 권한 설정
$ mysql -u root < board/webhack.sql
$ mysql -u root -p
MariaDB > ALTER USER 'root'@'localhost' IDENTIFIED BY 'admin'; FLUSH PRIVILEGES;

3. 주요 웹 취약점 실증 (Vulnerability Exploitation)

1. 정보 유출 (Information Disclosure)

존재하지 않는 파일 호출 시 발생하는 404 Error (파일을 찾을 수 없음) 페이지 하단에서 서버의 엔진 정보를 확인합니다.

GET /pds/abcd.html HTTP/1.1
Server: Apache/2.4.41 (Ubuntu) ...

2. SQL Injection (인증 우회)

논리적 참을 유도하는 쿼리를 삽입하여 Authentication (인증) 절차를 우회합니다.

# PW 입력값에 쿼리 삽입
' OR ''='
WHERE id='admin' AND pw='' OR ''=''
=> 항상 참(True)으로 판명되어 로그인 성공

3. XSS (Cross-Site Scripting)

게시판 제목이나 본문에 JavaScript (자바스크립트) 코드를 삽입하여 사용자의 브라우저에서 실행되게 합니다.

<SCRIPT>alert("XSS hack!");</SCRIPT>
// 게시물 클릭 시 공격자가 의도한 스크립트 팝업 발생

4. Webshell Upload (서버 장악)

첨부파일 업로드 기능을 악용하여 서버의 명령어를 실행할 수 있는 Webshell (웹쉘) 파일을 전송하고 실행합니다.

File: webshell.php
http://192.168.100.80/board/pds/webshell.php
=> 서버의 모든 자원 제어권 획득

4. 아키텍트 인사이트 (Expert Insight)

권한 관리의 중요성

웹 서버의 소유권을 www-data (아파치 기본 사용자 계정)로 설정하는 이유는 웹 서비스가 필요 이상의 시스템 권한을 갖지 못하게 하기 위함입니다. 하지만 웹쉘이 업로드되면 이 권한조차 탈취될 수 있으므로 파일 업로드 경로의 실행 권한을 제한하는 설정이 필수적입니다.

실습 시 유의 사항

현재 구축된 서버는 보안이 매우 취약하게 설정되어 있습니다. 외부망에서 접근 가능한 공인 IP 환경에서 가동할 경우 순식간에 실제 해킹에 노출될 수 있으므로 반드시 NAT (Network Address Translation; 네트워크 주소 변환) 기반의 폐쇄된 가상 네트워크 환경에서만 실습하십시오.
