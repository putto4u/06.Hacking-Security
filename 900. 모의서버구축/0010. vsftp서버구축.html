Ubuntu Linux
FTP 서버(vsftpd) 구축 및 보안

본 문서는 최신 우분투(Ubuntu) 환경에서 파일 전송 프로토콜(FTP) 데몬인 vsftpd를 APT 패키지 매니저를 통해 설치하고, 안전한 디렉토리 격리(Chroot) 및 방화벽 정책을 구성하는 실무 절차를 기술합니다.

Chapter 1. vsftpd 패키지 설치 및 기동

우분투의 기본 패키지 저장소에서 제공하는 vsftpd(Very Secure FTP Daemon)는 보안과 성능 면에서 가장 권장되는 FTP 서버 소프트웨어입니다.

# 패키지 목록 업데이트 및 vsftpd 설치
$ sudo apt update
$ sudo apt install -y vsftpd

# 서비스 구동 상태 확인
$ sudo systemctl status vsftpd
● vsftpd.service - vsftpd FTP server
   Loaded: loaded (/lib/systemd/system/vsftpd.service; enabled; vendor preset: enabled)
   Active: active (running)

부팅 시 자동 실행 설정

일반적으로 APT를 통한 설치 시 서비스가 자동으로 활성화(enabled)되나, 재부팅 시에도 데몬이 자동으로 시작되도록 명시적으로 설정하려면 다음 명령을 사용합니다.
sudo systemctl enable vsftpd

Chapter 2. vsftpd 핵심 보안 설정

기본 설정만으로는 보안 취약점이 존재할 수 있으며, 파일 업로드가 불가능할 수 있습니다. 메인 설정 파일인 /etc/vsftpd.conf를 수정하여 보안 정책을 강화합니다.

원본 설정 파일 백업

설정 파일을 수정하기 전, 원상 복구를 위해 반드시 원본 파일을 백업해야 합니다.
sudo cp /etc/vsftpd.conf /etc/vsftpd.conf.bak

1. vsftpd.conf 설정값 변경

sudo nano /etc/vsftpd.conf 명령어로 파일을 열고, 아래의 지시자들을 찾아서 주석(#)을 해제하거나 값을 변경합니다.

# 1. 익명 사용자 접속 차단 (보안)
anonymous_enable=NO

# 2. 로컬 계정(우분투 시스템 사용자) 접속 허용
local_enable=YES

# 3. 파일 업로드 및 디렉토리 생성 등 쓰기 권한 허용
write_enable=YES

# 4. 사용자 생성 파일의 기본 권한 설정 (022 설정 시 755/644로 생성됨)
local_umask=022

# 5. 접속한 사용자가 자신의 홈 디렉토리 상위로 이동하지 못하도록 격리 (Chroot Jail)
chroot_local_user=YES

# [추가] Chroot 사용 시 홈 디렉토리 쓰기 권한 충돌 에러(500 OOPS) 방지 구문 (파일 맨 아래 직접 추가)
allow_writeable_chroot=YES

# [추가] 패시브 모드(Passive Mode) 포트 범위 지정 (방화벽 설정을 위해 필요)
pasv_min_port=40000
pasv_max_port=50000

2. 서비스 재시작 및 적용

# 변경된 설정 파일을 데몬에 적용하기 위해 서비스 재시작
$ sudo systemctl restart vsftpd

# 데몬이 정상적으로 구동되었는지 재확인
$ sudo systemctl status vsftpd

Chapter 3. UFW 방화벽 정책 구성

우분투의 기본 방화벽인 UFW(Uncomplicated Firewall)가 활성화되어 있다면, FTP 통신에 필요한 제어 포트와 데이터 전송 포트(패시브 모드)를 허용해야 합니다.

# 1. FTP 제어 포트 허용 (명령어 및 제어 신호 전달용)
$ sudo ufw allow 20/tcp
$ sudo ufw allow 21/tcp

# 2. 패시브 모드 데이터 전송 포트 허용 (vsftpd.conf에 설정한 범위)
$ sudo ufw allow 40000:50000/tcp

# 3. 방화벽 규칙 리로드 및 적용 상태 확인
$ sudo ufw reload
$ sudo ufw status
Status: active
To Action From
-- ------ ----
20/tcp ALLOW Anywhere
21/tcp ALLOW Anywhere
40000:50000/tcp ALLOW Anywhere

액티브(Active) vs 패시브(Passive) 모드

클라이언트가 NAT(공유기) 환경 뒤에 있을 경우, 액티브 모드 사용 시 서버에서 클라이언트로의 데이터 연결이 방화벽에 의해 차단되어 디렉토리 목록을 불러오지 못하는 현상이 발생합니다. 이를 방지하기 위해 클라이언트가 서버의 특정 포트(40000~50000)로 직접 연결하는 패시브 모드를 구성하는 것이 현대 네트워크 아키텍처의 표준입니다.

Chapter 4. 접근 테스트 및 계정 관리

1. FTP 전용 사용자 계정 생성 (권장)

root나 기존 관리자 계정 대신, FTP 전용으로 사용할 격리된 계정을 생성하는 것이 안전합니다.

# 'ftpuser'라는 새로운 계정 생성 및 비밀번호 할당
$ sudo adduser ftpuser
Adding user `ftpuser' ...
New password: # 비밀번호 입력

# 업로드 테스트를 위한 디렉토리 생성 및 권한 부여
$ sudo mkdir -p /home/ftpuser/ftp_dir
$ sudo chown nobody:nogroup /home/ftpuser # chroot 격리를 위한 루트 권한 제거
$ sudo chown ftpuser:ftpuser /home/ftpuser/ftp_dir # 실제 데이터 디렉토리 소유권 부여

2. 클라이언트 접속 테스트

FileZilla, 윈도우 탐색기, 또는 CLI 터미널을 사용하여 구축한 FTP 서버에 접속을 확인합니다.

# CLI 환경에서의 접속 테스트
$ ftp [서버의_IP_주소]
Connected to [서버의_IP_주소].
220 (vsFTPd 3.0.5)
 Name: ftpuser
Password: 패스워드입력
230 Login successful.
Remote system type is UNIX.
Using binary mode to transfer files.
ftp> pwd
257 "/" is the current directory. # Chroot 적용으로 상위 경로가 / 로 표시됨

본 매뉴얼은 Ubuntu 기반 서버에서 안정적인 FTP 서비스 구축을 위한 표준 절차를 제공합니다. 평문(Cleartext) 전송 프로토콜인 FTP의 근본적인 취약점을 보완하기 위해, 실무 환경에서는 SSL/TLS 암호화가 적용된 FTPS 또는 SSH 프로토콜 기반의 SFTP 사용을 적극 권장합니다.
