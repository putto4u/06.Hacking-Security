실전 클라우드 인프라 보안 및 모의 해킹 시나리오 연구 자료 저장소

smbclient를 통한 접속 시 NT_STATUS_ACCESS_DENIED 에러가 발생하는 것은 인증(ID/PW)은 통과했으나, 해당 공유 폴더(PublicShare)에 접근할 파일 시스템 권한이나 Samba 공유 권한이 부족할 때 발생하는 전형적인 메시지입니다.

1. 원인 분석 및 체크리스트

이 에러는 보통 다음 세 가지 중 하나에서 걸립니다.

구분 확인 사항 해결 방법
Samba 설정 smb.conf 내 read only = yes 여부 read only = no 또는 writable = yes 설정
리눅스 권한 실제 디렉토리의 퍼미션 (755, 777 등) chmod 및 chown으로 권한 부여
유저 매핑 Samba 유저와 리눅스 유저의 일치 여부 smbpasswd -a로 유저 등록 확인

2. 단계별 해결 방법

[Step 1] 실제 디렉토리 권한 확인 및 수정

공유하려는 로컬 디렉토리(예: /srv/samba/public)의 권한이 sambashare 유저에게 있는지 확인합니다.

# 공유 폴더의 소유권을 Samba 유저로 변경 (예시 경로)
sudo chown -R sambashare:sambashare /srv/samba/public

# 또는 테스트를 위해 모든 권한 부여
sudo chmod -R 777 /srv/samba/public

[Step 2] smb.conf 설정 재점검

/etc/samba/smb.conf 파일에서 [PublicShare] 섹션의 설정을 확인하세요.

[PublicShare]
   path = /srv/samba/public
   browseable = yes
   writable = yes
   guest ok = no
   valid users = sambashare
   create mask = 0755

주의: 설정을 수정한 후에는 반드시 sudo systemctl restart smbd nmbd로 서비스를 재시작해야 합니다.

[Step 3] 홈 디렉토리 및 상위 폴더 권한

만약 공유 폴더가 특정 유저의 홈 디렉토리 내부에 있다면, 상위 폴더들에 대해 외부인이 접근할 수 있는 x(실행) 권한이 있어야 경로를 찾아 들어갈 수 있습니다.

3. 디버깅 팁

정확한 원인을 파악하기 위해 Samba 로그를 실시간으로 확인하며 접속을 시도해 보세요.

# 로그 확인 (접속 시도 시 에러 내용이 출력됨)
sudo tail -f /var/log/samba/log.smbd
  • 비용 안내: Samba 서비스 이용은 무료입니다. 다만, AWS 환경에서 AWS Storage Gateway(AWS 스토리지 게이트웨이)의 File Gateway(파일 게이트웨이) 모드를 사용하여 SMB 공유를 구성할 경우, 게이트웨이 시간당 비용과 데이터 처리 비용이 발생하므로 설정 시 주의가 필요합니다.
