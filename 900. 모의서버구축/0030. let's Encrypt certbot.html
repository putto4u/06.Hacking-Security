1. 아키텍처 개요 (Architecture Overview)

HTTPS(HyperText Transfer Protocol Secure, 보안 하이퍼텍스트 전송 프로토콜)는 SSL(Secure Sockets Layer, 보안 소켓 계층) 또는 TLS(Transport Layer Security, 전송 계층 보안) 프로토콜을 사용하여 클라이언트와 서버 간의 데이터를 암호화합니다. Let's Encrypt는 ACME(Automated Certificate Management Environment, 자동화된 인증서 관리 환경) 프로토콜을 사용하여 무료로 신뢰할 수 있는 SSL/TLS 인증서를 발급하는 비영리 CA(Certificate Authority, 인증 기관)입니다.

[도식: ACME 프로토콜 기반 인증서 발급 메커니즘]

Apache2 Server
(Certbot Client)
CSR 요청 & 검증
Let's Encrypt CA
(인증 기관)

* CSR(Certificate Signing Request, 인증서 서명 요청): 인증서 발급을 위해 CA에 제출하는 신청서 형식의 데이터입니다.

4. 아키텍트 인사이트 (Expert Insight)

인증서 자동 갱신 (Auto-Renewal)

Let's Encrypt 인증서는 유효기간이 90일입니다. Certbot은 설치 시 `systemd` 타이머나 `cron` 작업을 자동으로 등록하여 만료 전 인증서를 갱신합니다. `sudo certbot renew --dry-run` 명령어로 갱신 프로세스가 정상인지 미리 시뮬레이션할 수 있습니다.

HSTS(HTTP Strict Transport Security) 보안

인증서 적용 후에는 브라우저가 항상 HTTPS로만 접속하도록 강제하는 HSTS(보안 전송 강제 설정) 옵션을 활성화하는 것이 좋습니다. Apache 설정 파일 내 `Header always set Strict-Transport-Security` 구문을 추가하여 중간자 공격(MITM)을 방어하십시오.
