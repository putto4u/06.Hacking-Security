Senior Security Architect

1. 아키텍처 개요 (Architecture Overview)

메타스플로잇 3(Metasploitable 3)는 보안 전문가와 학습자가 실제 환경과 유사한 취약점 공격(Exploit) 및 모의 해킹(Penetration Testing)을 실습할 수 있도록 설계되었습니다. 윈도우 버전은 Windows Server 2008 R2 기반으로 구축되며, 단순한 단일 취약점을 넘어 웹 서비스, 데이터베이스, 시스템 설정 오류 등 다계층 취약점을 제공합니다. 특히 자동화 도구인 베이그런트(Vagrant, 가상 머신 생성 및 설정 자동화 도구)를 통해 환경을 구축함으로써 인프라 구성의 일관성을 유지합니다.

Kali Linux / OS X / Win

2. 실습 구성 요소 및 요구사항 (Prerequisites)

BUILDER

필수 소프트웨어 (Host Side)

  • VirtualBox (버추얼박스): 오픈 소스 하이퍼바이저. [Download]
  • Vagrant (베이그런트): 가상 머신 배포 관리 도구. [Download]
  • Packer (패커): 이미지 빌드 자동화 도구(직접 빌드 시 필요).
VM SPEC

가상 머신 권장 사양 (Target Spec)

  • CPU: 2 Cores 이상 (동시 서비스 처리용)
  • Memory: 2048MB (2GB) 이상 권장
  • Disk: 최소 60GB 여유 공간 (가변 할당)
  • Network: Host-only + NAT (이중 구성)

3. 실증 및 구현 (Implementation)

STEP 01

프로젝트 디렉토리 초기화 및 Vagrant 설정

작업을 위한 폴더를 생성하고, Vagrant Cloud(베이그런트 클라우드, 이미 빌드된 이미지를 공유하는 저장소)에서 Rapid7 공식 이미지를 불러옵니다.

PowerShell / Terminal
$ mkdir metasploitable3-workspace; cd metasploitable3-workspace
$ vagrant init rapid7/metasploitable3-win2k8
# Vagrantfile has been placed in this directory...
STEP 02

가상 머신 프로비저닝 (Provisioning)

명령어를 실행하면 자동으로 ISO 이미지를 다운로드(약 2~4GB)하고 VirtualBox에 VM을 생성 및 구성합니다.

Console
$ vagrant up
Bringing machine 'default' up with 'virtualbox' provider...
==> default: Importing base box 'rapid7/metasploitable3-win2k8'...
... (This may take 10-30 minutes depending on your internet speed)
STEP 03

VirtualBox 세부 설정 검토

Vagrant가 자동으로 설정하지만, 안정적인 실습을 위해 다음 항목을 수동으로 확인하십시오.

  • 네트워크(Network) 어댑터 1: NAT / 어댑터 2: Host-only
  • 가속(Acceleration) VT-x/AMD-V 활성화 (BIOS 필수 체크)
  • 사용자 계정(Credentials) ID: vagrant / PW: vagrant

4. 아키텍트 인사이트 (Expert Insight)

효율적인 빌드를 위한 팁

만약 빌드 과정에서 네트워크 오류가 자주 발생한다면, vagrant-vbguest 플러그인을 설치하여 게스트 확장 도구의 호환성을 높이십시오. 또한, 직접 빌드를 원할 경우 공식 GitHub의 빌드 스크립트를 사용하여 최신 패치를 포함할 수 있습니다.

보안 주의 사항 (Critical Warning)

메타스플로잇 3는 의도적으로 취약하게 설계되었습니다. 절대로 이 VM을 공인 IP에 노출시키거나, 브리지 모드(Bridged Mode)를 통해 실제 사무실/가정 네트워크에 연결하지 마십시오. 악의적인 외부 공격자에게 시스템 제어권을 탈취당할 수 있는 위험이 매우 큽니다.

Hyper-V 충돌 이슈

윈도우 호스트 시스템에서 Hyper-V가 활성화되어 있으면 VirtualBox가 64비트 가상 머신을 실행하지 못할 수 있습니다. `bcdedit /set hypervisorlaunchtype off` 명령을 통해 비활성화 후 재부팅이 필요할 수 있습니다.
