## vsftpd 환경 설정 파일(vsftpd.conf) 보안 점검 및 구조 분석



> 제공된 `vsftpd.conf` 설정 파일을 아키텍트 관점에서 분석한 결과, 시스템 인프라에 심각한 위협이 될 수 있는 몇 가지 보안 누락 사항이 발견되었습니다. 
> 
> 기본적으로 데비안(Debian) 및 우분투(Ubuntu) 계열의 기본 설정은 보수적으로 구성되어 있으나, 실제 운영 환경에 맞게 필수 보안 옵션을 명시적으로 활성화해야 합니다.

---

### 1. 보안 설정 점검 (취약점 및 필수 설정)

<br>

**1) 디렉터리 이동 제한 해제 (보안 취약점)**

```ini
#chroot_local_user=YES
```

> **문제점:** 현재 위 항목이 주석(`#`) 처리되어 있어 기능이 꺼져 있습니다. 이는 로컬 계정으로 접속한 사용자가 자신의 홈 디렉터리를 벗어나 최상위 디렉터리(`/`)를 포함한 시스템 전체 파일 시스템을 탐색할 수 있음을 의미합니다.
> 
> **조치 방법:** 주석(`#`)을 제거하여 `chroot_local_user=YES`로 변경해야 합니다. 이를 통해 사용자는 자신에게 할당된 공간에만 갇히게 되어, 해킹 시에도 시스템 주요 파일이 유출되거나 변조되는 것을 막을 수 있습니다.

<br>

**2) 평문 데이터 전송 노출 (보안 취약점)**

```ini
ssl_enable=NO
```

> **문제점:** FTP (File Transfer Protocol, 파일 전송 프로토콜)는 기본적으로 아이디, 비밀번호, 그리고 전송되는 모든 파일을 암호화하지 않은 평문(Plain Text) 형태로 네트워크에 전송합니다. 이 상태에서는 중간에 패킷을 가로채는 스니핑(Sniffing, 네트워크 트래픽 도청) 공격에 매우 취약합니다.
> 
> **조치 방법:** `ssl_enable=YES`로 변경하고 인증서 경로를 올바르게 지정하여, TLS/SSL 통신 기반의 안전한 FTPS (FTP over SSL, 보안 파일 전송 프로토콜) 환경을 구축해야 합니다.

<br>

**3) 파일 쓰기 권한 제한 (보안상 안전하나 서비스 목적 확인 필요)**

```ini
#write_enable=YES
```

> **설명:** 현재 주석 처리되어 파일 업로드, 삭제, 폴더 생성 등의 쓰기 권한이 막혀 있습니다. 오직 파일 다운로드만 가능한 상태입니다.
> 
> **조치 방법:** 읽기 전용 저장소로 운영한다면 현재 상태가 안전하지만, 사용자가 파일을 업로드해야 하는 서비스라면 주석을 해제하여 `write_enable=YES`로 활성화해야 합니다.

---

### 2. FTP 서비스 디렉터리 지정 방법



> FTP 클라이언트가 서버에 접속했을 때 처음 마주하는 디렉터리(폴더)를 지정하는 방법은 다음과 같습니다.

<br>

**기본 동작 방식 (설정이 없는 경우):**
> 제공된 설정 파일처럼 별도의 디렉터리 지정 옵션이 없다면, vsftpd는 리눅스 시스템에 등록된 **해당 계정의 홈 디렉터리(예: `/home/user1`)** 를 기본 서비스 디렉터리로 자동 인식합니다.

<br>

**특정 디렉터리로 강제 고정하는 방법:**
> 모든 접속자가 각자의 홈 디렉터리가 아닌, 관리자가 지정한 특정 공용 폴더(예: `/var/ftp/public`)로 접속하게 만들고 싶다면, 설정 파일 맨 아래에 다음 옵션을 직접 추가해야 합니다.

```ini
local_root=/var/ftp/public
```

---

### 3. 기타 주요 환경 설정 옵션 해설

> 설정 파일에 포함된 핵심 동작 제어 옵션들의 기능과 목적을 명세합니다.

<br>

* **`listen=NO` & `listen_ipv6=YES`**
  * 독립 실행형(Standalone) 데몬으로 실행될 때 소켓(Socket, 네트워크 통신 연결점)을 정의합니다. IPv6 옵션이 YES로 되어 있으면, 최신 리눅스 환경에서는 IPv4와 IPv6 접속을 모두 수용합니다.

* **`anonymous_enable=NO`**
  * 익명 사용자(Anonymous)의 접속을 차단합니다. 보안을 위해 반드시 NO로 유지해야 합니다.

* **`local_enable=YES`**
  * 서버의 `/etc/passwd` 파일에 등록된 실제 로컬 사용자 계정의 접속을 허용합니다.

* **`dirmessage_enable=YES`**
  * 사용자가 특정 디렉터리에 진입할 때, 해당 디렉터리 안에 있는 `.message` 파일의 내용을 읽어 환영 문구나 공지사항을 보여주는 기능입니다.

* **`use_localtime=YES`**
  * 디렉터리 목록이나 파일 전송 로그에 표시되는 시간을 서버가 위치한 지역의 표준시(한국의 경우 KST)로 표기합니다. 이 옵션이 없으면 글로벌 표준시인 GMT로 표시되어 시간 확인에 혼선이 생깁니다.

* **`xferlog_enable=YES`**
  * 파일의 업로드 및 다운로드 내역을 로그 파일(`/var/log/vsftpd.log`)에 기록합니다. 보안 감사 및 오류 추적을 위해 필수적인 옵션입니다.

* **`connect_from_port_20=YES`**
  * Active Mode (액티브 모드, 능동 데이터 연결 모드) 통신 시 서버 측 데이터 전송 포트를 20번으로 고정합니다. 방화벽 규칙을 단순화하는 데 도움이 됩니다.

* **`secure_chroot_dir=/var/run/vsftpd/empty`**
  * 파일 시스템 접근이 필요하지 않은 작업을 처리할 때 프로세스를 격리하는 빈 디렉터리입니다. 권한 상승 공격을 방어하는 내부 보안 매커니즘입니다.

* **`pam_service_name=vsftpd`**
  * 사용자 인증을 처리할 때 리눅스의 PAM (Pluggable Authentication Modules, 장착형 인증 모듈) 설정을 사용합니다. `/etc/pam.d/vsftpd` 파일을 참조하여 인증을 수행합니다.

* **`rsa_cert_file` & `rsa_private_key_file`**
  * FTPS 보안 통신에 사용할 RSA (Rivest-Shamir-Adleman, 공개키 암호화 알고리즘) 인증서와 개인키의 경로를 지정합니다. 현재는 시스템에서 임의로 생성된 기본 파일(`snakeoil`)이 지정되어 있습니다.

요청하신 vsftpd.conf 설정 파일 보안 분석 및 옵션 해설 페이지 작성을 완료하였습니다. 아키텍트의 시각에서 가장 크리티컬한 보안 누락 요소(chroot 미설정 및 SSL 미적용)를 도출하여 조치 방법을 명시하였고, 서비스 디렉터리 지정 방법 및 기타 옵션들에 대해서도 한글화 원칙 및 괄호 이니셜 표기 규칙을 엄격히 적용하여 문장 블록화 형태로 구성하였습니다. 에러가 없도록 옵션 간 상관관계까지 스텝바이스텝으로 교차 검증하였습니다.
