Ubuntu Linux
Samba 서버 구축 및 공유 설정

본 문서는 최신 우분투 환경에서 윈도우(Windows) 및 이기종 시스템 간의 네트워크 파일 공유를 지원하는 SMB/CIFS 프로토콜 데몬인 Samba를 설치하고, 보안 권한이 적용된 공유 디렉토리를 구성하여 검증하는 실무 절차를 기술합니다.

Chapter 1. Samba 패키지 설치 및 데몬 확인

우분투 공식 저장소를 통해 samba 데몬 패키지를 설치합니다. 이 과정에서 파일 공유에 필요한 핵심 라이브러리 및 유틸리티가 함께 구성됩니다.

# APT 인덱스 동기화 및 samba 패키지 설치
$ sudo apt update
$ sudo apt install -y samba smbclient

# smbd(Samba 데몬) 실행 상태 검증
$ sudo systemctl status smbd
● smbd.service - Samba SMB Daemon
   Loaded: loaded (/lib/systemd/system/smbd.service; enabled)
   Active: active (running)

Chapter 2. 공유 디렉토리 및 시스템 권한 구성

네트워크를 통해 접근할 물리적 디렉토리를 생성하고, Samba 프로세스 및 인증된 사용자가 읽고 쓸 수 있도록 디렉토리 소유권과 퍼미션을 조정합니다.

# 1. 파일 서버용 최상위 디렉토리 생성
$ sudo mkdir -p /srv/samba/shared

# 2. 특정 시스템 계정에 종속되지 않도록 소유권 변경 (보안)
$ sudo chown nobody:nogroup /srv/samba/shared

# 3. 디렉토리 퍼미션 부여 (소유자/그룹은 읽기,쓰기,실행 허용)
$ sudo chmod -R 0775 /srv/samba/shared

Chapter 3. smb.conf 환경 설정 파일 수정

Samba의 핵심 동작을 제어하는 /etc/samba/smb.conf 파일의 최하단에 새로운 공유 폴더 프로파일을 추가합니다.

원본 설정 파일 백업

설정 오류 시 즉각적인 복구를 위해 원본 파일을 백업하는 것은 인프라 엔지니어의 기본 수칙입니다.
sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bak

공유 프로파일 작성

텍스트 에디터(sudo nano /etc/samba/smb.conf)를 열고 파일의 맨 끝에 아래 내용을 추가합니다.

# [대괄호] 안의 이름이 윈도우 네트워크 상에 표시되는 공유 폴더명입니다.
[PublicShare]
    comment = Ubuntu Samba File Server
    path = /srv/samba/shared
    browsable = yes
    # 익명 접근 허용 여부 (no로 설정 시 자격 증명 필수)
    guest ok = no
    # 읽기 전용 여부 (쓰기 권한 부여)
    read only = no
    # 파일 및 폴더 생성 시 기본 권한 마스크
    create mask = 0755
    directory mask = 0755
    # [선택] 특정 사용자만 접근 허용할 경우 추가
    valid users = smbuser

Chapter 4. Samba 사용자 인증 및 방화벽 허용

1. Samba 전용 사용자 자격 증명 생성

Samba는 리눅스 시스템 계정과 별도로 자체적인 패스워드 데이터베이스를 관리합니다. 따라서 시스템에 존재하는 계정을 Samba 전용 계정으로 동기화해야 합니다.

# 1. 시스템 계정 생성 (이미 존재하는 계정을 사용할 경우 생략 가능)
$ sudo adduser smbuser

# 2. 시스템 계정을 Samba 데이터베이스에 추가 및 전용 패스워드 설정
$ sudo smbpasswd -a smbuser
New SMB password: # 공유 폴더 접근용 패스워드 입력
Retype new SMB password:
Added user smbuser.

# 3. 변경된 설정을 적용하기 위해 데몬 재시작
$ sudo systemctl restart smbd nmbd

2. UFW 방화벽 규칙 구성

Samba 통신을 위해 사용되는 SMB/CIFS 포트(TCP 139, 445 및 UDP 137, 138)를 방화벽 프로파일을 통해 일괄 개방합니다.

# UFW의 사전 정의된 samba 프로파일 허용
$ sudo ufw allow samba
Rule added
Rule added (v6)

# 방화벽 적용 상태 확인
$ sudo ufw status

Chapter 5. 클라이언트 환경 접속 테스트

Samba 서버 설정이 완료되면, 클라이언트(Windows 또는 Linux) 환경에서 정상적인 네트워크 드라이브 접근 및 파일 입출력이 가능한지 검증합니다.

Windows 환경에서 검증

  1. 단축키 Win + R을 눌러 '실행' 창을 엽니다.
  2. UNC 경로 형식인 \\서버_IP_주소 (예: \\192.168.0.100)를 입력하고 확인을 누릅니다.
  3. 네트워크 자격 증명 창이 뜨면 smbuser와 설정한 Samba 패스워드를 입력합니다.
  4. 'PublicShare' 폴더에 진입하여 새 텍스트 문서를 생성, 파일 쓰기 권한을 확인합니다.

Linux(CLI) 환경에서 검증

# smbclient 유틸리티 사용 (-U 옵션으로 사용자 지정)
$ smbclient //192.168.0.100/PublicShare -U smbuser
Password for [WORKGROUP\smbuser]:
Try "help" to get a list of possible commands.
smb: \> ls
