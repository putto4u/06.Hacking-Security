Technical Manual Infrastructure Setup Ver 8.1

Ubuntu Linux
Samba 서버 구축 및 공유 설정

본 문서는 최신 우분투 환경에서 윈도우(Windows) 및 이기종 시스템 간의 네트워크 파일 공유를 지원하는 SMB/CIFS 프로토콜 데몬인 Samba를 설치하고, 보안 권한이 적용된 공유 디렉토리를 구성하여 검증하는 실무 절차를 기술합니다.

Chapter 1. Samba 패키지 설치 및 데몬 확인

우분투 공식 저장소를 통해 samba 데몬 패키지를 설치합니다. 이 과정에서 파일 공유에 필요한 핵심 라이브러리 및 유틸리티가 함께 구성됩니다.

# APT 인덱스 동기화 및 samba 패키지 설치
$ sudo apt update
$ sudo apt install -y samba smbclient

# smbd(Samba 데몬) 실행 상태 검증
$ sudo systemctl status smbd
● smbd.service - Samba SMB Daemon
   Loaded: loaded (/lib/systemd/system/smbd.service; enabled)
   Active: active (running)

Chapter 2. 공유 디렉토리 및 시스템 권한 구성

네트워크를 통해 접근할 물리적 디렉토리를 생성하고, Samba 프로세스 및 인증된 사용자가 읽고 쓸 수 있도록 디렉토리 소유권과 퍼미션을 조정합니다.

# 1. 파일 서버용 최상위 디렉토리 생성
$ sudo mkdir -p /srv/samba/shared

# 2. 특정 시스템 계정에 종속되지 않도록 소유권 변경 (보안)
$ sudo chown nobody:nogroup /srv/samba/shared

# 3. 디렉토리 퍼미션 부여 (소유자/그룹은 읽기,쓰기,실행 허용)
$ sudo chmod -R 0775 /srv/samba/shared

Chapter 3. smb.conf 환경 설정 파일 수정

Samba의 핵심 동작을 제어하는 /etc/samba/smb.conf 파일의 최하단에 새로운 공유 폴더 프로파일을 추가합니다.

원본 설정 파일 백업

설정 오류 시 즉각적인 복구를 위해 원본 파일을 백업하는 것은 인프라 엔지니어의 기본 수칙입니다.
sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bak

공유 프로파일 작성

텍스트 에디터(sudo nano /etc/samba/smb.conf)를 열고 파일의 맨 끝에 아래 내용을 추가합니다.

# [대괄호] 안의 이름이 윈도우 네트워크 상에 표시되는 공유 폴더명입니다.
[PublicShare]
    comment = Ubuntu Samba File Server
    path = /srv/samba/shared
    browsable = yes
    # 익명 접근 허용 여부 (no로 설정 시 자격 증명 필수)
    guest ok = no
    # 읽기 전용 여부 (쓰기 권한 부여)
    read only = no
    # 파일 및 폴더 생성 시 기본 권한 마스크
    create mask = 0755
    directory mask = 0755
    # [선택] 특정 사용자만 접근 허용할 경우 추가
    valid users = smbuser

설정 파일 무결성 검증 (testparm)

설정 파일(smb.conf) 수정 후에는 반드시 Samba 내장 유틸리티인 testparm을 실행하여 문법적 오류(Syntax Error)를 사전에 차단해야 합니다. 오타가 존재할 경우 데몬이 오작동하거나 자기 자신조차 접속이 거부(Connection Refused)될 수 있습니다.

# 파라미터 테스트 실행
$ testparm
Load smb config files from /etc/samba/smb.conf
Loaded services file OK.
Server role: ROLE_STANDALONE
# 'Loaded services file OK.' 메시지가 출력되어야 정상입니다.

Chapter 4. Samba 사용자 인증 및 방화벽 허용

1. Samba 전용 사용자 자격 증명 생성

Samba는 리눅스 시스템 계정과 별도로 자체적인 패스워드 데이터베이스를 관리합니다. 따라서 시스템에 존재하는 계정을 Samba 전용 계정으로 동기화해야 합니다.

# 1. 시스템 계정 생성 (이미 존재하는 계정을 사용할 경우 생략 가능)
$ sudo adduser smbuser

# 2. 시스템 계정을 Samba 데이터베이스에 추가 및 전용 패스워드 설정
$ sudo smbpasswd -a smbuser
New SMB password: # 공유 폴더 접근용 패스워드 입력
Retype new SMB password:
Added user smbuser.

# 3. 변경된 설정을 적용하기 위해 데몬 재시작
$ sudo systemctl restart smbd nmbd

2. UFW 방화벽 규칙 구성

Samba 통신을 위해 사용되는 SMB/CIFS 포트(TCP 139, 445 및 UDP 137, 138)를 방화벽 프로파일을 통해 일괄 개방합니다.

# UFW의 사전 정의된 samba 프로파일 허용
$ sudo ufw allow samba
Rule added
Rule added (v6)

# 방화벽 적용 상태 확인
$ sudo ufw status

Chapter 5. 클라이언트 환경 접속 및 파일 제어 실습

Samba 서버 설정이 완료되면, 클라이언트(Windows 또는 Linux) 환경에서 정상적인 네트워크 드라이브 접근 및 파일 입출력(생성 및 삭제)이 가능한지 검증합니다.

Windows 환경에서 검증 (GUI)

  1. 단축키 Win + R을 눌러 '실행' 창을 엽니다.
  2. UNC 경로 형식인 \\서버_IP_주소 (예: \\192.168.0.100)를 입력하고 확인을 누릅니다.
  3. 네트워크 자격 증명 창이 뜨면 smbuser와 설정한 Samba 패스워드를 입력합니다.
  4. 'PublicShare' 폴더에 진입하여 마우스 우클릭 후 새 텍스트 문서를 생성(파일 넣기)하여 쓰기 권한을 확인합니다.
  5. 생성한 문서를 선택 후 Delete 키를 눌러 삭제(파일 지우기) 권한을 확인합니다.

Linux(CLI) 환경에서 검증 (smbclient)

자기 자신(Localhost) 접속 실패 원인 및 해결 (Troubleshooting)

서버 구축 직후 자기 자신(Samba가 설치된 서버)에서 로컬 테스트를 진행할 때 접속이 거부되거나 에러가 발생하는 가장 큰 원인 2가지는 IP 지정 오류와 프로토콜 버전 충돌입니다.

  • 루프백(Loopback) IP 사용: 앞선 예제의 192.168.0.100을 그대로 복사하면 본인 네트워크 환경과 달라 타임아웃(Timeout)이 발생합니다. 자신을 테스트할 땐 반드시 localhost 또는 127.0.0.1을 지정해야 합니다.
  • 최신 SMB 프로토콜 강제: 최신 우분투는 보안상 취약한 SMBv1(NT1) 프로토콜을 시스템 차원에서 차단합니다. CLI 클라이언트 접속 시 -m SMB3 옵션을 명시하여 최신 프로토콜 통신을 강제해야 협상 실패(Protocol negotiation failed) 에러를 방지할 수 있습니다.
# 1. 테스트용 로컬 파일 생성 및 접속 (서버 자신에서 테스트 시 localhost 및 SMB3 강제 옵션 적용)
$ touch smb_test.txt
$ smbclient //localhost/PublicShare -U smbuser -m SMB3
Password for [WORKGROUP\smbuser]:

# 2. 파일 업로드 (Put 명령어로 파일 넣기)
smb: \> put smb_test.txt
putting file smb_test.txt as \smb_test.txt (0.0 kb/s)
smb: \> ls
. D 0 Wed Mar 18 10:00:00 2026
  smb_test.txt A 0 Wed Mar 18 10:01:00 2026

# 3. 파일 삭제 (Rm 명령어로 파일 지우기)
smb: \> rm smb_test.txt
smb: \> exit

1. Windows 환경 네트워크 드라이브 영구 매핑(Mapping) 실무

단순 1회성 접근이 아닌, PC 재부팅 후에도 지속적으로 연결을 유지하기 위해 운영체제(OS)의 네트워크 드라이브(Z: 등)로 할당하는 실무 표준 절차입니다.

  • GUI(Graphical User Interface, 그래픽 사용자 인터페이스) 방식: 파일 탐색기를 열고 [내 PC(This PC)] 우클릭 ➔ [네트워크 드라이브 연결(Map network drive)] 선택 ➔ 드라이브 문자(예: Z:) 선택 ➔ 폴더 란에 UNC(Universal Naming Convention, 범용 명명 규칙) 경로인 \\192.168.0.100\PublicShare 입력 ➔ '로그온할 때 다시 연결' 체크 후 마침.
  • CLI(Command Line Interface, 명령줄 인터페이스) 방식: 관리자 권한으로 명령 프롬프트(CMD)를 실행하여 아래의 명령어를 입력합니다. 자동화를 위한 스크립트 작성 시 주로 활용됩니다.
# Windows CMD에서 Z 드라이브로 영구 마운트 설정
C:\> net use Z: \\192.168.0.100\PublicShare /user:smbuser "패스워드" /persistent:yes
명령을 잘 실행했습니다.

# 자격 증명(Credential) 캐시 초기화가 필요한 경우 (접속 장애 시 트러블슈팅)
C:\> cmdkey /delete:192.168.0.100

Windows 마운트 해제(Unmount, 네트워크 드라이브 연결 끊기)

연결된 네트워크 자원의 사용이 종료되었거나, 접속 계정을 변경하기 위해 기존 마운트를 해제하는 절차입니다.

# CLI 방식: 연결된 Z 드라이브 논리적 연결 해제 및 삭제
C:\> net use Z: /delete

2. Linux 환경 CIFS 마운트(Mount) 및 파일 제어 실무

리눅스 클라이언트 서버 간의 대용량 데이터 백업이나 도커(Docker) 볼륨(Volume)으로 활용하기 위해 운영체제 레벨에서 CIFS(Common Internet File System, 공통 인터넷 파일 시스템) 마운트(Mount, 논리적 장치 연결)를 수행하고 파일을 직접 생성/삭제하는 방법입니다.

# 1. CIFS 파일 시스템 지원 유틸리티 패키지 설치
$ sudo apt update && sudo apt install -y cifs-utils

# 2. 로컬 마운트 포인트(Mount Point, 연결 지점) 디렉토리 생성
$ sudo mkdir -p /mnt/samba_client

# 3. 수동 마운트 테스트 (uid, gid는 로컬 시스템 사용자의 ID 번호)
$ sudo mount -t cifs //192.168.0.100/PublicShare /mnt/samba_client -o username=smbuser,password=사용자암호,uid=1000,gid=1000

# 4. 부팅 시 자동 마운트를 위한 fstab(File System Table, 파일 시스템 정보 테이블) 등록
$ sudo nano /etc/fstab
# 파일 최하단에 아래 라인 추가 (보안을 위해 credentials 파일 생성 방식 권장)
//192.168.0.100/PublicShare /mnt/samba_client cifs username=smbuser,password=사용자암호,uid=1000,gid=1000 0 0

# 5. 마운트된 공유 폴더 내 파일 생성(쓰기) 및 삭제 테스트
$ cd /mnt/samba_client
$ echo "Samba Mount Test" > hello.txt # 파일 넣기 (생성)
$ ls -l hello.txt
-rwxrwxr-x 1 smbuser smbgroup 17 Mar 18 09:20 hello.txt
$ rm hello.txt # 파일 지우기 (삭제)

* 보안 팁(Security Tip): /etc/fstab에 평문 패스워드를 직접 기재하는 것은 보안 위협이 될 수 있으므로, 별도의 .smbcredentials 파일을 생성하고 권한을 600으로 부여한 뒤, fstab 옵션에 credentials=/path/to/.smbcredentials 형식으로 연결하는 아키텍처 구성을 적극 권장합니다.

Linux 마운트 해제(Unmount, 시스템 연결 해제)

리눅스 시스템에서 마운트된 공유 디렉토리의 연결을 안전하게 분리하는 명령어입니다. 파일 작업(읽기/쓰기)이 완전히 종료된 후 실행해야 합니다.

# 마운트 포인트 정상 연결 해제 (디렉토리를 빠져나온 후 실행)
$ cd ~
$ sudo umount /mnt/samba_client

# 강제 해제가 필요한 경우 (Device is busy 에러 발생 시 -l(지연 해제) 옵션 사용)
$ sudo umount -l /mnt/samba_client

본 매뉴얼은 Ubuntu 기반 서버에서 안정적인 인트라넷 파일 공유 서비스를 구축하기 위한 표준 절차를 제공합니다. 기업의 실제 운영 환경에서는 Active Directory(AD) 연동이나 세밀한 ACL(Access Control List) 설정을 추가로 적용하여 데이터 보안성을 강화해야 합니다.
