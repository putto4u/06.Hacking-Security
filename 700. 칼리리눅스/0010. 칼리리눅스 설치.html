칼리 리눅스(Kali Linux) 가상 머신 구축 및 심층 설치 가이드

모의 해킹 및 침투 테스트 인프라 구성을 위한 하이퍼바이저(Hypervisor) 기반 운영체제 배포 아키텍처

1 가상화(Virtualization) 아키텍처 및 준비 사항

보안 진단 및 모의 해킹을 수행하기 위해서는 호스트 운영체제(Host Operating System, Host OS)를 보호하면서 다양한 공격 도구를 안전하게 실행할 수 있는 격리된 환경이 필수적입니다. 이를 위해 버추얼박스(VirtualBox)와 같은 하이퍼바이저(Hypervisor) 소프트웨어를 사용하여 논리적인 가상 머신(Virtual Machine, VM)을 생성하고, 그 위에 침투 테스트 전용 운영체제인 칼리 리눅스(Kali Linux)를 설치합니다.

[도식 1] Hosted(Type 2) 하이퍼바이저 기반의 가상화 아키텍처

용어 및 사전 준비 상세 설명:
  • ISO 이미지 파일(ISO Image File): 광학 디스크(CD/DVD)의 내용을 파일 하나로 묶어놓은 표준 포맷입니다. 칼리 리눅스 공식 홈페이지에서 Installer Image 버전을 다운로드해야 처음부터 OS 설정 과정을 실습할 수 있습니다.
  • 하이퍼바이저(Hypervisor): 물리적 하드웨어 자원을 가상화하여 여러 개의 운영체제가 동시에 실행될 수 있도록 제어하는 중간 관리 소프트웨어입니다.
  • 호스트(Host)와 게스트(Guest): 실제 물리적 PC에 설치된 원래의 운영체제를 호스트(주인)라 부르고, 가상 머신 내부에 설치되는 운영체제를 게스트(손님)라 부릅니다.

2 가상 머신(VM) 생성 및 하드웨어 자원 할당

버추얼박스(VirtualBox)를 실행하고 새로운 가상 머신을 생성하기 위해 새로 만들기(New) 버튼을 클릭합니다. 아래의 설정 값은 실무 환경과 교육 목적에 맞게 최적화된 기준입니다.

1단계: 이름 및 운영체제 종류 선택

  • 이름(Name): Kali_Linux_2026 (원하는 이름 지정)
  • 종류(Type): Linux 선택.
    ※ 칼리는 리눅스 커널 기반이므로 반드시 Linux를 선택해야 호환성 모듈이 올바르게 로드됩니다.
  • 버전(Version): Debian (64-bit) 선택.
    ※ 칼리 리눅스는 데비안(Debian) 리눅스 배포판을 기반으로 파생되었기 때문입니다. Ubuntu나 RedHat을 선택하면 확장 팩 설치 시 오류가 발생할 수 있습니다.

2단계: 메모리(RAM) 및 프로세서(CPU) 할당

  • 기본 메모리(Base Memory): 최소 2048 MB (2GB) ~ 권장 4096 MB (4GB)
    ※ 호스트 PC 메모리의 절반 이상을 할당하지 마십시오. 초록색 바(Bar) 영역 내에서만 설정해야 호스트 OS가 멈추는(Freezing) 현상을 방지할 수 있습니다.
  • 프로세서(Processors): 최소 2 CPUs
    ※ 비밀번호 크래킹 등 연산이 많이 필요한 해킹 툴을 원활히 실행하려면 다중 코어 할당이 필수적입니다.

[도식 2] 가상 하드 디스크 할당 방식의 차이점

3단계: 가상 하드 디스크 생성 옵션 세부 설명

  • 하드 디스크 파일 종류: VDI (VirtualBox Disk Image)를 선택합니다. VDI는 버추얼박스의 네이티브 포맷으로 성능이 가장 좋습니다. (VMDK는 VMware 호환용, VHD는 Windows Hyper-V 호환용입니다.)
  • 물리적 하드 드라이브에 저장: 동적 할당(Dynamically allocated)을 선택합니다. 초기에 50GB를 설정하더라도, 실제 설치된 파일 용량(약 10GB)만큼만 호스트 PC의 용량을 차지하며 서서히 증가합니다.
  • 파일 위치 및 크기: 디스크 크기는 최소 30GB ~ 50GB를 권장합니다. 칼리 리눅스의 수많은 도구를 설치하고 로그를 저장하기 위해서는 여유 공간이 필요합니다.

3 칼리 리눅스 OS 설치 시퀀스 (Boot & Setup)

실전 팁(Expert Tip): 생성된 VM을 시작하기 전에, 설정(Settings) > 저장소(Storage) 메뉴에서 '비어 있음(Empty)'으로 표시된 CD 아이콘을 클릭하고, 우측 상단의 디스크 아이콘을 눌러 다운로드한 Kali Linux ISO 파일을 마운트(Mount, 삽입)해야 합니다.

① 부트 메뉴 (Boot Menu)

VM을 시작하면 칼리 부트 로더 화면이 나타납니다.

선택: Graphical Install (그래픽 설치)
설명: 마우스와 그래픽 사용자 인터페이스(GUI)를 지원하여 설치가 직관적입니다.
(참고: 'Install'을 선택하면 텍스트 기반 인터페이스(CLI 형태)로 설치가 진행되어 초보자에게는 다소 까다로울 수 있습니다.)

② 언어, 위치, 키보드 설정

  • Language (언어): Korean (한국어) 선택 (영어 학습 목적이라면 English 권장)
  • Location (위치): 대한민국 (Korea, Republic of) 선택 (시간대 설정을 위해 필요)
  • Keyboard (키보드): 한국어 (Korean) 선택

③ 네트워크 설정 (Network Configuration)

Hostname (호스트 이름): kali (기본값 유지)
설명: 네트워크 상에서 이 컴퓨터를 식별하는 이름입니다.
Domain name (도메인 이름): 빈칸으로 두고 계속 진행
설명: 회사나 학교 내부의 도메인(예: example.com)에 속해 있을 때 입력하지만, 독립적인 실습 환경이므로 생략합니다.

④ 사용자 및 암호 설정 (매우 중요)

과거 버전과 달리 최신 칼리 리눅스는 보안상 최고 관리자(root)로 직접 로그인하는 것을 금지하고, 일반 사용자 계정을 생성하도록 강제합니다.

  • 사용자의 실제 이름 (Full name): student (임의 지정)
  • 계정의 사용자 이름 (Username): student (로그인할 때 사용하는 실제 ID입니다. 반드시 기억해야 합니다.)
  • 암호 (Password): 강력한 비밀번호 입력 (실습용으로는 기억하기 쉬운 암호 지정)

[도식 3] 리눅스 파티션(Partition) 구조

⑤ 디스크 파티션 (Partition Disks)

선택 1: Guided - use entire disk (자동 - 디스크 전체 사용)
설명: 수동(Manual)으로 파티션을 나누는 복잡한 과정 없이, 생성한 가상 디스크(VDI) 전체를 리눅스 포맷으로 자동 할당합니다.

선택 2: All files in one partition (모든 파일을 하나의 파티션에 저장)
설명: 초보자에게 권장되는 설정으로, 시스템 파일, 사용자 파일(/home) 등을 분리하지 않고 한 공간에 설치하여 공간 부족 오류를 방지합니다.

선택 3: Finish partitioning and write changes to disk (파티션 나누기 마치고 디스크에 변경 사항 쓰기) -> 'Yes(예)' 선택
설명: 포맷을 최종 승인하는 단계입니다. 가상 디스크 안에서 일어나는 일이므로 실제 호스트 PC의 데이터는 안전합니다.

⑥ 소프트웨어 선택 (Software Selection)

설치할 데스크톱 환경(GUI 테마)과 기본 도구를 선택합니다. 기본 체크된 항목을 그대로 유지하는 것이 가장 좋습니다.

  • XFCE (기본값): 매우 가볍고 리소스 소모가 적어 가상 머신 환경에 최적화된 데스크톱 환경입니다.
  • GNOME / KDE Plasma: 미려한 그래픽을 제공하지만 시스템 자원을 많이 소모하므로 가상 머신에서는 속도 저하를 유발할 수 있어 선택을 해제해 두는 것이 좋습니다.
  • default -- recommended tools: 상위 10개의 핵심 해킹 툴 모음입니다. 반드시 체크가 유지되어야 합니다.

⑦ GRUB 부트 로더 설치 (Install the GRUB boot loader)

선택: 'Yes(예)' 선택 후, 디바이스 목록에서 `/dev/sda` 를 수동으로 클릭하여 선택.
설명: GRUB(GRand Unified Bootloader)은 운영체제를 메모리로 불러오는 부팅 관리 프로그램입니다. 이것을 마스터 부트 레코드(Master Boot Record, MBR)가 있는 첫 번째 디스크(`/dev/sda`)에 설치해야 칼리 리눅스가 정상적으로 부팅됩니다. 'Enter device manually'를 선택하지 말고 반드시 디스크 이름을 직접 클릭하세요.

4 설치 완료 후 필수 초기 설정 (CLI 패키지 업데이트)

설치가 완료되고 재부팅되면, 앞에서 생성한 계정(예: student)과 암호를 입력하여 로그인합니다. 로그인 직후 터미널(Terminal, 검은색 명령창 아이콘)을 열고, 최신 보안 패치 및 해킹 도구 업데이트를 위해 패키지 저장소를 동기화해야 합니다.

student@kali:~$ sudo apt update && sudo apt upgrade -y
명령어(Command) 세부 분석:
  • sudo (Superuser Do): 뒤에 오는 명령어를 최고 관리자(root)의 강력한 권한으로 임시 실행하겠다는 의미입니다. 실행 시 사용자의 암호를 묻습니다.
  • apt (Advanced Packaging Tool): 데비안 계열 리눅스에서 소프트웨어를 다운로드하고 설치, 삭제하는 공식 패키지 관리자 도구입니다.
  • update: 인터넷에 있는 공식 패키지 저장소(Repository) 서버에 접속하여, "새로 나온 소프트웨어나 패치 목록"만 가져와서 내 PC의 리스트를 최신화합니다. (실제 프로그램 설치는 안 됨)
  • && (논리 AND 연산자): 앞의 명령(update)이 오류 없이 성공적으로 끝나면, 즉시 뒤의 명령(upgrade)을 이어서 자동 실행하라는 연결 기호입니다.
  • upgrade: update로 가져온 최신 목록을 바탕으로, 내 PC에 설치된 구버전 프로그램들을 실제로 다운로드하여 최신 버전으로 덮어쓰기(업그레이드) 합니다.
  • -y (Yes 플래그): 업그레이드 중간에 "정말 설치하시겠습니까? (Y/n)" 라고 묻는 확인 창이 뜰 때, 무조건 'Y(예)'로 자동 응답하여 중간에 멈추지 않고 끝까지 설치되게 하는 옵션입니다.

최종 요약

본 문서에서는 하이퍼바이저 기반 가상화 환경에서 칼리 리눅스를 구축하는 엔드투엔드(End-to-End) 아키텍처를 살펴보았습니다. 올바른 디스크 할당, 네트워크 격리, 파티션 구성은 안정적인 모의 해킹 환경을 유지하는 근간이 됩니다. 특히 리눅스의 권한 분리 원칙에 따라 루트(root) 계정이 아닌 일반 사용자 계정을 생성하고, 필요시 sudo를 활용하는 방식은 현대 보안 컴플라이언스(Compliance)의 핵심 준수 사항입니다.
