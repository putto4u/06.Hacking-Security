칼리 리눅스(Kali Linux) 라이브(Live) USB 환경 구축

흔적을 남기지 않는 극강의 휴대성과 하드웨어 직접 접근을 위한 베어메탈(Bare-Metal) 침투 테스트 인프라 구성

1 라이브(Live) OS의 개념과 가상 머신(VM)과의 차이점

라이브 부팅(Live Booting)이란 하드 디스크에 운영체제를 영구적으로 설치하지 않고, USB(Universal Serial Bus, 범용 직렬 버스) 드라이브 자체를 부팅 매체로 사용하여 컴퓨터의 RAM(Random Access Memory, 임의 접근 기억 장치) 위에서 운영체제를 실행하는 방식입니다.

장점 1: 하드웨어 직접 제어 (베어메탈)

가상 머신(VM) 환경에서는 호스트 OS(윈도우 등)가 무선 랜카드나 GPU(Graphics Processing Unit, 그래픽 처리 장치)를 독점하므로, 칼리 리눅스에서 무선 해킹(Wi-Fi 패킷 주입)이나 GPU를 활용한 초고속 패스워드 크래킹을 수행하기 어렵습니다. USB로 직접 부팅하면 컴퓨터의 모든 물리적 하드웨어 자원을 100% 직접 제어할 수 있습니다.

장점 2: 안티 포렌식 (Anti-Forensics)

라이브 환경은 기본적으로 휘발성(Volatile) 메모리인 RAM에서 구동됩니다. 즉, 시스템 전원을 끄거나 USB를 뽑는 순간, 수행했던 모든 작업 내역과 침투 흔적이 물리적으로 완전히 소멸됩니다. 이는 디지털 포렌식 조사를 회피해야 하는 고도화된 침투 테스트 시나리오에서 필수적입니다.

[도식 1] 하드 디스크를 우회하여 RAM에서 직접 실행되는 라이브 USB의 논리 구조

관련 클라우드 실무 팁(Cloud Operations Tip): AWS(Amazon Web Services)나 Azure와 같은 클라우드 환경에서 가상 머신(EC2 등)을 띄워 칼리 리눅스를 사용하면 시간당 컴퓨팅 과금(Compute Billing)이 발생합니다. 하지만 개인 노트북에 라이브 USB를 만들어 사용하면 물리적인 노트북의 전력 외에는 클라우드 인프라 운영 비용이 전혀 발생하지 않습니다.

2 준비물 및 부팅 USB 생성 도구(Rufus) 설정

가장 자주 하는 치명적 실수(Critical Mistake): ISO 파일 다운로드 오류

칼리 공식 홈페이지(kali.org)에서 다운로드할 때, 가상 머신용 Installer Image를 다운로드하여 USB에 굽는 실수가 매우 빈번합니다. 라이브 USB를 만들려면 반드시 Live Boot (라이브 부트) 전용 ISO 이미지를 다운로드해야 합니다. 파일명에 live라는 단어가 포함되어 있는지 꼭 확인하십시오. (예: kali-linux-202x.x-live-amd64.iso)

1단계: 준비물

  • USB 드라이브: 최소 8GB 이상 권장 (퍼시스턴스 기능 사용 시 16GB ~ 32GB 이상 권장).
    ※ 작업 중 USB 내부의 모든 기존 데이터는 완전히 삭제(포맷)됩니다.
  • 이미지 플래싱(Flashing) 툴: 윈도우 환경에서는 Rufus(루퍼스) 또는 BalenaEtcher(발레나 에처)를 사용합니다. 본 교재에서는 실무에서 가장 세밀한 제어가 가능한 Rufus를 기준으로 설명합니다.

2단계: Rufus(루퍼스) 세부 설정 및 파티션 스킴 이해

Rufus를 실행하고 USB 드라이브와 다운로드한 Kali Live ISO 파일을 선택한 후, 아래와 같이 아키텍처를 설정합니다.

설정 항목 (Menu) 선택할 값 (Value) 선택 시 발생하는 결과 및 기술적 이유 (Explanation)
파티션 방식
(Partition scheme)		 GPT 또는 MBR 대상 PC가 최신 컴퓨터(2015년 이후)라면 GPT(GUID Partition Table, 전역 고유 식별자 파티션 테이블)를 선택합니다. 구형 컴퓨터나 호환성을 극대화하려면 MBR(Master Boot Record, 마스터 부트 레코드)를 선택합니다.
대상 시스템
(Target system)		 UEFI (비 CSM) 최신 메인보드의 펌웨어 표준인 UEFI(Unified Extensible Firmware Interface, 통일 확장 펌웨어 인터페이스) 모드로 부팅되도록 설정합니다.
파일 시스템
(File system)		 FAT32 (기본값) UEFI 시스템은 부팅 파일을 읽기 위해 반드시 FAT32(File Allocation Table 32) 파일 시스템을 요구합니다. NTFS를 선택하면 최신 PC에서 USB 자체를 부팅 디스크로 인식하지 못합니다.

실무 중요 옵션: ISO 모드 vs DD 모드

Rufus에서 '시작(Start)' 버튼을 누르면 "ISO 이미지 모드로 쓰기"와 "DD 이미지 모드로 쓰기" 중 선택하라는 팝업이 나타납니다.
칼리 리눅스 공식 문서에서는 DD 이미지 모드(Disk Dump Mode)를 강력히 권장합니다. DD 모드는 원본 ISO 파일의 파티션 구조를 비트(Bit) 단위로 USB에 똑같이 복제하는 방식(Raw Copy)으로, 데비안 기반 리눅스에서 부팅 오류를 99% 예방할 수 있는 가장 완벽한 방법입니다. (단, 완료 후 윈도우에서 USB 용량이 이상하게 보이거나 접근이 안 될 수 있으나 정상입니다.)

3 GRUB 부트 로더 메뉴 분석

USB를 꽂고 PC를 재부팅하면서 F2, F12, Del 키 등을 눌러 BIOS/UEFI 부트 메뉴로 진입한 뒤, USB 드라이브를 최우선 부팅 순위로 선택합니다. 정상적으로 부팅되면 칼리 리눅스의 GRUB(GRand Unified Bootloader, 통합 부트 로더) 화면이 나타나며 여러 선택지가 제공됩니다.

  • ① Live system (amd64) 기본값

    가장 기본적인 라이브 모드입니다. 재부팅 시 저장했던 파일, 바탕화면 설정, 다운로드한 해킹 툴 등이 모두 삭제됩니다. 타인의 PC나 공용 PC에서 안전하게 진단 작업을 수행하고 흔적 없이 빠져나와야 할 때 선택합니다.

  • ② Live system (persistence)

    USB 내부에 별도로 생성해 둔 퍼시스턴스(Persistence, 지속성/영구 저장) 파티션에 작업 내역을 실시간으로 기록하고 보존하는 모드입니다. 자신만의 휴대용 해킹 OS로 만들기 위해 가장 많이 사용하는 방식입니다. (단, 사전 파티션 세팅 작업이 필요하며 다음 섹션에서 다룹니다.)

  • ③ Live system (encrypted persistence)

    엔터프라이즈 레드팀(Red Team)의 표준 방식입니다. 퍼시스턴스 파티션을 LUKS(Linux Unified Key Setup, 리눅스 통합 키 설정) 방식의 강력한 암호화 기술로 보호합니다. 침투 테스터가 현장에서 USB를 분실하더라도, 비밀번호를 모르면 내부의 민감한 해킹 결과물이나 고객사의 취약점 리포트를 절대 열람할 수 없게 만듭니다.

4 데이터 영구 저장을 위한 퍼시스턴스(Persistence) 파티션 구축

단순히 부트 메뉴에서 Live system (persistence)를 선택한다고 해서 바로 데이터가 저장되는 것은 아닙니다. USB의 남는 빈 공간을 리눅스 전용 파일 시스템으로 포맷하고, 칼리 OS에게 "이 공간을 저장소로 써라"라고 명령을 내려주는 사전 작업이 반드시 1회 필요합니다.

[도식 2] 칼리 라이브 USB의 논리적 파티션 분할 구조

퍼시스턴스 구축 터미널(CLI) 작업 시퀀스

먼저 일반 Live system (amd64) 모드로 부팅한 후, 터미널을 열고 아래 명령어들을 순차적으로 입력합니다.

# 1. 루트(최고 관리자) 권한으로 전환
kali@kali:~$ sudo su -

# 2. 파티션 관리 도구(fdisk)를 열어 남은 공간에 새 파티션 생성 (보통 USB는 /dev/sdb 또는 /dev/sdc로 잡힘)
# (주의: fdisk 내에서 'n' 입력 -> Enter(디폴트) -> Enter(디폴트) -> Enter(디폴트) -> 'w' 입력으로 저장)
root@kali:~# fdisk /dev/sdb

# 3. 새로 생성된 파티션(/dev/sdb3 가정)을 리눅스 전용 파일 시스템인 ext4로 포맷하고, 라벨(Label) 이름을 'persistence'로 지정
root@kali:~# mkfs.ext4 -L persistence /dev/sdb3

# 4. 포맷된 파티션을 임시 디렉토리(/mnt/my_usb)에 마운트(연결)
root@kali:~# mkdir -p /mnt/my_usb
root@kali:~# mount /dev/sdb3 /mnt/my_usb

# 5. 퍼시스턴스 환경설정 파일 생성: USB 전체('/')를 통합(union)하여 사용하겠다는 명령을 파일에 기록
root@kali:~# echo "/ union" > /mnt/my_usb/persistence.conf

# 6. 마운트 해제 후 시스템 재부팅
root@kali:~# umount /dev/sdb3
root@kali:~# reboot
명령어 및 용어 분석:
  • sudo su - : 일반 사용자 계정에서 잠시 root(최고 관리자) 권한으로 세션을 완전히 전환합니다. (디스크 포맷 등은 시스템 코어 작업이므로 root 권한이 필수입니다.)
  • mkfs.ext4 (Make File System ext4): 리눅스 표준 파일 시스템인 ext4 형식으로 디스크를 포맷하는 명령어입니다. -L persistence 옵션은 칼리 OS가 부팅 시 저장소로 인식할 수 있도록 파티션의 명찰(Label)을 붙여주는 핵심입니다.
  • mount (마운트): 물리적인 하드 디스크나 USB 파티션을 리눅스 폴더 트리의 특정 폴더(예: /mnt/my_usb)에 연결하여 파일 접근을 가능하게 하는 명령어입니다.
  • echo "/ union" > persistence.conf : 칼리 리눅스의 마운트 시스템(OverlayFS)에게 원래의 읽기 전용 OS 영역과 방금 만든 쓰기 가능 파티션을 union(병합)하여 투명하게 하나처럼 사용하라고 지시하는 설정 파일을 생성합니다.
검증(Validation) 및 실무 적용: 재부팅 후 부트 메뉴에서 Live system (persistence)로 접속합니다. 바탕화면에 test.txt 파일을 만들고 다시 재부팅했을 때 그 파일이 그대로 남아있다면 퍼시스턴스 환경 구축이 완벽하게 성공한 것입니다. 이제 이 USB 하나만 있으면 전 세계 어떤 PC에 꽂아도 나만의 해킹 설정이 적용된 워크스테이션으로 변신합니다.

최종 요약

라이브(Live) USB 환경은 하드웨어를 직접 제어하여 무선 해킹과 같은 고급 기법을 가능하게 하고, 호스트 OS에 흔적을 남기지 않는 탁월한 익명성을 제공합니다. 여기에 퍼시스턴스(Persistence) 파티션 기술을 결합하면, 휘발성이라는 라이브 OS의 한계를 극복하고 일관된 침투 테스트 작업 환경을 유지하는 가장 이상적인 휴대용 무기고(Portable Arsenal)가 완성됩니다.
