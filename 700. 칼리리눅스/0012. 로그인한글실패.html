칼리 리눅스(Kali Linux) 한글 깨짐 및 로그인 장애(Login Failure) 트러블슈팅 가이드

인프라스트럭처 아키텍트 관점에서의 OS 부팅 장애 원인 분석 및 GRUB 커널 복구 기법

운영체제 설치 직후 로그인 화면의 문자가 모두 네모 박스(□□□)로 깨져서 출력되고, 키보드의 숫자 키패드가 작동하지 않으며, 심지어 TTY(Teletypewriter: 텍스트 기반 가상 콘솔)로 전환해도 설정한 비밀번호가 인식되지 않는 현상은 리눅스 입문자들이 겪는 가장 대표적이고 당혹스러운 장애입니다. 본 교재에서는 재설치(Reinstallation) 없이 시스템 코어에 접근하여 문제를 복구하는 전문적인 방법과, 이러한 장애를 원천 차단하는 아키텍트의 설치 Best Practice(모범 사례)를 함께 제시합니다.

1 장애 발생의 아키텍처적 원인 분석 (Root Cause Analysis)

비밀번호가 틀렸다고 나오는 현상과 한글이 깨지는 현상은 서로 다른 두 가지 시스템적 요인이 결합된 결과입니다. 시스템의 오류가 아니라 현지화(Localization) 패키지 누락과 디스플레이 매니저(Display Manager)의 기본 보안 정책 때문입니다.

[그림 1] CJK(중국어/일본어/한국어) 폰트가 누락되어 문자가 깨지는 폰트 렌더링 오류 화면

원인 1: 한글 폰트(fonts-nanum) 패키지 누락

설치 과정에서 언어를 '한국어'로 선택했지만, 설치 이미지가 오프라인(Offline) 전용 ISO이거나 네트워크 미러(Network Mirror) 사이트 연결에 실패했을 경우 발생합니다. 리눅스 코어는 한국어로 출력하려 하지만, 화면에 글자를 그릴 '나눔 고딕' 등의 한글 폰트 파일이 하드디스크에 없기 때문에 모든 글자가 □□□(일명 깍두기 현상, Mojibake)로 출력됩니다.

원인 2: 키보드 레이아웃 맵핑 및 NumLock 해제

칼리 리눅스의 기본 로그인 화면 관리자인 LightDM(Light Display Manager)은 보안 및 시스템 호환성을 이유로 부팅 시 키보드의 NumLock을 기본적으로 OFF(해제) 상태로 만듭니다. 또한 한국어 자판으로 설정된 경우 Shift 키를 누르고 입력하는 특수문자(예: @, ! 등)의 맵핑(Mapping)이 영문 키보드와 미세하게 다르게 인식되어 사용자는 맞게 쳤다고 생각하지만 시스템에는 전혀 다른 문자열이 입력되는 상태가 됩니다.

* 디스플레이 매니저(Display Manager): 리눅스 부팅 프로세스 마지막 단계에서 사용자에게 그래픽 로그인 화면을 띄워주고, 아이디/비밀번호를 검증한 뒤 데스크톱 환경(XFCE 등)으로 세션을 넘겨주는 프로그램입니다. (칼리 리눅스는 LightDM을 사용합니다.)

2 해결 1단계: GRUB(그럽) 부트 로더를 통한 싱글 유저 모드 패스워드 초기화

비밀번호를 계속 틀려서 TTY(텍스트 콘솔)에서도 로그인이 불가능하다면 재설치를 고려하기 전, 리눅스 커널(Kernel)에 직접 개입하여 패스워드를 강제로 초기화하는 기법을 사용합니다. 이는 시스템 관리자 및 해커들이 물리적 서버의 권한을 탈취할 때 사용하는 기초적인 싱글 유저 모드(Single User Mode) 진입 기술입니다.

[그림 2] 부팅 시 GRUB 메뉴에서 커널 파라미터(Kernel Parameters)를 수정하는 화면

커널 레벨 패스워드 초기화 시퀀스

  1. 부팅 시점 개입: 가상 머신(VM)을 재부팅하고, 파란색 칼리 부팅 선택 메뉴인 GRUB(Grand Unified Bootloader: 통합 부트 로더) 화면이 나타나면 방향키를 눌러 카운트다운을 멈춥니다.
  2. 편집 모드 진입: 첫 번째 메뉴(Kali GNU/Linux)가 선택된 상태에서 키보드의 e (edit의 약자) 키를 누릅니다.
  3. 커널 파라미터 수정: 복잡한 영문 텍스트 화면이 나타납니다. 방향키를 아래로 내려 linux 로 시작하는 긴 줄을 찾습니다. 그 줄의 맨 끝(보통 ro quiet splash라고 적혀 있음)으로 이동합니다.
  4. 명령어 치환: ro(Read-Only: 읽기 전용)를 rw(Read-Write: 읽고 쓰기)로 변경하고, 맨 끝에 init=/bin/bash 를 추가합니다.
    변경 전: linux /boot/vmlinuz-... root=UUID=... ro quiet splash
    변경 후: linux /boot/vmlinuz-... root=UUID=... rw quiet splash init=/bin/bash
  5. 부팅 강제 실행: Ctrl + x 또는 F10 키를 눌러 수정된 커널 설정으로 부팅합니다.
# GUI 로그인 창 없이 검은색 화면에 root@(none):/# 프롬프트가 떨어집니다.
# 이제 시스템의 신적인 권한(root)을 얻었으므로 일반 사용자(예: student)의 비밀번호를 매우 단순하게(예: 1234) 바꿉니다.
root@(none):/# passwd student
# 새로운 비밀번호 입력 (화면에 보이지 않으나 입력되고 있음)
root@(none):/# sync
root@(none):/# exec /sbin/init

🚨 보안 아키텍처 관점에서의 취약점 경고

위와 같이 관리자 아이디나 비밀번호를 몰라도 재부팅만으로 시스템을 장악할 수 있는 이유는 물리적 접근(Physical Access) 통제가 뚫렸기 때문입니다. 실무 서버 환경에서는 이러한 공격을 막기 위해 GRUB 부트 로더 자체에 강력한 비밀번호(GRUB Password)를 설정하여 함부로 e 키를 눌러 편집 모드에 진입하지 못하도록 방어 아키텍처를 구성해야 합니다.

3 해결 2단계: 네트워크를 통한 한글 폰트(fonts-nanum) 패키지 복구

비밀번호를 특수문자가 없는 단순한 숫자(예: 1234)로 초기화한 뒤 정상 부팅을 하면, 로그인 시 키보드 레이아웃 오류의 영향을 받지 않고 데스크톱 환경으로 진입할 수 있습니다. 여전히 글자는 깨져있지만, 눈치껏 검은색 터미널(Terminal) 아이콘을 찾아 실행한 뒤 폰트를 다운로드해야 합니다.

[그림 3] 폰트가 깨진(□□□) 상태에서 터미널을 열어 나눔 고딕 폰트 패키지를 설치하는 복구 과정
# 1. 터미널을 열고 인터넷 저장소에서 최신 패키지 목록을 업데이트합니다.
student@kali:~$ sudo apt update

# 2. 시스템의 기본 한글 폰트인 '나눔 폰트' 패키지를 강제로 설치합니다.
student@kali:~$ sudo apt install -y fonts-nanum

# 3. 설치가 완료되면 시스템을 재부팅합니다.
student@kali:~$ sudo reboot

💡 네트워크 연결 실패 시 대처법

만약 apt update 실행 시 오류가 발생한다면, 가상 머신(VirtualBox)의 네트워크 설정이 인터넷이 단절된 '호스트 전용(Host-Only)'으로 되어 있는지 확인하십시오. 폰트를 다운로드하려면 일시적으로 네트워크 설정을 'NAT(Network Address Translation: 네트워크 주소 변환)' 모드로 변경하여 가상 머신이 외부 인터넷과 통신할 수 있도록 만들어야 합니다.

4 근본적인 해결책: 인프라스트럭처 아키텍트의 OS 설치 Best Practice

복구 과정이 너무 복잡하여 결국 재설치(Reinstallation)를 결심했다면, 동일한 오류를 반복하지 않기 위한 글로벌 IT 엔지니어들의 모범 사례(Best Practice)를 따라야 합니다. 핵심 원칙은 "시스템 핵심부는 무조건 영어로 설치하고, 사용자 환경만 나중에 한국어로 튜닝한다"는 것입니다.

[그림 4] OS 설치 언어 및 키보드 레이아웃을 반드시 영문(English)으로 선택하는 단계

안정성 극대화를 위한 설치 3원칙

  • 설치 언어(Language) 및 키보드: 무조건 English(영어) 선택
    리눅스의 주요 시스템 로그(Log) 파일 분석, 오류 메시지 구글링, 각종 해킹 툴의 출력 결과는 모두 영어를 기반으로 설계되어 있습니다. 처음부터 한국어를 강제하면 폴더명(/home/student/바탕화면)까지 한글로 생성되어 터미널(CLI) 작업 시 디렉터리 이동에 엄청난 스트레스를 유발합니다.
  • 비밀번호 생성 규칙: 특수문자 배제 또는 키패드 레이아웃 확인
    초기 설치 시 계정 비밀번호에는 @, #, ! 등의 특수문자 사용을 자제하고, NumLock의 영향을 받는 우측 숫자 키패드 대신 키보드 자판 상단에 위치한 숫자키를 사용하여 아이디와 비밀번호를 입력해야 합니다.
  • 설치 완료 후 사후 현지화 (Post-Installation Localization)
    영문으로 완벽하게 시스템이 구동됨을 확인한 뒤, 터미널에서 dpkg-reconfigure locales 명령어를 통해 시스템 언어 팩(ko_KR.UTF-8)을 추가하고 한글 입력기(fcitx 등)를 세팅하는 것이 아키텍처 관점에서 가장 안정적인 배포 방식입니다.

과정 요약

본 문서를 통해 운영체제 부팅 시 발생하는 폰트 렌더링 및 키보드 맵핑 오류의 원인을 분석하고, GRUB 부트 로더를 통한 싱글 유저 모드(Single User Mode)의 강력한 패스워드 초기화 기법을 학습했습니다. 시스템에 장애가 발생했을 때 당황하여 맹목적으로 재설치를 하기보다는, 리눅스 커널 아키텍처의 허점을 역이용하여 문제를 해결하는 엔지니어의 논리적 트러블슈팅(Troubleshooting) 역량을 기르시길 바랍니다.
