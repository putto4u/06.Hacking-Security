1. 아키텍처 개요 (Architecture Overview)

칼리 리눅스(Kali Linux) 환경에서 메타스플로잇 프레임워크(Metasploit Framework, 침투 테스트 자동화 도구)를 포함한 다양한 보안 도구의 결과는 대개 PostgreSQL(PostgreSQL, 오픈 소스 객체-관계형 데이터베이스 시스템)에 저장됩니다. 이를 통해 방대한 스캔 결과와 취약점 데이터를 체계적으로 관리하고 조회할 수 있습니다.

[도식: 보안 도구와 PostgreSQL 데이터 연동 구조]

Nmap, Metasploit, Vuln Scan
Save Results
PostgreSQL DB Engine
Storage & Management
Analysis
보안 분석가 (Security Analyst)
psql, Report Generation
* 주석: PostgreSQL은 고급 기능을 지원하는 객체-관계형 데이터베이스 시스템(ORDBMS)으로, 보안 도구의 대규모 데이터를 안전하게 처리하기 위한 백엔드 엔진으로 사용됩니다.

2. 실습 구성 요소 (Components)

NODE 01

Kali Linux Desktop

  • Operating System: Kali Linux 2024.x
  • CLI Tool: psql (PostgreSQL 대화형 터미널)
  • Database Client: Metasploit Database Integration
NODE 02

PostgreSQL Server

  • Service Engine: PostgreSQL v15+
  • Default Port: TCP/5432
  • Listening Mode: Localhost (Default)

3. 실증 및 구현 (Implementation)

STEP 01

PostgreSQL 서비스 관리 및 접속 설정

서비스의 활성화 상태를 점검하고 필요시 가동합니다.

Console
# 서비스 상태 확인
$ sudo systemctl status postgresql
Active: active (exited) since ...
# 포트 리스닝 상태 확인
$ ss -ant | grep 5432
LISTEN 0 244 127.0.0.1:5432 0.0.0.0:*
⚠️ 주의(Caution): AWS RDS 등 클라우드 환경에서는 Hourly Rate(시간당 요금) 및 Data Transfer(데이터 전송) 비용이 발생하므로 로컬 환경인지 반드시 대조하십시오.
STEP 02

psql 터미널 접속 및 메타 명령어 탐색

psql(PostgreSQL Interactive Terminal)을 통해 DB 내부 구조를 파악합니다.

Console
$ sudo -u postgres psql
postgres=# \l -- DB 목록
postgres=# \c msf_db -- DB 연결
msf_db=# \dt -- 테이블 목록
명령어 기능 설명 비고
\l 전체 데이터베이스 목록 출력 시스템 DB 포함
\c [name] 특정 데이터베이스로 연결 전환 세션 이동
\dt 현재 DB 내의 사용자 테이블 목록 스캔 결과 테이블 확인용
\d [table] 특정 테이블의 스키마 상세 보기 컬럼명 및 데이터 타입 확인
STEP 03

SQL을 이용한 결과 조회 및 분석

스캔된 호스트 및 취약점 정보를 SQL(Structured Query Language) 구문으로 조회합니다.

SQL Query
SELECT address, os_name, state FROM hosts;
address | os_name | state
---------+---------+-------
192.168.1.10 | Linux | up
SELECT name, info FROM vulns ORDER BY created_at DESC;
📝 주석: Schema(스키마)
데이터베이스에서 자료의 구조, 표현 방법, 관계 등을 정의한 형식을 말합니다. 이를 숙지해야 효율적인 분석 쿼리 작성이 가능합니다.
STEP 04

데이터 추출 및 외부 리포트화 (Export)

분석 결과를 보고서 작성을 위해 CSV(Comma Separated Values, 쉼표로 구분된 값) 파일로 내보냅니다.

Export Command
$ psql -U postgres -d msf_db -c "COPY (SELECT * FROM hosts) TO '/tmp/hosts_report.csv' WITH CSV HEADER;"

* 팁: \copy 명령어는 psql 클라이언트 세션 내에서도 실행 가능합니다.

4. 아키텍트 인사이트 (Expert Insight)

GUI 기반 관리 도구: pgAdmin4

CLI(Command Line Interface) 환경이 익숙하지 않다면 pgAdmin4(PostgreSQL Administration Tool)를 설치하십시오. 웹 브라우저 기반의 시각적 인터페이스를 통해 복잡한 테이블 관계를 한눈에 파악하고 데이터를 관리할 수 있습니다.

백업 정책 및 인덱싱(Indexing)

중요 침투 테스트 데이터가 저장된 DB는 pg_dump 명령으로 주기적 백업을 수행해야 합니다. 또한, 데이터가 수만 건 이상 누적될 경우 특정 컬럼에 인덱스(Index)를 생성하여 조회 속도를 최적화하는 것이 아키텍트의 필수 역량입니다.
