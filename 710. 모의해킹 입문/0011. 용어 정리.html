모의 해킹(Penetration Testing) 핵심 용어 및 메커니즘 심층 해설서

사이버 킬 체인(Cyber Kill Chain) 전 단계에 걸친 오펜시브 시큐리티 전문 용어 및 아키텍처 원리 분석

인프라스트럭처 보안을 설계하고 방어 체계를 구축하기 위해서는, 공격자가 사용하는 전문 용어와 그 이면에 숨겨진 기술적 메커니즘을 정확히 통찰해야 합니다. 본 교재는 모의 해킹 실무에서 반드시 알아야 할 핵심 용어들을 공격 단계별로 분류하고, 직관적인 아키텍처 도식과 함께 심층적으로 해설합니다.

Phase 1. 정보 수집 및 정찰 (Reconnaissance)

OSINT (Open Source Intelligence: 공개 출처 정보)

공격 대상(타겟)의 네트워크에 직접 접속하여 패킷을 보내지 않고, 합법적으로 접근 가능한 인터넷상의 공개된 데이터들을 수집하여 타겟의 취약점을 유추하는 활동입니다. 검색 엔진(Google, Shodan), 소셜 미디어(LinkedIn, Facebook), 공공 데이터베이스(WHOIS, DNS 기록), 다크웹 유출 정보 등을 조합하여 타겟 기업의 인프라 지도와 임직원 명부를 작성하는 사전 정찰의 핵심입니다.

[도식 1] 타겟에 직접 접근하지 않고 외곽에서 정보를 긁어모으는 OSINT 수집 구조

* 패킷(Packet): 네트워크를 통해 전송되는 데이터의 기본 단위입니다. 큰 데이터는 여러 개의 패킷으로 잘게 쪼개져 전송됩니다.
* 다크웹(Dark Web): 일반적인 검색 엔진으로는 찾을 수 없고, 특수한 웹 브라우저(Tor 등)를 사용해야만 접근할 수 있는 익명성이 보장된 인터넷 영역입니다.

Scanning (Scanning: 스캐닝 및 탐색)

OSINT를 통해 수집된 타겟의 IP 주소에 능동적(Active)으로 네트워크 패킷을 보내어, 현재 살아있는 서버(Host)가 무엇인지, 방화벽이 어느 포트(Port)를 열어두었는지, 해당 포트에서 돌아가는 서비스(데몬)의 종류와 버전이 무엇인지 매핑(Mapping)하는 작업입니다. 모의 해킹 실무에서는 거의 100% Nmap(엔맵)이라는 도구를 사용합니다.

[도식 2] Nmap을 이용해 타겟 서버의 열린 포트와 동작 중인 서비스 버전을 식별하는 스캐닝

💰 클라우드 인프라 실무 경고: AWS, Azure 등의 퍼블릭 클라우드 환경에 공격 서버를 두고 타겟을 향해 광범위한 포트 스캐닝(전체 65,535개 포트 스캔 등)을 수행할 경우, 아웃바운드 트래픽 폭증으로 인해 막대한 네트워크 과금(1GB당 약 $0.09)이 발생할 수 있으므로 주의가 필요합니다.
* 포트(Port): 하나의 컴퓨터 안에서 웹, 이메일, 파일 전송 등 여러 가지 서비스가 동시에 돌아갈 때, 데이터가 찾아가야 할 애플리케이션의 논리적 출입구 번호입니다. (예: 80번=웹 서비스, 22번=원격 접속)
* 데몬(Daemon): 리눅스 서버의 백그라운드에서 사용자의 요청을 기다리며 항상 실행되고 있는 서비스 프로그램입니다. (윈도우의 '서비스'와 동일한 개념)
Phase 2. 취약점 분석 (Vulnerability Analysis)

CVE & CVSS (공통 취약점 및 노출 / 공통 취약점 등급 시스템)

스캐닝을 통해 타겟 서버에서 'Apache 2.4.49' 버전이 돌아간다는 것을 알았다면, 해당 버전에 해킹 가능한 구멍이 있는지 찾아야 합니다. CVE(Common Vulnerabilities and Exposures)는 전 세계에 공개된 보안 취약점들에 부여되는 고유한 일련번호(예: CVE-2021-41773)입니다. CVSS(Common Vulnerability Scoring System)는 그 취약점이 얼마나 위험한지를 0점에서 10점 사이의 숫자로 정량화한 등급 시스템입니다.

[도식 3] 고유 식별자(CVE)와 그 위험성을 정량화한 점수(CVSS)의 관계

Zero-Day (Zero-Day Vulnerability: 제로 데이 취약점)

소프트웨어 개발사가 자사 프로그램에 보안 구멍이 있다는 사실을 인지하지 못했거나, 아직 패치(Patch, 방어 업데이트)를 개발하지 못하여 대응할 수 있는 시간이 '0일(Zero Day)'인 상태의 초고위험 취약점을 뜻합니다. 이 상태에서 가해지는 해킹 공격을 제로 데이 공격(Zero-Day Attack)이라 하며, 아무리 뛰어난 방화벽이나 백신이라도 알려지지 않은 공격이므로 방어하기가 극도로 어렵습니다.

💡 아키텍트 실전 팁 (Defense Strategy)

방어자(블루팀) 입장에서 제로 데이 공격을 100% 차단하는 것은 불가능합니다. 따라서 실무에서는 단일 방어선에 의존하지 않고, 심층 방어(Defense in Depth) 아키텍처를 구현해야 합니다. 웹 서버가 뚫리더라도, 내부 DB 서버로 접근하는 망을 분리하고 최소 권한의 원칙을 적용하여 침해 사고의 반경을 최소화하는 설계가 필수입니다.

Phase 3. 익스플로이테이션 (Exploitation: 시스템 공격)

Exploit & Payload (익스플로잇 & 페이로드)

익스플로잇(Exploit)은 소프트웨어의 버그나 보안 취약점을 찔러 시스템의 방어막을 뚫어내는 '공격 코드' 또는 '행위 자체'를 뜻합니다. 반면, 페이로드(Payload)는 익스플로잇이 성공하여 방어막이 뚫린 후, 타겟 시스템 내부에서 실제로 실행되는 악의적인 '핵심 목적 코드'입니다. 비유하자면, 익스플로잇은 목표물을 타격하는 미사일(운반체)이고, 페이로드는 미사일 안에 탑재된 폭약(살상 무기)입니다.

[도식 4] 미사일(Exploit)을 이용해 타겟의 방어막을 뚫고 폭약(Payload)을 터뜨리는 개념도

* 버퍼 오버플로우(Buffer Overflow): 프로그램이 허용된 메모리 공간(버퍼)보다 더 많은 데이터를 입력받을 때, 넘쳐흐른 데이터가 다른 중요 메모리 영역을 덮어써서 공격자의 코드가 실행되게 만드는 대표적인 시스템 해킹 기법입니다.

Reverse Shell vs Bind Shell (리버스 셸과 바인드 셸)

시스템의 제어권을 얻어낸 뒤(Exploit 성공), 해커의 PC와 타겟 서버 간에 명령을 주고받을 통로(Shell)를 연결하는 두 가지 방식입니다.

바인드 셸(Bind Shell): 타겟 서버의 특정 포트를 열어두고(Bind), 해커가 타겟으로 정방향 접속을 시도하는 방식입니다. 하지만 대부분의 기업 방화벽은 외부에서 내부로 들어오는 낯선 접속(Inbound)을 철저히 차단하므로 실무에서는 성공하기 매우 어렵습니다.
리버스 셸(Reverse Shell): 해커가 자신의 PC 포트를 열어두고 기다리면, 공격당한 타겟 서버가 방화벽 안에서 밖으로(Outbound) 역방향으로 해커에게 접속을 시도하는 방식입니다. 일반적인 방화벽은 내부에서 외부로 나가는 통신은 관대하게 허용한다는 맹점을 찌른 기법으로, 현대 모의 해킹의 99%는 이 방식을 사용합니다.

[도식 5] 방화벽(Inbound 차단)을 우회하기 위해 타겟이 스스로 공격자에게 접속하게 만드는 리버스 셸 아키텍처

Phase 4. 웹 애플리케이션 공격 (Web Application Attacks)

SQL Injection (SQLi: SQL 인젝션)

사용자가 입력하는 웹사이트의 폼(로그인 창, 검색 창 등)에 일반적인 글자 대신 악의적인 데이터베이스 조작 언어(SQL)를 삽입(Injection)하여, 백엔드 데이터베이스 서버가 해커의 의도대로 오작동하게 만드는 치명적인 웹 공격입니다. 비밀번호를 몰라도 관리자로 로그인하거나(인증 우회), 쇼핑몰의 전체 고객 개인정보를 통째로 빼내는(데이터 탈취) 등 파괴력이 엄청납니다.

[도식 6] 논리적 허점을 찔러 무조건 참(True)을 만들어 인증을 우회하는 SQL 인젝션 공격 구조

* 백엔드(Back-end): 사용자의 눈에 보이지 않는 서버와 데이터베이스(DB) 환경을 말하며, 데이터의 저장과 핵심 논리 처리를 담당합니다.
* Prepared Statement (준비된 문장): 사용자의 입력값을 SQL 명령어(코드)로 인식하지 않고, 단순한 '문자열(데이터)'로만 처리하도록 분리하는 가장 강력한 SQL 인젝션 방어 시큐어 코딩 기법입니다.

XSS & CSRF (교차 사이트 스크립팅 / 사이트 간 요청 위조)

웹 해킹의 양대 산맥입니다. 두 공격 모두 사용자의 브라우저를 매개로 하지만 타겟이 다릅니다.
XSS (Cross-Site Scripting): 공격자가 웹 게시판 등에 악성 자바스크립트(JavaScript)를 삽입해 둡니다. 일반 사용자가 그 게시글을 클릭하여 읽는 순간, 사용자의 브라우저에서 스크립트가 강제 실행되어 사용자의 신분증 역할을 하는 세션 쿠키(Cookie)가 공격자에게 털리게 됩니다. (타겟: 사용자)
CSRF (Cross-Site Request Forgery): 사용자가 웹사이트에 로그인된 상태(인증된 상태)를 악용합니다. 해커가 보낸 악성 링크를 누르면, 사용자의 의도와 무관하게 사용자의 권한으로 비밀번호 변경, 자금 이체 등의 요청(Request)이 서버로 전송됩니다. 서버는 정상 사용자가 요청한 것으로 속게 됩니다. (타겟: 서버의 권한 검증)

Phase 5. 포스트 익스플로이테이션 (Post-Exploitation: 침투 후 후속 작업)

Privilege Escalation & Lateral Movement (권한 상승과 측면 이동)

시스템에 최초로 침투(Initial Access)하더라도 보통은 권한이 제한된 '일반 웹 사용자' 권한만 얻게 됩니다.
Privilege Escalation (권한 상승): 획득한 일반 계정의 권한을 관리자(Linux의 root, Windows의 SYSTEM) 권한으로 끌어올리는 작업입니다. 시스템 파일의 취약한 권한 설정(SUID 등)이나 커널 버그를 악용하여 모든 시스템을 통제할 수 있는 신적인 권한을 확보합니다.
Lateral Movement (측면 이동 / 내부망 이동): 하나의 서버를 장악한 뒤, 이를 교두보(Pivot) 삼아 내부망에 연결된 데이터베이스 서버, 인사팀 PC, 파일 서버 등 다른 내부 시스템으로 마치 게처럼 옆으로 전염되어 퍼져나가는 행위입니다. (APT 공격의 핵심)

[도식 7] 수직적인 권한 상승과 내부망을 휘젓고 다니는 수평적인 측면 이동의 2차원 침투 구조

C2 & Backdoor (명령 및 제어 서버와 백도어)

Backdoor (백도어): 해커가 시스템을 한 번 장악한 뒤, 나중에 로그인 과정 없이 다시 쉽게 들어오기 위해 보안 통제를 무력화하여 몰래 만들어둔 '뒷문'입니다. 정상적인 시스템 파일로 위장하여 숨어있습니다.
C2 (Command and Control Server, C&C 서버): 내부망에 잠입한 악성코드나 좀비 PC(Botnet)들에게 "데이터를 압축해서 보내라", "특정 서버를 공격하라" 등 명령을 내리고 탈취한 데이터를 수집하는 해커의 중앙 사령부 서버입니다. 침해 사고 대응(IR) 시 이 C2 서버와의 통신 로그를 찾아 차단하는 것이 방어자의 최우선 과제입니다.

💡 아키텍트 실전 팁 (Defense Strategy)

클라우드 환경이나 온프레미스에서 백도어와 C2 통신을 차단하려면, 외부에서 들어오는 통신(Inbound) 차단뿐만 아니라 내부 서버가 외부 인터넷으로 나가는 통신(Outbound) 정책을 화이트리스트(Whitelist) 기반으로 강력히 통제해야 합니다. 즉, 웹 서버가 외부의 알 수 없는 IP(C2 서버)로 몰래 데이터를 빼돌리려고 역방향 통신을 시도할 때 방화벽이 이를 원천 차단해야 합니다.
