Nmap(Network Mapper) 심층 분석 및 실전 스캐닝 기법

정보 수집(Reconnaissance)의 핵심 정찰 자산이자 가장 강력한 오픈소스 네트워크 탐색 및 보안 감사 도구의 완벽한 이해

1 Nmap의 소개와 핵심 논리 아키텍처 🌐

정의: Nmap이란 무엇인가?

Nmap(Network Mapper, 네트워크 매퍼)은 고든 라이온(Gordon Lyon)이 개발한 오픈소스 기반의 네트워크 탐색 및 보안 진단 도구입니다. 단순히 IP가 살아있는지 확인하는 것을 넘어, 대상 시스템에 정교하게 조작된 원시 패킷(Raw Packet)을 전송하여 운영체제(OS), 제공되는 서비스, 방화벽 상태 등을 심층적으로 분석해 냅니다.

실무적 위상과 가치

모의 해킹의 5단계 중 취약점 분석(Scanning & Enumeration) 단계에서 99% 이상의 비중으로 사용되는 산업 표준(De facto standard) 도구입니다. 레드팀(Red Team, 공격자 관점)에게는 타겟의 약점을 찾는 나침반이며, 블루팀(Blue Team, 방어자 관점)에게는 내부 자산의 취약점을 선제적으로 진단하는 필수 감사 도구입니다.

[도식 1] Nmap의 원시 패킷(Raw Packet) 프로빙(Probing) 및 응답 분석 아키텍처

Nmap이 진단하는 4가지 핵심 정보

  • Host Discovery (호스트 탐색): 해당 IP 대역에 실제로 켜져 있는 컴퓨터(Host)가 어느 것인지 찾아냅니다.
  • Port Scanning (포트 스캐닝): 발견된 호스트에 어떤 논리적 포트(Port)가 열려있는지(Open) 검사합니다.
  • Version Detection (버전 탐지): 열린 포트에서 어떤 애플리케이션 서비스가 돌아가고 있으며, 그 소프트웨어의 버전(Version)은 무엇인지 식별합니다. (취약점 매칭의 핵심 데이터)
  • OS Detection (운영체제 탐지): TCP/IP 스택의 고유한 지문(Fingerprint)을 분석하여, 대상 시스템의 운영체제가 Windows인지, Linux인지, Mac인지 등을 유추합니다.

칼리 리눅스 설치 직후 Nmap 첫 실행 가이드

칼리 리눅스에는 Nmap이 기본적으로 탑재되어 있어 별도의 설치 과정이 필요하지 않습니다. 시스템 부팅 후 CLI(Command Line Interface, 명령 줄 인터페이스) 환경인 터미널(Terminal)을 열고 즉시 진단을 시작할 수 있습니다.

# 1. 설치된 Nmap의 버전 및 컴파일 정보 확인
kali@kali:~$ nmap -V

# 2. 도움말 및 주요 옵션 리스트 요약 확인
kali@kali:~$ nmap -h

# 3. Nmap 재단에서 허용하는 합법적 테스트 서버(scanme.nmap.org) 대상 기본 스캔
# (주의: 스텔스 스캔 등 패킷을 직접 조작하는 고급 기능을 사용하려면 반드시 sudo를 붙여야 합니다.)
kali@kali:~$ sudo nmap scanme.nmap.org
핵심 용어 설명:
  • CLI (Command Line Interface, 명령 줄 인터페이스): 마우스를 사용하는 그래픽 화면(GUI)이 아닌, 키보드로 텍스트 명령어를 입력하여 컴퓨터를 제어하는 환경입니다.
  • sudo (Superuser Do, 최고 관리자 권한 실행): 리눅스에서 일반 사용자가 시스템 코어(네트워크 패킷 제어 등)에 접근하기 위해 임시로 최고 관리자(root)의 권한을 빌려 명령을 실행하게 해주는 명령어입니다.

실행 결과 분석: 위 기본 명령어를 실행하면 약 수십 초 내외로 대상 서버의 열린 포트(Port) 번호, 프로토콜 상태(State), 그리고 서비스(Service) 명칭이 화면에 출력됩니다. 이는 타겟 시스템의 대문이 어디에 열려있는지 확인하는 정보 수집의 가장 첫 번째 단계입니다.

2 호스트 탐색 기법 (Host Discovery) 🔍

본격적인 포트 스캐닝을 하기 전에 대상 네트워크 대역(예: 192.168.1.0/24)에서 살아있는(Alive) 호스트만 빠르게 추려내는 과정입니다. 이를 핑 스캔(Ping Scan)이라고도 부릅니다.

# 포트 검사 없이 핑(Ping) 테스트만으로 살아있는 호스트 찾기 (-sn 옵션)
kali@kali:~$ nmap -sn 192.168.1.0/24

# 방화벽이 ICMP(Ping)를 막았을 때 우회하여 TCP 80 포트로 살아있는지 확인하기 (-PS 옵션)
kali@kali:~$ nmap -sn -PS80 192.168.1.0/24

💡 실전 팁: 방화벽에 막힌 Ping 우회

대부분의 최신 윈도우(Windows) 운영체제나 기업용 방화벽은 기본적으로 ICMP(Internet Control Message Protocol, 인터넷 제어 메시지 프로토콜)의 Echo Request(일반적인 Ping 명령어)를 무시(Drop)하도록 설정되어 있습니다. 따라서 단순히 ping이 안 나간다고 호스트가 죽어있다고 판단하면 안 됩니다. -PS (TCP SYN Ping)나 -PA (TCP ACK Ping)를 사용하여 웹 포트(80, 443) 등을 두드려 반응을 확인하는 우회 기법이 실무에서 필수적입니다.

3 포트 상태 분석 (Port States) 및 스캐닝 타입 🚪

Nmap은 대상 시스템의 포트 상태를 단순히 "열림/닫힘"이 아니라 6가지의 디테일한 상태(States)로 분류하여 리포팅합니다.

포트 상태 (State) 상태의 의미 및 기술적 원인 (Description)
Open (열림) 애플리케이션(서비스)이 해당 포트에서 연결을 적극적으로 수락(Accepting)하고 있는 상태입니다. 해커들의 주된 타겟이 됩니다.
Closed (닫힘) 포트에 접근은 가능하지만(방화벽이 막고 있지 않음), 현재 연결을 받아주는 서비스 프로그램이 실행되고 있지 않은 상태입니다.
Filtered (필터링됨) Nmap이 포트의 열림/닫힘 여부를 판단할 수 없는 상태입니다. 중간에 있는 방화벽(Firewall)이나 라우터 룰에 의해 패킷이 차단(Drop)되어 응답이 돌아오지 않기 때문입니다.
Unfiltered 포트에 접근은 가능하지만(방화벽 통과), 열려있는지 닫혀있는지 명확히 구분되지 않는 상태입니다. (주로 ACK 스캔 시 발생)

* 그 외 open|filtered 및 closed|filtered 상태가 있으나 주로 응답 부족 시 나타나는 불확실한 상태입니다.

[도식 2] 실무에서 가장 많이 비교되는 TCP SYN 스캔(-sS)과 Connect 스캔(-sT)의 차이

4 버전 탐지 및 운영체제 식별 심화 🕵️‍♂️

단순히 Port 80이 열려있다는 정보만으로는 해킹을 할 수 없습니다. 그 포트에서 구동되는 서비스가 Apache 2.4.49 인지, Nginx 1.18.0 인지를 알아야 해당 버전에 존재하는 고유한 취약점(CVE)을 찾아 익스플로잇(Exploit) 할 수 있습니다.

버전 탐지 (-sV)

열려있는 포트에 수천 개의 프로브(Probe) 패킷을 보내고, 돌아오는 배너(Banner) 문자열과 응답 패턴을 분석하여 정확한 소프트웨어와 버전 번호를 식별합니다.

# nmap -sV 192.168.1.100

운영체제(OS) 탐지 (-O)

운영체제마다 네트워크 패킷을 처리하는 고유한 버릇(TCP/IP Fingerprinting)이 있습니다. Nmap은 이를 자사의 방대한 데이터베이스와 대조하여 Windows 11인지, Linux Kernel 5.4인지 유추합니다.

# nmap -O 192.168.1.100

💡 실전 팁: 어그레시브 스캔 (-A)

시간이 촉박할 때 실무에서 가장 즐겨 쓰는 옵션은 -A (Aggressive Scan, 강력한 스캔) 입니다. 이 옵션 하나만으로 운영체제 탐지(-O), 버전 탐지(-sV), 스크립트 스캐닝(-sC), 라우팅 경로 추적(traceroute)을 한 번에 모두 수행합니다. 단, 대상 시스템에 엄청난 양의 패킷과 로그를 발생시키므로 발각될 확률이 매우 높습니다.

5 마법의 무기: Nmap 스크립팅 엔진 (NSE) ⚙️

NSE(Nmap Scripting Engine, Nmap 스크립팅 엔진)는 Nmap을 단순한 포트 스캐너에서 강력한 취약점 진단 도구로 격상시켜 주는 핵심 기능입니다. Lua(루아) 프로그래밍 언어로 작성된 스크립트들을 통해 다양한 해킹/진단 작업을 자동화합니다.

※ Nmap 스크립트 기본 저장 경로: /usr/share/nmap/scripts/

주요 NSE 카테고리 (Categories)

  • vuln
    대상 시스템에 알려진 취약점(Vulnerability)이 존재하는지 실제로 검사합니다. (예: WannaCry 랜섬웨어 취약점인 MS17-010 진단)
  • exploit
    취약점을 단순히 확인하는 것을 넘어, 시스템에 실제 공격(Exploit)을 시도해 버리는 매우 위험한 카테고리입니다.
  • safe
    타겟 시스템을 다운시키거나 트래픽 폭주를 일으키지 않는 안전한(Safe) 정보 수집 스크립트들만 모아둔 카테고리입니다.
# 취약점(vuln) 카테고리의 모든 스크립트를 동원하여 타겟 검사하기
kali@kali:~$ nmap --script vuln 192.168.1.100

# 특정 스크립트(ftp-anon)만 지정하여 익명 FTP 로그인 허용 여부 점검하기
kali@kali:~$ nmap -p 21 --script ftp-anon 192.168.1.100

6 방화벽 우회 기법 및 AWS 클라우드 환경 실무 주의사항 ☁️

방화벽/IDS 회피 기법 (Evasion)

보안 장비의 감시를 피하기 위한 여러 기법이 존재합니다.

  • 디코이 스캔 (-D): 가짜 IP 주소(Decoy, 미끼)들을 섞어서 보내 방어자가 진짜 공격자 IP를 찾기 어렵게 만듭니다. (예: -D 10.0.0.1,10.0.0.2,ME)
  • 패킷 단편화 (-f): TCP 헤더를 잘게 쪼개어 전송함으로써(Fragmentation), 시그니처 기반의 구형 IDS(침입 탐지 시스템)가 패킷의 내용을 조립하고 분석하지 못하도록 회피합니다.

💰 AWS 클라우드 인프라 활용 시 과금 및 정책 경고

실무에서 AWS EC2 인스턴스 내부에 칼리 리눅스를 띄워 외부 타겟을 Nmap으로 스캐닝할 때 발생할 수 있는 크리티컬한 이슈 2가지입니다.

1. 아웃바운드 트래픽 요금 폭탄: Nmap의 -p- (전체 포트 스캔)이나 -A 옵션을 수십 개의 IP 대역에 걸면 막대한 패킷이 외부로 전송됩니다. AWS는 데이터 전송 아웃바운드 시 1GB당 약 $0.09의 과금을 부과하므로, 무분별한 스캔은 비용 상승을 초래합니다.
2. AWS 이용 약관(AUP) 위반: 과거에는 AWS 자산을 대상으로 한 모의 해킹 시 사전에 폼(Form)을 통해 신고해야 했습니다. 현재는 정책이 완화되어 사전 승인 없이 스캔이 가능해졌으나, 디도스(DDoS) 수준의 막대한 패킷 전송, 봇넷 시뮬레이션 등은 여전히 엄격히 금지되며 적발 시 계정이 즉각 영구 정지(Suspend) 될 수 있습니다.

출력 결과물 저장 자동화 옵션

장시간 스캔한 결과가 터미널을 끄면서 날아가는 참사를 방지하기 위해 실무에서는 출력(Output) 옵션을 상시 사용합니다.

  • -oN [파일명]: 터미널에 보이는 그대로 저장 (Normal 형식)
  • -oX [파일명]: 파이썬 등 타 프로그램과 연동하기 쉬운 XML 포맷으로 저장
  • -oA [파일명]: Normal, XML, Grepable 3가지 버전을 한 번에 모두 저장 (가장 권장)

과정 요약

Nmap은 현대 모의 해킹과 네트워크 보안 인프라 감사를 지탱하는 가장 근원적인 척추와도 같은 도구입니다. 포트의 6가지 상태를 정확히 인지하고, TCP SYN과 Connect의 차이 등 프로토콜의 본질을 이해하며 옵션을 자유롭게 통제할 수 있어야 진정한 인프라스트럭처 아키텍트이자 보안 전문가라 할 수 있습니다. 배운 기술은 항상 합법적이고 윤리적인 테두리 내에서만 사용되어야 함을 다시 한번 명심하십시오.

IP 직업훈련 전문가 과정 인프라스트럭처 아키텍트 교재 시리즈
