최신 우분투 환경에서의 Nmap 스캐닝 실시간 탐지 및 로그 분석

현업 시스템 관리 및 방어 관점에서, 공격자가 정보 수집 단계에서 수행하는 포트 스캐닝(Port Scanning)을 식별하는 것은 침해 사고를 예방하는 첫걸음입니다. 본 장에서는 최신 Ubuntu(우분투) 환경에서 내장된 패킷 캡처 도구와 방화벽 로깅 기능을 활용하여 Nmap(Network Mapper, 네트워크 매퍼) 스캐닝을 실시간으로 감지하고 분석하는 방법을 다룹니다.

1. tcpdump를 활용한 실시간 패킷(Packet) 기반 탐지

공격자가 가장 많이 사용하는 Nmap의 SYN 스캔(Stealth Scan, -sS)은 완전한 연결(3-Way Handshake)을 맺지 않아 일반적인 애플리케이션 로그에 남지 않습니다. 이를 탐지하기 위해서는 네트워크 계층에서 패킷을 직접 모니터링해야 합니다.

1-1. 실시간 탐지 명령어 실행

우분투 터미널에서 아래 명령어를 실행하여 들어오는 모든 TCP SYN 요청(연결 시도)을 모니터링합니다.

Ubuntu (Target) 터미널
# 인터페이스(eth0)로 들어오는 패킷 중 TCP SYN 플래그만 설정된 패킷을 실시간 캡처 # -n: IP 주소를 도메인으로 변환하지 않음 (속도 향상) / -nn: 포트 번호를 서비스 이름으로 변환하지 않음 ubuntu@server:~$ sudo tcpdump -i eth0 -nn 'tcp[tcpflags] == tcp-syn'

1-2. Nmap 스캔 탐지 결과 로그 및 해설

공격자가 nmap -sS 192.168.1.100 명령을 실행했을 때, 위 tcpdump 터미널에 실시간으로 출력되는 결과 예시와 각 줄에 대한 상세 분석입니다.

tcpdump 실시간 탐지 로그 출력 화면
# [형식] 시간 | 출발지 IP.포트 > 도착지 IP.포트 | 플래그 | 시퀀스 번호 | 윈도우 크기 14:22:15.123456 IP 192.168.1.50.48123 > 192.168.1.100.22: Flags [S], seq 123456789, win 1024, length 0 // 해설: [14시 22분] 공격자(1.50)가 임의의 포트(48123)에서 대상(1.100)의 SSH 포트(22번)로 SYN([S]) 패킷 전송. 14:22:15.123500 IP 192.168.1.50.48123 > 192.168.1.100.80: Flags [S], seq 123456790, win 1024, length 0 // 해설: [0.000044초 후] 동일 공격자가 대상의 HTTP 포트(80번)로 즉각적인 SYN 패킷 전송. 14:22:15.123555 IP 192.168.1.50.48123 > 192.168.1.100.443: Flags [S], seq 123456791, win 1024, length 0 // 해설: [0.000055초 후] 연이어 대상의 HTTPS 포트(443번)로 SYN 패킷 전송. # [분석 결론] 단일 IP(192.168.1.50)에서 1초 미만의 극히 짧은 시간에 서로 다른 다수의 포트(22, 80, 443 등)로 연결 요청([S])이 발생하는 패턴은 전형적인 '포트 스캐닝(Port Scanning)' 공격 징후입니다.

2. UFW(방화벽) 커널 로그를 통한 스캔 탐지

우분투의 기본 방화벽인 UFW(Uncomplicated Firewall, 복잡하지 않은 방화벽)의 로깅 기능을 활성화하면, 차단된 스캔 패킷을 시스템 커널 로그에서 실시간으로 추적할 수 있습니다.

Ubuntu (Target) 터미널
# 1. UFW 방화벽 활성화 및 로깅 수준을 'low'로 설정 (차단된 패킷 기록) ubuntu@server:~$ sudo ufw enable ubuntu@server:~$ sudo ufw logging low # 2. 커널 로그(/var/log/kern.log)를 실시간으로 추적하여 차단된 스캔 시도 모니터링 ubuntu@server:~$ tail -f /var/log/kern.log | grep -i "UFW BLOCK"
UFW 차단 로그 결과 화면
# [형식] 날짜/시간 | 호스트명 | 커널: [UFW 차단] | 입력 인터페이스 | 출발지 MAC | 목적지 MAC | 출발지 IP | 목적지 IP | 프로토콜 | 도착지 포트 Oct 25 14:30:10 server-kernel: [UFW BLOCK] IN=eth0 OUT= MAC=00:11:22... SRC=192.168.1.50 DST=192.168.1.100 PROTO=TCP DPT=3306 WINDOW=1024 RES=0x00 SYN URGP=0 // 해설: 공격자(192.168.1.50)가 데이터베이스 포트(3306)를 스캔하려 했으나 UFW 방화벽 룰에 의해 차단(BLOCK)되었으며, 해당 패킷이 SYN 플래그를 달고 있음이 기록됨.

3. 주요 용어 및 이니셜 풀이

이니셜 / 용어 전체 영어명 (Full Name) 한글 설명
Nmap Network Mapper (네트워크 매퍼) 네트워크 탐색 및 보안 감사를 위해 사용되는 강력한 오픈소스 스캐닝 도구입니다.
TCP Transmission Control Protocol (전송 제어 프로토콜) 신뢰성 있는 데이터 전송을 보장하기 위해 연결을 맺고 데이터를 주고받는 통신 규약입니다.
UFW Uncomplicated Firewall (복잡하지 않은 방화벽) 우분투에서 iptables(아이피테이블즈)를 보다 쉽게 관리할 수 있도록 제공하는 기본 방화벽 설정 도구입니다.
[주석]
* tcpdump (티씨피덤프): 명령 줄에서 네트워크 트래픽을 가로채고 분석하여 패킷의 헤더 정보를 텍스트로 출력해주는 가장 대표적인 패킷 스니퍼(Packet Sniffer) 도구입니다.
* SYN 플래그 (Synchronize): TCP 통신을 시작할 때 양단 간의 세션을 연결하기 위해 가장 처음 보내는 '연결 요청' 신호입니다. 스캐너들은 대상의 포트가 열려있는지 확인하기 위해 이 SYN 패킷만을 보내고 응답을 기다립니다.
[클라우드 과금 및 실무 인프라 팁]
온프레미스(On-Premise) 환경이나 개인 가상머신에서는 tcpdump를 자유롭게 사용할 수 있습니다. 그러나 AWS EC2 인스턴스에서 대규모 스캐닝 공격을 감지하려면, 개별 서버의 tcpdump 대신 AWS VPC Flow Logs (VPC 플로우 로그)를 활성화하여 전체 네트워크 흐름을 중앙 저장소(S3 또는 CloudWatch Logs)에 수집하는 아키텍처를 설계해야 합니다. 이때 VPC Flow Logs는 수집 및 저장되는 데이터 용량(GB)에 따라 매월 별도의 과금이 발생하므로, 모니터링 대상 ENI(탄력적 네트워크 인터페이스)와 필터 조건을 명확히 설정해야 합니다.
