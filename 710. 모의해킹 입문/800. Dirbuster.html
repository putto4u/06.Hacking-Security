1. 도구 정의 및 작동 원리 (Tool Definition & Logic)

Dirbuster (디렉터리 버스터)는 웹 애플리케이션 내의 숨겨진 디렉터리와 파일을 찾기 위해 설계된 Java (자바) 기반의 Multi-threaded (다중 스레딩) 응용 프로그램입니다. 공격자가 사전에 정의된 Wordlist (사전 파일)의 단어들을 타겟 URL 뒤에 하나씩 붙여보며 응답 코드를 확인하는 Brute-force (무차별 대입) 공격 방식을 사용합니다.

[도식: Dirbuster의 무차별 대입 메커니즘]

Wordlist (사전 파일)

admin, config, test, backup...

ENGINE
Multi-threading (다중 스레딩)

병렬 요청 처리

Response Code (응답 코드)

200 OK / 403 Forbidden

2. 상세 실행 및 타겟 설정 (Execution & Setup)

STEP 01

도구 실행 (Launch)

칼리 리눅스 메뉴에서 선택하거나 터미널에서 직접 명령어를 입력합니다.

Kali Terminal
$ dirbuster

* 주석: 실행 시 자바 래퍼 스크립트가 구동되므로 터미널 창이 먼저 나타나는 것은 정상입니다.

STEP 02

파라미터 구성 (Parameters)

1. Target URL (대상 주소)

스캔하려는 타겟 서버의 주소를 입력합니다. (예: http://192.168.100.80)

2. Threads (스레드 수)

동시에 보낼 요청 수를 설정합니다. 50~100 사이를 권장하며, 너무 높으면 서버 부하 및 차단 위험이 있습니다.

3. Wordlist Selection (사전 파일 선택)

Browse 버튼을 눌러 칼리에 내장된 사전을 선택합니다.

/usr/share/dirbuster/wordlists/directory-list-2.3-medium.txt

3. 심화 스캔 설정 및 분석 (Advanced Scan & Analysis)

주요 옵션 상세 (Options)

File Extensions (확장자 지정)

찾고자 하는 파일의 확장자를 쉼표로 구분하여 입력합니다. (예: php, html, txt, bak)

Recursive (재귀적 탐색)

새로운 디렉터리를 발견하면 그 하위 경로에 대해 자동으로 스캔을 다시 수행하는 기능입니다.

응답 코드별 의미 (Response Codes)
200 OK 접근 가능 리소스
301/302 Redirect 경로 이동 (리다이렉트)
403 Forbidden 권한 제한 (경로 존재 확인 가능)
404 Not Found 존재하지 않는 경로

Tree View (트리 뷰) 시각화 분석

스캔이 시작되면 'Tree View' 탭을 선택하여 웹 서버의 논리적 구조를 직관적으로 파악할 수 있습니다.

// TARGET STRUCTURE DISCOVERED
/ (root)
├── admin/ [!] Sensitive Area
│   ├── login.php
│   └── setup.php
├── backup/ CRITICAL
│   └── config.php.bak
└── pds/
    └── webshell.php

4. 아키텍트 인사이트 (Expert Insight)

발자취 수집의 핵심 (Footprinting)

Dirbuster를 통해 얻은 트리 구조는 모의해킹 보고서의 Footprinting (정보 수집) 섹션에서 가장 핵심적인 근거 자료가 됩니다. 관리자가 의도하지 않게 노출한 백업 파일이나 설정 페이지를 찾아내는 것이 목표입니다.

탐지 회피 전략 (Detection Evasion)

현대적인 WAF (Web Application Firewall, 웹 방화벽)는 과도한 HTTP 요청을 DoS (Denial of Service, 서비스 거부) 공격으로 간주하여 IP를 차단합니다. 실무에서는 스레드 수를 낮추고 랜덤 지연 시간을 추가하는 것이 좋습니다.
