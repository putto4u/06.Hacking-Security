1. 데이터베이스 환경 구성 및 초기화 (DB Environment Setup & Initialization)

메타스플로잇은 **PostgreSQL(포스트그레SQL, 오픈소스 관계형 데이터베이스)**을 백엔드로 사용합니다. 메타스플로잇을 실행하기 전, 반드시 데이터베이스 서비스를 활성화하고 초기화(Initialization) 과정을 거쳐야 연동이 정상적으로 이루어집니다.

STEP 01

서비스 활성화 (Service Start)

칼리 리눅스 프롬프트에서 PostgreSQL 서비스를 시작하고 부팅 시 자동 실행되도록 설정합니다.

# PostgreSQL 서비스 시작 및 활성화
$ sudo systemctl start postgresql
$ sudo systemctl enable postgresql
STEP 02

데이터베이스 초기화 (DB Init)

메타스플로잇 전용 데이터베이스 스키마와 사용자를 생성하기 위해 초기화 명령을 실행합니다.

# 메타스플로잇 DB 초기화 (최초 1회 필수)
$ sudo msfdb init
# msfconsole 실행 후 연결 상태 확인
msf6 > db_status

[도식: 메타스플로잇 데이터 연동 구조 (MSF-DB Architecture)]

PostgreSQL 데몬 기동
msfdb init (초기화)
DB 연동 완료 (Persistence)

2. 자산 및 서비스 정보 조회 (Assets & Services Retrieval)

HOSTS

호스트 리스트 (Host List)

데이터베이스에 저장된 모든 대상 IP 주소와 운영체제(OS, Operating System), 장치 정보를 확인하여 공격 대상 인벤토리를 관리합니다.

msf6 > hosts
# 특정 칼럼(주소, OS명, 용도)만 지정해서 보기
msf6 > hosts -c address,os_name,purpose
# 현재 활성화된(Up) 호스트만 보기
msf6 > hosts -u
SERVICES

서비스 리스트 (Services List)

탐지된 열린 포트와 실행 중인 서비스(HTTP, SSH 등)를 확인하여 침투 경로(Port, 네트워크 서비스 지점)를 식별합니다.

msf6 > services
# 80번 포트가 열린 대상만 필터링
msf6 > services -p 80
# 현재 구동 중인 TCP 프로토콜 서비스만 보기
msf6 > services -u -r tcp

3. 취약점 및 탈취 정보 관리 (Vulns & Credentials)

01

취약점 리스트 (Vulnerabilities List)

스캔 도구(Nmap, Nessus 등)를 통해 발견되었거나 실제 공격 성공으로 기록된 약점 목록을 확인합니다.

msf6 > vulns
# 특정 포트와 연관된 상세 정보(Reference) 보기
msf6 > vulns -p 445 -i
02

탈취 계정 정보 (Credentials)

공격 성공 후 획득한 사용자 이름(Username)과 비밀번호(Password/Hash)를 통합 관리합니다.

msf6 > creds
# 향후 횡적 이동(Lateral Movement, 인접 시스템 확장) 시 재사용
msf6 > creds -v

4. 모듈 검색 및 작업 공간 분리 (Module Search & Workspace)

공격 모듈 검색 (Search & Show)

# 특정 취약점(MS17-010, EternalBlue) 관련 모듈 검색
msf6 > search ms17-010
# 조건 필터 검색 (Windows용 Exploit 모듈만 검색)
msf6 > search platform:windows type:exploit
# 전체 목록 카테고리별 조회
msf6 > show exploits
msf6 > show auxiliary
msf6 > show payloads

Workspace (워크스페이스, 작업 공간)

여러 프로젝트나 고객사를 대상으로 침투 테스트를 수행할 때 데이터가 섞이는 것을 방지하기 위해 작업 공간을 논리적으로 분리합니다.

# 새로운 작업 공간 추가 및 자동 전환
msf6 > workspace -a project_alpha
# 현재 작업 공간 목록 및 선택된 공간 확인
msf6 > workspace

AWS Environment Precautions

AWS 환경에서 다수의 인스턴스를 대상으로 db_nmap 스캔을 수행할 경우, 짧은 시간에 많은 패킷이 발생하여 **AWS Shield (DDoS 방어 서비스)**에 의해 차단되거나 보안 경고를 받을 수 있습니다.

  • 실습 시 스캔 속도를 조절하는 -T3 이하의 옵션을 권장합니다.
  • 과도한 트래픽 발생 시 API 호출 및 트래픽 비용이 추가로 산정될 수 있음을 유의하십시오.
