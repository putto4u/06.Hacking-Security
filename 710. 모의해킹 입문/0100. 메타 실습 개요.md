
## Metasploit Framework (메타스플로잇 프레임워크, MSF) 개요
**Metasploit(메타스플로잇)**은 전 세계적으로 가장 많이 사용되는 Penetration Testing(침투 테스트, 모의 해킹) 플랫폼입니다. 시스템의 취약점을 탐지하고, 실제 공격 코드(Exploit)를 실행하여 보안 상태를 점검하는 데 최적화되어 있습니다.
1. 주요 모듈의 구성
Metasploit은 기능별로 모듈화되어 있으며, 각 역할은 다음과 같습니다.
| 모듈명 | 설명 |
|---|---|
| Exploit (익스플로잇, 취약점 공격 코드) | 대상 시스템의 약점(Bug)을 이용하여 침투하는 실제 공격 코드입니다. |
| Payload (페이로드, 공격 수행 코드) | 침투 성공 후, 대상 시스템에서 실행할 동작(예: 리버스 쉘 연결)을 정의합니다. |
| Auxiliary (오조릴리어리, 보조 도구) | 스캐닝, 스니핑, 서비스 거부 공격(DoS) 등 공격 전 단계의 정보 수집을 돕습니다. |
| Post (포스트, 침투 후 작업) | 침투 성공 이후 정보 수집, 권한 상승, 흔적 제거 등을 수행합니다. |
| Encoder (인코더, 암호화 도구) | 백신이나 IDS/IPS의 탐지를 피하기 위해 페이로드를 변조합니다. |
2. 기본 사용 흐름 (Step-by-Step)
모의 해킹의 표준 절차인 정보 수집 -> 취약점 분석 -> 침투 -> 결과 보고 과정을 MSF 내부 명령어로 구현하면 다음과 같습니다.
단계 1: 서비스 실행 및 접속
터미널에서 메타스플로잇 콘솔을 실행합니다. (관리자 권한 필요)
sudo msfconsole

단계 2: 모듈 검색 및 선택
공격하려는 서비스나 취약점(예: smb, 로그4제이 등)을 검색하고 사용할 모듈을 선택합니다.
search [취약점 이름 또는 서비스명]
use [선택한 모듈 경로]

단계 3: 옵션 설정 (Payload 및 IP/Port)
공격 대상(RHOSTS)과 내 컴퓨터(LHOST)의 정보를 설정합니다.
show options              # 설정 가능한 옵션 확인
set RHOSTS [대상 IP]      # Remote Host (공격 대상)
set LHOST [내 IP]         # Local Host (공격자 IP)
show payloads             # 사용 가능한 페이로드 확인
set PAYLOAD [페이로드 경로]

단계 4: 공격 실행
설정이 완료되면 공격을 수행합니다.
exploit                   # 또는 run 입력

3. 실무 활용 팁 및 주의사항
 * Meterpreter (미터프리터): MSF의 전용 페이로드로, 메모리 상에서 동작하며 대상 시스템을 강력하게 제어할 수 있는 대화형 쉘입니다.
 * Database 연동: msfdb init을 통해 PostgreSQL과 연동하면 스캔 결과와 전리품(Creds)을 체계적으로 관리할 수 있습니다.
 * 법적 고지: 허가받지 않은 시스템에 대한 테스트는 정보통신망법에 의해 처벌받을 수 있으므로, 반드시 가상 환경(VirtualBox, VMware) 내의 실습용 타겟을 대상으로만 사용해야 합니다.
