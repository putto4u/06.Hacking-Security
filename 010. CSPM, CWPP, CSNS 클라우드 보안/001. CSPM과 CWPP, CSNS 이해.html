클라우드 통합 보안
프레임워크의 정석

CSPM,CWPP,CSNS 결합 차세대 보안 플랫폼(CNAPP) 심층 분석

보안의 대전환: 제로 트러스트와 심층 방어

기존의 On-premise (온프레미스, 자체 구축 서버) 환경은 성벽을 높게 쌓는 경계 보안(Perimeter Security)에 의존했습니다. 하지만 클라우드는 경계가 모호합니다. 인프라가 코드로 정의(IaC)되고, 수천 개의 자산이 수시로 생성 및 삭제되는 환경에서는 "결코 신뢰하지 말고, 항상 검증하라(Never Trust, Always Verify)"는 Zero Trust (제로 트러스트) 원칙이 지배합니다.

전문가 실무 팁

클라우드 보안 사고의 99%는 사용자의 설정 실수(Misconfiguration)에서 비롯됩니다. 공격자는 시스템의 취약점보다 관리자의 실수로 열린 API나 S3 버킷을 먼저 노립니다. 이것이 CSPM이 최우선 보안 솔루션으로 꼽히는 이유입니다.

CSPM: 설정의 무결성 확보

Cloud Security Posture Management
(클라우드 보안 태세 관리)

CSPM은 클라우드 관리 평면인 Control Plane (컨트롤 플레인, 관리 계층)을 감시합니다. 클라우드 서비스 제공자(CSP)가 제공하는 모든 리소스의 설정값이 기업의 보안 정책 및 글로벌 표준에 부합하는지 실시간으로 대조합니다.

기술적 핵심 메커니즘

API 기반 스캐닝 (Agentless)

대상 서버 내부에 보안 소프트웨어를 설치하지 않습니다. 클라우드 API를 호출하여 자산의 메타데이터(예: 방화벽 규칙, 암호화 설정)를 수집하므로 성능 저하가 없습니다.

보안 드리프트(Drift) 탐지

최초 승인된 보안 기준선(Baseline)에서 벗어난 임의의 설정을 즉시 탐지합니다. 누군가 임의로 DB 포트를 개방하면 몇 초 내에 관리자에게 알림이 전송됩니다.

주요 실무 서비스와 비용 모델:

  • AWS Security Hub유료: 보안 검사 항목당 과금 ($0.001/평가). 여러 보안 서비스의 결과를 중앙 집중식으로 시각화합니다.
  • AWS Config유료: 리소스 변경 기록 및 규칙 평가당 과금 ($0.003/평가). 시간대별 자산의 변화를 기록(History)합니다.

주석: CSPM은 "인프라의 설계 도면이 법적/기술적 기준을 충족하는지" 외부에서 검사하는 정적 보안 관리 도구입니다.

CWPP: 워크로드 런타임 방어

Cloud Workload Protection Platform
(클라우드 워크로드 보호 플랫폼)

CWPP는 클라우드에서 실행되는 실제 연산 작업인 Workload (워크로드) 내부의 보안을 책임집니다. 가상 서버(EC2), 컨테이너(Pod), 서버리스(Lambda) 환경의 "안전한 실행"을 보장하며, 제로데이(Zero-day) 공격이나 지능형 지속 위협(APT)으로부터 시스템을 보호합니다.

보호 대상 및 기술적 구분

구분 가상 서버 (EC2) 컨테이너 (Kubernetes/ECS) 서버리스 (Lambda)
구현 방식 에이전트 (Agent) 기반 사이드카 (Sidecar) / DaemonSet 런타임 라이브러리 주입
핵심 감시 OS 커널 콜, 파일 무결성 이미지 취약점, 이종 격리 보안 실행 코드 로직, API 호출 권한
보안 가치 안티-맬웨어, IPS/IDS 컨테이너 탈출(Escape) 방지 코드 레벨 인젝션 방어

주요 실무 서비스와 비용 모델:

  • Amazon GuardDuty유료: 데이터 분석량 기준 과금 ($5.00/GB - VPC Flow log 등). 기계학습(ML)을 통해 계정 내 위협적인 활동을 감지합니다.
  • Amazon Inspector유료: 인스턴스 또는 이미지 스캔당 과금. 취약점을 지속적으로 관리(Vulnerability Management)합니다.

주석: CWPP는 "건물 내부(워크로드)에 CCTV를 설치하고 보안 요원을 배치하여 실시간 이상 행위를 감시"하는 동적 보안 도구입니다.

CSNS: 다계층 네트워크 방어

Cloud Service Network Security
(클라우드 서비스 네트워크 보안)

CSNS는 클라우드의 네트워크 인프라 자체를 보호합니다. 외부 인터넷 경계에서의 North-South (외부와 내부 통신) 트래픽뿐만 아니라, 내부 서버 간의 East-West (내부 자산 간 통신) 트래픽을 심층 분석하여 정교한 공격을 차단합니다.

심층 방어(Defense in Depth) 아키텍처

주요 실무 서비스와 비용 모델:

  • AWS Network Firewall유료: 시간당 사용료 (~$0.395) + 트래픽 처리량 기준 과금. VPC 경계에서 L3-L7 전체 계층에 대한 필터링을 제공합니다.
  • AWS WAF유료: 웹 ACL 및 규칙당 월 정액 + 요청 수 기준 과금. SQL Injection 및 XSS 공격 차단에 특화되어 있습니다.

주석: CSNS는 "마을(VPC) 입구의 검문소와 각 건물(Subnet/Host) 앞의 금속 탐지기"처럼 통로 자체를 지키는 네트워크 방어 체계입니다.

미래 보안의 중심: CNAPP

과거에는 CSPM, CWPP, CSNS를 개별 솔루션으로 분리 운영했지만, 이제는 이를 통합하여 클라우드 애플리케이션의 생명 주기 전체를 보호하는 CNAPP (Cloud-Native Application Protection Platform)이 표준이 되었습니다.

Post-Deployment

설계된 인프라가 배포된 직후의 상태를 상시 감시하여 '설정 오류'를 제거합니다.

CSPM

Real-time Execution

애플리케이션이 실행 중인 런타임 환경에서 '공격자의 행위'를 실시간으로 차단합니다.

CWPP

Traffic In-transit

데이터가 전송되는 모든 경로를 감시하여 '침입 및 데이터 유출'을 원천 봉쇄합니다.

CSNS
