Modern Cloud Security: CSPM, CWPP, and CSNS Architectures

1. CSPM (Cloud Security Posture Management, 클라우드 보안 태세 관리)

CSPM은 클라우드 관리 평면(Control Plane, 컨트롤 플레인)의 설정 상태를 상시 감시합니다. 인프라의 '뼈대'가 되는 설정값을 API(Application Programming Interface, 애플리케이션 프로그래밍 인터페이스)를 통해 수집하고, 보안 표준 프레임워크와 비교하여 위험 요소를 즉각 탐지합니다.

운영 원리 및 핵심 기능

  • 1
    Agentless(에이전트리스) 모니터링: 별도 소프트웨어 설치 없이 API를 통해 구성 정보(Metadata)를 수집합니다.
  • 2
    Compliance Scan(컴플라이언스 스캔): 수집된 설정값을 ISMS-P, CIS Benchmark 등의 보안 표준과 실시간 대조합니다.
  • 3
    Remediation(조치 및 치료): 잘못된 설정 발견 시 관리자에게 알리거나, 미리 정의된 정책에 따라 자동으로 정상화합니다.
  • 4
    관련 서비스: AWS Security Hub유료, AWS Config유료

2. CWPP (Cloud Workload Protection Platform, 클라우드 워크로드 보호 플랫폼)

CWPP는 클라우드에서 실행되는 실제 '작업(Workload, 워크로드)' 내부를 보호합니다. 서버 내부의 프로세스 행위, 파일 무결성, 네트워크 세션을 실시간으로 감시하며 위협을 탐지하고 방어하는 기능을 제공합니다.

작동 메커니즘 및 주요 기능

  • 1
    Runtime Visibility(런타임 가시성): 서버 내부에서 발생하는 권한 상승, 비인가 통신, 파일 변조를 실시간 감시합니다.
  • 2
    Vulnerability Management(취약점 관리): 설치된 소프트웨어의 CVE(Common Vulnerabilities and Exposures, 취약점 식별 번호)를 상시 점검합니다.
  • 3
    관련 서비스: Amazon GuardDuty유료, Amazon Inspector유료

3. CSNS (Cloud Service Network Security, 클라우드 서비스 네트워크 보안)

CSNS는 클라우드 경계(Perimeter, 페리미터)와 내부망의 트래픽을 보호합니다. 외부 공격을 최전방에서 차단하고, 내부 서버 간 통신을 필터링하여 보안 사고의 확산을 방지하는 네트워크 방어 체계입니다.

작동 메커니즘 및 주요 기능

  • 1
    Defense in Depth(심층 방어): 외부 경계부터 개별 인스턴스까지 다중 보안 필터를 두어 공격을 단계적으로 차단합니다.
  • 2
    DPI (Deep Packet Inspection, 심층 패킷 분석): 전송되는 데이터 본문(Payload)의 유해 여부를 정밀 분석합니다.
  • 3
    관련 서비스: AWS Network Firewall유료, AWS WAF유료

통합 보안 모델: CNAPP (Cloud-Native Application Protection Platform)
