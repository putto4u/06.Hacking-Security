1. 사회 공학적 공격 아키텍처 (Attack Vector Architecture)

Social-Engineer Toolkit (SET, 사회 공학 도구 세트)은 기술적 방화벽이 아닌 '사람'이라는 시스템의 가장 약한 고리를 공략합니다. 공격자는 타겟이 신뢰하는 환경(로그인 페이지, 업데이트 알림, 문서 파일 등)을 Spoofing (스푸핑, 위장)하여 자발적인 권한 부여나 정보 입력을 유도합니다. 본 실습은 Metasploitable 2 서버를 대상으로 다양한 시나리오를 전개합니다.

[도식: 사회 공학적 침투 및 권한 상승 흐름]

Deception (기만)
Phishing/Cloning
Execution (실행)
User Interaction
Access (접근)
Reverse Shell

3. 실전 공격 벡터 상세 가이드 (10 Attack Scenarios)

01

자격 증명 수집 (Credential Harvester Attack Method)

Menu Path (메뉴 선택 경로)

1) Social-Engineering Attacks (사회 공학 공격)
2) Website Attack Vectors (웹사이트 공격 벡터)
3) Credential Harvester Attack Method (자격 증명 수집 공격 방식)
2) Site Cloner (사이트 복제 도구)

공격 해설 (Description)

Metasploitable 2의 DVWA 로그인 페이지를 그대로 복제하여 공격자의 로컬 서버에 띄우는 방식입니다. 사용자가 익숙한 디자인의 로그인 창에 ID와 Password를 입력하는 순간, 데이터는 타겟 서버가 아닌 공격자의 터미널로 즉시 전송됩니다.

[기술적 취약점 및 기법 분석]

  • 사회 공학 요소: 사용자의 시각적 신뢰(Visual Trust)를 악용합니다. 평소 사용하던 관리 페이지와 동일한 UI를 제공하여 의심을 제거합니다.
  • 대상 취약점: HTTP 프로토콜의 POST Method (전송 방식) 데이터 캡처를 이용합니다.
  • 핵심 기술: 웹사이트의 HTML/CSS 소스를 크롤링하여 `action` 태그를 공격자 서버로 강제 변조하는 기술이 적용됩니다.
02

웹 자킹 (Web Jacking Attack Method)

Menu Path (메뉴 선택 경로)

1) Social-Engineering Attacks (사회 공학 공격)
2) Website Attack Vectors (웹사이트 공격 벡터)
5) Web Jacking Attack Method (웹 자킹 공격 방식)

공격 해설 (Description)

사용자에게 "이 사이트는 안전하지 않습니다. 여기를 클릭하여 안전한 페이지로 이동하세요"와 같은 허위 메시지를 띄웁니다. 사용자가 링크를 클릭하면 브라우저의 주소창에는 정상 주소가 보이지만, 실제로는 공격자의 악성 페이지가 백그라운드에서 실행됩니다.

[기술적 취약점 및 기법 분석]

  • 사회 공학 요소: 보안에 대한 공포심(Fear)과 업데이트에 대한 강박을 이용합니다.
  • 대상 취약점: 브라우저의 Status Bar Spoofing (상태 표시줄 위장) 및 앵커 태그 조작 취약점을 타겟팅합니다.
  • 핵심 기술: JavaScript를 이용하여 클릭 시점에 `window.open`과 `location.replace`를 교묘하게 섞어 사용자의 시선을 분산시킵니다.
03

자바 애플릿 공격 (Java Applet Attack Method)

Menu Path (메뉴 선택 경로)

1) Social-Engineering Attacks (사회 공학 공격)
2) Website Attack Vectors (웹사이트 공격 벡터)
1) Java Applet Attack Method (자바 애플릿 공격 방식)

공격 해설 (Description)

공격자의 웹페이지 접속 시 "Java 업데이트가 필요합니다"라는 팝업을 띄우고 사용자가 'Run(실행)'을 누르도록 유도합니다. 사용자가 승인하면 즉시 공격자의 Kali Linux와 통신하는 Meterpreter (미터프리터) 세션이 형성됩니다.

[기술적 취약점 및 기법 분석]

  • 사회 공학 요소: 기술적 지식이 부족한 사용자가 시스템 경고창을 신뢰하고 무심코 '확인'을 누르는 습관을 악용합니다.
  • 대상 취약점: 클라이언트 측의 Java Runtime Environment (JRE, 자바 실행 환경)의 신뢰되지 않은 코드 실행 허용 취약점을 이용합니다.
  • 핵심 기술: 악성 페이로드가 포함된 `.jar` 파일을 생성하고 Self-Signed Certificate(자가 서명 인증서)로 위장하여 보안 경고를 우회합니다.
04

탭내빙 (Tabnabbing Attack Method)

1) Social-Engineering Attacks
2) Website Attack Vectors
4) Tabnabbing Attack Method

사용자가 브라우저 탭을 여러 개 띄워놓고 다른 작업을 할 때, 비활성화된 공격 페이지를 유명 사이트(Gmail, Facebook 등)의 로그인 페이지로 교체합니다. 사용자가 나중에 해당 탭으로 돌아왔을 때 세션이 만료된 줄 알고 다시 로그인하게 만드는 지능형 기법입니다.

  • 사회 공학 요소: 사용자의 망각과 멀티태스킹 습관을 공략합니다.
  • 대상 취약점: 브라우저 탭의 Focus/Blur Event (포커스 변화 감지)를 이용한 동적 콘텐츠 교체 가능성을 이용합니다.
05

HTA 공격 (HTA Attack Vector)

1) Social-Engineering Attacks
2) Website Attack Vectors
8) HTA Attack Method

HTML Application (HTA, HTML 애플리케이션) 파일을 생성하여 배포합니다. 윈도우 환경에서 HTA는 일반 웹페이지보다 높은 권한을 가지며, 사용자가 파일을 실행하는 순간 파워쉘(PowerShell) 코드가 동작하여 리버스 쉘을 연결합니다.

  • 사회 공학 요소: 파일 다운로드 및 실행에 대한 거부감을 낮추기 위해 유용한 도구나 패치로 위장합니다.
  • 핵심 기술: `mshta.exe`를 통해 브라우저 샌드박스를 우회하고 운영체제 명령어를 직접 수행합니다.

추가 5종 공격(Metasploit Exploit, Info Gathering, Spear-Phishing, Media Generator, QR Code)은 위와 동일한 분석 프로세스를 따르며, 각각 브라우저 취약점, 이메일 스푸핑, 물리적 보안 허점을 공략합니다.

4. 전문가적 대응 방안 (Offensive Insight)

지능형 위협 탐지 (Advanced Threat Detection)

사회 공학 공격은 기술적 패턴보다 '사용자 행위'에 집중합니다. UEBA (User and Entity Behavior Analytics, 사용자 및 엔터티 행동 분석) 솔루션을 도입하여 평소와 다른 시간에 비정상적인 사이트에 로그인하거나 대량의 데이터를 외부로 전송하는 행위를 실시간으로 차단해야 합니다.

제로 트러스트 원칙 (Zero Trust Principles)

"절대 믿지 말고 항상 검증하라"는 원칙을 적용해야 합니다. MFA (Multi-Factor Authentication, 다요소 인증)는 자격 증명 탈취 공격을 무력화하는 가장 강력한 수단이며, 링크 클릭 전 도메인의 인증서와 평판을 자동으로 검사하는 브라우저 보안 확장이 필수적입니다.
