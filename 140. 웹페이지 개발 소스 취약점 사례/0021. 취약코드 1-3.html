취약 소스코드 분석 및 시뮬레이션

실제 개발 환경에서 발생할 수 있는 보안 약점(Security Weakness)이 포함된 소스코드를 분석하고, 공격자의 관점에서 이를 악용했을 때의 결과를 실시간으로 확인합니다.

01

SQL Injection (SQL 삽입)

Database Query Manipulation

Vulnerable Source Code 

// [취약 코드 분석] 사용자 입력을 쿼리 문자열에 직접 결합하는 전형적인 방식
public User login(String userId, String userPw) {
    // 1. 외부에서 전달된 userId 변수가 검증 없이 SQL 구문에 포함됨
    String sql = "SELECT * FROM users WHERE id = '" + userId + "' AND pw = '" + userPw + "'";
    
    // 2. Statement 객체는 쿼리 구조를 미리 정의하지 않고 전체를 실행함
    Statement stmt = conn.createStatement();
    ResultSet rs = stmt.executeQuery(sql); // 공격자의 입력값이 쿼리 구조를 변경 가능
    
    return rs.next() ? new User(rs.getString("name")) : null;
}

Attack Simulation

Exploit Success (인증 우회 성공)

생성된 쿼리: SELECT * FROM users WHERE id = '' OR '1'='1' --' AND pw = '...'
설명: 입력값 내의 -- 주석 기호가 패스워드 확인 부분을 무력화시켰으며, '1'='1' 조건이 항상 참(True)이 되어 비밀번호 없이 **관리자 계정으로 로그인이 성공**되었습니다.

Secure Implementation (정상화 코드)

// 매개변수화된 질의문(Parameterized Query)을 사용한 방어
String sql = "SELECT * FROM users WHERE id = ? AND pw = ?";
PreparedStatement pstmt = conn.prepareStatement(sql);
// 입력값은 쿼리의 '구조'가 아닌 단순 '데이터'로만 취급됨
pstmt.setString(1, userId); 
pstmt.setString(2, userPw);
02

Stored XSS (저장형 교차 사이트 스크립팅)

Malicious Script Injection

Unsafe Data Persistence 

// [취약 지점] 사용자가 보낸 데이터를 필터링 없이 DB에 저장하고 그대로 출력
String userComment = request.getParameter("comment");
// DB 저장 시점에서 어떤 검증도 거치지 않음
boardDao.insertComment(userComment); 

// JSP/HTML 렌더링 지점
<div class="comment-item">
    <%= board.getComment() %> <!-- 브라우저가 스크립트를 즉시 실행 -->
</div>

Payload Injection Simulation

[브라우저 경고창]

"당신의 세션 쿠키가 탈취되었습니다: SESSION_ID=7f8e...44a"
Exploit Reality

공격자가 게시판에 남긴 스크립트가 데이터베이스에 저장(Stored)된 후, **일반 사용자가 게시글을 읽는 시점에 실행**되었습니다. 이는 단순 알림창을 넘어 사용자 계정 탈취로 이어집니다.
