1. 코드 분석 아키텍처 (Code Analysis Architecture)

본 연구는 Source Code(소스 코드) 내에서 외부 입력값이 처리되는 전 과정을 추적합니다. Untrusted Data(신뢰할 수 없는 데이터)가 검증 없이 Sink(취약한 함수 호출 지점)에 도달할 때 발생하는 논리적 결함을 중점적으로 다룹니다.

4. 아키텍트 인사이트 (Expert Insight)

SAST (Static Application Security Testing)

정적 애플리케이션 보안 테스트(SAST) 도구는 코드 실행 없이 구조를 분석하여 잠재적 위험 패턴을 찾아냅니다. 하지만 로직 결함을 완벽히 탐지하기 위해서는 수동 코드 리뷰(Manual Code Review)가 병행되어야 합니다.

Principle of Least Privilege

소스 코드 레벨에서의 방어뿐만 아니라, 애플리케이션 실행 계정이 시스템 리소스에 접근할 때 최소 권한 원칙(Principle of Least Privilege)을 준수하도록 환경을 구성하는 것이 2차 피해를 막는 핵심입니다.
