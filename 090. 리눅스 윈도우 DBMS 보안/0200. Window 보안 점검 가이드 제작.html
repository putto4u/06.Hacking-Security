Module 01: Account Security

1.1 관리자 및 게스트 계정 관리 (Administrator & Guest Account)

윈도우 서버 설치 시 생성되는 기본 관리자 계정(Administrator)은 공격자의 주요 타겟입니다. 계정 이름을 변경하고 사용하지 않는 게스트(Guest) 계정은 반드시 비활성화해야 합니다.

[도식: 계정 이름 변경을 통한 난독화 모델 (Account Obfuscation)]

Administrator
RENAME TARGET
SEC_Admin_2026
NEW IDENTITY

GUI Menu Path

로컬 보안 정책 (secpol.msc)
로컬 정책 > 보안 옵션
- 계정: 관리자 계정 이름 바꾸기: [사용자 지정 이름]
- 계정: 게스트 계정 상태: [사용 안 함]

PowerShell Command

# 관리자 계정 이름 변경
Rename-LocalUser -Name "Administrator" -NewName "SEC_Admin"

# 게스트 계정 비활성화
Disable-LocalUser -Name "Guest"

1.2 패스워드 복잡성 및 계정 잠금 정책 (Password & Lockout)

패스워드 무작위 대입 공격(Brute Force Attack)을 방어하기 위해 복잡성 요건을 충족하고, 일정 횟수 이상 로그인 실패 시 계정을 강제 잠금하도록 설정합니다.

보안 설정값 (Audit Values)

  • - 최소 암호 길이: 12자 이상
  • - 암호 복잡성 요건 충족: 사용
  • - 계정 잠금 임계값: 5회
  • - 계정 잠금 기간: 30분 이상

Menu Path

로컬 보안 정책 (secpol.msc)
계정 정책 > 암호 정책
- 복잡성 요건 충족: 사용
계정 정책 > 계정 잠금 정책
- 계정 잠금 임계값: 5회

Module 02: Network Security

2.1 원격 데스크톱 서비스 보안 (RDP Hardening)

RDP(Remote Desktop Protocol, 원격 데스크톱 프로토콜)는 외부 공격의 주된 경로입니다. NLA(Network Level Authentication, 네트워크 수준 인증)를 강제하고 기본 포트(3389)를 변경하여 노출을 최소화합니다.

RDP Port Modification

# 레지스트리 경로
HKLM\SYSTEM\CurrentControlSet\Control\
Terminal Server\WinStations\RDP-Tcp

PortNumber 값을 3389에서 33890 등으로 변경

Menu Path (NLA)

시스템 속성 > 원격
- 네트워크 수준 인증을 사용하여 원격 데스크톱을 실행하는 컴퓨터에서만 연결 허용: 체크(Enable)

2.2 고급 보안이 설정된 Windows 방화벽 (Windows Firewall)

방화벽(Firewall, 네트워크 장벽)을 항상 활성화하고, 불필요한 인바운드 규칙(Inbound Rule, 들어오는 트래픽 규칙)을 제거합니다. 특히 ICMP(Ping) 응답 거부 설정을 통해 정보 수집 공격을 차단합니다.

PowerShell
# 모든 프로필의 방화벽 활성화
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True

# 특정 포트(33890) 허용 규칙 추가
New-NetFirewallRule -DisplayName "Custom RDP" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 33890
RE-APPLY METHOD

설정 적용 후 gpupdate /force 명령어를 실행하여 도메인/로컬 그룹 정책을 즉시 갱신하십시오.

Module 03: System Hardening

3.1 사용자 계정 컨트롤 최적화 (UAC Optimization)

UAC(User Account Control, 사용자 계정 컨트롤)는 악성 코드가 관리자 권한으로 자동 실행되는 것을 방지합니다. 보안 수준을 '항상 알림'으로 유지해야 합니다.

[도식: UAC 권한 승인 프로세스 (UAC Authorization Flow)]

App Execution
UAC PROMPT
Consent UI
Elevated Privilege
Menu Path 제어판 > 사용자 계정 > 사용자 계정 컨트롤 설정 변경
최상단 (항상 알림) 으로 설정

Module 04: Audit & Event Log

4.1 로컬 감사 정책 구성 (Audit Policy Configuration)

누가 시스템에 접속하고 파일을 수정했는지 기록하기 위해 감사 정책(Audit Policy)을 활성화합니다. 실패한 로그인 시도는 반드시 기록되어야 합니다.

Audit Categories

로컬 보안 정책 > 로컬 정책 > 감사 정책
- 로그온 이벤트 감사: 성공, 실패
- 계정 관리 감사: 성공, 실패
- 시스템 이벤트 감사: 성공, 실패

Event Viewer

# 로그 용량 및 보관 정책 설정
$ Limit-EventLog -LogName Security -MaximumSize 128MB -RetentionDays 30

Architect Insight (보안 아키텍트 조언)

Active Directory 환경에서의 주의사항

AD(Active Directory) 환경에서는 로컬 정책보다 그룹 정책(GPO, Group Policy Object)이 우선순위를 가집니다. 로컬에서 설정을 변경해도 도메인 컨트롤러에 의해 원복될 수 있으므로, 대규모 환경에서는 반드시 GPMC.msc를 통해 중앙 집중 관리를 수행하십시오.

Public Cloud Security (AWS/Azure)

AWS EC2 윈도우 인스턴스의 경우, 운영체제 내부 방화벽 외에도 Security Group(SG, 보안 그룹)에서 33890 포트가 오픈되어 있는지 확인해야 합니다. 인스턴스 외부 계층 보안과 내부 보안 정책을 동기화(Synchronization)하는 것이 핵심입니다.
