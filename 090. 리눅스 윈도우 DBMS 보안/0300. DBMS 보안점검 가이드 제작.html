Module 01: Account Hardening

1.1 기본 계정 및 패스워드 정책 (Default Account & Password Policy)

DBMS 설치 시 기본으로 생성되는 관리자 계정(Administrative Account) 및 테스트용 익명 계정은 공격자의 침투 경로가 됩니다. **계정 난독화(Account Obfuscation)** 및 **패스워드 복잡성(Password Complexity)** 강제가 필수적입니다.

[도식: 데이터베이스 인증 계층 (DB Auth Layers)]

Default Users
Drop/Disable
Password Policy
Complexity Enforced
Access Control
Role-based Auth
MySQL / MariaDB Configuration & SQL
-- 익명 계정 삭제 및 Root 비밀번호 강화
DROP USER ''@'localhost';
INSTALL COMPONENT 'file://component_validate_password';
ALTER USER 'root'@'localhost' IDENTIFIED BY 'Strong_P@ssw0rd!';
Oracle Database PL/SQL Command
-- 프로필을 통한 패스워드 정책 적용
ALTER PROFILE DEFAULT LIMIT FAILED_LOGIN_ATTEMPTS 5;
ALTER PROFILE DEFAULT LIMIT PASSWORD_LIFE_TIME 90;
ALTER USER SCOTT ACCOUNT LOCK; -- 불필요한 기본 계정 잠금
PostgreSQL SQL & Config
-- 암호 암호화 방식 설정 (scram-sha-256 권장)
# postgresql.conf 설정
password_encryption = scram-sha-256
ALTER ROLE postgres WITH PASSWORD 'Secure_DB_Admin!';

Module 02: Network Hardening

2.1 수신 주소 제한 및 포트 변경 (Binding Address & Port)

데이터베이스가 모든 인터페이스(`0.0.0.0`)에서 대기할 경우 외부 노출 위험이 매우 큽니다. **포트 바인딩(Port Binding, 특정 주소 연결)** 및 **SSL/TLS 암호화(Transport Layer Security, 전송 계층 보안)** 강제가 필수입니다.

MySQL / MariaDB

# my.cnf 설정
[mysqld]
bind-address = 127.0.0.1
port = 3307 # 포트 변경
require_secure_transport = ON

Oracle DB

# sqlnet.ora 설정
TCP.VALIDNODE_CHECKING = YES
TCP.INVITED_NODES = (192.168.1.10)
# 허용된 IP 외 접속 차단

PostgreSQL

# pg_hba.conf 설정
hostssl all all 10.0.0.0/24 scram-sha-256
# SSL 연결 및 특정 대역 강제

Module 03: Privilege Control

3.1 최소 권한 할당 원칙 (Principle of Least Privilege, PoLP)

사용자에게 업무에 필요한 최소한의 권한(Minimal Permissions)만을 부여합니다. 특히 일반 서비스 계정이 `SUPER`, `DBA`, `Superuser` 권한을 가지는 것은 치명적인 보안 결함입니다.

[MySQL]
-- 특정 DB에 읽기 권한만 부여
GRANT SELECT ON service_db.*
TO 'app_user'@'192.168.1.%';
[Oracle]
-- 권한 회수 및 롤 할당
REVOKE DBA FROM HR_USER;
GRANT CONNECT, RESOURCE TO HR_USER;
[PostgreSQL]
-- 스키마 접근 제한
REVOKE ALL ON SCHEMA public FROM PUBLIC;
GRANT USAGE ON SCHEMA public TO user;

Module 04: Encryption & Audit

4.1 데이터 저장 암호화 (Data-at-Rest Encryption)

**TDE(Transparent Data Encryption, 투명한 데이터 암호화)**를 적용하여 데이터 파일이나 백업본이 유출되더라도 내용을 보호합니다. 또한 **감사(Auditing, 행위 기록)**를 통해 모든 쿼리 실행 내역을 추적합니다.

Oracle TDE (Enterprise Feature)

테이블스페이스 단위로 강력한 AES 암호화를 지원하여 스토리지 레벨의 유출에 대응합니다.

CREATE TABLESPACE crypt_ts DATAFILE '...' ENCRYPTION USING 'AES256' DEFAULT STORAGE(ENCRYPT);

PostgreSQL pgaudit (Extension)

데이터 조작 및 정의 언어(DML/DDL) 실행 내역을 세부적으로 기록하여 규제 준수(Compliance)를 달성합니다.

shared_preload_libraries = 'pgaudit'; pgaudit.log = 'read, write, ddl';

Architect Insight (보안 아키텍트 조언)

Managed DB 서비스 과금 유의 (AWS RDS)

AWS RDS 환경에서 보안을 위해 Enhanced Monitoring이나 CloudWatch Logs로 쿼리 로그를 전송할 경우, 데이터 보관 용량 및 요청 횟수에 따라 예상치 못한 과금이 발생할 수 있습니다. 보존 기간(Retention Period) 설정을 최적화하십시오.

SQL Injection (SQL 주입 공격)

아무리 서버 보안이 강력해도 애플리케이션 레벨의 **SQL 인젝션** 취약점이 있다면 무용지물입니다. **Prepared Statements(준비된 문장)** 사용을 강제하고, DB 계정에 DML 권한만 제한적으로 부여하여 피해 범위를 최소화하십시오.
