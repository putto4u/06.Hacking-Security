Module 01: Account Security

1.1 원격 Root 접속 제한 (Restrict Remote Root Access)

관리자 계정(Root User)으로의 직접 원격 접속은 무차별 대입 공격(Brute Force Attack)의 1순위 표적입니다. 일반 사용자 계정으로 접속 후 관리자 권한을 획득하는 다단계 인증 구조를 강제해야 합니다.

[도식: 원격 인증 다중화 모델 (Multi-stage Auth Model)]

Direct Root Login
DENIED
Normal User (SSH)
STEP 1
SUDO Privilege
STEP 2

Configuration File

Path: /etc/ssh/sshd_config

# PermitRootLogin을 no로 설정
PermitRootLogin no

Execution Command

# 설정 확인
$ grep "PermitRootLogin" /etc/ssh/sshd_config

# 서비스 재시작
$ sudo systemctl restart sshd

1.2 패스워드 복잡성 설정 (Password Complexity Policy)

패스워드 취약점을 이용한 무작위 대입 공격(Dictionary Attack)을 방어하기 위해 최소 길이와 문자 조합을 PAM(Pluggable Authentication Modules, 탈착 가능한 인증 모듈)을 통해 강제합니다.

정책 상세 (Policy Details)

  • - minlen=12: 최소 길이 12자
  • - dcredit=-1: 숫자 최소 1자 포함
  • - ucredit=-1: 대문자 최소 1자 포함
  • - lcredit=-1: 소문자 최소 1자 포함
  • - ocredit=-1: 특수문자 최소 1자 포함

Implementation

# 파일: /etc/security/pwquality.conf
$ sudo vi /etc/security/pwquality.conf

# 내용 수정
minlen = 12
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1

1.3 계정 잠금 임계값 설정 (Account Lockout Threshold)

연속적인 인증 실패 시 해당 계정을 일정 시간 동안 잠금 처리하여 자동화된 공격 도구의 성공 가능성을 차단합니다.

PAM Config
# 파일: /etc/pam.d/password-auth & system-auth

auth required pam_tally2.so deny=5 unlock_time=600
account required pam_tally2.so
IMPORTANT

Root 계정은 잠금에서 제외하거나, 예외 처리를 정교하게 설계하여 관리자의 실수로 서버 접근이 원천 차단되는 사고를 방지하십시오.

Module 02: Network Security

2.1 불필요한 서비스 비활성화 (Disable Unnecessary Services)

공격 표면(Attack Surface, 공격 가능한 지점)을 줄이기 위해 현재 사용하지 않는 프로토콜이나 서비스를 중지하고 삭제합니다. 특히 취약한 레거시 서비스(Telnet, FTP, R-services)가 주요 점검 대상입니다.

Telnet Unencrypted Port 23
FTP Vulnerable Data Transfer
Finger Info Leakage Risk
# 실행중인 포트 및 서비스 확인
$ sudo netstat -tulpn

# 서비스 중지 및 비활성화
$ sudo systemctl stop [service_name]
$ sudo systemctl disable [service_name]

2.2 호스트 방화벽 설정 (Host-based Firewall Hardening)

IPtables 또는 Firewalld를 사용하여 명시적으로 허용되지 않은 모든 인바운드 트래픽(Inbound Traffic, 들어오는 트래픽)을 기본적으로 차단(Default Deny)합니다.

Firewall Rules (White-list Policy)
ALLOW HTTP/HTTPS (80, 443) ENABLED
ALLOW SSH (Custom Port) ENABLED
ALL OTHER TRAFFIC REJECTED
# Firewalld 사용 예시
$ sudo firewall-cmd --permanent --add-service=http
$ sudo firewall-cmd --permanent --add-service=https
$ sudo firewall-cmd --reload

Module 03: Permission Hardening

3.1 시스템 중요 파일 접근 권한 (Critical File Permissions)

/etc/passwd, /etc/shadow 등 계정 정보 파일은 시스템 핵심 자산입니다. 소유자 외의 사용자가 쓰기 권한을 가지거나, 일반 사용자가 읽기 권한을 가지지 않도록 엄격히 관리합니다.

/etc/passwd 644 (-rw-r--r--)
/etc/shadow 400 (-r--------)
# 권한 일괄 수정 명령
$ sudo chmod 644 /etc/passwd
$ sudo chmod 400 /etc/shadow
$ sudo chown root:root /etc/passwd /etc/shadow

3.2 불필요한 SUID/SGID 설정 파일 제거

SUID(Set User ID) 파일은 일반 사용자가 파일을 실행할 때 파일 소유자(보통 Root)의 권한으로 실행되게 합니다. 이는 권한 상승 공격(Privilege Escalation)의 주요 경로이므로 정기적인 전수 조사가 필요합니다.

[도식: SUID 실행 메커니즘 (SUID Execution Flow)]

Low Priv User
SUID
Binaries
Root Access
# SUID 파일 검색
$ find / -user root -perm -4000 -print 2>/dev/null

# 불필요한 파일 권한 제거
$ chmod -s [파일명]

Module 04: Audit & Logging

4.1 시스템 감사 기록 (System Auditing with Auditd)

사고 발생 후 원인 분석 및 추적을 위해 시스템에서 발생하는 주요 이벤트(로그인, 파일 변경, 시스템 콜)를 Auditd(Linux Audit Daemon, 감사 데몬)를 통해 기록합니다.

Monitoring Rules

Path: /etc/audit/rules.d/audit.rules

# 파일 수정 감시
-w /etc/passwd -p wa -k passwd_changes
-w /etc/shadow -p wa -k shadow_changes

Apply Config

# 서비스 재시작
$ sudo service auditd restart

# 감사 로그 조회
$ ausearch -k passwd_changes

Expert Insight (보안 아키텍트 조언)

Cloud Security Warning (AWS 과금 주의)

AWS 환경에서 리눅스 보안을 자동화하기 위해 Amazon GuardDuty나 AWS Config를 활성화할 경우, 데이터 처리량에 따라 상당한 비용이 발생할 수 있습니다. 특히 대용량 로그를 분석 대상으로 설정할 때 프리티어 초과 여부를 반드시 확인하십시오.

Infrastructure as Code (IaC)
