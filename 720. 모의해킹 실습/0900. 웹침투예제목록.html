아키텍처 개요 (Architecture Overview)

Metasploit(메타스플로잇)은 전 세계적으로 가장 많이 사용되는 침투 테스트(Penetration Testing, 페너트레이션 테스팅) 플랫폼입니다. 대상 시스템의 취약점을 탐지하고, 실제 공격 코드를 실행하여 보안 상태를 점검하는 데 최적화되어 있습니다. 무료 오픈소스 버전과 상용 프로 버전이 존재하며, 실무 보안 점검의 표준 도구로 자리매김하고 있습니다.

주요 모듈 구성 (Core Modules)

Metasploit은 기능별로 모듈화(Modularized)되어 있어 높은 확장성을 제공합니다. 모듈 구조를 이해하는 것은 프레임워크를 자유자재로 다루기 위한 첫걸음입니다.

ATTACK

Exploit

(익스플로잇, 취약점 공격 코드)
대상 시스템의 운영체제, 애플리케이션, 또는 네트워크 서비스의 약점(Bug)이나 설계 결함을 이용하여 시스템 권한을 획득하는 실제 공격 코드입니다.

EXECUTE

Payload

(페이로드, 공격 수행 코드)
익스플로잇을 통해 침투에 성공한 직후, 대상 시스템의 메모리나 디스크에서 실행될 동작(예: 리버스 쉘 연결, 백도어 설치, 사용자 계정 추가 등)을 정의합니다.

SCAN

Auxiliary

(옥질리어리, 보조 도구)
익스플로잇처럼 쉘(Shell)을 획득하지는 않지만, 스캐닝(Scanning), 스니핑(Sniffing), 포트 스캔, 서비스 거부 공격(DoS) 등 정보 수집과 보조 작업을 돕는 모듈입니다.

CONTROL

Post

(포스트, 침투 후 작업)
시스템 침투 성공 및 세션 획득 이후, 시스템 내부 정보 수집, 권한 상승(Privilege Escalation), 해시 덤프(Hash Dump), 로그 삭제 등 후속 작업을 수행합니다.

EVADE

Encoder

(인코더, 암호화/난독화 도구)
백신(Anti-Virus)이나 IDS/IPS의 시그니처 기반 탐지를 회피하기 위해 페이로드의 형태를 변조하고 난독화(Obfuscation)하는 역할을 수행합니다.

* 침입 탐지/방지 시스템(Intrusion Detection/Prevention System, IDS/IPS): 네트워크나 시스템의 비정상적인 트래픽과 활동을 모니터링하고 실시간으로 차단하는 엔터프라이즈 보안 솔루션입니다.
* 서비스 거부 공격(Denial of Service, DoS): 타겟 시스템의 네트워크 대역폭이나 메모리 등 리소스를 고갈시켜 정상적인 서비스 제공을 불가능하게 만드는 공격 기법입니다.

Metasploitable 2 기반 웹 모의 해킹 실습 10선

모의 해킹 실습용 리눅스 서버인 Metasploitable 2(우분투 기반)를 대상으로, Kali Linux(칼리 리눅스)에서 수행할 수 있는 웹 취약점(Web Vulnerability) 중심의 실습 예제 10가지를 정리합니다. 웹 애플리케이션의 구조적 결함을 파악하고 공격 벡터를 수립하는 훈련에 최적화되어 있습니다.

번호 실습 주제 (Vulnerability) 취약점 개요 및 실무 아키텍처 설명
01 PHP-CGI Query String RCE CVE-2012-1823 취약점을 이용하여 웹 서버(Apache)에서 원격 코드 실행(Remote Code Execution, RCE)을 유도하고 시스템 쉘(Shell)을 획득합니다.
02 DVWA: SQL Injection
(Error-based)		 웹 애플리케이션의 입력값 검증 미비로 인해 발생하며, DB의 민감한 데이터를 강제로 추출하거나 관리자 권한 계정을 탈취하는 구조화된 질의어 삽입(SQL Injection) 공격입니다.
03 DVWA: Command Injection 웹 페이지의 파라미터 입력창을 통해 리눅스 OS 시스템 명령어(예: cat /etc/passwd)를 주입하여 서버 내부 파일을 무단으로 조회합니다.
04 Mutillidae: Persistent XSS 게시판 등에 악성 자바스크립트를 영구적으로 저장시켜, 해당 글을 열람하는 방문자의 세션 쿠키(Session Cookie)를 탈취하는 크로스 사이트 스크립팅(Cross-Site Scripting, XSS) 공격을 실습합니다.
05 phpMyAdmin Authenticated RCE 기본 관리자 계정(root/공백)의 설정 결함을 이용해 접속한 후, SQL 쿼리를 통해 서버 내부에 백도어(Backdoor) 파일을 강제로 생성합니다.
06 TikiWiki 원격 코드 실행 서버에 설치된 위키 소프트웨어의 특정 모듈(Module) 취약점을 이용하여 메타스플로잇 악성 페이로드를 메모리에 주입합니다.
07 WebDAV 파일 업로드 공격 웹 서버 확장 기능인 웹 분산 저작 및 버전 관리(Web Distributed Authoring and Versioning, WebDAV)의 인증 및 권한 설정 오류를 통해 PHP 웹 쉘(Web Shell)을 직접 업로드합니다.
08 Directory Traversal (LFI) URL 파라미터의 경로를 조작(../../)하여 서버 내부의 민감한 파일인 /etc/passwd 등을 강제로 읽어오는 로컬 파일 포함(Local File Inclusion, LFI) 취약점입니다.
09 TWiki 원격 코드 실행 펄(Perl) 언어 기반의 엔터프라이즈 협업 도구 취약점을 악용해 침투 테스트를 수행하고 역방향 세션(Reverse Session)을 연결합니다.
10 Brute Force (HTTP Auth) 웹 페이지의 HTTP 기본 인증(Basic Authentication) 창을 대상으로 패스워드 사전 대입 공격(Dictionary Attack)을 수행하여 유효한 관리자 계정 정보를 알아냅니다.

주요 취약점 심층 분석 및 익스플로잇 (Exploitation)

1. PHP-CGI 원격 코드 실행 (RCE)

원리 및 아키텍처: PHP 인터프리터가 공통 게이트웨이 인터페이스(Common Gateway Interface, CGI) 모드에서 실행될 때, URL의 쿼리 스트링(Query String)에 전달된 특정 인자(Arguments)를 적절히 이스케이프(Escape) 처리하지 못하여 발생하는 치명적인 결함입니다. 이를 통해 공격자는 소스 코드 노출이나 임의의 바이너리를 실행할 수 있습니다.

msf6 > use exploit/multi/http/php_cgi_arg_injection msf6 exploit(multi/http/php_cgi_arg_injection) > set RHOSTS 192.168.1.100 msf6 exploit(multi/http/php_cgi_arg_injection) > exploit
[*] Meterpreter session 1 opened meterpreter > getuid Server username: www-data

결과: 타겟 시스템의 웹 서비스 구동 계정인 www-data 권한으로 즉각적인 리버스 쉘(Reverse Shell)을 획득하게 됩니다.

2. SQL 인젝션 (DVWA 실습 환경)

원리 및 아키텍처: 웹 애플리케이션 프론트엔드에서 입력된 데이터가 백엔드 데이터베이스로 전달될 때, 입력값에 대한 특수문자 검증(Sanitization)이 누락되어 악의적인 SQL 구문이 쿼리의 일부로 실행(Injection)되는 치명적인 데이터 침해 공격입니다.

# SQLmap 연동 또는 MSF Auxiliary 모듈 활용 예시
kali@attacker:~$ sqlmap -u "http://192.168.1.100/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="security=low; PHPSESSID=..." --dbs
[INFO] fetching database names available databases [2]: [*] dvwa [*] information_schema

결과: 데이터베이스 구조를 매핑한 후, users 테이블의 ID와 비밀번호 해시(Hash) 값을 덤프하여 관리자 권한을 영구적으로 획득할 수 있습니다.

3. 커맨드 인젝션 (Command Injection)

원리 및 아키텍처: 네트워크 Ping 테스트 페이지 등 백엔드에서 리눅스 운영체제 명령어(OS Command)를 직접 호출하는(exec(), system()) 기능이 존재할 때, 세미콜론(;)이나 파이프(|), 논리 연산자(&&)를 통해 다중 명령어를 강제로 주입하는 공격 기법입니다.

# DVWA 웹 페이지 폼(Form) 입력창에 다음 페이로드 삽입
IP Address: 127.0.0.1; cat /etc/passwd
PING 127.0.0.1 (127.0.0.1) 56(84) bytes of data. ... root:x:0:0:root:/root:/bin/bash daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin msfadmin:x:1000:1000:msfadmin,,,:/home/msfadmin:/bin/bash

결과: 본래 목적인 Ping 결과 출력 직후, 공격자가 주입한 명령어 cat /etc/passwd가 실행되어 시스템의 주요 사용자 계정 정보가 평문으로 노출됩니다.

실전 스캐닝 전략 및 아키텍트 인사이트

실전 팁: 자동화 스캐너 연계 전략

Metasploit으로 특정 모듈을 공격하기 전, 전체적인 공격 지점(Attack Surface)을 식별하는 습관이 중요합니다. 실무에서는 자동화된 웹 취약점 스캐너인 Nikto(닉토)나 OWASP ZAP(제드 어택 프록시)을 먼저 구동하여 서버에 열려있는 디렉터리 구조와 오래된 플러그인 목록을 체계적으로 수집해야 합니다.

# 예: 타겟 URL을 파악 후 직접 접속하여 실습 준비
http://[Target_IP]/dvwa
http://[Target_IP]/mutillidae

잦은 실수 및 인가(Authorization)의 중요성

해킹을 처음 배우는 수많은 초보자들이 자신의 로컬 PC가 아닌, 타인의 퍼블릭 웹사이트에 SQLmap이나 Nikto 스캐너를 무단으로 구동하는 치명적인 실수를 범합니다. 단지 "취약점이 있는지 스캔만 했을 뿐이다"라는 변명은 법적으로 통용되지 않으며, 인가받지 않은 스캐닝 행위 자체만으로도 정보통신망법 위반으로 강력한 처벌 대상이 됩니다. 반드시 격리된 Metasploitable 2 가상 머신에서만 실습해야 합니다.

* CGI (Common Gateway Interface, 공통 게이트웨이 인터페이스): 웹 서버와 외부 프로그램(예: PHP 인터프리터) 사이에서 정보를 주고받는 표준화된 프로토콜입니다.
* DVWA (Damn Vulnerable Web Application): 보안 전문가들이 의도적으로 취약하게 코딩하여 배포한 PHP/MySQL 기반의 웹 해킹 훈련용 애플리케이션입니다.
