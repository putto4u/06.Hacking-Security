타겟 분석 및 NSE 아키텍처 개요

본 실습은 Metasploitable 2의 아파치(Apache) 웹 서버 취약점을 정밀 타격하여 탐지할 수 있는 NSE (Nmap Scripting Engine, 엔맵 스크립팅 엔진) 활용법과 맞춤형 Lua (루아, 임베디드 스크립팅 언어) 스크립트 작성 가이드를 다룹니다. NSE는 단순한 포트 스캔을 넘어, 대상 네트워크 서비스의 버전 결함 및 보안 설정 오류를 자동으로 진단하고 검증하는 강력한 자동화 프레임워크입니다.

Metasploitable 2 아파치 타겟 집중 분석

가상의 취약한 환경인 Metasploitable 2에 설치된 아파치 웹 데몬은 고전적이고 치명적인 취약점들을 다수 포함하고 있도록 의도적으로 설계되었습니다. 특히 보안 진단 시 다음 세 가지 항목이 주요 1차 탐지 대상입니다.

  • Apache 2.2.8 (Old version): 수많은 공개된 메모리 손상 버그와 DoS (Denial of Service, 서비스 거부 공격) 취약점을 내포한 심각한 구버전 시스템.
  • PHP-CGI (CVE-2012-1823): URL의 쿼리 스트링(Query String)을 통해 소스 코드를 탈취하거나 RCE (Remote Code Execution, 원격 코드 실행)를 유발하는 치명적인 결함.
  • WebDAV 설정 오류: 인증되지 않은 사용자가 웹 서버 경로에 직접 파일을 생성하고 업로드할 수 있게 허용하는 WebDAV (Web Distributed Authoring and Versioning, 웹 분산 저작 및 버전 관리) 프로토콜 권한 설정 오류.

* CGI (Common Gateway Interface, 공통 게이트웨이 인터페이스): 웹 서버가 외부 프로그램(예: PHP 인터프리터)을 실행하여 동적 웹 페이지를 생성할 때 사용하는 표준 통신 규약입니다.

내장 NSE 스크립트 활용 (Built-in Scripts)

새로운 스크립트를 직접 짜기 전, Nmap 프레임워크에 이미 내장되어 있는 방대한 취약점 DB 스크립트를 먼저 활용하는 것이 실무에서 가장 권장되는 효율적인 방법입니다.

# 1. 아파치 서버의 모든 알려진 취약점(vuln) 카테고리 스캔 kali@attacker:~$ nmap -p 80 --script vuln 192.168.1.100 # [해설] Nmap의 스크립트 카테고리 중 'vuln(Vulnerability, 취약점)' 태그가 지정된 수백 개의 스크립트를 80번 포트에 일괄 구동시켜 알려진 CVE 결함을 스캔합니다.
# 2. HTTP 열거 및 취약한 디렉터리(WebDAV 등) 집중 탐지 kali@attacker:~$ nmap -p 80 --script http-enum,http-methods,http-webdav-scan 192.168.1.100 # [해설] 쉼표(,)를 통해 여러 개의 HTTP 전용 진단 스크립트를 동시 실행합니다. 디렉터리 구조를 열거(enum)하고, 허용된 HTTP 메서드(PUT, DELETE 등)를 확인하며, WebDAV 활성화 여부를 정밀 스캔합니다.
Starting Nmap 7.93... PORT STATE SERVICE 80/tcp open http | http-php-cgi-unset-env: | VULNERABLE: PHP-CGI Remote Code Execution (CVE-2012-1823) | State: VULNERABLE (Exploitable) |_ IDs: CVE:CVE-2012-1823 # [결과 분석] Nmap 내장 스크립트가 대상 시스템에 치명적인 PHP-CGI 원격 코드 실행 취약점이 존재함을 성공적으로 탐지했습니다. 'Exploitable' 상태는 즉시 악용 가능하다는 최고 수준의 위험도를 의미합니다.
* Enumeration (열거): 대상 시스템에 어떠한 파일, 디렉터리, 사용자 계정 등이 존재하는지 무작위 대입이나 패턴 매칭을 통해 샅샅이 파악하는 스캐닝 기법입니다.

맞춤형 NSE 스크립트 제작 (Custom Script)

기존 DB에 없는 새로운 취약점이나 자사만의 특정 보안 정책 위반(예: 특정 관리자 페이지 노출 여부)을 점검하기 위해 Lua (루아) 언어를 사용하여 맞춤형 NSE 스크립트를 제작할 수 있습니다. 아래는 PHP-CGI 취약점 징후를 직접 확인하는 apache-check.nse 스크립트의 소스 코드 구조입니다.

apache-check.nse
-- Metasploitable 2의 특정 아파치 취약 경로를 확인하는 맞춤형 Lua 스크립트 local http = require "http" local shortport = require "shortport" -- [해설] 스크립트 구동에 필요한 HTTP 통신 모듈과 포트 판별(shortport) 라이브러리를 메모리에 불러옵니다(require). description = [[ Metasploitable 2의 Apache 및 PHP-CGI 취약점(CVE-2012-1823) 존재 여부를 자체적으로 체크합니다. ]] author = "Sung-doo Won" license = "Same as Nmap" categories = {"vuln", "discovery"} -- [해설] 스크립트의 메타데이터를 정의합니다. 카테고리를 설정하면 추후 nmap --script vuln 명령어 실행 시 자동 포함될 수 있습니다. -- 80번 포트나 HTTP 서비스가 열려 있을 때만 스크립트가 실행되도록 트리거(Trigger) 규칙 설정 portrule = shortport.http action = function(host, port) -- PHP-CGI 취약점(소스코드 노출)을 강제로 유도하는 특수 쿼리 스트링 파라미터(-s) 정의 local path = "/index.php?-s" local response = http.get(host, port, path) -- [해설] 타겟 웹 서버로 GET 방식의 HTTP 요청을 전송하고 결과를 response 변수에 담습니다. -- 응답 바디(Body)에 PHP 소스 코드 하이라이트 HTML 태그()가 평문으로 보인다면 취약한 것임 if ( response.status == 200 and response.body:match("") ) then return "Critical: PHP-CGI RCE (CVE-2012-1823) 취약점이 발견되었습니다!" end return "정상: 해당 취약점이 발견되지 않았습니다." end -- [해설] 메인 실행 함수(action)입니다. HTTP 상태 코드가 200(정상)이고, 반환된 응답 본문에 <code> 문자열 패턴 매칭(match)이 성공하면 경고 메시지를 터미널에 출력합니다.
* Lua (루아): 프로그램 내부에 쉽게 이식(Embedded)할 수 있도록 가볍고 빠르게 설계된 스크립팅 언어로, 게임 엔진이나 Nmap과 같은 보안 도구에서 확장 모듈을 작성할 때 널리 쓰입니다.
* Pattern Matching (패턴 매칭): 텍스트 데이터 내에서 특정 문자열이나 정규 표현식 규칙에 부합하는 구조를 찾아내는 데이터 검사 기법입니다.

커스텀 스크립트 실행 및 로그 검증

작성한 Lua 기반 커스텀 스크립트(apache-check.nse)를 파일로 저장한 후, Kali Linux의 Nmap 엔진에 직접 로드하여 실행하는 실증 단계입니다.

# 1. 스크립트 실행 권한 부여 및 타겟 대상 실행 kali@attacker:~$ chmod +x apache-check.nse kali@attacker:~$ nmap -p 80 --script ./apache-check.nse 192.168.1.100 # [해설] 스크립트 파일이 위치한 현재 경로(./)를 명시하여 Nmap이 기본 내장 스크립트 폴더 대신 해당 파일을 로컬에서 직접 불러오도록 지시합니다.
Starting Nmap 7.93... Nmap scan report for 192.168.1.100 PORT STATE SERVICE 80/tcp open http |_apache-check: Critical: PHP-CGI RCE (CVE-2012-1823) 취약점이 발견되었습니다! # [결과 분석] 제작한 커스텀 NSE 스크립트가 타겟 시스템의 HTTP 응답에서 PHP 소스 하이라이트 태그(<code>) 노출을 성공적으로 패턴 매칭하여, 관리자에게 취약 상태를 보고했습니다. 커스텀 자동화 툴이 정상 동작함을 입증합니다.

아키텍트 대응 방안 (Remediation)

이러한 NSE 자동화 탐지 및 해커의 스캐닝 정보 획득을 원천적으로 차단하고 시스템을 보호하기 위해, 서버 관리자(인프라 아키텍트)는 아파치 환경에서 다음과 같은 필수 보안 강화(Hardening) 조치를 적용해야 합니다.

1. 버전 정보 은닉 (Banner Grabbing 방어)

httpd.conf 파일에서 웹 서버의 운영체제와 상세 버전 정보를 외부에 노출하지 않도록 수정합니다. 이는 해커의 공격 벡터 식별을 지연시킵니다.

ServerTokens Prod
ServerSignature Off

2. 취약한 CGI 실행 제한

오래되고 보안 결함이 많은 CGI 모듈(mod_cgi) 사용을 중단하고 비활성화하십시오. 성능과 보안이 개선된 현대화된 PHP-FPM(FastCGI Process Manager) 방식으로 아키텍처를 전환해야 합니다.

3. 시스템 보안 패치 적용

가장 근본적인 해결책입니다. 지원이 종료된 구버전 아파치(2.2.x 계열) 사용을 폐기하고, 최신 보안 패치와 메모리 보호 기법이 적용된 최신 안정화 버전(Apache 2.4.x 이상)으로 인프라를 전면 마이그레이션(Migration) 해야 합니다.

* Banner Grabbing (배너 그래빙): 서비스 포트에 접속하여 서버가 응답하는 초기 환영 메시지(배너)를 통해 해당 서버의 소프트웨어 종류와 버전을 수집하는 정보 획득 기법입니다.
* Remediation (복구 및 개선): 발견된 보안 취약점을 제거하고 향후 동일한 위협이 발생하지 않도록 시스템 구조를 안전하게 개선하는 조치를 뜻합니다.
