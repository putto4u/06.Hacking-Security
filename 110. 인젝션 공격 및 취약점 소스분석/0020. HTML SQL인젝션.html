1. PostgreSQL 엔진 가동 및 자동화 설정 (DB Persistence)

메타스플로잇의 성능을 극대화하려면 PostgreSQL(포스트그레스큐엘, 오픈 소스 객체-관계형 데이터베이스 시스템) 서비스가 먼저 활성화되어야 합니다. 특히 시스템 재시작 시에도 자동으로 가동되도록 설정하는 것이 실무의 핵심입니다.

[도식: 데이터베이스 초기화 및 자동화 워크플로우]

START
systemctl start

엔진 즉시 구동

ENABLE
systemctl enable

부팅 시 자동 가동

INITIALIZE
msfdb init

스키마 및 계정 생성

[Database Service Commands]

# 서비스 시작
sudo systemctl start postgresql
# 부팅 시 자동 시작 등록
sudo systemctl enable postgresql
# MSF 데이터베이스 초기화
sudo msfdb init

[Verification in msfconsole]

msf6 > db_status
[*] Connected to msf. Connection type: postgresql.
INFO: DB 연결 시 `hosts`, `services` 명령어로 정보를 관리할 수 있습니다.
PostgreSQL (포스트그레스큐엘) 메타스플로잇의 방대한 공격 모듈 정보와 수집된 타겟 데이터를 저장하는 고성능 객체-관계형 데이터베이스입니다.
Persistence (지속성) 시스템이 재시작(Reboot)되어도 서비스 상태가 유지되도록 설정하는 것으로, `systemctl enable`이 이 역할을 수행합니다.

2. 모듈 정밀 탐색 전략 (Search & Filtering)

실전 침투 시 수천 개의 모듈 중 목적에 맞는 모듈을 찾는 것은 매우 중요합니다. 단순히 키워드만 검색하는 것이 아니라, 필터(Filter)를 활용하여 공격 대상(Platform)과 유형(Type)을 좁혀 나갑니다.

Strategic Module Search

# SQL 인젝션 스캐너를 찾기 위한 최적의 쿼리
msf6 > search type:auxiliary name:sql_injection

# 결과 분석 (Output Analysis)
Matching Modules
================
   #  Name                             Rank    Check  Description
   -  ----                             ----    -----  -----------
   0  auxiliary/scanner/http/sql_injection  normal  No     HTTP SQL Injection Scanner
Type Filter

exploit(공격), auxiliary(보조), post(후속) 등 모듈 유형별 필터링

type:auxiliary
Platform Filter

linux, windows, android 등 대상 운영체제별 필터링

platform:linux
Rank Filter

excellent, great 등 공격 신뢰도가 높은 순서로 정렬

rank:excellent
Auxiliary Module (보조 모듈) 실제 공격(Exploit)을 수행하기 전 단계에서 스캐닝, 정보 수집, 취약점 탐지 등을 수행하는 도구 모음입니다.
Rank (랭크) 해당 모듈이 타겟 시스템에 미치는 영향과 성공 가능성을 나타내며, `excellent`는 시스템 충돌 위험이 거의 없음을 의미합니다.
