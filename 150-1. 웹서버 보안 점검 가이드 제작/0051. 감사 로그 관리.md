## 5.1 감사 로그 관리 및 중앙 집중화

---

### **1. 감사 로그(Audit Log)의 정의와 중요성**

**Audit Log (감사 로그, 시스템 및 사용자 활동 기록)**는 정보 시스템 내에서 발생하는 모든 활동(접속 기록, 데이터 수정, 오류 발생 등)을 시간 순서대로 기록한 데이터입니다. 침해 사고 발생 시 **Forensics (포렌식, 디지털 증명 및 분석)**의 핵심 증거로 사용되며, 법적 규제 준수(Compliance)를 위해 반드시 안전하게 관리되어야 합니다.



* **기밀성 및 무결성**: 로그는 공격자에 의해 변조되거나 삭제되지 않도록 별도의 격리된 저장소에 보관해야 합니다.
* **추적성**: "누가, 언제, 어디서, 무엇을 했는지"에 대한 **5W1H (Who, When, Where, What, Why, How)** 정보를 포함해야 합니다.

> **설명**: 웹 서버 보안의 완성은 침입을 막는 것뿐만 아니라, 침입 후의 행적을 완벽히 기록하여 재발을 방지하는 시스템을 갖추는 것입니다.

---

### **2. 웹 서버 로그 설정 (Nginx 기준)**

웹 서버의 상세한 로그 설정은 공격 시도의 패턴을 분석하는 기초 자료가 됩니다.

* **Access Log (접근 로그)**: 모든 HTTP 요청에 대한 기록을 남깁니다.
* **Error Log (에러 로그)**: 서버 내부 오류나 비정상적인 요청에 대한 기록을 남깁니다.

```bash
# Nginx 로그 포맷 설정 (/etc/nginx/nginx.conf)
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                  '$status $body_bytes_sent "$http_referer" '
                  '"$http_user_agent" "$http_x_forwarded_for"';

# 로그 저장 경로 및 포맷 적용
access_log /var/log/nginx/access.log main;
error_log /var/log/nginx/error.log warn;
```

* **Log Rotation (로그 로테이션, 로그 순환)**: 로그 파일이 너무 커져 디스크 용량을 소모하는 것을 방지하기 위해 주기적으로 파일을 압축하고 교체합니다.

```bash
# 리눅스 logrotate 설정 확인
cat /etc/logrotate.d/nginx
```

### **3. 로그 중앙 집중화 및 SIEM 연동**

개별 서버에 로그를 두면 서버 탈취 시 로그가 삭제될 위험이 있습니다. 따라서 외부의 **Log Collector (로그 수집기)**를 통해 중앙 저장소로 전송해야 합니다.

* **ELK Stack (Elasticsearch, Logstash, Kibana)**: 오픈소스 기반의 실시간 로그 수집, 분석 및 시각화 도구입니다.
* **SIEM (Security Information and Event Management, 보안 정보 및 이벤트 관리)**: 여러 시스템의 로그를 실시간으로 통합 분석하여 위협을 탐지하는 보안 솔루션입니다.



> **설명**: **SIEM (보안 정보 및 이벤트 관리)**은 단순한 로그 저장을 넘어, 서로 다른 장비에서 발생하는 로그들 사이의 상관관계를 분석하여 고도화된 공격을 탐지합니다.

---

### **4. [AWS 환경 보안 서비스 및 과금 안내]**

AWS에서는 로그 수집과 분석을 자동화해주는 강력한 관리형 서비스를 제공합니다.

* **Amazon CloudWatch Logs (아마존 클라우드워치 로그)**
    * EC2 인스턴스의 로그를 실시간으로 수집하고 저장합니다.
    * **과금 안내**: 수집되는 데이터량(GB당 **$0.50**)과 보관되는 데이터량(GB당 **$0.03/월**)에 따라 비용이 발생합니다. [공식 요금 페이지](https://aws.amazon.com/ko/cloudwatch/pricing/)
* **Amazon S3 (아마존 S3)**
    * 장기 보관이 필요한 로그를 저렴한 비용으로 저장하는 용도로 사용됩니다.
* **Amazon OpenSearch Service (아마존 오픈서치 서비스)**
    * 대량의 로그 데이터를 고속으로 검색하고 시각화합니다. (유료 인스턴스 시간당 과금)

**상세 설명 주석**
1. **Forensics**: 디지털 매체에 기록된 정보를 수집, 식별, 분석하여 법적 증거 능력을 확보하는 기술입니다.
2. **Log Rotation**: 로그 파일의 무한 증식을 막기 위해 특정 주기로 기존 파일을 보관하고 새 파일을 생성하는 관리 방식입니다.
3. **Logstash**: 다양한 소스로부터 데이터를 수집하고 변환하여 원하는 저장소로 전송하는 데이터 파이프라인 도구입니다.
4. **CloudWatch Agent**: 서버 내부의 로그 파일(access.log 등)을 읽어 CloudWatch Logs 서비스로 전송하기 위해 설치하는 에이전트 소프트웨어입니다.
5. **Retention Policy (보존 정책)**: 로그 데이터를 법적/업무적 필요에 따라 얼마나 오래 보관할지 결정하는 기준입니다.
