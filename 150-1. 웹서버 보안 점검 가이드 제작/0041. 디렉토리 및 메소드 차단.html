1. Directory Listing (디렉터리 리스팅)의 보안 위협

Directory Listing (디렉터리 리스팅, 서버 내 파일 목록 노출)은 브라우저가 인덱스 파일(index.html 등)이 없는 디렉터리에 접근했을 때, 서버가 해당 디렉터리에 포함된 모든 파일과 폴더 목록을 보여주는 기능입니다. 이는 공격자에게 서버의 디렉터리 구조, 백업 파일, 소스 코드, 설정 파일 등을 노출시키는 심각한 Information Disclosure (정보 노출, 인가되지 않은 정보의 외부 유출) 취약점이 됩니다.

[도식: Information Disclosure via Directory Listing (정보 노출 시나리오)]

Index of /backup
  • config.php.bak
  • users_dump.sql
  • .git/
VULNERABLE: RISKY EXPOSURE
취약점 영향 (Impact)

관리자가 실수로 남겨둔 .bak, .sql, .git 등의 민감한 파일이 공격자에 의해 다운로드될 수 있습니다.

보안 목표 (Security Goal)

파일 목록 대신 403 Forbidden (접근 금지) 에러를 반환하도록 명시적으로 설정합니다.

설명: 웹 서버는 요청받은 경로에 기본 페이지가 없을 경우 자동으로 목록을 생성하여 응답하는 기능을 내장하고 있습니다. 보안을 위해 이 자동 생성 기능을 명시적으로 비활성화해야 합니다.

autoindex (오토인덱스): Nginx(엔진엑스)에서 디렉터리 내의 파일 목록을 HTML 형태로 자동 생성하여 보여주는 모듈입니다.
Indexes (인덱스 옵션): Apache(아파치)에서 인덱스 파일이 없을 때 파일 목록을 출력하도록 허용하는 옵션입니다.

2. 웹 서버별 디렉터리 리스팅 차단 설정

가장 널리 쓰이는 두 웹 서버에서 해당 기능을 끄는 방법은 다음과 같습니다.

Nginx (엔진엑스) 설정

# 1. Nginx 기본 설정 파일 편집
sudo vi /etc/nginx/sites-available/default # 설정 파일을 텍스트 에디터로 엽니다.

location / {
autoindex off; # 디렉터리 파일 목록 자동 생성 기능을 끕니다.
}

# 2. 변경된 설정 적용
sudo systemctl restart nginx # 엔진엑스 서비스를 재시작하여 반영합니다.

Apache (아파치) 설정

# 1. 아파치 메인 설정 파일 편집
sudo vi /etc/apache2/apache2.conf # 시스템 설정 파일을 편집기로 엽니다.

<Directory /var/www/html>
Options -Indexes # 하이픈(-)을 붙여 인덱싱 기능을 명시적으로 제거합니다.
</Directory>

# 2. 서비스 재시작
sudo systemctl restart apache2 # 아파치 서버를 재시작하여 설정을 확정합니다.
Options -Indexes: 아파치 설정에서 특정 디렉터리에 대해 파일 목록 출력을 금지하는 가장 확실한 방법입니다.

3. 불필요한 HTTP Methods (HTTP 메소드) 제한

HTTP Methods (HTTP 메소드, 하이퍼텍스트 전송 프로토콜 요청 방식)는 클라이언트가 서버에 요청하는 동작의 종류를 정의합니다. 일반적인 웹 서비스는 GET (데이터 조회)과 POST (데이터 제출)만으로 충분합니다.

[도식: HTTP Method Filtering Policy (허용/차단 정책)]

GET / POST
HEAD / OPTIONS
Security Barrier
PUT / DELETE
TRACE / CONNECT

Nginx 메소드 제한 구현 (Implementation)

server { # 서버 가상 호스트 설정을 시작합니다.
# 지정된 메소드(GET, HEAD, POST)가 아닐 경우 조건문 실행
if ($request_method !~ ^(GET|HEAD|POST)$ ) {
return 405; # 405 Method Not Allowed 코드를 응답하며 요청을 거부합니다.
}
}

설명: HTTP 405 Method Not Allowed 응답은 클라이언트가 사용한 요청 방식이 서버에서 금지되었음을 의미합니다. 이를 통해 공격 벡터를 최소화할 수 있습니다.

HTTP TRACE (HTTP 트레이스): 클라이언트의 요청 메시지를 그대로 응답 본문에 담아 보내는 진단용 메소드로, 보안상 위험하여 반드시 차단해야 합니다.
405 Method Not Allowed (405 허용되지 않은 메소드): 요청한 메소드가 서버에서 지원되지만 보안 정책상 명시적으로 금지되었을 때 발생하는 상태 코드입니다.

[AWS 환경 보안 및 과금 안내]

클라우드 환경에서는 스토리지 서비스와 네트워크 레벨에서 리스팅과 메소드 제어를 병행해야 합니다.

Amazon S3 (아마존 S3, 객체 스토리지 서비스)

ListBucket 권한이 퍼블릭으로 열려 있을 경우 전 세계 누구나 파일을 리스팅할 수 있습니다. 반드시 '퍼블릭 액세스 차단' 설정을 활성화해야 합니다.

Billing (과금 안내)
  • • 저장 용량 및 데이터 전송량(DTO) 기준 과금
  • • API 요청 건수 (PUT, GET, LIST)에 따라 비용 발생
  • • 비정상적인 LIST 요청 폭주 시 비용 증가 주의

AWS WAF (웹 애플리케이션 방화벽)

WAF의 Custom Rule (사용자 정의 규칙)을 사용하여 특정 HTTP 메소드(예: DELETE, PUT)를 서버 도달 전 원천적으로 차단할 수 있습니다.

Billing (유료 서비스)
  • • 활성화된 웹 ACL 및 규칙 수 기준 월 고정 비용
  • • 처리된 100만 건당 약 $0.60의 요청 처리 비용
ListBucket (리스트버킷, 버킷 내 객체 목록 조회 권한): S3 버킷 내의 파일 목록을 볼 수 있는 권한으로, 보안 사고의 주요 원인이므로 엄격히 관리되어야 합니다.
