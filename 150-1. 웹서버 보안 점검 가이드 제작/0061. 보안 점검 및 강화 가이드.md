## 웹 서버 보안 점검 및 강화 가이드 (Web Server Security Hardening Guide)

---

### **[PART 1] 1.1 OS Hardening (운영체제 요새화)**

**OS Hardening**은 운영체제의 설정 오류와 불필요한 요소를 제거하여 **Attack Surface (공격 표면, 공격 가능한 영역)**를 최소화하는 공정입니다. 최신 리눅스 커널 및 패키지 관리 기준(2026년 가이드)을 적용하여 시스템의 기밀성과 무결성을 확보합니다.



* **불필요한 네트워크 서비스 식별**
    현재 외부로 노출된 포트와 해당 프로세스를 정확히 파악하여 미사용 서비스를 격리합니다.
* **보안 업데이트 자동화**
    발표된 **CVE (Common Vulnerabilities and Exposures, 알려진 보안 취약점 식별자)**에 대응하기 위해 보안 패치를 최신 상태로 유지합니다.
* **SSH (Secure Shell) 강화**
    원격 접속 시 **Root (루트, 최고 관리자)** 직접 접속을 차단하여 계정 탈취 위험을 방지합니다.

> **설명**: 시스템의 가장 하단인 OS 레벨에서 보안 구멍을 메우는 것은 상위 웹 애플리케이션 보안을 위한 필수 전제 조건입니다.

```bash
# 1. 실행 중인 리스닝 포트 및 프로세스 확인 (최신 ss 명령어 사용)
sudo ss -tulpn

# 2. 불필요한 서비스 중지 및 비활성화 (예: rpcbind)
sudo systemctl stop rpcbind
sudo systemctl disable rpcbind

# 3. 최신 패키지 업데이트 및 보안 패치 적용
sudo apt update && sudo apt upgrade -y

# 4. SSH Root 로그인 차단 (sed를 이용한 안전한 수정)
sudo sed -i 's/^#PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart ssh
```

* **AWS 비용 안내**: **Amazon Inspector (아마존 인스펙터)** 사용 시, EC2 인스턴스 스캔당 비용이 발생합니다. (월별 스캔 횟수에 따라 과금)

### **[PART 1] 1.2 리눅스 커널 파라미터 최적화**

커널 파라미터 조정은 네트워크 스택 수준에서 발생할 수 있는 **DDoS (Distributed Denial of Service, 분산 서비스 거부 공격)** 및 **IP Spoofing (IP 스푸핑, IP 주소 위변조)**을 방어하는 효과적인 방법입니다.



* **SYN Cookies (SYN 쿠키) 활성화**
    **TCP SYN Flood (TCP SYN 플러드, 연결 요청 폭주 공격)** 시 서버 자원 고갈을 방지합니다.
* **ICMP (Internet Control Message Protocol) 제어**
    서버의 위치 노출 및 네트워크 진단을 악용한 공격(Smurf Attack 등)을 차단합니다.
* **Reverse Path Filtering (역경로 필터링)**
    위조된 소스 IP를 가진 패킷이 유입되지 않도록 경로 검증을 강화합니다.

> **설명**: **sysctl (System Control, 커널 매개변수 제어)**은 재부팅 없이 커널 설정을 변경할 수 있는 도구이며, 설정 후에는 반드시 파일을 저장하여 영구 적용해야 합니다.

```bash
# 1. SYN Flood 방어 설정
sudo sysctl -w net.ipv4.tcp_syncookies=1

# 2. ICMP Redirect 차단 (MITM 공격 방지)
sudo sysctl -w net.ipv4.conf.all.accept_redirects=0
sudo sysctl -w net.ipv4.conf.default.accept_redirects=0

# 3. IP 스푸핑 방지 (RP Filter 활성화)
sudo sysctl -w net.ipv4.conf.all.rp_filter=1
sudo sysctl -w net.ipv4.conf.default.rp_filter=1

# 4. 설정 파일 영구 적용
sudo sysctl -p
```

* **AWS 비용 안내**: **AWS Shield Standard (아마존 실드 스탠다드)**는 기본 무료 제공되나, **Shield Advanced (고급형)**는 월 $3,000의 고정비용과 데이터 전송 비용이 발생합니다.

### **[PART 2] 2.1 정보 노출 방지 및 배너 숨기기**

공격자는 공격 전 **Banner Grabbing (배너 그래빙, 서비스 정보 수집)**을 통해 서버 종류와 버전을 파악합니다. 이를 숨김으로써 특정 버전에 특화된 **Exploit (익스플로잇, 취약점 공격 코드)** 시도를 사전에 차단합니다.



* **Nginx 버전 정보 은폐**
    HTTP 응답 헤더에서 `Server: nginx/1.x.x`와 같은 버전 노출을 차단합니다.
* **Apache 서버 서명 제거**
    에러 페이지 하단에 노출되는 시스템 정보를 비활성화합니다.

> **설명**: **Footprinting (풋프린팅, 사전 조사)** 단계를 어렵게 만드는 것만으로도 자동화된 공격 도구의 타겟에서 제외될 확률이 높아집니다.

```bash
# [Nginx] 설정 파일 수정
sudo vi /etc/nginx/nginx.conf
# http { ... } 블록 내에 추가
server_tokens off;

# [Apache] 설정 파일 수정 (Ubuntu 기준)
sudo vi /etc/apache2/conf-enabled/security.conf
# 아래 항목 수정
ServerTokens Prod
ServerSignature Off

# 서비스 재시작
sudo systemctl restart nginx
sudo systemctl restart apache2
```

### **[PART 3] 3.1 SSL/TLS (보안 소켓 계층/전송 계층 보안) 적용**

최신 웹 표준인 **TLS 1.3 (Transport Layer Security 1.3)**을 적용하여 통신 과정의 기밀성과 데이터 무결성을 보장합니다. 취약한 암호화 알고리즘을 제거하는 것이 핵심입니다.



* **Cipher Suite (사이퍼 스위트, 암호 알고리즘 조합) 최적화**
    안전하지 않은 DES, RC4, MD5 등을 제거하고 강력한 알고리즘만 허용합니다.
* **HSTS (HTTP Strict Transport Security) 활성화**
    클라이언트가 항상 HTTPS로만 접속하도록 강제하여 **SSL Stripping (SSL 스트리핑, 하향 유도 공격)**을 방어합니다.

> **설명**: **Forward Secrecy (전방 향후 보안)**가 보장되는 암호화 방식을 선택하면, 미래에 서버 개인키가 유출되어도 과거에 암호화된 통신 내용을 복구할 수 없습니다.

```bash
# Nginx 가상 호스트 보안 설정 예시
server {
    listen 443 ssl;
    ssl_protocols TLSv1.2 TLSv1.3; # 최신 버전만 허용
    
    # 강력한 암호화 알고리즘 세트 (Mozilla 권장 가이드 반영)
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
    ssl_prefer_server_ciphers on;

    # HSTS 설정 (1년 유지)
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
}
```

* **AWS 비용 안내**: **ACM (AWS Certificate Manager)**을 통한 퍼블릭 인증서 발급은 **무료**이나, **Private CA (사설 인증기관)** 운영 시 월별 고정 비용이 발생합니다.

### **[PART 4] 4.1 디렉터리 리스팅 및 메소드 차단**

불필요한 디렉터리 목록 노출과 위험한 **HTTP Methods (HTTP 메소드, 요청 방식)**를 차단하여 서버 내부 구조가 드러나는 것을 방지합니다.



* **Directory Indexing (디렉터리 인덱싱) 비활성화**
    `index.html` 파일이 없는 폴더 접근 시 파일 목록이 나열되는 현상을 차단합니다.
* **위험한 메소드(PUT, DELETE, TRACE) 제한**
    서버 자원을 수정하거나 탈취할 수 있는 요청 방식을 원천 차단합니다.

> **설명**: **HTTP 405 Method Not Allowed** 응답을 활용하여 허용된 방식(GET, POST) 외의 접근을 차단하는 것이 보안의 기본입니다.

```bash
# 1. [Nginx] 디렉터리 리스팅 차단 (기본값이 off이나 명시적 설정 권장)
location / {
    autoindex off;
}

# 2. [Nginx] 허용되지 않은 메소드 일괄 차단
if ($request_method !~ ^(GET|HEAD|POST)$ ) {
    return 405;
}

# 3. [Apache] 디렉터리 리스팅 차단
<Directory /var/www/html>
    Options -Indexes
</Directory>
```

* **AWS 비용 안내**: **Amazon S3 (아마존 S3)** 버킷 사용 시, '퍼블릭 액세스 차단' 설정은 무료이나 데이터를 리스팅하는 API 호출 건수에 따라 과금이 발생합니다.

### **용어 설명 및 주석 (Footnotes)**
1. **CVE (Common Vulnerabilities and Exposures)**: 보안 취약점을 관리하는 국제 표준 식별 번호입니다.
2. **HSTS (HTTP Strict Transport Security)**: 브라우저에 해당 웹사이트는 HTTPS로만 접속해야 함을 알리는 보안 정책입니다.
3. **SS (Socket Statistics)**: 리눅스에서 네트워크 소켓 정보를 확인하는 최신 명령어로, 구형인 `netstat`을 대체합니다.
4. **MITM (Man-In-The-Middle)**: 중간자 공격으로, 통신 당사자 사이에서 데이터를 도청하거나 변조하는 행위입니다.
5. **ACM (AWS Certificate Manager)**: AWS 서비스(ELB, CloudFront 등)에서 사용할 SSL/TLS 인증서를 통합 관리하는 서비스입니다.
