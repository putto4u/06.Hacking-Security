1. 감사 로그(Audit Log)의 정의와 중요성

Audit Log (감사 로그, 시스템 및 사용자 활동 기록)는 정보 시스템 내에서 발생하는 모든 활동(접속 기록, 데이터 수정, 오류 발생 등)을 시간 순서대로 기록한 데이터입니다. 침해 사고 발생 시 Forensics (디지털 포렌식, 디지털 증명 및 분석)의 핵심 증거로 사용되며, 법적 규제 준수(Compliance, 컴플라이언스)를 위해 반드시 안전하게 관리되어야 합니다.

[도식: 5W1H Principle of Audit Logging (감사 로그의 6하원칙 구조)]

WHO
사용자 계정 및 IP
WHEN
정확한 타임스탬프
WHERE
시스템 및 서비스 위치
WHAT
수행된 구체적 행위
WHY
행위의 목적 및 배경
HOW
사용된 프로토콜/도구

기밀성 및 무결성

로그는 공격자에 의해 변조되거나 삭제되지 않도록 별도의 격리된 저장소에 보관해야 합니다.

추적성 (Traceability)

"누가, 언제, 어디서, 무엇을 했는지"에 대한 5W1H 정보를 포함하여 사후 추적을 가능케 합니다.

설명: 웹 서버 보안의 완성은 침입을 막는 것뿐만 아니라, 침입 후의 행적을 완벽히 기록하여 재발을 방지하는 시스템을 갖추는 것입니다.

Forensics (디지털 포렌식): 디지털 매체에 기록된 정보를 수집, 식별, 분석하여 법적 증거 능력을 확보하는 기술입니다.
Log Rotation (로그 순환): 로그 파일의 무한 증식을 막기 위해 특정 주기로 기존 파일을 보관하고 새 파일을 생성하는 관리 방식입니다.

2. 웹 서버 로그 설정 (Nginx 기준)

웹 서버의 상세한 로그 설정은 공격 시도의 패턴을 분석하는 기초 자료가 됩니다.

Access & Error Logging

모든 HTTP 요청(Access)과 시스템 비정상 동작(Error)을 분리하여 정교한 로그 포맷으로 기록합니다.

# Nginx 로그 포맷 정의 (/etc/nginx/nginx.conf)
log_format main '$remote_addr - $remote_user [$time_local] "$request" ' # 클라이언트 주소, 사용자, 시간을 기록합니다.
'$status $body_bytes_sent "$http_referer" ' # 응답 상태, 전송량, 유입 경로를 기록합니다.
'"$http_user_agent" "$http_x_forwarded_for"'; # 브라우저 정보와 경유 IP를 포함합니다.

# 로그 저장 경로 및 정의된 포맷 적용
access_log /var/log/nginx/access.log main; # 접근 로그를 main 포맷으로 저장합니다.
error_log /var/log/nginx/error.log warn; # 경고 이상의 에러만 로그로 남깁니다.

Log Rotation Policy

파일이 무한정 커지는 것을 방지하기 위해 주기적으로 파일을 압축하고 순환(Rotation) 관리합니다.

# 시스템 로그 순환 설정 확인
cat /etc/logrotate.d/nginx # 엔진엑스용 로그 로테이션 설정을 출력합니다.
/var/log/nginx/*.log {
  daily
  rotate 14
  compress
}
Logstash (로그스태시): 다양한 소스로부터 데이터를 실시간으로 수집, 변환하여 원하는 저장소로 전송하는 데이터 파이프라인 도구입니다.

3. 로그 중앙 집중화 및 SIEM 연동

개별 서버에 로그를 두면 서버 탈취 시 로그가 삭제될 위험이 있습니다. 따라서 외부의 Log Collector (로그 수집기)를 통해 중앙 저장소로 전송해야 합니다.

ELK Stack (엘크 스택)

Elasticsearch, Logstash, Kibana를 결합한 오픈소스 기반의 실시간 로그 수집, 분석 및 시각화 도구 세트입니다.

SIEM (보안 정보 및 이벤트 관리)

여러 시스템의 로그를 실시간으로 통합 분석하여 보안 위협 및 비정상 행위를 탐지하는 전문 보안 솔루션입니다.

설명: SIEM (보안 정보 및 이벤트 관리)은 단순한 로그 저장을 넘어, 서로 다른 장비에서 발생하는 로그들 사이의 상관관계를 분석하여 고도화된 공격을 탐지합니다.

SIEM (Security Information and Event Management): 정보 시스템의 보안 로그와 이벤트를 실시간으로 통합하여 이상 징후를 탐지하는 체계입니다.

CloudWatch Agent (클라우드워치 에이전트): 서버 내부 로그 파일(access.log 등)을 실시간으로 읽어 AWS 클라우드로 전송하기 위한 소프트웨어입니다.
Retention Policy (보존 정책): 법적 근거 또는 비즈니스 필요에 따라 로그 데이터를 얼마나 보관할지 정의하는 기준입니다.
