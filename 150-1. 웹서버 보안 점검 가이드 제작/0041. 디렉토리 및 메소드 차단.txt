## 4.1 디렉터리 리스팅 및 불필요한 HTTP 메소드 차단

---

### **1. Directory Listing (디렉터리 리스팅)의 보안 위협**

**Directory Listing (디렉터리 리스팅, 서버 내 파일 목록 노출)**은 브라우저가 인덱스 파일(index.html 등)이 없는 디렉터리에 접근했을 때, 서버가 해당 디렉터리에 포함된 모든 파일과 폴더 목록을 보여주는 기능입니다. 이는 공격자에게 서버의 디렉터리 구조, 백업 파일, 소스 코드, 설정 파일 등을 노출시키는 심각한 **Information Disclosure (정보 노출, 인가되지 않은 정보의 외부 유출)** 취약점이 됩니다.



* **취약점 영향**: 관리자가 실수로 남겨둔 `.bak`, `.sql`, `.git` 등의 민감한 파일이 공격자에 의해 다운로드될 수 있습니다.
* **보안 목표**: 인덱스 파일이 없는 경우에도 파일 목록을 보여주지 않고 **403 Forbidden (접근 금지)** 에러를 반환하도록 설정합니다.

> **설명**: 웹 서버는 요청받은 경로에 기본 페이지가 없을 경우 자동으로 목록을 생성하여 응답하는 기능을 내장하고 있습니다. 보안을 위해 이 자동 생성 기능을 명시적으로 비활성화해야 합니다.

---

### **2. 웹 서버별 디렉터리 리스팅 차단 설정**

가장 널리 쓰이는 두 웹 서버에서 해당 기능을 끄는 방법은 다음과 같습니다.

* **Nginx (엔진엑스)**
    기본적으로 Nginx는 이 기능이 꺼져 있으나, 설정 파일에서 `autoindex` 속성이 `on`으로 되어 있는지 반드시 확인해야 합니다.

```bash
# Nginx 설정 파일 수정
sudo vi /etc/nginx/sites-available/default

# server 또는 location 블록 내 설정
location / {
    autoindex off;  # 디렉터리 리스팅 비활성화
}

# 설정 적용
sudo systemctl restart nginx
```

* **Apache (아파치)**
    Apache는 기본적으로 리스팅이 활성화된 경우가 많으므로 `Options` 지시어에서 `Indexes`를 제거해야 합니다.

```bash
# Apache 보안 설정 또는 사이트 설정 수정
sudo vi /etc/apache2/apache2.conf

# 해당 디렉터리 설정에서 Indexes 옵션 제거 (-)
<Directory /var/www/html>
    Options -Indexes
</Directory>

# 설정 적용
sudo systemctl restart apache2
```

### **3. 불필요한 HTTP Methods (HTTP 메소드) 제한**

**HTTP Methods (HTTP 메소드, 하이퍼텍스트 전송 프로토콜 요청 방식)**는 클라이언트가 서버에 요청하는 동작의 종류를 정의합니다. 일반적인 웹 서비스는 **GET (데이터 조회)**과 **POST (데이터 제출)**만으로 충분합니다. **PUT (데이터 수정)**, **DELETE (데이터 삭제)**, **TRACE (루프백 진단)** 등의 메소드가 활성화되어 있으면 공격자가 서버의 파일을 무단으로 수정하거나 정보를 탈취할 위험이 커집니다.



* **제한 대상**: PUT, DELETE, OPTIONS, TRACE, CONNECT, PATCH 등 업무상 불필요한 모든 메소드.
* **Nginx에서의 메소드 제한 설정**:

```bash
# Nginx에서 특정 메소드만 허용하고 나머지는 차단
server {
    if ($request_method !~ ^(GET|HEAD|POST)$ ) {
        return 405; # Method Not Allowed (허용되지 않은 메소드) 반환
    }
}
```

> **설명**: **HTTP 405 Method Not Allowed** 응답은 클라이언트가 사용한 요청 방식이 서버에서 금지되었음을 의미합니다. 이를 통해 공격 벡터를 최소화할 수 있습니다.

### **4. [AWS 환경 보안 및 과금 안내]**

클라우드 환경에서는 스토리지 서비스와 네트워크 레벨에서 리스팅과 메소드 제어를 병행해야 합니다.

* **Amazon S3 (아마존 S3, 객체 스토리지 서비스)**
    * S3 버킷의 **ListBucket (리스트버킷, 버킷 내 객체 목록 조회 권한)** 권한이 퍼블릭으로 열려 있을 경우 전 세계 누구나 파일을 리스팅할 수 있습니다. 반드시 '퍼블릭 액세스 차단' 설정을 활성화해야 합니다.
    * **과금 안내**: S3는 저장 용량, 데이터 전송량, **API 요청 건수 (PUT, GET, LIST)**에 따라 비용이 발생합니다. LIST 요청이 과도하게 발생하면 비용이 증가할 수 있습니다. [공식 요금 페이지](https://aws.amazon.com/ko/s3/pricing/)
* **AWS WAF (웹 애플리케이션 방화벽)**
    * WAF의 **Custom Rule (사용자 정의 규칙)**을 사용하여 특정 HTTP 메소드(예: DELETE)를 원천적으로 차단할 수 있습니다.
    * **과금 안내**: 활성화된 규칙 수와 처리된 웹 요청 수에 따라 월별 과금이 발생합니다.

**상세 설명 주석**
1. **autoindex**: Nginx에서 디렉터리 내의 파일 목록을 HTML 형태로 자동 생성하여 보여주는 모듈입니다.
2. **Indexes**: Apache에서 인덱스 파일이 없을 때 파일 목록을 출력하도록 허용하는 옵션입니다. 앞에 `-`를 붙여 기능을 제거합니다.
3. **HTTP TRACE**: 클라이언트의 요청 메시지를 그대로 응답 본문에 담아 보내는 메소드로, **XSS (Cross-Site Scripting)**와 결합하여 쿠키 정보를 탈취하는 **XST (Cross-Site Tracing)** 공격에 악용될 수 있습니다.
4. **ListBucket**: S3 버킷 내의 파일 목록을 볼 수 있는 권한으로, 보안 사고의 주요 원인 중 하나이므로 엄격히 관리되어야 합니다.
5. **405 Method Not Allowed**: 요청한 메소드가 서버에서 지원되지만 명시적으로 금지되었을 때 발생하는 HTTP 상태 코드입니다.
