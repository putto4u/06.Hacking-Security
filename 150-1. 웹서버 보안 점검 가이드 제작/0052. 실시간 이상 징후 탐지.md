## 5.2 실시간 이상 징후 탐지

---

### **1. IDS/IPS (Intrusion Detection System / Intrusion Prevention System, 침입 탐지 시스템 / 침입 방지 시스템)**

**IDS (Intrusion Detection System, 침입 탐지 시스템)**는 네트워크나 시스템에서 발생하는 비정상적인 활동이나 알려진 공격 패턴을 실시간으로 감시하고 경고를 발생시키는 보안 솔루션입니다. 한 단계 더 나아가 **IPS (Intrusion Prevention System, 침입 방지 시스템)**는 탐지된 공격을 실시간으로 차단하는 능동적인 방어 기능을 수행합니다.



* **탐지 방식**:
    * **Signature-based (시그니처 기반)**: 이미 알려진 공격 패턴(데이터베이스)과 대조하여 탐지.
    * **Anomaly-based (이상 현상 기반)**: 평상시의 정상적인 트래픽 패턴(Baseline)을 학습한 뒤, 이를 벗어나는 비정상적 행위를 탐지.
* **보안 목표**: 공격자가 시스템 내부에 침투하거나 데이터를 유출하기 전에 즉각적으로 대응하여 피해를 최소화합니다.

> **설명**: **Signature (시그니처, 서명/특징)**는 특정 바이러스나 해킹 공격이 가진 고유한 흔적입니다. 경찰이 범죄자의 지문을 대조하여 잡는 것이 시그니처 방식이라면, 평소와 다른 수상한 행동을 보고 검문하는 것이 이상 현상 방식입니다.

---

### **2. 호스트 기반 이상 징후 탐지 및 차단 (Fail2ban)**

서버 내부 로그를 실시간으로 모니터링하여 **Brute-force (무차별 대입 공격, 모든 조합을 시도하는 공격)**나 반복적인 로그인 실패를 유발하는 IP를 자동으로 차단하는 도구를 활용합니다.

* **Fail2ban (페일투밴, 침입 차단 소프트웨어)** 활용:
    * 로그 파일을 분석하여 사전에 설정된 횟수 이상 실패가 발생하면 해당 IP를 **iptables (리눅스 커널 방화벽 도구)**나 **NFTables (리눅스 차세대 패킷 필터링 프레임워크)**를 통해 일정 시간 동안 차단합니다.

```bash
# Fail2ban 설치 (Ubuntu 기준)
sudo apt update && sudo apt install fail2ban -y

# 기본 설정 파일 복사 및 편집
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
sudo vi /etc/fail2ban/jail.local

# SSH 서비스 보호 설정 예시 (5회 실패 시 10분 차단)
[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 5
bantime = 600
```



> **설명**: **Ban (밴, 금지)**은 특정 사용자나 IP 주소의 접근을 기술적으로 막는 행위입니다. Fail2ban은 로그라는 '흔적'을 보고 방화벽이라는 '자물쇠'를 자동으로 채우는 역할을 합니다.

### **3. 네트워크 트래픽 이상 징후 모니터링**

평소보다 과도하게 발생하는 네트워크 트래픽이나 비정상적인 연결 요청은 **DoS (Denial of Service, 서비스 거부 공격)**의 신호일 수 있습니다.

* **실시간 모니터링 항목**:
    * 초당 HTTP 요청 수 (Requests Per Second)의 급격한 증가.
    * 특정 국가나 특정 IP 대역에서의 과도한 접속 시도.
    * 서버의 **CPU (Central Processing Unit, 중앙 처리 장치)** 및 메모리 점유율의 비정상적 상승.

> **설명**: 트래픽 모니터링은 서버의 '맥박'을 체크하는 것과 같습니다. 갑작스러운 맥박의 상승은 외부 공격이나 시스템 오류의 강력한 지표가 됩니다.

### **4. [AWS 환경 보안 서비스 및 과금 안내]**

클라우드 환경에서는 머신러닝 기반의 지능형 탐지 서비스를 통해 수동으로 잡기 힘든 위협을 자동으로 찾아낼 수 있습니다.

* **Amazon GuardDuty (아마존 가드듀티, 지능형 위협 탐지 서비스)**
    * **VPC (Virtual Private Cloud, 가상 사설 클라우드)** 흐름 로그, DNS 로그 등을 분석하여 악성 IP와의 통신이나 비정상적인 계정 활동을 탐지합니다.
    * **과금 안내**: 분석된 로그의 총량(GB)에 따라 비용이 청구됩니다. 30일 무료 체험판을 제공하므로 실습 시 활용 가능합니다. [공식 요금 페이지](https://aws.amazon.com/ko/guardduty/pricing/)
* **AWS Config (AWS 컨피그, 리소스 구성 관리 서비스)**
    * 보안 그룹 설정이나 인스턴스 구성이 사전에 정의된 보안 가이드라인에서 벗어날 경우 실시간으로 알림을 보냅니다.
    * **과금 안내**: 기록된 리소스 수와 규칙 평가 횟수에 따라 과금됩니다. [공식 요금 페이지](https://aws.amazon.com/ko/config/pricing/)

**상세 설명 주석**
1. **IDS (Intrusion Detection System)**: 네트워크 통로에 설치되어 패킷을 복사(Mirroring)하여 분석하며, 흐름에 직접적인 영향을 주지 않고 감시하는 시스템입니다.
2. **IPS (Intrusion Prevention System)**: 네트워크 통로 중간에 직접 위치(In-line)하여 공격을 실시간으로 차단하므로 오탐(False Positive) 시 정상 트래픽까지 차단될 위험이 있어 주의 깊은 설정이 필요합니다.
3. **VPC (Virtual Private Cloud)**: AWS 클라우드 내에서 논리적으로 격리된 가상 네트워크 공간을 의미합니다.
4. **Drift (드리프트, 구성 편차)**: 원래 설정된 보안 기준값에서 실제 시스템 설정이 임의로 변경된 상태를 의미합니다.
5. **False Positive (오탐)**: 정상적인 행위를 공격으로 잘못 판단하여 알람을 울리거나 차단하는 오류를 말합니다.

OS Hardening, Kernel Tuning, Banner Grabbing, Permission Isolation, SSL/TLS, Security Headers, Directory Listing, WAF, SIEM, IDS/IPS
