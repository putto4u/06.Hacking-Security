1. 웹 위협 분석 아키텍처 (Web Threat Analysis Architecture)

본 보고서는 기존의 L3/L4 Network Firewall (네트워크 방화벽)이 탐지하지 못하는 L7 Application Layer (애플리케이션 계층)의 비정상적인 HTTP/HTTPS 트래픽을 정밀 분석합니다. 네트워크 방화벽은 IP와 포트를 제어하지만, WAF (Web Application Firewall, 웹 애플리케이션 방화벽)는 데이터 본체인 Payload (페이로드, 데이터 본체)를 실시간으로 전수 검사하여 웹 취약점 공격을 원천 차단하는 핵심 방어 체계입니다.

[도식: L4 Firewall vs L7 WAF 검사 심도 비교]

Network FW

IP/Port Filter Only

ALLOW TCP/80 FROM ANY
Web Application Firewall

Deep Packet Inspection (DPI)

DENY IF Payload CONTAINS 'OR 1=1'
  • L7 (Application Layer, 애플리케이션 계층): OSI 7계층의 최상위 계층으로 HTTP, HTTPS, FTP 등 최종 사용자 서비스가 동작하는 영역입니다.
  • DPI (Deep Packet Inspection, 심층 패킷 검사): 패킷의 헤더뿐만 아니라 데이터 본체(Payload)까지 검사하여 숨겨진 악성 코드를 찾아내는 기술입니다.

3. 침해 사고 분석 로그 (WAF Detection Logs)

ModSecurity(Nginx 연동)에서 실제 공격을 탐지했을 때의 로데이터(Raw Data) 로그 형식입니다.

/var/log/nginx/modsec_audit.log
[2024-03-27 14:32:01] [CRITICAL] TRANSACTION ID: 1711517521
--- Message: SQL Injection Attack Detected via libmodsecurity
--- Rule ID: 942100 (OWASP CRS)
--- Client IP: 192.168.45.10 (Host PC)
--- Requested URL: /api/v1/auth?user=' OR '1'='1'
--- Action: Intervention - Status 403 (Forbidden)
# 분석: 공격자의 패턴이 SQL Injection 코어 규칙에 매칭되어 즉시 403 응답으로 차단됨.
  • OWASP CRS (Core Rule Set, 핵심 규칙 집합): 전 세계 보안 전문가들이 공동으로 관리하는 ModSecurity용 범용 공격 탐지 규칙 라이브러리입니다.
  • Intervention (중재/개입): WAF 엔진이 비정상 트래픽을 발견하고 요청을 중단시키거나 차단 페이지로 유도하는 동작입니다.

4. 실전 보안 대응 가이드 (Security Hardening Guide)

01

ModSecurity Nginx 엔진 활성화

기존에 설치된 Nginx를 삭제하지 않고 모듈화된 libmodsecurity를 연동하여 트래픽 필터링을 활성화합니다.

Configuration
# /etc/nginx/nginx.conf
load_module modules/ngx_http_modsecurity_module.so;
http {
modsecurity on;
modsecurity_rules_file /etc/nginx/modsec/main.conf;
}
02

OWASP CRS 규칙 배포 및 업데이트

최신 웹 위협에 대응하기 위해 주기적으로 CRS v4.0 (최신 버전)을 다운로드하여 적용합니다.

$ git clone https://github.com/coreruleset/coreruleset.git /etc/nginx/modsec/crs
$ cp crs/crs-setup.conf.example crs/crs-setup.conf
  • Security Hardening (보안 강화/경량화): 시스템의 불필요한 요소를 제거하고 보안 설정을 최적화하여 공격 가능 표면을 최소화하는 작업입니다.
  • vX.X (Version): 소프트웨어의 배포 단계를 나타내는 번호로, 보안 모듈의 경우 최신 버전일수록 방어 가능한 패턴이 많습니다.

5. 클라우드 전용 WAF 전략 (AWS WAF Specialization)

인프라 관리 부담을 줄이기 위해 AWS 관리형 서비스를 이용할 경우의 과금 정책 및 실행 전략입니다.

Managed Rule Groups

AWS 전문가가 관리하는 Core Rule Set (코어 규칙 집합)을 즉시 활성화하여 제로데이 공격에 선제 대응합니다.

Web ACL Hierarchical Logic

국가별 IP 차단(Geo-blocking), SQLi 방지, Rate Limit(속도 제한) 등을 계층화하여 적용합니다.

  • Zero-day Attack (제로데이 공격): 보안 취약점이 발견된 후 공식 패치가 나오기 전에 시도되는 미지의 공격입니다.
