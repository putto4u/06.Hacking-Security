1. 아키텍처 개요 (Architecture Overview)

WAF의 정의와 필요성

WAF (Web Application Firewall, 웹 애플리케이션 방화벽)는 일반적인 네트워크 방화벽이 탐지하지 못하는 L7 (Application Layer, 애플리케이션 계층)의 비정상적인 트래픽을 분석하고 차단하는 보안 솔루션입니다. 기존 방화벽이 IP 주소나 포트 번호를 기준으로 차단한다면, WAF는 HTTP 패킷의 내용(Payload)을 검사하여 웹 공격을 정교하게 방어합니다.

  • 필요성: 웹 애플리케이션은 항상 외부로 노출되어 있으며, 소스 코드의 취약점을 이용한 공격은 네트워크 방화벽만으로는 방어가 불가능합니다.
  • 보안 목표: OWASP (Open Web Application Security Project, 오픈 소스 웹 애플리케이션 보안 프로젝트) Top 10과 같은 주요 웹 취약점 공격을 실시간으로 차단합니다.

"Payload (페이로드, 데이터 본체)는 전송되는 데이터 중 실제 공격 코드가 포함될 수 있는 부분입니다. WAF는 이 페이로드를 하나하나 검사하는 '세밀한 검문소' 역할을 수행합니다."

2. 주요 탐지 및 차단 패턴 (Detection Patterns)

CRITICAL

SQL Injection

(SQL 인젝션, 데이터베이스 명령 삽입)

입력창에 SQL 구문을 삽입하여 데이터베이스 정보를 무단으로 조회하거나 변조하는 공격입니다.

HIGH

XSS

(Cross-Site Scripting, 교차 사이트 스크립팅)

악성 스크립트를 웹 페이지에 삽입하여 사용자의 세션 쿠키를 탈취하는 공격입니다.

MEDIUM

Path Traversal

(경로 탐색, 디렉터리 접근 공격)

`../` 와 같은 상대 경로를 이용하여 서버의 민감한 시스템 파일에 접근하는 공격입니다.

3. ModSecurity 오픈소스 WAF 연동 (Implementation)

호스트 기반 WAF인 ModSecurity를 Nginx나 Apache에 연동하여 서버 자체적인 방어력을 높일 수 있습니다.

STEP 01

커넥터 및 라이브러리 설치

Nginx용 ModSecurity 커넥터를 Ubuntu 시스템에 설치합니다.

Console
$ sudo apt update
$ sudo apt install libnginx-mod-http-modsecurity -y
STEP 02

Nginx 설정 파일 수정 (Activation)

nginx.conf 내 http { ... } 또는 server { ... } 블록에 규칙을 로드합니다.

Config File
modsecurity on;
modsecurity_rules_file /etc/nginx/modsec/main.conf;
STEP 03

서비스 재시작 및 적용

$ sudo systemctl restart nginx

"ModSecurity (모드시큐리티)는 전 세계적으로 가장 많이 사용되는 오픈소스 WAF 엔진으로, 정교한 규칙 세트인 OWASP CRS (Core Rule Set)를 적용하여 높은 보안 수준을 유지할 수 있습니다."

4. AWS WAF 활용 및 과금 (Managed Service & Pricing)

클라우드 환경에서는 별도의 서버 설치 없이 AWS WAF를 ALB (Application Load Balancer) 또는 CloudFront에 직접 연결하여 관리할 수 있습니다.

Managed Rule Groups (관리형 규칙 그룹)

AWS 보안 전문가가 미리 작성한 규칙(예: SQLi 방어, 봇 제어)을 클릭 한 번으로 적용할 수 있습니다.

Web ACL (웹 접근 제어 목록)

여러 규칙을 묶어 특정 리소스에 적용하는 최상위 보안 설정 단위입니다.

Pricing Policy (과금 안내)

Web ACL 월 $5.00 / 개당
Rule 월 $1.00 / 개당
Web Request $0.60 / 100만 건당

* Managed Rules: AWS 또는 마켓플레이스 판매자가 제공하는 유료 규칙 사용 시 추가 비용이 발생할 수 있습니다.

5. 상세 설명 주석 (Glossary)

1. L7 (Application Layer) OSI 7계층 중 최상위 계층으로, HTTP, FTP, SMTP 등 사용자가 직접 접하는 프로토콜이 동작하는 영역입니다.
2. OWASP Top 10 웹 애플리케이션 보안에서 가장 빈번하게 발생하는 10가지 위험 요소에 대한 표준 가이드라인입니다.
3. ModSecurity 웹 서버 모듈 형태로 동작하며 패킷의 헤더와 바디를 실시간으로 필터링하는 방화벽 엔진입니다.
4. Web ACL (Web Access Control List) 웹 요청을 허용, 차단 또는 모니터링할지 결정하는 규칙들의 집합입니다.
5. ALB (Application Load Balancer) 여러 대의 서버로 트래픽을 분산시켜주는 장치로, L7 계층에서 동작하므로 WAF와 직접 연동됩니다.
