1. WAF의 정의와 필요성 (Application Layer Defense)

WAF (Web Application Firewall, 웹 애플리케이션 방화벽)는 일반적인 네트워크 방화벽이 탐지하지 못하는 L7 (Application Layer, 애플리케이션 계층)의 비정상적인 트래픽을 분석하고 차단하는 보안 솔루션입니다. 기존 방화벽이 IP 주소나 포트 번호를 기준으로 차단한다면, WAF는 HTTP 패킷의 내용(Payload)을 검사하여 웹 공격을 정교하게 방어합니다.

필요성 (Necessity)

웹 애플리케이션은 항상 외부로 노출되어 있으며, 소스 코드의 취약점을 이용한 공격은 네트워크 방화벽만으로는 방어가 불가능합니다.

보안 목표 (Security Goal)

OWASP Top 10과 같은 주요 웹 취약점 공격을 실시간으로 탐지하고 원천적으로 차단합니다.

설명: Payload (페이로드, 데이터 본체)는 전송되는 데이터 중 실제 공격 코드가 포함될 수 있는 부분입니다. WAF는 이 페이로드를 하나하나 검사하는 '세밀한 검문소' 역할을 수행합니다.

L7 (Application Layer, 애플리케이션 계층): OSI 7계층 중 최상위 계층으로, 사용자가 직접 이용하는 서비스 프로토콜이 동작하는 영역입니다.
OWASP Top 10 (오픈 소스 웹 애플리케이션 보안 프로젝트 10대 위협): 웹 보안에서 가장 위험하고 빈번한 10가지 취약점에 대한 표준 리스트입니다.

2. 주요 탐지 및 차단 패턴 (Common Attack Patterns)

WAF는 사전에 정의된 규칙(Rule)을 바탕으로 다음과 같은 대표적인 웹 공격을 탐지합니다.

[도식: Key Attack Patterns & Detection (핵심 공격 탐지 로직)]

SQL Injection

입력창에 SQL 구문을 삽입하여 DB 정보를 무단 조회하거나 변조하는 시도입니다.

SELECT * FROM users--
Cross-Site Scripting

악성 스크립트를 페이지에 삽입하여 사용자의 세션 쿠키를 탈취하는 방식입니다.

<script>stealCookie()</script>
Path Traversal

상대 경로(../)를 이용하여 서버의 민감한 시스템 파일에 무단 접근하는 공격입니다.

../../etc/passwd
SQL Injection (SQL 인젝션): 데이터베이스 쿼리를 조작하여 비정상적인 데이터 접근을 유도하는 삽입 공격입니다.
Path Traversal (경로 탐색 공격): 시스템 경로를 거슬러 올라가 허가되지 않은 파일에 접근하는 기법입니다.

3. ModSecurity (모드시큐리티) 오픈소스 WAF 연동

호스트 기반 WAF인 ModSecurity를 Nginx나 Apache에 연동하여 서버 자체적인 방어력을 높일 수 있습니다.

ModSecurity 설치 및 설정 (Implementation)

# 1. 패키지 인덱스 갱신 및 ModSecurity 커넥터 설치
sudo apt update # 설치 전 최신 리스트를 불러옵니다.
sudo apt install libnginx-mod-http-modsecurity -y # Nginx용 WAF 모듈을 설치합니다.

# 2. 엔진엑스 설정 파일에 WAF 기능 활성화
sudo vi /etc/nginx/nginx.conf # 메인 설정 파일을 엽니다.

modsecurity on; # 필터링 엔진을 활성화 상태로 전환합니다.
modsecurity_rules_file /etc/nginx/modsec/main.conf; # 적용할 규칙 세트 파일 경로를 지정합니다.

# 3. 서비스 재시작을 통한 설정 적용
sudo systemctl restart nginx # 변경사항을 즉시 커널 프로세스에 반영합니다.

설명: ModSecurity (모드시큐리티)는 전 세계적으로 가장 많이 사용되는 오픈소스 WAF 엔진으로, 정교한 규칙 세트인 OWASP CRS (Core Rule Set)를 적용하여 높은 보안 수준을 유지할 수 있습니다.

ModSecurity (모드시큐리티): 웹 서버 내부 모듈로 동작하며 HTTP 트래픽을 실시간 분석하는 강력한 방화벽 엔진입니다.
OWASP CRS (Core Rule Set, 핵심 규칙 세트): ModSecurity에서 보편적으로 사용되는 표준 공격 탐지용 규칙 모음입니다.

4. AWS WAF (아마존 웹 애플리케이션 방화벽) 활용 및 과금

클라우드 환경에서는 별도의 서버 설치 없이 AWS WAF를 ALB 또는 CloudFront에 직접 연결하여 관리할 수 있습니다.

Core Components

  • Managed Rule Groups: AWS 보안 전문가가 미리 작성한 최신 공격 방어 규칙을 원클릭으로 적용합니다.
  • Web ACL: 여러 보안 규칙을 하나로 묶어 리소스에 배포하는 최상위 관리 단위입니다.
