1. OS Hardening의 정의 및 최신 보안 트렌드

OS Hardening은 운영체제의 설정 오류나 불필요한 기능을 제거하여 Attack Surface (공격 표면, 공격자가 침입할 수 있는 지점)를 최소화하는 보안 강화 공정입니다. 최신 보안 표준인 CIS Benchmark (Center for Internet Security Benchmark, 인터넷 보안 센터 인증 벤치마크) v1.0.0(Ubuntu 24.04 기준)을 준수하는 것이 권장됩니다.

2. 불필요한 서비스 및 취약 데몬(Daemon) 제거

SOCKET CHECK

포트 및 프로세스 점검

실제 외부로 노출된 포트와 해당 포트를 사용하는 프로세스를 실시간으로 확인하여 시스템 상태를 점검합니다.

$ sudo ss -tulpn
# 리스닝 상태의 모든 TCP/UDP 포트와 프로세스 ID 확인
SERVICE HARDENING

취약 프로토콜 제거

Telnet 및 FTP와 같은 취약한 서비스는 즉시 제거하고 SSH로 대체합니다.

$ sudo systemctl stop rpcbind
$ sudo systemctl disable rpcbind

Service Inspection Commands

systemctl list-units --type=service --state=running
# 서비스 리스트와 설명
systemctl list-unit-files --type=service | grep enabled
# 부팅 시 자동 실행 서비스 확인

3. 시스템 업데이트 및 패치 관리 자동화

01

최신 패키지 업데이트

CVE (Common Vulnerabilities and Exposures, 공개적으로 알려진 보안 취약점)가 발표되면 즉시 패치를 적용해야 합니다.

$ sudo apt update && sudo apt upgrade -y
02

보안 자동 업데이트

관리자의 수동 작업 없이도 치명적인 보안 패치를 즉시 적용하도록 Unattended-Upgrades를 구성합니다.

$ sudo dpkg-reconfigure -plow unattended-upgrades

4. 계정 보안 및 원격 접속 제한

SSH 강화 및 Root 로그인 차단

최고 관리자인 Root (루트, 시스템 최고 관리자)의 직접 접속을 차단하고 키 기반 인증을 사용합니다.

# SSH 설정에서 Root 로그인 금지 (PermitRootLogin no 확인)
sudo sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart ssh

DISABLED

AWS 환경 보안 가이드 및 과금 안내

AWS 환경에서 서버 보안을 자동화할 때는 다음 서비스를 참조하십시오.

Amazon Inspector (v2)

  • 중요 알림: 기존 Inspector Classic은 2026년 5월 20일부로 지원 종료됩니다. 반드시 최신 버전인 Amazon Inspector (v2)를 사용하십시오.
  • Billing 스캔된 EC2 인스턴스당 월별 비용 발생. 지속적 스캔 기능 활성화 시 추가 비용이 발생할 수 있습니다.

AWS Systems Manager (SSM)

  • Patch Manager: 여러 대의 인스턴스에 보안 패치를 일괄 적용할 수 있는 기능을 제공합니다.
  • Billing 기본 패치 관리는 무료이나, 하이브리드 노드나 고급 인스턴스 티어 사용 시 시간당 비용이 발생합니다.

5. 아키텍트 인사이트 (Expert Insight)

보안 전략의 본질

최신 서버 보안은 단순히 방화벽을 세우는 것을 넘어, OS 내부의 구성 요소 하나하나를 표준 가이드라인에 맞게 '단단하게' 만드는 과정입니다.

패치 자동화의 가치

보안 패치 자동화는 관리자의 실수를 방지하고, 제로 데이(Zero-day) 공격의 위험 노출 시간을 최소화하는 가장 효과적인 방법입니다.

주석 01. OS Hardening: 운영체제 수준에서 허점을 제거하는 행위로, 불필요한 계정 삭제, 서비스 차단, 파일 권한 강화 등을 포함합니다.

주석 02. CIS Benchmark: 전 세계 보안 전문가들이 정의한 시스템 보안 설정의 '골든 스탠다드'로, 기업 보안 감사 시 필수 기준이 됩니다.

주석 03. CVE: 보안 취약점의 표준 식별자로, 보안 패치가 나올 때마다 고유 번호(예: CVE-2024-XXXX)가 부여됩니다.

주석 04. Amazon Inspector v2: 에이전트 없이도 소프트웨어 취약점과 네트워크 노출을 실시간으로 감지하는 최신 보안 서비스입니다.

주석 05. Daemon (데몬, 백그라운드 상주 프로그램): 외부 요청을 항상 대기하므로, 취약점이 발견될 경우 공격자에게 시스템 권한을 넘겨주는 통로가 될 수 있습니다.
