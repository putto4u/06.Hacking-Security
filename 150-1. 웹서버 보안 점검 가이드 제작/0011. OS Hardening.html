1.1 OS Hardening (Operating System Hardening, 운영체제 요새화)

1. OS Hardening의 정의와 목표

OS Hardening은 운영체제의 보안 취약점을 최소화하여 Attack Surface (공격 표면, 공격 가능한 지점)을 줄이는 일련의 보안 강화 과정을 의미합니다. 시스템 설치 시 기본적으로 활성화된 불필요한 기능들을 제거하고, 보안 설정을 최적화하여 외부 침입으로부터 서버를 보호하는 것이 핵심 목적입니다.

  • 보안 원칙: Least Privilege (최소 권한 원칙)
    • 시스템은 업무 수행에 꼭 필요한 최소한의 서비스와 권한만을 유지해야 합니다.
  • 보안 원칙: Default Deny (기본 차단 원칙)
    • 명시적으로 허용된 것 외의 모든 통로는 차단하는 것을 원칙으로 합니다.

설명: 서버의 기반이 되는 OS 레벨에서 보안 구멍(Vulnerability)을 메우는 작업은 상위 애플리케이션 보안의 전제 조건입니다.

2. 불필요한 서비스 및 데몬(Daemon) 관리

사용하지 않는 서비스가 실행 중이면, 해당 서비스의 취약점이 시스템 침투 경로가 될 수 있습니다.

  • 실행 중인 네트워크 서비스 식별 현재 리스닝(Listening) 상태인 포트와 프로세스를 확인합니다.
# 리스닝 중인 모든 TCP/UDP 포트와 프로세스 확인
sudo ss -tulpn
  • 불필요한 서비스 중지 및 비활성화 구형 프로토콜인 Telnet (Telecommunication Network, 원격 터미널 접속 프로토콜)이나 FTP (File Transfer Protocol, 파일 전송 프로토콜) 등 보안에 취약한 서비스는 즉시 제거하거나 중지합니다.
# 특정 서비스 중지 (예: telnet)
sudo systemctl stop telnetd

# 부팅 시 자동 실행 비활성화
sudo systemctl disable telnetd

설명: Daemon (데몬, 백그라운드 프로세스)은 시스템에서 지속적으로 실행되며 특정 요청을 기다리는 프로그램입니다. 불필요한 데몬을 방치하는 것은 잠겨 있지 않은 뒷문을 열어두는 것과 같습니다.

3. 최신 보안 패치 및 업데이트 관리

알려진 취약점(Known Vulnerabilities)을 방어하기 위해 시스템을 최신 상태로 유지하는 것은 가장 기본적이면서도 강력한 보안 대책입니다.

  • 패키지 시스템 업데이트 보안 패치가 포함된 최신 패키지 정보를 가져오고 업그레이드를 수행합니다.
# 패키지 목록 업데이트 및 설치된 패키지 전체 업그레이드
sudo apt update && sudo apt upgrade -y
  • 보안 자동 업데이트 (Unattended-Upgrades) 설정 치명적인 보안 패치가 관리자 개입 없이 자동으로 적용되도록 구성할 수 있습니다.
# 자동 업데이트 패키지 설치
sudo apt install unattended-upgrades

# 자동 업데이트 활성화 상태 확인
sudo systemctl status unattended-upgrades

설명: Kernel (커널, 운영체제 핵심 거점) 업데이트 시에는 시스템 리부팅이 필요할 수 있으므로, 서비스 운영 가용성을 고려한 스케줄링이 필수적입니다.

4. 시스템 접근 통제 및 계정 보안

  • Root (루트, 최고 관리자) 직접 접속 차단 공격자의 표적이 되기 쉬운 Root 계정의 원격 로그인을 차단하고, 일반 계정 접속 후 관리자 권한을 획득하도록 설정합니다.
# SSH 설정 파일 수정 (PermitRootLogin no 설정)
sudo vi /etc/ssh/sshd_config

# 설정 변경 후 SSH 서비스 재시작
sudo systemctl restart ssh
  • Password Complexity (비밀번호 복잡성) 강제 강력한 비밀번호 정책을 적용하여 Brute-force Attack (무차별 대입 공격, 모든 조합을 시도하는 공격)을 예방합니다.

설명: 보안 정책에 따라 패스워드 최소 길이, 특수 문자 포함 여부, 유효 기간 등을 설정하여 계정 탈취 위험을 낮춥니다.
