OS Hardening
(Operating System Hardening, 운영체제 요새화)

VER: 2026.04

Chief System Architect

1. 정의 및 최신 보안 트렌드 (Definition & Trends)

OS Hardening은 운영체제의 설정 오류나 불필요한 기능을 제거하여 Attack Surface (공격 표면, 공격자가 침입할 수 있는 지점)를 최소화하는 보안 강화 공정입니다. 최신 보안 표준인 CIS Benchmark (Center for Internet Security Benchmark, 인터넷 보안 센터 인증 벤치마크) v1.0.0(Ubuntu 24.04 기준)을 준수하는 것이 권장됩니다.

[도식: Attack Surface Reduction (공격 표면 감소 매커니즘)]

RISK
기본 설치 상태 (불필요한 서비스 활성)
OS HARDENING PROCESS
SAFE
요새화 상태 (최소 기능 유지)

Defense in Depth (심층 방어)

단일 보안 솔루션에 의존하지 않고 OS, 네트워크, 어플리케이션 각 계층에서 방어벽을 구축합니다.

Zero Trust (제로 트러스트)

내부 사용자나 프로세스라 할지라도 적절한 인증과 최소 권한이 확인된 경우에만 접근을 허용합니다.

설명: 최신 서버 보안은 단순히 방화벽을 세우는 것을 넘어, OS 내부의 구성 요소 하나하나를 표준 가이드라인에 맞게 '단단하게' 만드는 과정입니다.

2. 취약 데몬(Daemon) 및 서비스 제거

최신 리눅스 배포판(Ubuntu 24.04 LTS 등)은 기본적으로 보안이 강화되어 있으나, 관리 편의를 위해 설치된 불필요한 패키지는 잠재적 위협이 됩니다.

CHECK

Network Sockets Analysis

실제 외부로 노출된 포트와 해당 포트를 사용하는 프로세스를 실시간으로 확인합니다.

$ sudo ss -tulpn
Netid State Recv-Q Send-Q Local Address:Port ...
ACTION

Disable Vulnerable Services

암호화되지 않은 프로토콜인 Telnet / FTP를 제거하고 SSH (Secure Shell, 보안 셸)로 대체합니다.

$ sudo systemctl stop rpcbind
$ sudo systemctl disable rpcbind

[메커니즘: Daemon & Attack Vector]

Attacker
Vulnerable Port
telnet(23) / ftp(21)
System Daemon

설명: Daemon (데몬, 백그라운드 상주 프로그램)은 외부 요청을 항상 대기하므로, 취약점이 발견될 경우 공격자에게 시스템 권한을 넘겨주는 통로가 될 수 있습니다.

3. 업데이트 및 패치 관리 자동화

CVE (Common Vulnerabilities and Exposures, 공개적으로 알려진 보안 취약점)가 발표되면 공격자는 즉시 공격 도구를 제작하므로, 신속한 패치 적용이 필수입니다.

수동 패치 적용

# 최신 보안 패치 일괄 적용
sudo apt update && sudo apt upgrade -y

자동 보안 업데이트

# Unattended-Upgrades 활성화
sudo dpkg-reconfigure -plow unattended-upgrades

설명: 보안 패치 자동화는 관리자의 실수를 방지하고, 제로 데이(Zero-day) 공격의 위험 노출 시간을 최소화하는 가장 효과적인 방법입니다.

4. 계정 보안 및 원격 접속 제한

SSH 강화 및 Root 로그인 차단

최고 관리자인 Root (루트, 시스템 최고 관리자)의 직접 접속을 차단하고 키 기반 인증을 사용합니다.

# SSH 설정 수정 (PermitRootLogin no 설정)
$ sudo sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin no/' /etc/ssh/sshd_config
$ sudo systemctl restart ssh

ACCESS CONTROL POLICY

  • Direct Root Login: DENY
  • SSH Key Auth: REQUIRED
  • Passwordless Sudo: OPTIONAL

상세 설명 주석 (Glossary)

1. OS Hardening

운영체제 수준에서 허점을 제거하는 행위로, 불필요한 계정 삭제, 서비스 차단, 파일 권한 강화 등을 포함합니다.

2. CIS Benchmark

전 세계 보안 전문가들이 정의한 시스템 보안 설정의 '골든 스탠다드'로, 기업 보안 감사 시 필수 기준이 됩니다.

3. CVE

보안 취약점의 표준 식별자로, 보안 패치가 나올 때마다 고유 번호(예: CVE-2024-XXXX)가 부여됩니다.

4. Amazon Inspector v2

에이전트 없이도 소프트웨어 취약점과 네트워크 노출을 실시간으로 감지하는 최신 보안 서비스입니다.
