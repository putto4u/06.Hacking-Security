1.1 OS Hardening (Operating System Hardening, 운영체제 요새화)

1. OS Hardening의 정의 및 최신 보안 트렌드

OS Hardening은 운영체제의 설정 오류나 불필요한 기능을 제거하여 Attack Surface (공격 표면, 공격자가 침입할 수 있는 지점)를 최소화하는 보안 강화 공정입니다. 최신 보안 표준인 CIS Benchmark (Center for Internet Security Benchmark, 인터넷 보안 센터 인증 벤치마크) v1.0.0(Ubuntu 24.04 기준)을 준수하는 것이 권장됩니다.

  • 보안 원칙: Defense in Depth (심층 방어, 다층적 보안 계층 구성)
    • 단일 보안 솔루션에 의존하지 않고 OS, 네트워크, 어플리케이션 각 계층에서 방어벽을 구축합니다.
  • 보안 원칙: Zero Trust (제로 트러스트, 아무도 신뢰하지 않는 보안 모델)
    • 내부 사용자나 프로세스라 할지라도 적절한 인증과 최소 권한이 확인된 경우에만 접근을 허용합니다.

설명: 최신 서버 보안은 단순히 방화벽을 세우는 것을 넘어, OS 내부의 구성 요소 하나하나를 표준 가이드라인에 맞게 '단단하게' 만드는 과정입니다.

2. 불필요한 서비스 및 취약 데몬(Daemon) 제거

최신 리눅스 배포판(Ubuntu 24.04 LTS 등)은 기본적으로 보안이 강화되어 있으나, 관리 편의를 위해 설치된 불필요한 패키지는 잠재적 위협이 됩니다.

  • 네트워크 소켓 및 서비스 상태 점검 실제 외부로 노출된 포트와 해당 포트를 사용하는 프로세스를 실시간으로 확인합니다.
# 리스닝 상태의 모든 TCP/UDP 포트와 프로세스 ID 확인
sudo ss -tulpn
  • 보안에 취약한 서비스 비활성화 암호화되지 않은 프로토콜인 Telnet (Telecommunication Network, 원격 접속 프로토콜) 및 FTP (File Transfer Protocol, 파일 전송 프로토콜)는 즉시 제거하고 SSH (Secure Shell, 보안 셸)로 대체해야 합니다.
# 불필요한 서비스 중지 및 자동 실행 방지 (예: rpcbind)
sudo systemctl stop rpcbind
sudo systemctl disable rpcbind

설명: Daemon (데몬, 백그라운드 상주 프로그램)은 외부 요청을 항상 대기하므로, 취약점이 발견될 경우 공격자에게 시스템 권한을 넘겨주는 통로가 될 수 있습니다.

systemctl list-units --type=service --state=running # 서비스 리스트와 설명 systemctl list-unit-files --type=service | grep enabled # 부팅 시 자동 실행되는 서비스 리스트 확인

3. 시스템 업데이트 및 패치 관리 자동화

CVE (Common Vulnerabilities and Exposures, 공개적으로 알려진 보안 취약점)가 발표되면 공격자는 즉시 공격 도구를 제작하므로, 신속한 패치 적용이 필수입니다.

  • 최신 패키지 업데이트 수행 운영체제 및 설치된 모든 소프트웨어를 최신 보안 패치 상태로 유지합니다.
# 패키지 인덱스 갱신 및 보안 업데이트 적용
sudo apt update && sudo apt upgrade -y
  • 보안 자동 업데이트 (Unattended-Upgrades) 설정 관리자의 수동 작업 없이도 치명적인 보안 패치를 즉시 적용하도록 구성합니다.
# 자동 보안 업데이트 설정 및 상태 확인
sudo dpkg-reconfigure -plow unattended-upgrades

설명: 보안 패치 자동화는 관리자의 실수를 방지하고, 제로 데이(Zero-day) 공격의 위험 노출 시간을 최소화하는 가장 효과적인 방법입니다.

4. 계정 보안 및 원격 접속 제한

  • SSH 강화 및 Root 로그인 차단 최고 관리자인 Root (루트, 시스템 최고 관리자)의 직접 접속을 차단하고 키 기반 인증을 사용합니다.
# SSH 설정에서 Root 로그인 금지 (PermitRootLogin no 확인)
sudo sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart ssh

[AWS 환경 보안 가이드 및 과금 안내]

AWS 환경에서 서버 보안을 자동화할 때는 다음 서비스를 참조하십시오.

  • Amazon Inspector (아마존 인스펙터, 자동화된 취약점 관리 서비스)
    • 중요 알림: 기존의 Inspector Classic은 2026년 5월 20일부로 지원이 종료됩니다. 반드시 최신 버전인 Amazon Inspector (v2)를 사용하십시오.
    • 과금: 스캔된 EC2 인스턴스당 월별 비용이 발생하며, 지속적 스캔 기능 활성화 시 비용이 추가될 수 있습니다. 공식 요금 페이지
  • AWS Systems Manager (SSM, 시스템즈 매니저 - 패치 관리자)
    • 여러 대의 인스턴스에 보안 패치를 일괄 적용할 수 있는 Patch Manager (패치 매니저) 기능을 제공합니다.
    • 과금: 기본 패치 관리 기능은 무료이나, 온프레미스 서버를 연결하는 하이브리드 노드나 고급 인스턴스 티어 사용 시 시간당 비용이 발생합니다. 공식 요금 페이지

Next Step: 리눅스 커널 파라미터 최적화

상세 설명 주석 1. OS Hardening: 운영체제 수준에서 허점을 제거하는 행위로, 불필요한 계정 삭제, 서비스 차단, 파일 권한 강화 등을 포함합니다. 2. CIS Benchmark: 전 세계 보안 전문가들이 정의한 시스템 보안 설정의 '골든 스탠다드'로, 기업 보안 감사 시 필수 기준이 됩니다. 3. CVE: 보안 취약점의 표준 식별자로, 보안 패치가 나올 때마다 고유 번호(예: CVE-2024-XXXX)가 부여됩니다. 4. Amazon Inspector v2: 에이전트 없이도 소프트웨어 취약점과 네트워크 노출을 실시간으로 감지하는 최신 보안 서비스입니다.
