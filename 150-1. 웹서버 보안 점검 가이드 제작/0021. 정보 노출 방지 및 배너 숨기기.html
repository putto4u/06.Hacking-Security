1. Banner Grabbing의 위험성 (Risk of Banner Grabbing)

Banner Grabbing (배너 그래빙, 서비스 정보 수집)은 공격자가 대상 서버에 접속하여 서버 소프트웨어의 이름, 버전, 운영체제 정보를 알아내는 기법입니다. 공격자는 이 정보를 바탕으로 해당 버전의 알려진 취약점(Exploit)을 검색하여 공격 대상을 구체화합니다.

Footprinting (풋프린팅, 사전 조사)

공격의 첫 번째 단계로, 시스템의 구조와 소프트웨어 정보를 수집하는 행위입니다.

보안 목표 (Security Goal)

서버가 응답 헤더(Response Header)나 에러 페이지에서 자신의 정체를 드러내지 않도록 설정하여 공격자의 분석을 방해합니다.

설명: 웹 서버의 이름과 구체적인 버전(예: Nginx 1.18.0)이 노출되면, 해당 버전에만 존재하는 특정 취약점을 이용한 정밀 공격에 무방비로 노출될 수 있습니다.

2. Nginx 서버 정보 은폐 설정 (Nginx Information Hiding)

Nginx는 기본적으로 응답 헤더의 Server 필드에 버전 정보를 포함합니다. 이를 비활성화하여 소프트웨어 이름만 노출되거나 아예 숨기도록 설정합니다.

STEP Server Tokens 비활성화

/etc/nginx/nginx.conf 파일의 http 블록 내에서 설정합니다.

# Nginx 설정 파일 수정
$ sudo vi /etc/nginx/nginx.conf

http {
server_tokens off;
}

# 설정 적용 후 서비스 재시작
$ sudo nginx -t
$ sudo systemctl restart nginx

설명: server_tokens off; 설정은 응답 헤더에서 버전 번호를 제거합니다. 더 나아가 more_set_headers 모듈(추가 설치 필요)을 사용하면 'Server' 필드 자체를 임의의 문자열로 변경할 수도 있습니다.

3. Apache 서버 정보 은폐 설정 (Apache Security Tuning)

Apache HTTP Server 역시 기본적으로 상세 정보를 노출합니다. ServerTokens와 ServerSignature 옵션을 조정하여 이를 차단합니다.

보안 강화 설정 (Security Config)

# /etc/apache2/conf-enabled/security.conf

# 버전 정보 노출 차단
ServerTokens Prod

# 에러 페이지 서명 제거
ServerSignature Off

[비교: ServerTokens 설정별 노출 범위]

Setting Example Output
Full Apache/2.4.41 (Ubuntu)
OS Apache/2.4.41 (Ubuntu)

설명: ServerSignature (서버 시그니처, 서버 서명)는 서버가 생성하는 에러 페이지 하단에 서버 버전과 호스트명을 자동으로 삽입하는 기능입니다. 이를 끄는 것은 정보 유출 방지의 기본입니다.

4. 사용자 정의 에러 페이지 (Custom Error Pages)

Anti-Info Leak

서버 내부 오류(500)나 페이지 없음(404) 발생 시, 기본 에러 페이지 대신 관리자가 만든 페이지를 보여주어 시스템 내부 경로 정보나 서버 특성이 노출되는 것을 방지합니다.

# Nginx 예시: 404 에러 시 특정 HTML 파일로 대체
error_page 404 /custom_404.html;

location = /custom_404.html {
root /var/www/html/errors;
internal;
}

[AWS 환경 보안 가이드 및 과금 안내]

인프라 수준에서 정보를 숨기거나 헤더를 조작하려면 AWS의 엣지(Edge) 서비스를 활용하는 것이 매우 효과적입니다.

상세 설명 주석 (Glossary)

1. Banner Grabbing

텔넷(Telnet)이나 넷캣(Netcat) 등을 이용해 웹 서버 포트(80, 443)에 연결하여 서버가 보내는 초기 응답 메시지를 가로채 정보를 얻는 행위입니다.

2. Server Tokens

Nginx에서 서버의 버전 정보를 응답 헤더에 포함할지 여부를 결정하는 지시어입니다.

3. Response Header

웹 서버가 브라우저의 요청에 응답할 때 보내는 메타데이터로, 여기에 서버의 소프트웨어 정보가 포함됩니다.

4. Internal 지시어

Nginx에서 외부 사용자가 직접 해당 경로(예: /custom_404.html)로 접속하는 것을 막고, 서버 내부적인 리다이렉션으로만 접근 가능하게 제한하는 설정입니다.
