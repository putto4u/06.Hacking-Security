1. Banner Grabbing (배너 그래빙)의 위험성

Banner Grabbing (배너 그래빙, 서비스 정보 수집)은 공격자가 대상 서버에 접속하여 서버 소프트웨어의 이름, 버전, 운영체제 정보를 알아내는 기법입니다. 공격자는 이 정보를 바탕으로 해당 버전의 알려진 취약점(Exploit)을 검색하여 공격 대상을 구체화합니다.

[도식: Footprinting (풋프린팅, 사전 조사) 메커니즘]

보안 목표 (Security Objectives)

서버가 응답 헤더(Response Header)나 에러 페이지에서 자신의 정체를 드러내지 않도록 설정하여 공격자의 분석을 방해합니다. 웹 서버의 이름과 구체적인 버전(예: Nginx 1.18.0)이 노출되면, 해당 버전에만 존재하는 특정 취약점을 이용한 정밀 공격에 무방비로 노출될 수 있습니다.

2. Nginx 서버 정보 은폐 설정

NGINX HARDENING

Server Tokens 비활성화

Nginx는 기본적으로 응답 헤더의 Server 필드에 버전 정보를 포함합니다. 이를 비활성화하여 소프트웨어 이름만 노출되거나 아예 숨기도록 설정합니다. `server_tokens off;` 설정은 응답 헤더에서 버전 번호를 제거합니다.

nginx.conf
# http { ... } 블록 안에 내용 추가
server_tokens off;
# 설정 적용 후 재시작
$ sudo nginx -t
$ sudo systemctl restart nginx

더 나아가 more_set_headers 모듈(추가 설치 필요)을 사용하면 'Server' 필드 자체를 임의의 문자열로 변경할 수도 있습니다.

3. Apache 서버 정보 은폐 설정

APACHE SECURITY

보안 강화 설정 (Security.conf)

Apache HTTP Server 역시 기본적으로 상세 정보를 노출합니다. ServerTokens와 ServerSignature 옵션을 조정하여 이를 차단합니다. ServerSignature (서버 시그니처, 서버 서명)를 끄는 것은 정보 유출 방지의 기본입니다.

security.conf
# 서버 정보 노출 최소화 (Product 이름만)
ServerTokens Prod
# 에러 페이지 하단 서명 비활성화
ServerSignature Off
$ sudo systemctl restart apache2

4. 사용자 정의 에러 페이지 (Custom Error Pages) 적용

서버 내부 오류(500)나 페이지 없음(404) 발생 시, 기본 에러 페이지 대신 관리자가 만든 페이지를 보여주어 시스템 내부 경로 정보나 서버 특성이 노출되는 것을 방지합니다. Information Leakage(정보 유출, 정보 노출)의 원인이 되는 기본 페이지를 교체하십시오.

웹 서버 종류 기본 에러 페이지 처리 방식 및 경로
Apache HTTP Server 내장된 메시지 출력. /usr/share/apache2/error/ 또는 /var/www/error/ 참조 가능.
Nginx 기본적으로 /usr/share/nginx/html/의 50x.html 관리.
IIS C:\inetpub\custerr\ 디렉터리 하위 상태 코드별 폴더 관리.

Nginx Implementation

error_page 404 /custom_404.html;
location = /custom_404.html {
root /var/www/html/errors;
internal;
}

Apache Implementation

ErrorDocument 404 /custom_404.html
ErrorDocument 500 /custom_500.html
# .htaccess 또는 sites-available/default 적용

주석 01. Banner Grabbing: 텔넷(Telnet)이나 넷캣(Netcat) 등을 이용해 웹 서버 포트(80, 443)에 연결하여 서버가 보내는 초기 응답 메시지를 가로채 정보를 얻는 행위입니다.

주석 02. Server Tokens: Nginx에서 서버의 버전 정보를 응답 헤더에 포함할지 여부를 결정하는 지시어입니다.

주석 03. Response Header: 웹 서버가 브라우저의 요청에 응답할 때 보내는 메타데이터로, 여기에 서버의 소프트웨어 정보가 포함됩니다.

주석 04. Internal 지시어: Nginx에서 외부 사용자가 직접 해당 경로(예: /custom_404.html)로 접속하는 것을 막고, 서버 내부적인 리다이렉션으로만 접근 가능하게 제한하는 설정입니다.

주석 05. Information Leakage (정보 유출, 정보 노출): 시스템의 종류, 버전, 내부 경로 등 민감한 정보가 에러 페이지 등을 통해 외부에 공개되는 취약점입니다.

주석 06. Hardcoded (하드코딩, 코드 내에 고정됨): 외부 파일이나 설정값을 참조하지 않고 프로그램 소스 코드 내에 데이터가 직접 기록된 상태를 의미합니다.
