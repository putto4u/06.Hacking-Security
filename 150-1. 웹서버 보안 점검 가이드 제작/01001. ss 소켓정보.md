네트워크 소켓 상태를 분석하고 시스템 성능을 최적화하기 위한 **ss(Socket Statistics, 소켓 통계)** 명령어의 종합 가이드입니다. 이 자료는 기본 옵션부터 전문가 수준의 분석 기법까지 체계적으로 정리하여 강의 교재로 활용할 수 있도록 구성되었습니다.

---

## 1. ss 명령어 핵심 옵션 일람표

**ss** 명령어는 다양한 옵션을 조합하여 필요한 정보만 정밀하게 타격하여 추출할 수 있습니다.

| 옵션 | 명칭 (Full Name) | 상세 기능 설명 |
| :--- | :--- | :--- |
| **-a** | **All** | 모든 소켓(Listening 및 Non-listening)을 표시합니다. |
| **-l** | **Listening** | 연결을 기다리고 있는 **Listening(리슨)** 상태의 소켓만 표시합니다. |
| **-t** | **TCP** | **TCP(Transmission Control Protocol)** 소켓만 표시합니다. |
| **-u** | **UDP** | **UDP(User Datagram Protocol)** 소켓만 표시합니다. |
| **-n** | **Numeric** | 서비스 이름 대신 포트 번호를 숫자로 표시합니다. (DNS 해석 생략으로 속도 빠름) |
| **-p** | **Process** | 소켓을 사용하는 **PID(Process ID)**와 프로세스 이름을 함께 출력합니다. |
| **-i** | **Internal** | **TCP**의 내부 성능 지표(RTT, CWND 등)를 상세히 출력합니다. |
| **-s** | **Summary** | 현재 활성화된 소켓의 프로토콜별 요약 통계를 보여줍니다. |
| **-m** | **Memory** | 각 소켓이 사용 중인 **Memory(메모리)** 점유량을 표시합니다. |
| **-e** | **Extended** | 상세한 소켓 정보를 추가로 표시합니다. |



> **포트 해석(DNS Resolution)의 차이**: `-n` 옵션을 사용하지 않으면 `80`번 포트는 `http`, `443`번은 `https`로 자동 변환되어 출력됩니다. 하지만 대규모 트래픽 분석 시에는 속도를 위해 숫자로 표기하는 `-n` 옵션이 권장됩니다.

## 2. 전문가의 실전 활용 시나리오

단순한 조회를 넘어 시스템 장애 진단 및 보안 분석에 활용되는 고급 기법입니다.

sudo ss -tulpn # 리스닝 상태의 모든 TCP/UDP 포트와 프로세스 ID 확인


### ① 요약 통계를 통한 시스템 부하 진단
서버의 전체적인 연결 상태를 한눈에 파악하여 **DDoS(Distributed Denial of Service, 분산 서비스 거부 공격)** 상황이나 리소스 고갈 여부를 확인합니다.
* **명령어**: `ss -s`
* **활용**: 전체 세션 중 `ESTABLISHED` 상태가 비정상적으로 높다면 서비스 부하를 의심해야 합니다.
`sudo ss -tulpn # 리스닝 상태의 모든 TCP/UDP 포트와 프로세스 ID 확인`

### ② 특정 TCP 상태(State) 필터링
특정 단계에 머물러 있는 연결을 찾아내어 네트워크 병목 지점을 파악합니다.
* **명령어**: `ss -t state time-wait`
* **설명**: `TIME-WAIT` 상태가 너무 많으면 소켓 고갈로 인해 새로운 연결이 거부될 수 있습니다.

### ③ 네트워크 품질(RTT) 분석
패킷의 왕복 시간을 확인하여 네트워크 지연의 원인이 내부 서버인지 외부 회선인지 판단합니다.
* **명령어**: `ss -ti`
* **핵심 지표**: **RTT(Round Trip Time, 왕복 시간)** 값이 높으면 회선 품질 저하나 네트워크 혼잡을 의미합니다.

## 3. 고급 조건부 필터링 (Filter Expressions)

복잡한 네트워크 환경에서 특정 조건에 맞는 데이터만 추출하는 강력한 기능입니다.

* **포트 대역 필터링**: `ss -atn sport >= :1024` (1024번 이상의 사용자 포트만 확인)
* **특정 IP 대역 제외**: `ss -at dst != 192.168.1.1` (특정 내부 서버를 제외한 외부 연결 확인)
* **웹 트래픽 전용**: `ss -atn '( dport = :80 or dport = :443 )'` (HTTP/HTTPS 트래픽만 집계)

## 4. 소켓 상태 구조 도식화

시스템 내부에서 프로세스가 생성한 소켓이 어떤 단계를 거쳐 외부와 연결되는지에 대한 논리적 흐름입니다.

> **[Application Process]** (데이터 생성)
>       ↓
> **[Socket Buffer]** (Send-Q / Recv-Q 메모리 할당)
>       ↓
> **[TCP/IP Stack]** (커널 내 프로토콜 처리)
>       ↓
> **[Network Interface]** (물리적 패킷 전송)
