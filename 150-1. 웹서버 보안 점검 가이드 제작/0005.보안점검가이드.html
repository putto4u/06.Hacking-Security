Cloud Security & Hacking Lab

실전 클라우드 인프라 보안 및 모의 해킹 시나리오 연구 자료 저장소

웹서버 보안 점검 가이드 강의 목차

[PART 1] 웹 인프라 및 운영체제 보안 환경 구축

  • 1.1 OS Hardening (Operating System Hardening, 운영체제 요새화)

    • 불필요한 서비스 및 데몬 제거를 통한 공격 표면 축소.
    • 최신 보안 패치 및 커널 업데이트 적용 절차.
    • 설명: 서버의 가장 기초가 되는 운영체제 수준에서 허점(Vulnerability)을 제거하는 단계입니다.

  • 1.2 리눅스 커널 파라미터 최적화

    • sysctl을 활용한 네트워크 스택 보안 강화.
    • ICMP (Internet Control Message Protocol, 인터넷 제어 메시지 프로토콜) 차단 및 DDoS (Distributed Denial of Service, 분산 서비스 거부 공격) 방어 설정.
    • 설명: 네트워크 계층에서 발생할 수 있는 비정상적인 패킷 공격을 커널 수준에서 차단합니다.

[PART 2] 웹 서버 소프트웨어 설정 보안 (Nginx/Apache)

  • 2.1 정보 노출 방지 및 배너 숨기기

    • Server Tokens 및 Server Signature 비활성화.
    • 웹 서버 버전 정보 은폐를 통한 Footprinting (풋프린팅, 사전 정보 수집) 차단.
    • 설명: 공격자가 서버의 종류와 버전을 알 수 없게 하여 특정 버전에 특화된 공격(Exploit)을 예방합니다.

  • 2.2 권한 관리 및 프로세스 격리

    • 루트(Root) 권한이 아닌 별도 계정으로 웹 프로세스 실행.
    • 디렉터리 권한 및 ACL (Access Control List, 접근 제어 목록) 설정.
    • 설명: 웹 서버 프로세스가 탈취되더라도 시스템 전체 권한을 뺏기지 않도록 격리하는 설정입니다.

[PART 3] HTTP 프로토콜 및 통신 보안 강화

  • 3.1 SSL/TLS (Secure Sockets Layer / Transport Layer Security, 보안 소켓 계층 / 전송 계층 보안) 적용

    • HSTS (HTTP Strict Transport Security, HTTP 엄격한 전송 보안) 설정.
    • 취약한 암호화 알고리즘(DES, RC4) 제거 및 최신 가이드 적용.
    • 설명: 모든 통신 데이터를 암호화하여 스니핑(Sniffing)에 의한 데이터 유출을 방지합니다.

  • 3.2 보안 헤더 (Security Headers) 설정

    • XSS (Cross-Site Scripting, 교차 사이트 스크립팅) 방지를 위한 Content-Security-Policy.
    • 클릭재킹 방지를 위한 X-Frame-Options.
    • 설명: 브라우저와 서버 간의 응답 헤더에 보안 정책을 삽입하여 클라이언트 측 공격을 방어합니다.

[PART 4] 액세스 제어 및 취약점 진단

  • 4.1 디렉터리 리스팅 및 불필요한 HTTP 메소드 차단

    • Directory Indexing 비활성화 및 PUT, DELETE 등 불필요한 메소드 제한.
    • 설명: 서버 내부 파일 구조가 노출되는 것을 막고, 비인가된 파일 수정을 차단합니다.

  • 4.2 WAF (Web Application Firewall, 웹 애플리케이션 방화벽) 연동

    • SQL Injection (SQL 인젝션, 데이터베이스 명령 삽입) 및 주요 웹 공격 탐지 패턴 적용.
    • 설명: 웹 페이로드(Payload)를 실시간으로 검사하여 애플리케이션 레벨의 공격을 차단합니다.
    • 유료 서비스 안내: AWS 사용 시 AWS WAF 서비스는 요청 수 및 규칙에 따라 과금이 발생합니다.

[PART 5] 로깅, 모니터링 및 감사

  • 5.1 감사 로그 관리 및 중앙 집중화

    • Access Log 및 Error Log 상세 설정.
    • SIEM (Security Information and Event Management, 보안 정보 및 이벤트 관리) 솔루션과의 연동.
    • 설명: 침해 사고 발생 시 원인을 추적하기 위한 증거 데이터를 확보하고 보존하는 과정입니다.

  • 5.2 실시간 이상 징후 탐지

    • IDS/IPS (Intrusion Detection System / Intrusion Prevention System, 침입 탐지 시스템 / 침입 방지 시스템) 설정.
    • 비정상적인 트래픽 폭주 및 반복적 로그인 실패 모니터링.
    • 설명: 공격 시도를 실시간으로 감지하여 관리자에게 경고하거나 즉각 차단합니다.
