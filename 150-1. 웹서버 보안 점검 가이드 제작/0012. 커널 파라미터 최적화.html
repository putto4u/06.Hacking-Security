1. Kernel 및 sysctl의 역할 (Role of Kernel & sysctl)

Kernel (커널, 운영체제의 핵심)은 하드웨어 자원을 관리하고 프로세스 간의 통신을 제어하는 리눅스의 심장부입니다. sysctl (System Control, 커널 매개변수 제어 도구) 명령어를 사용하면 시스템을 재부팅하지 않고도 실행 중인 커널의 네트워크 스택 및 가상 메모리 파라미터를 실시간으로 수정하여 보안을 강화할 수 있습니다.

[도식: sysctl Control Interface Structure]

sysctl Interface
Kernel Space

설정 파일 경로

  • /etc/sysctl.conf (전역 설정)
  • /etc/sysctl.d/*.conf (분할 관리)

보안 목표

네트워크 성능 최적화와 동시에 외부의 비정상적인 패킷 공격으로부터 시스템 자원을 보호합니다.

설명: Parameter (파라미터, 매개변수)는 소프트웨어의 동작 방식을 결정하는 설정값입니다. 커널 파라미터를 조정하는 것은 엔진의 세부 튜닝을 통해 차량의 안정성을 높이는 과정과 같습니다.

2. 네트워크 스택 보안 강화 (Network Stack Hardening)

웹 서버는 수많은 불특정 다수의 연결을 처리하므로, TCP (Transmission Control Protocol, 전송 제어 프로토콜) 연결 과정에서 발생하는 취약점을 방어해야 합니다.

DEFENSE 01

SYN Flood 방어 (SYN Cookies)

공격자가 대량의 연결 요청(SYN)만 보내고 응답(ACK)을 하지 않아 서버의 대기 자원을 고갈시키는 공격을 방어하기 위해 SYN Cookies (SYN 쿠키, 연결 요청 검증 매커니즘)를 활성화합니다.

Mechanism: SYN Cookies Validation

Client (SYN)
Server
Generate Cookie
SYN-ACK + Cookie
$ sudo sysctl -w net.ipv4.tcp_syncookies=1
DEFENSE 02

IP Spoofing 방지 (RP Filter)

공격자가 발신지 IP를 위조하여 접근하는 것을 막기 위해 Reverse Path Filtering (RP 필터링, 역경로 필터링)을 적용합니다.

$ sudo sysctl -w net.ipv4.conf.all.rp_filter=1
$ sudo sysctl -w net.ipv4.conf.default.rp_filter=1

설명: Filtering (필터링, 여과)은 들어오는 패킷의 경로가 정상적인 인터페이스를 통해 들어왔는지 검증하여 가짜 IP를 가진 패킷을 차단하는 기술입니다.

3. ICMP 및 DDoS (분산 서비스 거부 공격) 방어

ICMP (Internet Control Message Protocol, 인터넷 제어 메시지 프로토콜)는 네트워크 진단에 쓰이지만, 공격자에 의해 서버의 위치를 파악하거나 DDoS (Distributed Denial of Service, 분산 서비스 거부 공격) 통로로 악용될 수 있습니다.

ICMP Redirect 차단

공격자가 가짜 경로 정보를 보내 데이터를 가로채는 행위를 방지합니다.

sudo sysctl -w net.ipv4.conf.all.accept_redirects=0
sudo sysctl -w net.ipv4.conf.default.accept_redirects=0

Broadcast ICMP 무시

네트워크 전체에 응답을 유도하는 Smurf Attack을 차단합니다.

sudo sysctl -w net.ipv4.icmp_echo_ignore_broadcasts=1

[도식: Broadcast ICMP Amplification Risk]

Attacker (1 Req)
Network Broadcast
100+ Replies
Target Server (Overwhelmed)

설명: Broadcast (브로드캐스트, 일괄 전송)는 네트워크상의 모든 장비에 패킷을 전달하는 방식입니다. 이를 악용하면 작은 요청으로 서버에 거대한 부하를 줄 수 있습니다.

4. 파라미터 영구 적용 및 확인 (Persistence)

CRITICAL STEP

상세 설명 주석 (Glossary)

1. sysctl

리눅스에서 /proc/sys 디렉터리 밑에 있는 커널 변수들을 읽고 쓰기 위한 인터페이스입니다.

2. SYN Cookies

TCP 핸드셰이크 시 서버 측에 세션 정보를 저장하지 않고 암호화된 쿠키를 클라이언트에 보내 연결의 정당성을 확인하는 기술입니다.

3. RP Filter

패킷이 들어온 인터페이스가 해당 소스 IP로 패킷을 보낼 때 사용될 인터페이스와 일치하는지 확인하여 스푸핑을 방지합니다.

4. Smurf Attack

소스 IP를 서버 IP로 위조하여 네트워크 내의 모든 장비에 ICMP 에코 요청을 보내 서버가 수많은 응답 패킷에 파묻히게 만드는 공격입니다.
