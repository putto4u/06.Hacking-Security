1. 루트 권한 실행의 위험성과 최소 권한 원칙

웹 서버 프로세스가 Root (루트, 시스템 최고 관리자) 권한으로 실행될 경우, 애플리케이션의 취약점을 통해 서버가 해킹당하면 공격자는 즉시 시스템 전체에 대한 통제권을 갖게 됩니다. 이를 방지하기 위해 Principle of Least Privilege (최소 권한 원칙, 업무 수행에 필요한 최소한의 권한만 부여하는 보안 원칙)를 적용해야 합니다.

[도식: Principle of Least Privilege Isolation Model]

Full Privilege (Root)

해킹 시 전체 시스템 장악
(FS 쓰기, 계정 변경, 프로세스 Kill)

Privilege Dropping
Least Privilege (Service)

해킹 시 특정 디렉토리만 노출
(시스템 영역 접근 불가)

위험 요소 (Risk Factors)

  • • 파일 시스템 전체 읽기/쓰기
  • • 시스템 설정 변경 (커널/네트워크)
  • • 타 사용자 프로세스 간섭

해결 방안 (Solutions)

웹 서버 전용의 비관리자 계정을 생성하고, 해당 계정 범위 내에서만 프로세스가 동작하도록 제한합니다.

권한 전환 정책

포트 점유 시에만 루트 권한을 사용하고, 요청 처리 시에는 일반 계정으로 전환합니다.

설명: 웹 서버는 포트 점유를 위해 초기 구동 시에만 루트 권한을 사용하고, 이후 실제 요청을 처리하는 Worker Process (워커 프로세스, 실제 작업을 수행하는 프로세스)는 반드시 일반 사용자 권한으로 전환되어야 합니다.

Annotate 01. Worker Process: 부모 프로세스(Master)로부터 생성되어 실제 클라이언트의 HTTP 요청을 처리하는 자식 프로세스입니다.

2. 전용 서비스 계정 생성 및 설정 (Service Account Setup)

Nginx나 Apache와 같은 웹 서버는 시스템 로그인 권한이 없는 별도의 전용 계정을 사용하여 구동하는 것이 안전합니다.

01

보안 계정 생성 예시

로그인이 불가능한 셸(/usr/sbin/nologin)을 지정하여 계정 탈취 시에도 2차 피해를 막습니다.

# 그룹 및 로그인 불가 사용자 생성
sudo groupadd www-data
sudo useradd -g www-data -s /usr/sbin/nologin -d /var/www www-data
02

Nginx 설정 반영

실행 사용자 설정을 명시하여 프로세스 권한을 제어합니다.

# /etc/nginx/nginx.conf

user www-data;
worker_processes auto;
Annotate 02. nologin: 해당 계정으로 시스템에 직접 로그인하는 것을 기술적으로 차단하는 셸 경로입니다. 보안 계정에 필수적으로 설정합니다.

3. 디렉터리 권한 및 ACL (Access Control List, 접근 제어 목록) 강화

웹 콘텐츠가 저장된 디렉터리는 웹 서버 계정만 읽을 수 있어야 하며, 쓰기 권한은 엄격히 제한해야 합니다.

[도식: Web Root Permission Hierarchy (웹 루트 권한 계층)]

/var/www/html Owner: root (755)
index.html, static/* File: 644 (Read Only)
/uploads (Data) Writable: www-data (775)

소유권 및 권한 최적화 (Implementation)

# 웹 루트 디렉터리 소유권 변경 (관리자 소유)
$ sudo chown -R root:root /var/www/html
# 디렉터리는 755(drwxr-xr-x), 파일은 644(-rw-r--r--) 일괄 권한 부여
$ sudo find /var/www/html -type d -exec chmod 755 {} \;
$ sudo find /var/www/html -type f -exec chmod 644 {} \;
# 업로드 폴더 등 쓰기가 필요한 특정 디렉터리만 예외 설정
$ sudo chown -R www-data:www-data /var/www/html/uploads

설명: ACL (Access Control List, 접근 제어 목록)은 전통적인 유닉스 권한 체계보다 더 세밀하게 사용자나 그룹별로 권한을 부여할 수 있는 메커니즘입니다.

Annotate 03. chmod (Change Mode): 파일이나 디렉터리의 읽기(4), 쓰기(2), 실행(1) 권한을 변경하는 명령어입니다.
Annotate 04. ACL (Access Control List): 사용자나 그룹별로 파일/디렉토리 접근 권한을 정교하게 제어하기 위한 목록입니다.

4. Chroot Jail (Chroot 제일, 루트 변경 격리) 및 컨테이너 격리

프로세스가 특정 디렉터리를 루트(/)로 인식하게 하여, 해당 디렉터리 외부의 시스템 파일에 접근하지 못하도록 물리적으로 격리합니다.

격리 효과

공격자가 웹 서버를 장악하더라도 /etc/passwd와 같은 시스템 중요 파일에 접근할 수 없습니다.

최신 트렌드

최근에는 Docker (도커, 컨테이너 기반 가상화 플랫폼)를 사용하여 애플리케이션 실행 환경 자체를 완전히 격리합니다.

Chroot Jail Concept

Process believes
/var/www/jail is /
Annotate 05. Chroot: 'Change Root'의 약자로, 프로세스가 실행되는 루트 디렉터리 위치를 변경하여 시스템의 나머지 부분으로부터 격리시키는 보안 기법입니다.

[AWS 환경 보안 및 과금 안내]
