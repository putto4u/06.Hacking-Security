2.2 권한 관리 및 프로세스 격리 (Permission Management and Process Isolation)

1. 루트 권한 실행의 위험성과 최소 권한 원칙

웹 서버 프로세스가 Root (루트, 시스템 최고 관리자) 권한으로 실행될 경우, 애플리케이션의 취약점을 통해 서버가 해킹당하면 공격자는 즉시 시스템 전체에 대한 통제권을 갖게 됩니다. 이를 방지하기 위해 Principle of Least Privilege (최소 권한 원칙, 업무 수행에 필요한 최소한의 권한만 부여하는 보안 원칙)를 적용해야 합니다.

  • 위험 요소: 파일 시스템 전체 읽기/쓰기, 시스템 설정 변경, 다른 사용자 프로세스 간섭.
  • 해결 방안: 웹 서버 전용의 비관리자 계정을 생성하고, 해당 계정 범위 내에서만 프로세스가 동작하도록 제한합니다.

설명: 웹 서버는 포트 점유를 위해 초기 구동 시에만 루트 권한을 사용하고, 이후 실제 요청을 처리하는 Worker Process (워커 프로세스, 실제 작업을 수행하는 프로세스)는 반드시 일반 사용자 권한으로 전환되어야 합니다.

2. 전용 서비스 계정 생성 및 설정

Nginx나 Apache와 같은 웹 서버는 시스템 로그인 권한이 없는 별도의 전용 계정을 사용하여 구동하는 것이 안전합니다.

  • 보안 계정 생성 예시 로그인이 불가능한 셸(/usr/sbin/nologin)을 지정하여 계정 탈취 시에도 2차 피해를 막습니다.
# 웹 서비스 전용 그룹 및 사용자 생성 (로그인 불가 설정)
sudo groupadd www-data
sudo useradd -g www-data -s /usr/sbin/nologin -d /var/www www-data
  • Nginx 설정 반영 /etc/nginx/nginx.conf 파일 최상단에 실행 사용자를 명시합니다.
# 실행 사용자 설정 확인
user www-data;
worker_processes auto;

3. 디렉터리 권한 및 ACL (Access Control List, 접근 제어 목록) 강화

웹 콘텐츠가 저장된 디렉터리는 웹 서버 계정만 읽을 수 있어야 하며, 쓰기 권한은 엄격히 제한해야 합니다.

  • 소유권 및 권한 최적화 웹 루트 디렉터리의 소유자를 관리자로 설정하고, 웹 서버 계정은 읽기 권한만 갖도록 설정하는 것이 표준입니다.
# 웹 루트 디렉터리 소유권 변경 (관리자 소유)
sudo chown -R root:root /var/www/html

# 디렉터리는 755(drwxr-xr-x), 파일은 644(-rw-r--r--) 권한 부여
sudo find /var/www/html -type d -exec chmod 755 {} \;
sudo find /var/www/html -type f -exec chmod 644 {} \;

# 업로드 폴더 등 쓰기가 필요한 특정 디렉터리만 예외 설정
sudo chown -R www-data:www-data /var/www/html/uploads

설명: ACL (Access Control List, 접근 제어 목록)은 전통적인 유닉스 권한 체계보다 더 세밀하게 사용자나 그룹별로 권한을 부여할 수 있는 메커니즘입니다.

4. Chroot Jail (Chroot 제일, 루트 변경 격리) 및 컨테이너 격리

프로세스가 특정 디렉터리를 루트(/)로 인식하게 하여, 해당 디렉터리 외부의 시스템 파일에 접근하지 못하도록 물리적으로 격리합니다.

  • 격리 효과: 공격자가 웹 서버를 장악하더라도 /etc/passwd와 같은 시스템 중요 파일에 접근할 수 없습니다.
  • 최신 트렌드: 최근에는 Docker (도커, 컨테이너 기반 가상화 플랫폼)를 사용하여 애플리케이션 실행 환경 자체를 완전히 격리하는 방식을 선호합니다.

[AWS 환경 보안 및 과금 안내]

클라우드 환경에서는 OS 내부 권한뿐만 아니라 인프라 자원에 대한 권한 격리도 중요합니다.

  • IAM Role (Identity and Access Management Role, IAM 역할)
    • EC2 인스턴스에 직접 자격 증명(Access Key)을 저장하지 않고, Instance Profile (인스턴스 프로파일)을 통해 최소한의 AWS 자원 접근 권한만 부여합니다.
    • 과금 안내: IAM 서비스 자체는 무료입니다.
  • Amazon GuardDuty (아마존 가드듀티, 지능형 위협 탐지 서비스)
    • 비정상적인 권한 상승 시도나 비인가된 API 호출을 탐지합니다.
    • 과금 안내: 분석된 로그 양(VPC Flow Logs, CloudTrail 등)에 따라 비용이 발생합니다. 공식 요금 페이지

상세 설명 주석 1. Worker Process: 부모 프로세스(Master)로부터 생성되어 실제 클라이언트의 HTTP 요청을 처리하는 자식 프로세스입니다. 2. nologin: 해당 계정으로 시스템에 직접 로그인하는 것을 기술적으로 차단하는 셸 경로입니다. 보안 계정에 필수적으로 설정합니다. 3. chmod (Change Mode): 파일이나 디렉터리의 읽기(4), 쓰기(2), 실행(1) 권한을 변경하는 명령어입니다. 4. Chroot: 'Change Root'의 약자로, 프로세스가 실행되는 루트 디렉터리 위치를 변경하여 시스템의 나머지 부분으로부터 격리시키는 보안 기법입니다. 5. IAM Role: 특정 사용자에게 고정된 권한을 주는 것이 아니라, 필요할 때 서비스나 객체가 권한을 '수임'하여 사용할 수 있게 하는 임시 자격 증명 방식입니다.
