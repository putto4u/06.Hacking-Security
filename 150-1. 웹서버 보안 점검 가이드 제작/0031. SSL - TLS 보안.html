1. 아키텍처 개요 (Architecture Overview)

SSL/TLS (Secure Sockets Layer / Transport Layer Security, 보안 소켓 계층 / 전송 계층 보안)는 웹 브라우저와 웹 서버 간의 데이터를 암호화하여 전송하는 표준 보안 프로토콜입니다. 현대 웹 환경에서는 기존의 보안 취약점이 발견된 SSL을 대신하여 보다 안전한 TLS 프로토콜을 사용하며, 이를 통해 통신의 기밀성, 무결성, 가용성을 보장합니다.

Confidentiality (기밀성)

스니핑(Sniffing) 공격으로부터 데이터를 보호하기 위해 모든 통신 내용을 암호화합니다.

Integrity (무결성)

전송 중인 데이터가 변조되지 않았음을 보장합니다.

Authentication (인증)

접속하려는 서버가 신뢰할 수 있는 기관으로부터 인증받은 실제 서버임을 증명합니다.

3. 실증 및 구현 (Implementation)

STEP 01

Nginx 보안 설정 (HTTPS 활성화)

취약한 암호화 알고리즘(DES, RC4, MD5)을 제거하고 TLS 1.2 이상 버전만 사용하도록 설정합니다.

Nginx Config
server {
    listen 443 ssl;
    server_name example.com;

    # 인증서 경로 지정
    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    # 최신 보안 프로토콜만 허용 (TLSv1.2, TLSv1.3)
    ssl_protocols TLSv1.2 TLSv1.3;

    # 강력한 사이퍼 스위트(Cipher Suite) 설정
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256...';
    ssl_prefer_server_ciphers on;

    # HSTS 헤더 추가 (HTTPS 강제)
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
}
STEP 02

Apache 보안 설정 (HTTPS 활성화)

$ sudo a2enmod ssl && sudo a2ensite default-ssl
Enabling module ssl... Enabling site default-ssl.
Apache Config
<VirtualHost *:443>
    ServerName example.com
    SSLEngine on

    # 인증서 경로 지정
    SSLCertificateFile    /etc/ssl/certs/example.com.crt
    SSLCertificateKeyFile /etc/ssl/private/example.com.key

    # 보안 핵심: 최신 프로토콜만 허용 및 취약 프로토콜 제외(-)
    SSLProtocol all -SSLv3 -TLSv1 -TLSv1.1
    
    # 서버 정의 암호화 순서 우선
    SSLHonorCipherOrder on
    
    # HSTS 활성화 권장
    Header always set Strict-Transport-Security "max-age=63072000"
</VirtualHost>
STEP 03

CSR (인증서 서명 요청) 생성

공인 기관(CA)에 인증서를 신청하기 위해 Private Key (개인키)와 CSR (Certificate Signing Request, 인증서 서명 요청)을 생성합니다.

4. 아키텍트 인사이트 (Architect Insight)

AWS 환경 보안 가이드 및 과금 안내

ACM (AWS Certificate Manager)

Public SSL/TLS 인증서를 무료로 발급 및 갱신해 줍니다. 단, 사설 CA(Private CA) 구축 시 월별 고정 비용 및 발급 비용이 발생하므로 주의가 필요합니다.

ALB (Application Load Balancer)

SSL Termination 기능을 통해 서버 부하를 절감합니다. ALB는 시간당 사용 요금과 LCU(Load Balancer Capacity Units) 소비량에 따라 과금됩니다.

SSL Termination (SSL 종단)

로드 밸런서에서 SSL 복호화를 수행하고, 내부 서버와는 HTTP로 통신하여 서버 자원을 절약하는 방식입니다. 동일 VPC 내 신뢰할 수 있는 구간에서 주로 사용됩니다.

Forward Secrecy (전방 향후 보안)

서버의 개인키가 유출되더라도 과거에 통신했던 암호화된 데이터는 복구할 수 없도록 보장하는 성질입니다. 이를 위해 ECDHE와 같은 알고리즘 사용이 권장됩니다.

HSTS Max-Age 설정

브라우저에 HTTPS 접속 강제 기간을 초 단위로 지정합니다. 일반적으로 1년(31536000초)을 권장하지만, 초기 설정 시에는 짧은 기간으로 테스트하는 것이 안전합니다.

Self-signed Certificate 주의사항

자가 서명 인증서는 기술적으로 암호화 강도는 동일하나, 공인 인증기관의 신뢰 확인이 없으므로 브라우저 경고창이 발생합니다. 운영 환경에서는 반드시 CA 인증서를 사용하십시오.
